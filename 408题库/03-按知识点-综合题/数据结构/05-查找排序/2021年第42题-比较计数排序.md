# 2021年第42题

**真题出处**：[2021年第42题](../../../01-真题原卷/2021年408真题.md#42)

**考点**：排序算法 | 稳定性 | 比较排序

**题干**：
已知某排序算法如下：

```c
void cmpCountSort(int a[], int b[], int n)
{
    int i, j, *count;
    count = (int *)malloc(sizeof(int) *n);
    // C++ 语言：count = new int[n];
    for (i = 0; i < n; i++) count[i] = 0;
    for (i = 0; i < n - 1; i++)
        for (j = i + 1; j < n; j++)
            if (a[i] < a[j]) count[j]++;
            else count[i]++;
    for (i = 0; i < n; i++) b[count[i]] = a[i];
    free(count); // C++ 语言：delete count;
}
```

请回答下列问题。

1) 若有 int a[] = {25, -10, 25, 10, 11, 19}, b[6];，则调用 cmpCountSort(a, b, 6) 后数组 b 中的内容是什么？

2) 若 $a$ 中含有 $n$ 个元素, 则算法执行过程中, 元素之间的比较次数是多少?

3）该算法是稳定的吗？若是，则阐述理由；否则，修改为稳定排序算法。

**答案**：
## 42.【王道书8.6节】

## 1)【标准答案】

```txt
b[]={-10, 10, 11, 19, 25, 25}。 
```

## 【分析】

cmpCountSort 算法基于计数排序的思想, 对序列进行排序。cmpCountSort 算法遍历数组中的元素, count 数组记录比对应待排序数组元素下标大的元素个数, 例如, count[1]=3 的意思是数组 a 中有 3 个元素比 a[1] 小, 即 a[1] 是第 4 大元素, a[1] 的正确位置应是 b[3]。

## 2)【标准答案】

元素之间的比较次数是 $n(n-1)/2$ 。

## 【分析】

由代码 for(i=0;i<n-1;i++) 和 for(j=i+1;j<n;j++) 可知，在循环过程中，每个元素都与它后面的所有元素比较一次（即所有元素都两两比较一次），比较次数之和为 $(n-1)+(n-2)+\cdots+1$ 。

## 3)【标准答案】

不是。需要将程序中的 if 语句修改如下：

```txt
if(a[i]<=a[j]) count[j]++;
else count[i]++;
""" 
```

## 【分析】

如果不加等号，两个相等的元素比较时，前面元素的 count 值会加 1，导致原序列中靠前的元素在排序后的序列中处于靠后的位置。
---

## 解题思路

**算法原理**：count[i]记录比a[i]小的元素个数，b[count[i]]=a[i]完成排序

**第1问**：手算count[]={4,0,5,2,3,1}，b[]={-10,10,11,19,25,25}

**第2问**：两层循环，比较次数=n(n-1)/2

**第3问(稳定性)**：
- 原算法不稳定：a[i]<a[j]时count[j]++，相等元素前面的count值会大
- 修改为稳定：if(a[i]<=a[j]) count[j]++，相等时后面元素count++

## 易错点分析

### 易错点1：稳定性判定
❌ 错误理解：count记录个数就是稳定排序
✅ 正确理解：相等元素需保持原顺序，a[i]<a[j]改为a[i]<=a[j]

### 易错点2：count含义混淆
| 错误理解 | 正确理解 | 说明 |
|---------|---------|------|
| count[i]=元素出现次数 | count[i]=比a[i]小的元素个数 | 决定a[i]在有序序列中的位置 |
