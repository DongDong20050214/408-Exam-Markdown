# 2019年第41题

**考点**：单链表 | 链表重排 | 空间复杂度 | 算法设计

**题干**：
设线性表 $L = (a_{1}, a_{2}, a_{3}, \cdots, a_{n-2}, a_{n-1}, a_{n})$ 采用带头结点的单链表保存, 链表中的结点定义如下:

```c
typedef struct node
{ 
    int data;
    struct node *next;
} NODE;
```

请设计一个空间复杂度为 $O(1)$ 且时间上尽可能高效的算法，重新排列 $L$ 中的各结点，得到线性表 $L' = (a_{1}, a_{n}, a_{2}, a_{n-1}, a_{3}, a_{n-2}, \cdots)$ 。

要求：
1）给出算法的基本设计思想。
2）根据设计思想，采用 C 或 C++ 语言描述算法，关键之处给出注释。
3）说明你所设计的算法的时间复杂度。

**答案**：
41.【王道书2.3节】

1)【标准答案】

算法的基本设计思想

算法分 3 步完成。第 1 步，采用两个指针交替前行，找到单链表的中间结点；第 2 步，将单链表的后半段结点原地逆置；第 3 步，从单链表前后两段中依次各取一个结点，按要求重排。【分析】

先观察 $L = (a_{1}, a_{2}, a_{3}, \cdots, a_{n-2}, a_{n-1}, a_{n})$ 和 $L' = (a_{1}, a_{n}, a_{2}, a_{n-1}, a_{3}, a_{n-2}, \cdots)$ ，发现 $L'$ 是由 $L$ 取第一个元素，再取倒数第一个元素……依次合并而成的。为了方便链表后半段取元素，需要先将 $L$ 后半段原地逆置 [题目要求空间复杂度为 $O(1)$ ，不能借助栈]，否则每取最后一个结点都需要遍历一次链表。①先找出链表 $L$ 的中间结点，为此设置两个指针 $\mathbf{p}$ 和 $\mathbf{q}$ ，指针 $\mathbf{p}$ 每次走一步，指针 $\mathbf{q}$ 每次走两步，当指针 $\mathbf{q}$ 到达链尾时，指针 $\mathbf{p}$ 正好在链表的中间结点；②然后将 $L$ 的后半段结点原地逆置。③从单链表前后两段中依次各取一个结点，按要求重排。

2)【标准答案】

算法实现

```lisp
void change_list(NODE *h)
{
    NODE *p,*q,*r,*s;
    p=q=h;
    while(q->next!=NULL) //寻找中间结点
    {
    p=p->next; //p走一步
    q=q->next;
    if(q->next!=NULL) q=q->next; //q走两步
    }
    q=p->next; //p所指结点为中间结点，q为后半段链表的首结点
    p->next=NULL;
    while(q!=NULL) //将链表后半段逆置
    {
    r=q->next;
    q->next=p->next;
    p->next=q;
    q=r;
    }
}
```

```txt
s=h->next; //s 指向前半段的第一个数据结点，即插入点
q=p->next; //q 指向后半段的第一个数据结点
p->next=NULL;
while (q!=NULL) //将链表后半段的结点插入指定位置
{ r=q->next; //r 指向后半段的下一个结点
    q->next=s->next; //将 q 所指结点插入 s 所指结点之后
    s->next=q;
    s=q->next; //s 指向前半段的下一个插入点
    q=r;
}
```

## 3)【标准答案】

算法的时间复杂度为 $O(n)$ 。

## 【分析】

第1步找中间结点的时间复杂度为 $O(n)$ , 第2步逆置的时间复杂度为 $O(n)$ , 第3步合并链表的时间复杂度为 $O(n)$ , 所以该算法的时间复杂度为 $O(n)$ 。
---

## 解题思路

**核心思想**：快慢指针找中点→逆置后半段→前后交替合并

**解题步骤**：
1. 找中间节点：p走1步，q走2步，q到尾时p在中点。时间O(n)
2. 逆置后半段：从p.next开始头插法逆置。时间O(n)
3. 交替合并：从前半段取一个，后半段取一个，插入指定位置。时间O(n)

**关键操作**：头插法逆置 `q->next=p->next; p->next=q`

## 易错点分析

### 易错点1：空间复杂度超限
❌ 错误理解：用栈存储后半段元素再依次弹出合并
✅ 正确理解：题目要求O(1)空间，必须原地逆置后半段，不能使用辅助栈

### 易错点2：合并顺序混淆
| 错误做法 | 正确做法 | 区别 |
|---------|---------|------|
| 先移动前半段指针再插入 | 先插入后半段节点，再移动前半段指针 | 正确做法保证s始终指向下一个插入点 |
