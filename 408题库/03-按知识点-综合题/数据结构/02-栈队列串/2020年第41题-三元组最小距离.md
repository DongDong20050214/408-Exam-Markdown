# 2020年第41题

**真题出处**：[2020年第41题](../../../01-真题原卷/2020年408真题.md#41)

**考点**：贪心算法 | 多路归并 | 数组遍历

**题干**：
定义三元组 $(a, b, c)$ （其中 $a, b, c$ 均为整数）的距离 $D = |a - b| + |b - c| + |c - a|$ 。给定3个非空整数集合 $S_{1}$ 、 $S_{2}$ 和 $S_{3}$ ，按升序分别存储在3个数组中。设计一个尽可能高效的算法，计算并输出所有可能的三元组 $(a, b, c)$ ( $a \in S_{1}, b \in S_{2}, c \in S_{3}$ ) 中的最小距离。例如 $S_{1} = \{-1, 0, 9\}$ ， $S_{2} = \{-25, -10, 10, 11\}$ ， $S_{3} = \{2, 9, 17, 30, 41\}$ ，则最小距离为2，相应的三元组为(9, 10, 9)。

要求：
1）给出算法的基本设计思想。
2）根据设计思想，采用 C 或 C++ 语言描述算法，关键之处给出注释。
3）说明你所设计算法的时间复杂度和空间复杂度。

**答案**：
41.【王道书2.2节】

1)【标准答案】

算法的基本设计思想

① 使用 $D_{min}$ 记录当前所有已处理过的三元组的最小距离，初值为 C 语言能表示的最大整数 INT MAX;

② 若集合 $S_{1}$ 、 $S_{2}$ 和 $S_{3}$ 分别保存在数组 A、B、C 中。数组下标变量 $i = j = k = 0$ ，当 $i < |S_{1}|$ 且 $j < |S_{2}|$ 且 $k < |S_{3}|$ 时（ $|S|$ 表示集合 $S$ 中的元素个数），循环执行 a)～c):

a) 计算（A[i], B[j], C[k]）的距离 D;

b) 若 $D < D_{\min}$ ，则 $D_{\min} = D;$ 

c) 将 A[i], B[j], C[k] 中的最小值的下标 +1;

③ 输出 $D_{min}$ ，结束。

【分析】

由 $D = |a - b| + |b - c| + |c - a| \geqslant 0$ 得出如下结论：

① 当 a=b=c 时，距离最小。

② 其余情况。不失一般性，假设 $a \leqslant b \leqslant c$ ，观察下面的数轴：

![image](https://cdn-mineru.openxlab.org.cn/result/2026-06-09/9eb26cc9-225d-405f-b367-78f7bfef7dae/810cb39806836dddf12f81fc456427c43ec3f23629de20d6bb5ff480313d6f29.jpg)


$$
L _ {1} = | a - b |, L _ {2} = | b - c |, L _ {3} = | c - a |, D = | a - b | + | b - c | + | c - a | = L _ {1} + L _ {2} + L _ {3} = 2 L _ {3}
$$

由 D 的表达式可知，事实上决定 D 大小的关键是 a 和 c 之间的距离，于是问题就可以简化为每次固定 c 找一个 a，使得 $L_{3} = |c - a|$ 最小。

2)【标准答案】

算法实现

```c
#define INT_MAX 0x7fffffff
int abs_(int a){//计算绝对值
    if(a<0) return -a;
    else return a;
}
bool xls_min(int a, int b, int c){//a 是否是三个数中的最小值
    if(a<=b&&a<=c) return true;
    return false;
}
int findMinofTrip(int A[], int n, int B[], int m, int C[], int p){
    //D_min 用于记录三元组的最小距离，初值赋为 INT_MAX
    int i=0, j=0, k=0, D_min=INT_MAX, D;
    while(i<n&&j<m&&k<p&&D_min>0){
    D=abs_(A[i]-B[j])+abs_(B[j]-C[k])+abs_(C[k]-A[i]);    //计算 D
    if(D<D_min) D_min=D;    //更新 D
    if(xls_min(A[i], B[j], C[k])) i++;    //更新 a
    else if(xls_min(B[j], C[k], A[i])) j++;
    else k++;
    }
    return D_min;
}
```

## 3)【标准答案】

设 $n = |S_1| + |S_2| + |S_3|$ ，算法的时间复杂度为 $O(n)$ ，空间复杂度为 $O(1)$ 。

## 解题思路

**核心思想**：贪心策略，三指针协同移动，每次移动最小值

**解题步骤**：
1. 初始化i=j=k=0，D_min=INT_MAX
2. 计算当前D=|A[i]-B[j]|+|B[j]-C[k]|+|C[k]-A[i]|
3. 更新D_min，将A[i]、B[j]、C[k]中最小值的下标+1

**关键公式**：假设a≤b≤c，D=2|c-a|，移动最小值可减小距离

## 易错点分析

### 易错点1：移动策略错误
❌ 错误理解：每次移动中间值或最大值
✅ 正确理解：由D=2|max-min|可知，移动最小值可缩小max-min差距，有可能减小D

### 易错点2：循环终止条件
❌ 错误理解：当D_min=0时才终止
✅ 正确理解：任一指针到达数组末尾即终止，D_min>0时加速终止
