# 2023年第41题

**考点**：有向图 | 出度入度 | 图的遍历

**题干**：
已知有向图 $G$ 采用邻接矩阵存储，类型定义如下：

```c
typedef struct // 图的类型定义
{
    int numVertices, numEdges; // 图的顶点数和有向边数
    char VerticesList[MAXV]; // 顶点表，MAXV为已定义常量
    int Edge[MAXV][MAXV]; // 邻接矩阵
```

将图中出度大于入度的顶点称为 K 顶点。例如在题 41 图中，顶点 a 和 b 为 K 顶点。

1)【标准答案】

算法的基本设计思想：

采用邻接矩阵表示有向图时，一行中1的个数为该行对应顶点的出度，一列中1的个数为该列对应顶点的入度。使用一个初值为零的计数器记录K顶点数。对图 $G$ 的每个顶点，根据邻接矩阵计算其出度outdegree和入度indegree。若outdegree-indegree $>0$ ，则输出该顶点且计数器加1。最后返回计数器的值。

2)【标准答案】

用 C 语言描述的算法:

```txt
int printVertices(MGraph G){
//采用邻接矩阵存储，输出K顶点，返回个数
int indegree, outdegree, k, m, count = 0;
for (k = 0; k < G.numVertices; k++) {
    indegree = outdegree = 0;
    for (m = 0; m < G.numVertices; m++) // 计算顶点的出度
    outdegree += G.Edge[k][m];
    for (m = 0; m < G.numVertices; m++) // 计算顶点的入度
    indegree += G.Edge[m][k];
    if (outdegree > indegree) {
    printf("%c", G.VerticesList[k]);
    count++;
    }
}
```

## 解题思路

**核心思想**：遍历每个顶点，计算出度和入度，判断出度>入度

**解题步骤**：
1. 对每个顶点k，计算出度：累加Edge[k][m](第k行)
2. 计算入度：累加Edge[m][k](第k列)
3. 若outdegree>indegree，输出顶点，count++
4. 返回count

**邻接矩阵性质**：有向图第i行元素和=出度，第j列元素和=入度

## 易错点分析

### 易错点1：出度入度计算混淆
❌ 错误理解：出度=第k列元素和
✅ 正确理解：出度=第k行元素和，入度=第k列元素和

### 易错点2：时间复杂度分析
| 错误分析 | 正确分析 | 说明 |
|---------|---------|------|
| O(n) | O(n²) | 每个顶点需遍历矩阵一行一列，共n个顶点 |
