# 2021年第41题

**真题出处**：[2021年第41题](../../../01-真题原卷/2021年408真题.md#41)

**考点**：欧拉路径 | 图的连通性 | 度数判定

**题干**：
已知无向连通图 $G$ 由顶点集 $V$ 和边集 $E$ 组成, $|E| > 0$, 当 $G$ 中度为奇数的顶点个数为不大于 2 的偶数时，G 存在包含所有边且长度为 $|E|$ 的路径（称为 EL 路径）。设图 G 采用邻接矩阵存储，类型定义如下：

```c
typedef struct{ //图的定义
    int numVertices,numEdges; //图中实际的顶点数和边数
    char VerticesList[MAXV]; //顶点表。MAXV为已定义常量
    int Edge[MAXV][MAXV]; //邻接矩阵
}MGraph;
```

请设计算法：int IsExistEL(MGraph G)，判断 $G$ 是否存在EL路径，若存在，则返回1，否则，返回0。要求：

1）给出算法的基本设计思想。

2）根据设计思想，采用 C 或 C++ 语言描述算法，关键之处给出注释。

3）说明你所设计算法的时间复杂度和空间复杂度。

**答案**：
41.【王道书6.2节】

1)【标准答案】

算法的基本设计思想

对于采用邻接矩阵存储的无向图，邻接矩阵每行（列）中非零元素的个数为本行（列）对应顶点的度。可以依次计算连通图 G 中各顶点的度，并记录度为奇数的顶点数，若个数为 0 或 2，则返回 1，否则返回 0。

2)【标准答案】

算法实现

```txt
int IsExistEL(MGraph G) {
    //采用邻接矩阵存储，判断图是否存在 EL 路径
    int degree, i, j, count = 0;
    for (i = 0; i < G.numVertices; i++) {
    degree = 0;
    for (j = 0; j < G.numVertices; j++)
    degree += G.Edge[i][j];    //依次计算各个顶点的度
    if (degree % 2 != 0)
    count++;    //对度为奇数的顶点计数
    }
}
```

}
if(count==0 || count==2)

## 3)【标准答案】

算法需要遍历整个邻接矩阵，时间复杂度是 $O(n^{2})$ ，空间复杂度是 $O(1)$ 。

## 解题思路

**核心思想**：EL路径=欧拉路径，判定条件是奇度顶点数为0或2

**解题步骤**：
1. 遍历邻接矩阵，计算每个顶点度数(第i行非零元素个数)
2. 统计度为奇数的顶点数count
3. count=0或2返回1(存在EL路径)，否则返回0

**欧拉路径定理**：连通图存在欧拉路径⇔奇度顶点数为0(欧拉回路)或2

## 易错点分析

### 易错点1：度数计算错误
❌ 错误理解：有向图度数=出度+入度
✅ 正确理解：题目是无向图，度数=邻接矩阵第i行(或列)非零元素个数

### 易错点2：EL路径条件混淆
| 欧拉回路 | 欧拉路径 | 区别 |
|---------|---------|------|
| 奇度顶点数=0 | 奇度顶点数=0或2 | 回路起点=终点 |
