# 2013年第45题

**考点**：进程同步 | 信号量 | PV操作 | 互斥访问

**题干**：
某博物馆最多可容纳 500 人同时参观，有一个出入口，该出入口一次仅允许一个人通过。参观者的活动描述如下:

```txt
cobegin
    参观者进程 i:
    {
    ...
    进门;
    ...
    参观;
    ...
    出门;
    ...
    }
coend
```

请添加必要的信号量和 P、V [或 wait()、signal()] 操作，以实现上述过程中的互斥与同步。要求写出完整的过程，说明信号量的含义并赋初值。

**答案**：
## 45.【王道书2.3节】

## 【标准答案】

定义两个信号量

```txt
Semaphore empty = 500; //博物馆可以容纳的最多人数（2分）
Semaphore mutex =1; //用于出入口资源的控制（2分）
cobegin
参观者进程 i:
{
    ...
    P(empty);
    P(mutex);
    进门;
    V(mutex);
```

```txt
参观；  
P(mutex);  
出门；  
V(mutex);  
V(empty);  
...  
}  
coend（3分）
```

【分析】

出入口一次仅允许一个人通过，设置互斥信号量 mutex，初值为 1。博物馆最多可同时容纳 500 人，故设置信号量 empty，初值为 500。

【评分说明】

① 信号量初值给 1 分，说明含义给 1 分，两个信号量的初值和含义共 4 分。

② 对 mutex 的 P、V 操作正确给 2 分。

③ 对 empty 的 P、V 操作正确给 1 分。

④ 其他答案，参照①～③的标准给分。

## 解题思路

### 第一步：分析两类资源
- 互斥资源：出入口（一次一人通过）→ mutex=1
- 有限资源：博物馆容量（最多500人）→ empty=500

### 第二步：确定PV操作位置
- 进门：P(empty)减少容量 → P(mutex)占用出入口 → 进门 → V(mutex)释放出入口
- 出门：P(mutex)占用出入口 → 出门 → V(mutex)释放出入口 → V(empty)增加容量

### 第三步：理解empty和mutex的作用
empty控制总人数上限，mutex保证出入口互斥使用

## 易错点分析

### 易错点1：出门时遗漏V(empty)
❌ 错误理解：认为出门只需要对出入口互斥，只需P(mutex)→出门→V(mutex)，忘记释放容量。这会导致empty一直减少，最终归0后无人能进，即使博物馆内已空。
✅ 正确理解：empty=500代表博物馆可容纳人数，进门P(empty)消耗容量，出门V(empty)归还容量。这是资源计数器，进出必须配对。如果只进不归还，empty会耗尽，违反实际逻辑（人已出去但容量未恢复）。

### 易错点2：进门时empty和mutex的PV顺序
❌ 错误做法：先P(mutex)占用出入口，再P(empty)检查容量。当博物馆满员时，参观者持有mutex并等待empty，馆内参观者想出门需要mutex，形成死锁。
✅ 正确理解：必须先P(empty)检查容量，再P(mutex)使用出入口。这遵循"先判断条件，再占用资源"的原则，避免持有互斥锁等待条件导致的死锁。empty是同步信号量（控制容量），mutex是互斥信号量（保护临界区），同步在外互斥在内。
