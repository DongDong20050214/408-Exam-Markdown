# 2024年第46题

**真题出处**：[2024年第46题](../../../01-真题原卷/2024年408真题.md#46)

**考点**：进程同步 | 信号量 | 临界区

**题干**：
计算机系统中的进程之间往往需要相互协作以完成一个任务。在某网络系统中, 缓冲区B用于存放一个数据分组, 对B的操作有C1、C2和C3。C1将一个数据分组写入B, C2从B中读出一个数据分组, C3对B中的数据分组进行修改。要求B为空时才能执行C1, B非空时才能执行C2和C3。请回答下列问题。

1）假设进程P1和P2均需要执行C1，实现C1的代码是否为临界区？为什么？（2分）

2) 假设B初始为空, 进程P1执行C1一次, 进程P2执行C2一次。请定义尽可能少的信号量, 并用wait()、signal()操作描述进程P1和P2之间的同步或互斥关系, 说明所用信号量的作用及其初值。(3分)

3) 假设B初始不为空, 进程P1和P2各执行C3一次。请定义尽可能少的信号量, 并用wait()、signal()操作描述进程P1和P2之间的同步或互斥关系, 说明所用信号量的作用及其初值。(3分)

**答案**：
## 46. 【王道书 2.3 节】

## 1）【标准答案】

实现C1的代码是临界区。因为代码C1执行对B的写操作，且P1和P2需要互斥执行C1。

## 2）【标准答案】

进程 P1 和 P2 的同步伪代码如下。

<table><tr><td colspan="2">Semaphore S = 0; //实现进程 P1 与 P2 的同步</td></tr><tr><td>P1...C1;signal(S);...</td><td>P2...wait(S);C2;...</td></tr></table>

## 【分析】

有一组互斥关系和一组同步关系。互斥关系为：P1 和 P2 需要互斥访问缓冲区 B；同步关系为：P1 执行完 C1 后，P2 才能执行 C2。缓冲区 B 初始为空，且最大容量为 1，因此可不用定义互斥信号量 mutex。因为在缓冲区大小为 1 的条件下，同步信号量 S 就可同时保证 P1 和 P2 互斥访问 B，符合题意“尽可能定义少的信号量”的要求。

## 3）【标准答案】

进程 P1 和 P2 的互斥伪代码如下。

<table><tr><td colspan="2">Semaphore mutex = 1; //实现 P1 与 P2 互斥执行 C3</td></tr><tr><td>P1...wait(mutex);C3;signal(mutex);...</td><td>P2...wait(mutex);C3;signal(mutex);...</td></tr></table>

## 【分析】

缓冲区B初始非空，因此仅需一个互斥信号量mutex来保证P1和P2互斥访问B即可。
---

## 解题思路

### 第一步：判断C1是否为临界区
C1对共享缓冲区B执行写操作，P1和P2需要互斥执行C1，因此C1是临界区。

### 第二步：分析情况2（B初始为空）
P1执行C1写入，P2执行C2读出。同步关系：P1→P2。缓冲区大小为1，同步信号量S=0同时保证互斥，无需额外互斥信号量。

### 第三步：分析情况3（B初始不为空）
P1和P2各执行C3修改。只有互斥关系，无同步依赖。需要mutex=1保证互斥访问。

## 易错点分析

### 易错点1：情况2中额外定义互斥信号量
❌ 错误做法：定义S=0（同步）和mutex=1（互斥），认为需要两个信号量分别处理同步和互斥。P1:P(mutex)→C1→V(mutex)→V(S); P2:P(S)→P(mutex)→C2→V(mutex)。
✅ 正确理解：题目要求"尽可能少的信号量"。缓冲区容量为1时，同步信号量S隐含了互斥。S=0保证P2必须等P1完成，同时只有一个进程能访问B（P1执行时S=0，P2被阻塞；P2执行时S变0，P1无法再次执行）。容量为1的生产者-消费者问题，同步信号量兼具互斥功能。

### 易错点2：情况3中定义同步信号量
❌ 错误做法：认为P1和P2都执行C3存在先后顺序，定义S=0让P1先执行C3再让P2执行。P1:wait(mutex)→C3→signal(mutex)→signal(S); P2:wait(S)→wait(mutex)→C3→signal(mutex)。
✅ 正确理解：题目说"P1和P2各执行C3一次"，没有指定顺序，只要求互斥访问B。两个进程是对等的，不存在前驱关系，只有竞争关系。只需mutex=1保证互斥即可。强行添加同步关系会限制并发性，违背题意。同步用于有依赖的操作，互斥用于无序的竞争。
