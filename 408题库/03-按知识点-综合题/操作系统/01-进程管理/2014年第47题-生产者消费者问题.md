# 2014年第47题

**真题出处**：[2014年第47题](../../../01-真题原卷/2014年408真题.md#47)

**考点**：生产者消费者 | PV操作 | 信号量 | 进程同步

**题干**：
某系统中有n个生产者进程和m个消费者进程，每个生产者进程不断地生产产品并放入缓冲区，每个消费者进程不断地从缓冲区中取出产品并消费。缓冲区有1000个空位，每次只能放入或取出一个产品。消费者每次需要连续取10个产品才能进行消费，不足10个则需等待。

请添加必要的信号量和P、V（或wait()、signal()）操作，实现生产者进程和消费者进程之间的同步与互斥。要求写出完整的过程，说明信号量的含义并赋初值。

**答案**：
## 47.【王道书2.3节】

【标准答案】

伪代码如下：

```c
semaphore empty = 1000; //空缓冲区数  
semaphore full = 0; //非空缓冲区数（1分）  
semaphore mutex1 = 1; //用于实现生产者之间的互斥  
semaphore mutex2 = 1; //用于实现消费者之间的互斥（1分）  
int in = 0;  
int out = 0;
```

生产者进程（2分）

```c
while (TRUE)
{ 
    produce;
    P(empty);
    P(mutex1);
    put an item into buf[in];
    in=(in+1) mod n;
    V(mutex1);
    V(full);
} 
```

消费者进程（4分）

```c
while (TRUE)
{    
    P(mutex2);
    for(int i = 0; i < 10 ; i++)
    {    
        P(full);
        get an item from buf[out];
        out=(out+1) mod n;
        V(empty);
    }
    V(mutex2);
    Consume;
} 
```

【分析】

这是典型的生产者和消费者问题，只对典型问题加了一个条件，只需在标准模型上新加一个信号量，即可完成指定要求。

设置4个变量mutex1、mutex2、empty和full，mutex1用于一个控制一个消费者进程一个周期（10次）内对于缓冲区的控制，初值为1；mutex2用于进程单次互斥的访问缓冲区，初值为1；empty代表缓冲区的空位数，初值为1000；full代表缓冲区的产品数，初值为0。

【评分说明】

① 信号量的初值和含义都正确给2分。

② 生产者之间的互斥操作正确给1分；生产者与消费者之间的同步操作正确给2分；消费者之间互斥操作正确给1分。

③ 控制消费者连续取产品数量正确给2分。

## 解题思路

### 第一步：识别同步互斥关系
- 互斥关系：生产者之间、消费者之间互斥访问缓冲区
- 同步关系：缓冲区空满状态（empty/full）
- 特殊约束：消费者需连续取10个产品

### 第二步：设计信号量
- mutex1/mutex2：生产者/消费者互斥（初值1）
- empty：空缓冲区数（初值1000）
- full：产品数（初值0）

### 第三步：编写PV操作顺序
- 生产者：P(empty) → P(mutex1) → 放产品 → V(mutex1) → V(full)
- 消费者：P(mutex2) → for循环10次{P(full)→取产品→V(empty)} → V(mutex2)

## 易错点分析

### 易错点1：消费者互斥信号量位置错误
❌ 错误理解：把P(mutex2)放在循环内部，认为每次取产品都需要申请互斥锁。这会导致消费者取第一个产品后释放锁，其他消费者可能插入，无法保证连续取10个。
✅ 正确理解：P(mutex2)必须在for循环外部，确保一个消费者一次性连续取10个产品，期间其他消费者不能介入。mutex2的作用是保护"连续取10个"这个原子操作。

### 易错点2：PV操作顺序颠倒导致死锁
❌ 错误做法：生产者先P(mutex1)再P(empty)。当缓冲区满时，生产者持有mutex1并等待empty，消费者无法获取mutex1来取产品释放empty，形成死锁。
✅ 正确理解：必须先对同步信号量（empty/full）进行P操作，再对互斥信号量（mutex）进行P操作。这是生产者-消费者模型的铁律：同步在外，互斥在内。
