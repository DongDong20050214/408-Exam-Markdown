# 2023年第45题

**考点**：临界区 | 互斥 | swap指令

**题干**：
现要求学生使用 swap 指令和布尔型变量 lock 实现临界区互斥。lock 为线程间共享的变量，lock 的值为 TRUE 时线程不能进入临界区，为 FALSE 时线程能够进入临界区。某同学编写的实现临界区互斥的伪代码如题 45(a) 图所示。

某同学编写的伪代码：
```
bool lock = FALSE;    // 共享变量
......
bool key = TRUE;
if (key == TRUE)
    swap key, lock;    // 交换 key 和 lock 的值
临界区;
lock = TRUE;
......
```
进入区、退出区

题45(b)图给出了newSwap()的代码：
```c
void newSwap(bool *a, bool *b)
{
    bool temp = *a;
    *a = *b;
    *b = temp;
}
```

请回答下列问题。

1) 题 45(a) 图的伪代码中哪些语句存在错误？将其改为正确的语句（不增加语句的条数）。

2) 题 45(b)图给出了交换两个变量值的函数 newSwap()的代码, 是否可以用函数调用语句 "newSwap(&key, &lock)" 代替指令 "swap key, lock" 以实现临界区互斥? 为什么?

**答案**：
## 45.【王道书2.3节】

## 1)【标准答案】

进入区中的语句 “if (key == TRUE) swap key, lock” 存在错误，修改为 “while (key == TRUE) swap key, lock”。退出区中的语句“lock = TRUE”存在错误，修改为 “lock = FALSE”。

## 【分析】

if 语句无法实现对临界区的互斥访问，因为 if 语句执行之后，不论结果如何，线程都能访问临界区。本题使用 swap 指令和 lock 变量来实现对临界区的互斥访问，当线程不能进入临界区时，本身并不会主动放弃 CPU，因此需要让线程在进入区中循环检查 lock 值，可以使用 while 循环，进入区的语句修改后，当 lock 值为 TRUE 时，线程会一直执行 while 循环的内容，直到 lock 值被修改为 FALSE 时，线程才能进入临界区。在退出区中，代表该线程对临界资源的访问已经结束，此时需要把 lock 值设为 FALSE，代表其他线程可以访问临界区。

## 2)【标准答案】

否。因为多个线程可以并发执行 newSwap(),newSwap()执行时传递给形参 b 的是共享变量 lock 的地址，在 newSwap()中对 lock 既有读操作又有写操作，并发执行时不能保证实现两个变量值的原子交换，从而会导致并发执行的线程同时进入临界区。

## 【分析】

例如，线程 A 和线程 B 并发执行，初始时 lock 值为 FALSE，当线程 A 执行完 $*a=^{*}b$ 之后发生了进程调度，切换到线程 B 执行，线程 B 执行完 newSwap 之后发生线程切换，则此时线程 A 和 B 都能进入临界区，不能实现互斥访问。

## 解题思路

### 第一步：分析swap指令实现互斥的原理
swap指令原子交换key和lock，若lock=FALSE则key变FALSE进入临界区，若lock=TRUE则key变TRUE继续循环等待。

### 第二步：识别错误
- if改为while：if只检查一次无法阻塞等待
- lock=TRUE改为FALSE：退出时应释放锁（FALSE），而非加锁

### 第三步：分析newSwap为何不可行
newSwap非原子操作，并发执行可能导致多线程同时进入临界区。

## 易错点分析

### 易错点1：用if而非while导致无法阻塞
❌ 错误理解：if(key==TRUE) swap key,lock足够实现互斥，因为lock=TRUE时会交换，线程会被阻止进入临界区。忽略了if只执行一次，交换后key变为TRUE但代码继续向下执行进入临界区。
✅ 正确理解：必须用while忙等。当lock=TRUE时，swap后key变TRUE，while循环继续，再次swap又把key和lock交换回去，反复循环直到其他线程释放lock（设为FALSE）。if只检查一次无法实现持续等待，线程会直接进入临界区破坏互斥。

### 易错点2：退出区设置lock=TRUE而非FALSE
❌ 错误理解：进入临界区时key变为FALSE，所以退出时应该将lock设为TRUE表示已使用。混淆了锁的语义。
✅ 正确理解：lock=FALSE表示临界区可用（未锁），lock=TRUE表示临界区被占用（已锁）。退出临界区必须释放锁，即设lock=FALSE，允许其他线程进入。设为TRUE会永久锁住临界区，其他线程无法进入。锁的语义：FALSE=空闲，TRUE=占用。
