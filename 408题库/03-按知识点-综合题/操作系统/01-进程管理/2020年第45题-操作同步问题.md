# 2020年第45题

**真题出处**：[2020年第45题](../../../01-真题原卷/2020年408真题.md#45)

**考点**：进程同步 | 信号量 | PV操作 | 前驱图

**题干**：
现有 5 个操作 A、B、C、D 和 E, 操作 C 必须在 A 和 B 完成后执行, 操作 E 必须在 C 和 D 完成后执行, 请使用信号量的 wait()、signal() 操作 (P、V 操作) 描述上述操作之间的同步关系，并说明所用信号量及其初值。

**答案**：
## 45.【王道书2.3节】

## 【标准答案】

Semaphore $S_{AC} = 0$ ; //控制操作A和C的执行顺序
Semaphore $S_{BC} = 0$ ; //控制操作B和C的执行顺序
Semaphore $S_{CE} = 0$ ; //控制操作C和E的执行顺序
Semaphore $S_{DE} = 0$ ; //控制操作D和E的执行顺序
CoBegin
Begin 操作A; signal( $S_{AC}$ ); End
Begin 操作B; signal( $S_{BC}$ ); End
Begin wait( $S_{AC}$ ); wait( $S_{BC}$ ); 操作C; signal( $S_{CE}$ ); End
Begin 操作D; signal( $S_{DE}$ ); End
Begin wait( $S_{CE}$ ); wait( $S_{DE}$ ); 操作E; End
CoEnd

【分析】

本题是一个典型的利用信号量实现前驱关系的同步问题。首先画出各个操作之间的执行顺序图，如下图所示，可以看出，A、B、D 的执行不需要任何前提条件。执行完 A 和 B 之后才能执行 C，存在两对同步关系： $A \rightarrow C$ 和 $B \rightarrow C$ ，设置两个同步变量 $S_{AC} = 0$ 和 $S_{BC} = 0$ ，在完成 A 和 B 之后分别执行 signal( $S_{AC}$ ) 和 signal( $S_{BC}$ )，表示 A 或 B 已完成；在执行 C 之前需要执行 wait( $S_{AC}$ ) 和 wait( $S_{BC}$ )，检查 A 和 B 是否完成。执行完 C 和 D 之后才能执行 E，也存在两对同步关系： $C \rightarrow E$ 和 $D \rightarrow E$ ，因此再设置两个同步变量 $S_{CE} = 0$ 和 $S_{DE} = 0$ ，在完成 C 和 D 之后分别执行 signal( $S_{CE}$ ) 和 signal( $S_{DE}$ )，表示 C 或 D 已完成；在执行 E 之前需要执行 wait( $S_{CE}$ ) 和 wait( $S_{DE}$ )，检查 C 和 D 是否完成。

## 解题思路

### 第一步：绘制前驱图
```
A   B   D
 \ /   /
  C   /
   \ /
    E
```

### 第二步：识别跨进程依赖
- A→C, B→C, C→E, D→E（4对前驱关系）
- A、B、D可并发执行，无前驱

### 第三步：为每对前驱关系设置信号量
需要4个信号量：S_AC, S_BC, S_CE, S_DE（均初值0）

## 易错点分析

### 易错点1：漏设信号量导致同步不完整
❌ 错误做法：只设置S_AC和S_CE，认为B和C的关系可以用A代替。这会导致B完成前C就可能执行（只要A完成），违反"C必须在A和B完成后执行"的约束。
✅ 正确理解：每条跨进程的前驱边都需要独立的信号量。C有两个前驱A和B，必须wait两次（wait(S_AC)和wait(S_BC)），确保A和B都完成后C才执行。不同前驱不能合并。

### 易错点2：wait()操作顺序理解错误
❌ 错误理解：认为C中wait(S_AC)和wait(S_BC)的顺序必须与A、B的执行顺序一致，担心顺序错误会死锁。
✅ 正确理解：C中两个wait()的顺序任意，不影响正确性。wait()只是检查前驱是否完成，若未完成则阻塞。无论先wait哪个，只要两个前驱都未完成，C都会被阻塞；两个都完成后，C就能通过两个wait()继续执行。这与互斥锁不同，不会产生死锁。
