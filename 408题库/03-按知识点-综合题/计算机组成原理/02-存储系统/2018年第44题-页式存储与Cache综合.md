# 2018年第44题

**考点**：二级页表 | TLB | Cache | 虚拟地址转换

**题干**：
某计算机采用页式虚拟存储管理方式，按字节编址。CPU进行存储访问的过程如题44图所示。

题44图

根据题44图回答下列问题。

1）主存物理地址占多少位？

2）TLB采用什么映射方式？TLB是用SRAM还是用DRAM实现？

3) Cache 采用什么映射方式？若 Cache 采用 LRU 替换算法和回写 (Write Back) 策略，则 Cache 每行中除数据 (Data)、Tag 和有效位，还应有哪些附加位？Cache 的总容量是多少？Cache 中有效位的作用是什么?

4) 若 CPU 给出的虚拟地址为 0008 C040H，则对应的物理地址是多少？是否在 Cache 中命中？说明理由。若 CPU 给出的虚拟地址为 0007 C260H，则该地址所在主存块映射到的 Cache组号是多少？

**答案**：
## 44.【王道书3.6节】

## 1)【标准答案】

物理地址位数是 $20 + 3 + 5 = 28$ （或 $16 + 12 = 28$ ）。

## 【分析】

物理地址由实页号和页内地址拼接得到。

## 2)【标准答案】

TLB 中每项都有一个比较器，故 TLB 采用全相联映射方式。

TLB 采用 SRAM 实现。

## 【分析】

TLB 中每项都有一个比较器，故采用全相联映射，可以把页表内容调入任一块空 TLB 项中。TLB 采用 SRAM，读写速度快，但成本高，多用于容量较小的高速缓冲存储器。

## 3)【标准答案】

Cache 中每组有两行，故采用 2 路组相联映射方式。

因为是 2 路组相联并采用 LRU 替换算法，所以每行（或每组）需要 1 位 LRU 位；因为采用回写策略，所以每行有 1 位修改位（脏位）。

28 位物理地址中 Tag 字段占 20 位，组索引字段占 3 位，块内偏移地址占 5 位，故 Cache 共有 $2^{3} = 8$ 组，每组 2 行，每行有 $2^{5} = 32$ 字节；Cache 总容量为 $8 \times 2 \times (20 + 1 + 1 + 1 + 32 \times 8) = 4464$ 位 = 558 字节。

(或者 $8 \times 2 \times (20 + 1 + 1 + 32 \times 8) + 8 = 4456$ 位 = 557 字节)

有效位用来指出所在 Cache 行中的信息是否有效。

## 4)【标准答案】

虚拟地址 0008 C040H 对应的物理地址是 004 0040H。

访问 Cache 不命中。

理由为: 主存物理地址为 004 0040H, 其中高 20 位 00400H 为标志字段, 低 5 位 00000B 为块内偏移量, 中间 3 位 010B 为组号 2, 因此将 00400H 与 Cache 中第 2 组两行中的标志字段同时比较, 可以看出, 虽然有一个 Cache 行中的标志字段与 00400H 相等, 但对应的有效位为 0, 而另一 Cache 行的标志字段与 00400H 不相等, 故访问 Cache 不命中。

因为物理地址的低 12 位与虚拟地址低 12 位相同，即为 0010 0110 0000B，其中 011B 是组号（组索引），因此，该地址所在的主存块映射到的 Cache 组号为 3。

【分析】

虚拟地址分为两部分：虚页号、页内地址；物理地址分为两部分：实页号、页内地址。利用虚拟地址的虚页号部分去查找 TLB 表（缺失时从页表调入），将实页号取出后和虚拟地址的页内地址拼接，就形成了物理地址。
---

## 解题思路
物理地址=实页号(20位)+页内偏移(12位)=28位。TLB每项有比较器，全相联映射；速度要求快用SRAM。Cache每组2行，2路组相联；附加位包括LRU位(1位)、脏位(1位)、有效位(1位)。虚地址查TLB获得实页号，与页内偏移拼接成物理地址，再查Cache。有效位0表示Cache行内容无效，不命中。

## 易错点分析

### 易错点1：Cache命中判断忽略有效位
❌ 错误判断：物理地址004 0040H组号2，Tag 00400H与Cache第2组某行Tag匹配，认为命中。
✅ 正确理解：Tag匹配还需检查有效位。该行有效位=0表示内容无效(可能是初始状态或被替换)，不能命中。命中条件：Tag匹配且有效位=1。

### 易错点2：TLB和Cache映射方式混淆
| TLB全相联 | Cache组相联 | 区别 |
|-------|-------|------|
| 每项有比较器，虚页号可映射任意项 | 按组号索引，块只能映射到对应组 | TLB容量小速度要求高用全相联 |
| SRAM实现，速度快成本高 | 每组2行，组间直接映射组内全相联 | Cache容量大用组相联平衡成本和命中率 |
