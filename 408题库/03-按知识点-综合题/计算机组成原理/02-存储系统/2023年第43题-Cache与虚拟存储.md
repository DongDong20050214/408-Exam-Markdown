# 2023年第43题

**真题出处**：[2023年第43题](../../../01-真题原卷/2023年408真题.md#43)

**考点**：Cache | 虚拟存储 | 局部性原理 | 命中率

**题干**：
已知计算机M字长为32位，按字节编址，采用请求调页策略的虚拟存储管理方式，虚拟地址为32位，页大小为4KB；数据Cache采用4路组相联映射方式，数据区大小为8KB，主存块大小为32B。现有C语言程序段如下：

```c
int a[24][64];
......
for (i = 0; i < 24; i++)
    for (j = 0; j < 64; j++) a[i][j] = 10;
```

已知二维数组 a 按行优先存放, 在虚拟地址空间中分配的起始地址为 0042 2000H, sizeof(int) = 4, 假定在 M 上执行上述程序段之前数组 a 不在主存, 且在该程序段执行过程中不会发生页面置换。请回答下列问题。

1) 数组 a 分布在几个页面中？对于数组 a 的访问，会发生几次缺页异常？页故障地址各是什么？

2）不考虑变量j和i，该程序段的数据访问是否具有时间局部性？为什么？

3) 计算机 M 的虚拟地址（A31～A0）中哪几位用作块内地址？哪几位用作 Cache 组号？a[1][0]的虚拟地址是多少？其所在主存块对应的 Cache 组号是多少？

4) 数组 a 占用多少主存块？假设上述程序段执行过程中数组 a 的访问不会和其他数据发生 Cache 访问冲突，则数组 a 的 Cache 命中率是多少？若将循环中 i 和 j 的次序按如下方式调换：

```c
for (j = 0; j < 64; j++)
    for (i = 0; i < 24; i++) a[i][j] = 10;
```

则数组 a 的 Cache 命中率又是多少？

**答案**：
## 43.【王道书3.6节】

## 1)【标准答案】

数组a分布在2个页面中。缺页异常次数为2。两个页故障地址分别是0042 2000H和0042 3000H。

【分析】

数组 a 的起始地址为 00422000H，页大小为 4KB，页内偏移量占 12 位，数组 a 共有 $24 \times 64 = 1536$ 个元素，每个 int 型数据占 4 字节，因此数组 a 共占 $1536 \times 4B = 6KB$ ，故分布在 2 个相邻的页面中，页号分别为 00422H 和 00423H，当访问这两个页面的第一个数组元素的地址时，因为页面尚未调入内存，所以会发生 2 次缺页异常。

## 2)【标准答案】

该程序段的数据访问没有时间局部性。因为每个数组元素仅访问1次。

【分析】

若不考虑变量i和j，该程序段的数据访问只涉及对数组元素的访问，每个数组元素只访问一次，在不久的将来不会再次被访问，因此数据访问不存在时间局部性。

## 3)【标准答案】

虚拟地址中低 5 位（A4～A0）用作块内地址；低 11 位虚拟地址中高 6 位（A10～A5）用作 Cache 组号。

a[1][0]的虚拟地址为 $0042\ 2000H + 1 \times 64 \times 4 + 0 \times 4 = 0042\ 2100H$ 。

a[1][0]所在主存块对应的 Cache 组号为 001000B = 8。

【分析】

在组相联映射方式下，物理地址结构为（Tag 标记 - Cache 组号 - 块内地址），主存块大小为 32B，故块内地址占 5 位；Cache 数据区共有 8KB÷32B = 256 行，采用 4 路组相联，共有 64 组，故 Cache 组号占 6 位。虚拟地址为 32 位，页框大小为 4KB，虚拟地址的低 12 位表示页内偏移量，因此物理地址的低 12 位和虚拟地址的低 12 位相同。

【注意】本题给出的是虚拟地址，需要分析转化为物理地址后与 Cache 的映射关系，不能直接用虚拟地址和 Cache 映射，因此认为虚拟地址的高 21 位是 Tag 标记的想法是错误的，虽然对本题的结果没有影响，但真题多次考查过相关考点。

## 4)【标准答案】

数组 a 占 $24 \times 64 \times 4B \div 32B = 192$ 个主存块。每个主存块存放 $32B \div 4B = 8$ 个数组元素，访问数组 a 的 Cache 命中率为 $(8 - 1)/8 = 87.5\%$ 。8 行数组元素占 $8 \times 64 \times 4B \div 32B = 64$ 个主存块，分别映射到 64 个 Cache 组的某 Cache 行，数组 a 共有 24 行，因此每个 Cache 组中只有 24/8 = 3 个 Cache 行存放数组 a 中的数据，而每个 Cache 组有 4 行，因此不会发生替换，访问数组 a 的 Cache 命中率为 $7/8 = 87.5\%$ 。

## 解题思路

**问题1：页面分布与缺页**
数组a起始0042 2000H，大小24×64×4B=6KB，页大小4KB，占2个页面（00422H和00423H）。首次访问每页第一个元素触发缺页，共2次，页故障地址0042 2000H和0042 3000H。

**问题2：时间局部性判断**
每个数组元素仅访问1次，不会在近期再次访问，无时间局部性。（有空间局部性，访问连续地址）

**问题3：Cache地址映射**
块大小32B，块内地址5位（A4~A0）。Cache 8KB÷32B=256行，4路组相联64组，组号6位（A10~A5）。a[1][0]地址=0042 2000H+1×64×4=0042 2100H，组号=001000B=8。

**问题4：命中率计算**
数组a占192块，每块8个元素。按行访问每块首次缺失后续7次命中，命中率7/8=87.5%。按列访问跨行跳跃，但8行数据占64块映射到64组不冲突，命中率仍7/8=87.5%。

## 易错点分析

### 易错点1：用虚拟地址高20位做Cache标记
❌ 错误理解：虚拟地址32位，Cache组号6位块内地址5位，剩余21位做标记。
✅ 正确理解：虚拟地址需先转换为物理地址再映射Cache。页内偏移12位不变，物理地址低12位=虚拟地址低12位，Cache映射用物理地址低12位（组号6位+块内5位），标记用物理地址高位。

### 易错点2：按列访问命中率大幅下降
❌ 错误理解：按列访问a[0][0]→a[1][0]跨行，空间局部性差，命中率降至1/24或更低。
✅ 正确理解：虽然按列访问跨行，但8行元素占64块分散映射到64个不同组，4路组相联每组可存4块，不会发生替换冲突。每块仍是首次缺失后续命中，命中率不变。
