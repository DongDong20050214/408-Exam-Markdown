# 2020年第44题

**考点**：Cache组织 | 组相联映射 | 直写策略 | LRU算法

**题干**：
假定主存地址为32位，按字节编址，指令Cache和数据Cache与主存之间均采用8路组相联映射方式，直写（Write Through）写策略和LRU替换算法，主存块大小为64B，数据区容量各为32KB。开始时Cache均为空。请回答下列问题。

1）Cache 每一行中标记（Tag）、LRU 位各占几位？是否有修改位？

2) 有如下 C 语言程序段:
```c
for (k = 0; k < 1024 ; k++)
    s[k] = 2*s[k];
```
若数组 s 及其变量 k 均为 int 型，int 型数据占 4B，变量 k 分配在寄存器中，数组 s 在主存中的起始地址为 0080 00C0H，则该程序段执行过程中，访问数组 s 的数据 Cache 缺失次数为多少？

3）若CPU最先开始的访问操作是读取主存单元0001 0003H中的指令，简要说明从Cache中访问该指令的过程，包括Cache缺失处理过程。

**答案**：
## 44.【王道书3.5节】

## 1)【标准答案】

主存块大小为 $64 \mathrm{~B} = 2^{6}$ 字节, 故主存地址低 6 位为块内地址, Cache 组数为 $32 \mathrm{~KB} \div (64 \mathrm{~B} \times 8) = 64 = 2^{6}$ , 故主存地址中间 6 位为 Cache 组号, 主存地址中高 $32 - 6 - 6 = 20$ 位为标记, 采用 8 路组相联映射, 故每行中 LRU 位占 3 位, 采用直写方式, 故没有修改位。

## 2)【标准答案】

因为数组 s 的起始地址最后 6 位全为 0，故 s 位于一个主存块开始处，占 $1024 \times 4B \div 64B = 64$ 个主存块；执行程序段过程中，每个主存块中的 $64B \div 4B = 16$ 个数组元素依次读、写 1 次，因此对于每个主存块，总是第一次访问缺失，以后每次命中。综上，数组 s 的数据 Cache 访问缺失次数为 64 次。

## 【分析】

0080 00C0H = 0000 0000 1000 0000 0000 0000 1100 0000B，低6位为块内地址。

## 3)【标准答案】

0001 0003H = 0000 0000 0000 0001 0000 000000 000011B，根据主存地址划分可知，组索引为0，故该地址所在主存块被映射到指令 Cache 第 0 组；因为 Cache 初始为空，所有 Cache 行的有效位均为 0，所以 Cache 访问缺失。此时，将该主存块取出后存入指令 Cache 第 0 组的任意一行，并将主存地址高 20 位（00010H）填入该行标记字段，设置有效位，修改 LRU 位，最后根据块内地址 000011B 从该行中取出相应内容。

## 解题思路

**问题1：Cache行结构**
主存块64B，块内地址6位。8路组相联64组，组号6位。物理地址32位，标记=32-6-6=20位。LRU位=log₂8=3位（8路需3位编码最久未用）。直写策略无需修改位。

**问题2：数组访问缺失次数**
数组s起始0080 00C0H（块边界），占1024×4B÷64B=64块。每块16个元素，按行访问首次缺失后续命中。缺失次数=块数=64次。

**问题3：指令Cache缺失处理**
0001 0003H，组号=000000B=0，标记=00010H。Cache为空有效位0，缺失。取主存块存入第0组任意行，填入标记00010H，置有效位，更新LRU位，按块内地址000011B取指令。

## 易错点分析

### 易错点1：直写策略需要修改位
❌ 错误理解：直写策略每次写操作都写主存和Cache，需要修改位标记哪些行被写过。
✅ 正确理解：修改位（脏位）用于回写策略，标记Cache行数据是否被修改需写回主存。直写策略每次写同时更新Cache和主存，数据始终一致，无需脏位判断是否写回。

### 易错点2：LRU位数按组数计算
❌ 错误理解：64组需log₂64=6位LRU位，标记最久未使用的组。
✅ 正确理解：LRU位针对组内各路，标记组内哪一路最久未用。8路组相联需log₂8=3位编码8个Cache行的使用顺序。LRU位数=log₂(每组路数)，与组数无关。
