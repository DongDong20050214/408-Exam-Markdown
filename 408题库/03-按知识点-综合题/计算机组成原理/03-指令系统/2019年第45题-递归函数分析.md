# 2019年第45题

**考点**：递归函数 | 函数调用 | 汇编分析 | 溢出判断 | 大小端

**题干**：
已知 $f(n) = n! = n \times (n - 1) \times (n - 2) \times \cdots \times 2 \times 1$ ，计算 $f(n)$ 的C语言函数f1的源程序及其在32位计算机M上的部分机器级代码如下：

```c
int f1(int n) {
    if (n > 1)
        return n*f1(n-1);
    else return 1;
}
```

机器级代码：
```
100401000 55              push ebp
...
10040101018 83 7D 08 01   cmp dword ptr [ebp+8], 1
10040101C 7E 17           jle f1+35h (00401035)
10040101E 8B 45 08        mov eax, dword ptr [ebp+8]
100401021 83 E8 01        sub eax, 1
100401024 50              push eax
100401025 E8 D6 FF FF FF  call f1 (00401000)
...
100401030 0F AF C1        imul eax, ecx
100401033 EB 05           jmp f1+3Ah (0040103a)
100401035 B8 01 00 00 00  mov eax, 1
...
100401040 3B EC           cmp ebp, esp
...
10040104A C3              ret
```

请回答下列问题：
1）计算 $f(10)$ 需要调用函数f1多少次？执行哪条指令会递归调用f1？
2）在上述代码中，哪条指令是条件转移指令？哪几条指令一定会使程序跳转执行？
3) 根据第 16 行的 call 指令, 第 17 行指令的虚拟地址应是多少? 已知第 16 行的 call 指令采用相对寻址方式, 该指令中的偏移量应是多少 (给出计算过程)? 已知第 16 行的 call 指令的后 4 字节为偏移量, M 是采用大端方式还是采用小端方式?
4） $f(13) = 6227020800$ ，但 f1(13) 的返回值为 1932053504，为什么两者不等？要使 f1(13) 能返回正确的结果，应如何修改 f1 源程序？
5) 第 19 行 imul 指令（带符号整数乘）的功能是 R[eax]←R[eax]×R[ecx]，当乘法器输出的高、低 32 位乘积之间满足什么条件时，溢出标志 OF = 1？要使 CPU 在发生溢出时转异常处理，编译器应在 imul 指令后应加一条什么指令？

**答案**：
## 45.【王道书4.3节】

## 1)【标准答案】

计算 $f(10)$ 需要调用函数fl共10次。执行第16行call指令会递归调用fl。

## 2)【标准答案】

第12行jle指令是条件转移指令。第16行call指令、第20行jmp指令、第30行ret指令一定会使程序跳转执行。

## 【分析】

jle 指令是条件转移指令，其含义为小于或等于时跳转至目标地址。call 指令为函数调用指令、jmp 指令为无条件转移指令、ret 指令为子程序的返回指令。

## 3)【标准答案】

第 16 行 call 指令的下一条指令的地址为 0040 1025H + 5 = 0040 102AH，故第 17 行指令的虚拟地址是 0040 102AH。call 指令采用相对寻址方式，即目标地址=(PC)+偏移量，call 指令的目标地址为 0040 1000H，所以偏移量 = 目标地址 - (PC) = 0040 1000H - 0040 102AH = FFFF FFD6H。根据第 16 行 call 指令的偏移量字段为 D6 FF FF FF，可确定 M 采用小端方式。

## 【分析】

M 按字节编址，call 指令长度为 5 字节，故下一条指令的地址 = call 指令的地址 + 5。

## 4)【标准答案】

因为 $f(13) = 6227020800$ ，大于32位int型数据可表示的最大值，所以f1(13)的返回值是一个发生了溢出的结果。

为使 f1(13)能返回正确结果, 可将函数 f1 的返回值类型改为 double(或 long long 或 long double 或 float)。

## 5)【标准答案】

若乘积的高 33 位为非全 0 或非全 1，则 OF = 1。

编译器应该在imul指令后加一条“溢出自陷指令”，使得CPU自动查询溢出标志OF，当

OF = 1 时调出 “溢出异常处理程序”。

## 解题思路

### 第一步：分析递归调用层数
f(10)=10×f(9)=10×9×...×f(1)，递归终止条件n=1。从f(10)到f(1)共调用10次。call指令(第16行)实现递归调用，每次将n-1作为参数压栈后跳转到函数入口。

### 第二步：识别跳转指令类型
jle(第12行)是条件转移(小于等于时跳转)；call(16行)强制跳转到子函数；jmp(20行)无条件跳转；ret(30行)返回上层。后三者必然跳转，jle取决于比较结果。

### 第三步：计算相对寻址偏移量
call下一条指令地址=00401025H+5=0040102AH(call长5字节)。目标地址00401000H，偏移量=目标-PC=00401000H-0040102AH=FFFFFFD6H。机器码D6 FF FF FF，低位在前为小端方式。

### 第四步：分析溢出原因和解决方案
f(13)=6227020800>2^31-1(int上界2147483647)，乘法溢出导致结果错误。修改返回类型为long long或double等更大范围类型。高33位非全0/非全1时OF=1，加入溢出自陷指令检测OF并跳转异常处理。

## 易错点分析

### 易错点1：相对寻址基准地址理解错误
❌ 错误计算：偏移量=目标地址-call指令地址=00401000H-00401025H=FFFFFFDBH，与机器码D6不符。
✅ 正确理解：call执行时PC已指向下一条指令(取指后PC自增)。基准=(call地址+指令长度)=00401025H+5=0040102AH，偏移量=00401000H-0040102AH=FFFFFFD6H。相对寻址基于PC当前值(已更新)，不是指令首地址。

### 易错点2：溢出标志OF判断条件不完整
❌ 错误理解：带符号乘法结果超过32位时OF=1，即高32位不为0时溢出。
✅ 正确理解：高33位(含符号位)非全0且非全1时OF=1。正数溢出高33位为00...01(高32位有1)，负数溢出高33位为11...10(高32位有0)。全0表示正数未溢出，全1表示负数未溢出，其他情况均溢出。

### 易错点3：大小端方式判断依据
| 大端方式 | 小端方式 | 本题情况 |
|-------|-------|------|
| 高字节存低地址 | 低字节存低地址 | 偏移量FFFFFFD6H存为D6 FF FF FF |
| 符合人类读写习惯 | x86架构标准 | D6存最低地址，为低字节在前=小端 |
