# 2020年第47题

**考点**：NAT技术 | 路由配置 | IP地址配置 | 网络互联

**题干**：
某校园网有两个局域网，通过路由器R1、R2和R3互联后接入Internet，S1和S2为以太网交换机。局域网采用静态IP地址配置，路由器部分接口以及各主机的IP地址如下图所示。

![image](https://cdn-mineru.openxlab.org.cn/result/2026-06-09/61a7441a-e21c-435b-9476-47f9ff01a858/aa183189b890fda563c84a81ea49d1729df36e6da34649248df01079c66caba1.jpg)

假设NAT转换表结构为

| 外网 |  | 内网 |  |
|------|------|------|------|
| IP 地址 | 端口号 | IP 地址 | 端口号 |
|  |  |  |  |

请回答下列问题:

1）为使H2和H3能够访问Web服务器（使用默认端口号），需要进行什么配置？

2) 若 H2 主动访问 Web 服务器时, 将 HTTP 请求报文封装到 IP 数据报 P 中发送, 则 H2 发送 P 的源 IP 地址和目的 IP 地址分别是什么? 经过 R3 转发后, P 的源 IP 地址和目的 IP 地址分别是什么? 经过 R2 转发后, P 的源 IP 地址和目的 IP 地址分别是什么?

**答案**：
## 47.【王道书4.2节】

## 1)【标准答案】

需要静态配置R2的NAT，实现NAT穿透，具体配置为

<table><tr><td colspan="2">外网</td><td colspan="2">内网</td></tr><tr><td>IP地址</td><td>端口号</td><td>IP地址</td><td>端口号</td></tr><tr><td>203.10.2.2</td><td>80</td><td>192.168.1.2</td><td>80</td></tr></table>

## 【分析】

H2 和 H3 与 Web 服务器处于不同的局域网，路由器 R2、R3 具有 NAT 功能。当 R2 从 WAN 端口收到来自 H2 或 H3 发来的 HTTP 请求时，根据 NAT 表发送给 Web 服务器的对应端口。为使外部主机能正常访问 Web 服务器，应在 R2 的 NAT 表中增加一项，外网的 IP 地址配置为路由器的外部 IP 地址，内网的 IP 地址配置为 Web 服务器的 IP 地址，HTTP 的服务器端的默认端口号为 80，因此外网和内网的端口号都需配置为 80。只需在 R2 中配置 Web 服务器的 NAT 表项，而不用在 R3 中配置 H2 和 H3 的 NAT 表项，原因在于我们只知道 Web 服务器的端口号 80，而客户端的端口号是随机分配的，无法做静态配置，只能通过自动动态配置实现。

## 2)【标准答案】

H2 发送的 P 的源 IP 地址和目的 IP 地址分别是：192.168.1.2 和 203.10.2.2；R3 转发后，P 的源 IP 地址和目的 IP 地址分别是：203.10.2.6 和 203.10.2.2；R2 转发后，P 的源 IP 地址和目的 IP 地址分别是：203.10.2.6 和 192.168.1.2。

## 【分析】

由于启用了 NAT 服务，H2 发送的 P 的源 IP 地址是 H2 的内网地址，目的地址是 R2 的外网 IP 地址。R3 转发后，将 P 的源 IP 地址改为 R3 的外网 IP 地址，目的 IP 地址仍然不变。R2 转发后，将 P 的目的 IP 地址改为 Web 服务器的内网地址，源地址仍然不变。

## 解题思路

**问题1：NAT静态配置实现穿透**
H2和H3访问Web服务器需穿透NAT，在R2配置静态映射：外网IP=203.10.2.2，端口80映射到内网IP=192.168.1.2端口80。客户端端口随机无需配置。

**问题2：NAT地址转换过程**
- H2发送：源IP=192.168.1.2，目的IP=203.10.2.2
- R3转发：源IP=203.10.2.6（R3外网），目的IP=203.10.2.2
- R2转发：源IP=203.10.2.6，目的IP=192.168.1.2（转换为Web服务器内网地址）

## 易错点分析

### 易错点1：认为需要在R3配置H2和H3的NAT表项
❌ 错误理解：局域网内所有主机访问外网都需要在路由器配置静态NAT映射。
✅ 正确理解：仅需在R2配置Web服务器的静态映射实现NAT穿透。H2/H3作为客户端，R3动态分配端口自动建立NAT映射。静态配置用于外部访问内网服务器（端口固定），客户端使用动态NAT（端口随机）。

### 易错点2：认为R2转发后源IP地址也改变
❌ 错误理解：R2转发时将源IP改为Web服务器IP，目的IP改为H2的IP，实现双向地址转换。
✅ 正确理解：NAT仅转换目的IP地址203.10.2.2→192.168.1.2，将外网地址映射到内网服务器。源IP保持203.10.2.6不变（R3的外网地址），这样Web服务器回复时知道发给谁。NAT转换方向：入站改目的IP，出站改源IP。
