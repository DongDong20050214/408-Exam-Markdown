# 二叉排序树真题汇总

## 2013年第06题

**元数据**
- 年份：[2013年第6题](../../../01-真题原卷/2013年408真题.md#06)
- 科目：#数据结构
- 知识点：#数据结构/树/二叉排序树
- 难度：⭐⭐⭐
- 关联知识：[知识库/数据结构/二叉排序树](知识库/数据结构/二叉排序树.md)

**题干**

在任意一棵非空二叉排序树 T1 中, 删除某结点 v 之后形成二叉排序树 T2, 再将 v 插入 T2 形成二叉排序树 T3。下列关于 T1 与 T3 的叙述中, 正确的是 ( )。I. 若 v 是 T1 的叶结点，则 T1 与 T3 不同  II，若 v 是 T1 的叶结点，则 T1 与 T3 相同  III. 若 v 不是 T1 的叶结点，则 T1 与 T3 不同  IV. 若 v 不是 T1 的叶结点，则 T1 与 T3 相同

A. 仅 I、III  B. 仅 I、IV  C. 仅II、III  D. 仅 II、IV

**答案**：C

**考点分析**
- 叶结点删除后再插入位置相同，非叶结点不同

---

## 解题思路

### 第一步：理解题目核心

**操作流程**：
1. T1：原始二叉排序树
2. 删除结点v → T2
3. 将v插入T2 → T3
4. 比较T1和T3是否相同

**关键分析**：
- 删除操作：删除v后，v的位置需要被其他结点替代（如果v不是叶子）
- 插入操作：按BST规则重新找位置插入v
- 叶结点删除后再插入位置不变
- 非叶结点删除后被替代，再插入位置可能不同

### 第二步：分析两种情况

**情况1：v是叶结点**
```
示例：
T1:     5
       / \
      3   7
     /
    2

删除叶结点2 → T2:
        5
       / \
      3   7

插入2 → T3:（按BST规则，2<5左，2<3左）
        5
       / \
      3   7
     /
    2

结论：T1 = T3 ✓
```

**情况2：v不是叶结点**
```
示例：
T1:     5
       / \
      3   8
     / \ /
    2  4 7

删除3（有两个孩子，用后继4或前驱2替代）→ T2:
假设用后继4替代3的位置：
        5
       / \
      4   8
     /   /
    2   7

插入3 → T3:（按BST规则，3<5左，3<4左，3>2右）
        5
       / \
      4   8
     /   /
    2   7
     \
      3

结论：T1 ≠ T3（3的位置不同）
```

### 第三步：逐项分析

**I. 若v是T1的叶结点，则T1与T3不同 ❌**
```
详细分析：
叶结点删除：直接删除，不影响其他结点
再插入：按BST规则，仍会插入到原来的位置（因为路径相同）

判断依据：叶结点删除再插入位置不变
结论：❌ 错误（T1与T3相同）
```

**II. 若v是T1的叶结点，则T1与T3相同 ✅**
```
详细分析：
叶结点v的插入路径由其他结点决定
删除v后，其他结点位置不变
再插入v时，比较路径完全相同，最终到达原位置

判断依据：叶结点删除不改变树结构，插入路径不变
结论：✅ 正确
```

**III. 若v不是T1的叶结点，则T1与T3不同 ✅**
```
详细分析：
非叶结点v删除时，需要用其前驱或后继替代v的位置
替代后，v原位置被占用
再插入v时，v会按BST规则找新位置，不会回到原位置

判断依据：非叶结点删除会改变树结构，v的原位置被占
结论：✅ 正确
```

**IV. 若v不是T1的叶结点，则T1与T3相同 ❌**
```
详细分析：
同III的分析，非叶结点删除后再插入不会回到原位置

判断依据：与III矛盾
结论：❌ 错误
```

### 第四步：确认答案

II和III正确 → 答案：**C**

## 易错点分析

### 易错点1：认为所有结点删除再插入都回到原位置
❌ 错误理解：BST插入是确定性算法，删除结点v后再插入v一定回到原位置
✅ 正确理解：只有叶结点删除再插入才回到原位置。叶结点删除时树其他部分结构不变，再插入比较路径相同，回到原位。非叶结点删除时必须用前驱或后继替代，原位置被占用，再插入会找到新位置

### 易错点2：BST删除操作的三种情况
| 删除情况 | 处理方式 | 影响 |
|---------|---------|------|
| 叶结点 | 直接删除 | 树结构不变 |
| 一个孩子 | 孩子替代 | 局部调整 |
| 两个孩子 | 前驱/后继替代 | 原位置被占 |

## 知识点对比

| 结点类型 | 删除后树结构 | 再插入位置 | T1与T3关系 |
|---------|------------|-----------|-----------|
| 叶结点 ✅ | 不变（仅少一个叶子） | 原位置 | 相同 |
| 非叶结点 | 改变（被替代） | 新位置 | 不同 |

| 操作 | 叶结点示例 | 非叶结点示例 |
|------|-----------|-------------|
| 原树T1 | 5→3→2 | 5→3(2,4) |
| 删除v | 删2，得5→3 | 删3，得5→4(2,-) |
| 插入v | 插2，回到5→3→2 | 插3，到5→4(2,3) |
| 结果 | T1=T3 ✓ | T1≠T3 ✓ |

| 命题 | 正误 | 理由 |
|------|------|------|
| I. 叶结点→T1≠T3 | ❌ | 叶结点回到原位 |
| II. 叶结点→T1=T3 | ✅ | 正确 |
| III. 非叶→T1≠T3 | ✅ | 非叶不回原位 |
| IV. 非叶→T1=T3 | ❌ | 与III矛盾 |
