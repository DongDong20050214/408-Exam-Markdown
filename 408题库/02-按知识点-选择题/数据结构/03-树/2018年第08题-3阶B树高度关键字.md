**题干**

高度为 5 的 3 阶 B 树含有的关键字个数至少是（）。

A. 15  B. 31  C. 62  D. 242

**答案**：B

**考点分析**
- 3阶B树最少：根1个，第2层2个，第3-5层每节点1个子树
- 节点数：1+2+2×2+2×2×2+2×2×2×2=31？需重算关键字数

---

## 解题思路

### 第一步：理解题目核心

**3阶B树性质**：
- m=3，每个结点最多2个关键字，最多3个子树
- 根结点：1≤关键字数≤2
- 非根结点：⌈m/2⌉-1 ≤ 关键字数 ≤ m-1
  即：⌈3/2⌉-1 = 1 ≤ 关键字数 ≤ 2

**最少关键字情况**：
- 根：1个关键字
- 其他结点：1个关键字（最少）

### 第二步：计算最少关键字数

**按层分析（高度为5）**：

**第1层（根）**：
- 1个结点，1个关键字
- 2个子树

**第2层**：
- 2个结点（根的2个孩子）
- 每个1个关键字 → 2个关键字
- 每个2个子树 → 共4个子树

**第3层**：
- 4个结点
- 每个1个关键字 → 4个关键字
- 每个2个子树 → 共8个子树

**第4层**：
- 8个结点
- 每个1个关键字 → 8个关键字
- 每个2个子树 → 共16个子树

**第5层**：
- 16个结点
- 每个1个关键字 → 16个关键字

**总关键字数**：
```
1 + 2 + 4 + 8 + 16 = 31
```

### 第三步：逐项分析

**A. 15 ❌**
```
分析：15 = 1+2+4+8，只算到第4层
结论：❌ 少算一层
```

**B. 31 ✅**
```
详细分析：
按等比数列求和：
第h层结点数：2^(h-1)
总关键字数：Σ(i=1 to 5) 2^(i-1) = 2^5 - 1 = 31

结论：✅ 正确
```

**C. 62 ❌**
```
分析：62 = 2×31，可能误认为每个结点2个关键字
但题目求最少，应该每个结点1个关键字

结论：❌ 算多了
```

**D. 242 ❌**
```
分析：242远大于31，可能混淆了其他计算
结论：❌ 错误
```

### 第四步：确认答案

答案：**B**

## 易错点分析

### 易错点1：混淆结点数和关键字数
❌ 错误理解：计算结点总数就是答案，忘记每个结点可以有多个关键字。按层计算结点数：1+2+4+8+16=31，所以答案是31。但这只是结点数，题目问的是关键字数，应该用结点数乘以每个结点的关键字数。

✅ 正确理解：B树的关键字存储在结点中，每个结点可以存储多个关键字，题目明确问的是"关键字个数"不是"结点个数"。对于3阶B树，每个结点最少1个关键字（根和非根都是），最多2个关键字。求最少关键字数时，每个结点取最少值1个。高度为5时，按最少结点数布局：第1层1个结点，第2层2个结点（根的2个孩子），第3层4个，第4层8个，第5层16个，共31个结点。每个结点恰有1个关键字，所以总关键字数=结点数=31。此题中结点数和关键字数数值相等是因为"每个结点最少1个关键字"这一巧合，但概念上必须区分清楚。若问4阶B树最多关键字数，每结点最多3个，则关键字数≠结点数。

### 易错点2：高度和层数概念混淆
❌ 错误理解：高度5表示根下面5层或第0-5层共6层
✅ 正确理解：408默认高度h=层数，高度5表示有5层(第1层到第5层)。根是第1层，最底层是第5层
