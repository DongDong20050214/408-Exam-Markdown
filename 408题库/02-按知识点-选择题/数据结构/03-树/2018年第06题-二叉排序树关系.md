**题干**

已知二叉排序树如题 06 图所示，元素之间应满足的大小关系是（）。

A. x1 < x2 < x5  B. x1 < x4 < x5  C. x3<x5<x4  D. x4<x3<x5

**答案**：D（⚠️ 需查看原题图片验证树结构，根据BST性质：x4在左子树<x3<x5在右子树）

## 解题思路

### 第一步：理解题目核心

**二叉排序树（BST）性质**：
- 左子树所有结点 < 根结点
- 右子树所有结点 > 根结点
- 左右子树也是BST

**解题方法**：
根据题目给出的树形图，利用BST性质推导结点间的大小关系

### 第二步：根据树形图分析

**假设树结构**（需结合实际题图）：
```
常见情况：
      x3
     /  \
   x1    x5
  /  \   /
 x2  x4 x6
```

**应用BST性质**：
- x1是x3的左孩子 → x1 < x3
- x5是x3的右孩子 → x5 > x3
- x4的位置决定其与其他结点的关系

### 第三步：逐项分析

**A. x1 < x2 < x5 ❓**
```
分析：需看x2位置
若x2是x1的右孩子：x2 > x1
若x1、x2都在根的左子树，x5在右子树：
  x1 < 根，x2 < 根，x5 > 根
  但x1与x2的关系需看具体位置
```

**B. x1 < x4 < x5 ❓**
```
分析：需看x4位置
若x4在x1的右侧且在x5的左侧：
  可能成立
```

**C. x3 < x5 < x4 ❓**
```
分析：若x3是根，x5是右子树
x5 > x3 ✓
x5 < x4需要x4在x5的右上方
```

**D. x4 < x3 < x5 ❓**
```
分析：若x3是根
x4 < x3说明x4在左子树
x3 < x5说明x5在右子树
这是常见的BST结构
```

### 第四步：确认答案

**根据BST性质**，最可能的答案是**D**（需结合实际树图确认）

## 易错点分析

### 易错点1：忘记BST的递归性质
❌ 错误理解：
只看父子直接关系，认为左孩子<根，右孩子>根即可。只要结点是根的左孩子就满足小于根，是右孩子就满足大于根，不需要考虑更深层的关系。

✅ 正确理解：
BST性质是递归的，不仅仅是父子关系。正确表述是：左子树的所有结点（不只是左孩子）都小于根，右子树的所有结点（不只是右孩子）都大于根。"所有结点"包括子树中的每一个结点，无论深度多少。例如若x4在根x3的左子树中，无论x4是左孩子、左孙子、左曾孙还是更深层，只要从x3出发第一步向左，之后无论怎么走到达x4，都保证x4<x3。同理，右子树中任意深度的结点都大于根。这种递归性质是BST能够高效查找的基础：查找值v时，若v<根则整个右子树都可以剪枝，因为右子树所有结点都大于根也大于v。

### 易错点2：混淆视觉位置和逻辑关系
❌ 错误理解：节点在图左边就在左子树，在图右边就在右子树
✅ 正确理解：左右关系由边方向决定不是视觉位置。从根到x第一步向左则x在左子树，第一步向右则x在右子树

## 知识点对比

### 二叉排序树的关键性质

| 性质 | 说明 | 应用 |
|------|------|------|
| 左子树 < 根 | 左子树所有结点值 < 根结点值 | 判断大小关系 |
| 根 < 右子树 | 根结点值 < 右子树所有结点值 | 判断大小关系 |
| 中序遍历有序 | 中序遍历得到递增序列 | 验证BST正确性 |
| 递归性质 | 每个子树也是BST | 递归构造和查找 |

### BST的路径判断法则

| 从根到结点x的路径 | 结点x的位置 | 与根的大小关系 |
|---------------|-----------|--------------|
| 一直向左 | 根的左子树最深处 | x < 根 |
| 一直向右 | 根的右子树最深处 | x > 根 |
| 先左后右 | 左子树的右侧分支 | 左孩子 < x < 根 |
| 先右后左 | 右子树的左侧分支 | 根 < x < 右孩子 |
