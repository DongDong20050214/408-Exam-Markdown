# 二叉排序树真题汇总

## 2020年第05题

**元数据**
- 年份：[[../../01-真题原卷/2020年408真题#05|2020年第5题]]
- 科目：#数据结构
- 知识点：#数据结构/树/二叉排序树 #BST构造
- 难度：⭐⭐
- 关联知识：[[知识库/数据结构/二叉排序树]]

**题干**

下列给定的关键字输入序列中，不能生成右侧二叉排序树的是（）。

A. 4, 5, 2, 1, 3  B. 4, 5, 1, 2, 3  C. 4, 2, 5, 3, 1  D. 4,2,1,3,5

**答案**：B

**解题思路**

题目给出一棵目标二叉排序树（需要根据题目图示），要找出哪个序列**不能**生成该树。

由于没有图示，我们假设目标树的结构，根据答案B反推：

**典型的目标树结构**（猜测）：
```
      4
     / \
    2   5
   / \
  1   3
```

**逐项验证**：

**A. 4, 5, 2, 1, 3**
1. 插入4：根节点
2. 插入5：5>4，放右边
3. 插入2：2<4，放左边
4. 插入1：1<4→左，1<2→左
5. 插入3：3<4→左，3>2→右

生成树：
```
      4
     / \
    2   5
   / \
  1   3
```
✅ 可以生成

**B. 4, 5, 1, 2, 3**
1. 插入4：根节点
2. 插入5：5>4，放右边
3. 插入1：1<4，放左边
4. 插入2：2<4→左，2>1→右
5. 插入3：3<4→左，3>1→右，3>2→右

生成树：
```
      4
     / \
    1   5
     \
      2
       \
        3
```
❌ 与目标树不同（2不是4的直接左孩子）

**C. 4, 2, 5, 3, 1**
1. 插入4：根节点
2. 插入2：2<4，放左边
3. 插入5：5>4，放右边
4. 插入3：3<4→左，3>2→右
5. 插入1：1<4→左，1<2→左

生成树：
```
      4
     / \
    2   5
   / \
  1   3
```
✅ 可以生成

**D. 4, 2, 1, 3, 5**
1. 插入4：根节点
2. 插入2：2<4，放左边
3. 插入1：1<4→左，1<2→左
4. 插入3：3<4→左，3>2→右
5. 插入5：5>4，放右边

生成树：
```
      4
     / \
    2   5
   / \
  1   3
```
✅ 可以生成

**选项分析**：
- ✅ A. 4, 5, 2, 1, 3 —— 可以生成目标树
- ❌ B. 4, 5, 1, 2, 3 —— 不能生成目标树（2的位置错误）
- ✅ C. 4, 2, 5, 3, 1 —— 可以生成目标树
- ✅ D. 4, 2, 1, 3, 5 —— 可以生成目标树

**易错点分析**

❌ **错误1：忘记二叉排序树的插入规则**
- 误认为可以任意调整节点位置
- ✅ 正确：每个节点插入时，按大小比较，唯一确定位置（小于向左，大于向右）

❌ **错误2：没有逐步模拟插入过程**
- 仅凭直觉判断，容易出错
- ✅ 正确：必须按序列顺序逐个插入，每步都按BST规则

❌ **错误3：混淆中序遍历和插入序列**
- 误以为中序遍历序列就是某种插入序列
- ✅ 正确：中序遍历是1,2,3,4,5，但不能直接作为插入序列判断依据

**知识点对比**

| 特性 | 二叉排序树（BST） | 说明 |
|------|-----------------|------|
| **构造唯一性** | 由插入序列唯一决定 | 不同序列可能生成不同的树 |
| **插入规则** | 小于根→左子树，大于根→右子树 | 递归应用 |
| **中序遍历** | 得到有序序列 | 所有BST的中序遍历都是升序 |
| **查找效率** | 平衡时O(log n)，退化时O(n) | 取决于树的形态 |

**插入顺序对树形态的影响**

| 插入序列类型 | 生成树形态 | 示例 |
|------------|-----------|------|
| **有序序列** | 退化为链表（单支树） | 1,2,3,4,5 → 右斜树 |
| **逆序序列** | 退化为链表（单支树） | 5,4,3,2,1 → 左斜树 |
| **中间元素先插** | 较平衡 | 4,2,5,1,3 → 平衡树 |
| **随机序列** | 一般情况 | 取决于具体顺序 |

**本题各序列生成的树对比**

| 序列 | 树的结构特点 | 是否符合目标 |
|------|------------|------------|
| A. 4,5,2,1,3 | 4为根，2-5分列，2下有1-3 | ✅ 符合 |
| B. 4,5,1,2,3 | 4为根，5在右，1-2-3右斜链在左 | ❌ 不符合 |
| C. 4,2,5,3,1 | 4为根，2-5分列，2下有1-3 | ✅ 符合 |
| D. 4,2,1,3,5 | 4为根，2-5分列，2下有1-3 | ✅ 符合 |

**BST构造过程可视化（序列B）**

```
步骤1: 4
       4

步骤2: 4,5
       4
        \
         5

步骤3: 4,5,1
       4
      / \
     1   5

步骤4: 4,5,1,2
       4
      / \
     1   5
      \
       2

步骤5: 4,5,1,2,3
       4
      / \
     1   5
      \
       2
        \
         3
```

最终结构：2是1的右孩子，而目标树中2是4的左孩子 → 不匹配

**关键理解**：
- BST的形态完全由插入序列决定
- 第一个插入的元素必然是根节点
- 之后每个元素的位置由其与已有节点的大小关系唯一确定
- 同一组数据，不同插入顺序会生成不同形态的BST

## 易错点补充

### 易错点4：未意识到第一个元素必是根
❌ 错误理解：认为根可能是序列中任意元素
✅ 正确理解：BST构造时，第一个插入的元素必然成为根结点

### 易错点5：混淆相同元素集合不同序列的树形态
❌ 错误理解：相同的5个数字一定生成相同的树
✅ 正确理解：插入顺序不同，树形态可能完全不同

## 知识点对比表

| 序列特点 | 第一个元素 | 树形态 | 示例 |
|---------|-----------|--------|------|
| 根先插 | 中间值 | 较平衡 | 4,2,5,1,3 |
| 升序 | 最小值 | 右斜链 | 1,2,3,4,5 |
| 降序 | 最大值 | 左斜链 | 5,4,3,2,1 |

| 判断方法 | 步骤 | 关键点 |
|---------|------|--------|
| 模拟插入法 | 逐个插入，比较大小 | 小左大右，唯一路径 |
| 根结点检查 | 第一个元素是否为目标树的根 | 不是则直接排除 |
| 子树递归 | 左子树元素<根，右子树元素>根 | 递归验证每个子树 |
