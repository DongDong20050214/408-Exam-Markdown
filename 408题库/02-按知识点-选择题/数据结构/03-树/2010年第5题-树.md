# 2010年第5题

**元数据**
- 年份：2010年第5题
- 科目：#数据结构
- 难度：⭐⭐

**题干**

在一棵度为 4 的树 T 中，若有 20 个度为 4 的结点，10 个度为 3 的结点，1 个度为 2 的结点，10 个度为 1 的结点，则树 T 的叶结点个数是（）。

A. 41  
B. 82  
C. 113  
D. 122

**答案**：B

**考点分析**

本题考查树的基本性质：节点数与边数的关系。

关键公式：在树中，边数 = 节点总数 - 1，且边数 = 各节点度数之和。

计算过程：
- 边数 = 20×4 + 10×3 + 1×2 + 10×1 = 80 + 30 + 2 + 10 = 122
- 节点总数 = 边数 + 1 = 122 + 1 = 123
- 叶结点数（度为0的结点）= 总节点数 - 度不为0的节点数
- 叶结点数 = 123 - (20 + 10 + 1 + 10) = 123 - 41 = 82

该题考查树的度与节点数量关系的计算，是树的基本性质的重要应用，需要熟练掌握边数与度数的关系。

---

## 解题思路

### 第一步：理解题目核心

**树的基本性质**：
- 边数 = 节点总数 - 1（树的连通性）
- 边数 = 所有结点的度数之和（每条边连接一个父结点和一个子结点）
- 叶结点 = 度为0的结点

**题目条件**：
- 度为4的树（每个结点最多4个孩子）
- 度为4的结点：20个
- 度为3的结点：10个
- 度为2的结点：1个
- 度为1的结点：10个
- 求：叶结点（度为0）个数

### 第二步：逐项分析

**计算过程**：

**步骤1：计算边数**
```
边数 = 各结点度数之和
= 20×4 + 10×3 + 1×2 + 10×1 + 叶结点数×0
= 80 + 30 + 2 + 10 + 0
= 122条边
```

**步骤2：计算总结点数**
```
在树中：边数 = 总结点数 - 1
总结点数 = 边数 + 1 = 122 + 1 = 123个
```

**步骤3：计算叶结点数**
```
总结点数 = 度为4的 + 度为3的 + 度为2的 + 度为1的 + 叶结点
123 = 20 + 10 + 1 + 10 + 叶结点数
叶结点数 = 123 - 41 = 82个
```

**A. 41 ❌**
```
详细分析：
41 = 20 + 10 + 1 + 10（非叶结点总数）

判断依据：这是度不为0的结点总数，不是叶结点数
结论：❌ 错误，混淆了非叶结点和叶结点
```

**B. 82 ✅**
```
详细分析：
按照正确计算过程：
边数 = 122
总结点数 = 123
叶结点数 = 123 - 41 = 82

判断依据：符合树的基本性质计算
结论：✅ 正确答案
```

**C. 113 ❌**
```
详细分析：
113 无法通过合理的树性质推导出来
可能错误：113 ≈ 总结点数123 - 某个误算值

判断依据：不符合任何合理的计算方法
结论：❌ 错误
```

**D. 122 ❌**
```
详细分析：
122 = 边数
这是把边数当成了叶结点数

判断依据：混淆了边数和叶结点数的概念
结论：❌ 概念错误
```

### 第三步：确认答案

叶结点数 = 82 → 答案：**B**

## 易错点分析

### 易错点1：混淆边数和结点数
❌ 错误理解：边数 = 结点数，或叶结点数 = 边数
✅ 正确理解：边数 = 结点数 - 1（树的基本性质）

| 概念 | 定义 | 本题数值 |
|------|------|---------|
| 边数 | 所有结点度数之和 | 122 |
| 总结点数 | 边数 + 1 | 123 |
| 叶结点数 | 度为0的结点数 | 82 |

### 易错点2：把非叶结点数当成叶结点数
❌ 错误计算：叶结点数 = 20+10+1+10 = 41
✅ 正确理解：41是**非叶结点数**，叶结点数 = 总数 - 非叶数 = 123 - 41 = 82

### 易错点3：忘记"边数=度数之和"的关系
❌ 错误想法：认为边数需要另外计算或通过层数推导
✅ 正确理解：每个结点的度 = 它作为父结点连出的边数，所以**所有度数之和 = 边数**

```
示例验证：
度为4的结点 → 连出4条边
度为1的结点 → 连出1条边
所有结点的度数之和 = 所有边的总数
```
