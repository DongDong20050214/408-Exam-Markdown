# 2012年第3题

**元数据**
- 年份：2012年第3题
- 难度：⭐⭐

**题干**

若一棵二叉树的前序遍历序列为 $a, e, b, d, c$ ，后序遍历序列为 $b, c, d, e, a$ ，则根结点的孩子结点（）。

A. 只有 $e$ 

B. 有 $e, b$ 

C. 有 $e$ 、 $c$ 

D. 无法确定

**答案**：A

**考点分析**

本题考查根据前序和后序遍历序列确定二叉树结构。前序遍历首个元素a是根结点，后序遍历最后一个元素a也是根结点。前序第二个元素e是根的左子树的根（如果存在左子树），后序倒数第二个元素e对应前序中的e。通过分析可知，e是a的唯一孩子。因为如果a有两个孩子，后序序列倒数第二个应该是右子树的根，但e在前序中紧跟a，说明e是左孩子且a没有右孩子。

---

## 解题思路

### 第一步：标识根节点

**前序遍历**：a, e, b, d, c（根-左-右）
**后序遍历**：b, c, d, e, a（左-右-根）

前序首元素 = 后序尾元素 = **a（根节点）**

### 第二步：确定a的子树结构

**前序规律**：根后面紧跟左子树的根（若存在）
- 前序：**a**, e, b, d, c
- a后面是e，说明e是a的左孩子或a的唯一孩子

**后序规律**：根前面是右子树的根（若存在右子树）
- 后序：b, c, d, **e**, a
- a前面是e，说明e是最后处理的子树根

**关键判断**：
- 如果a有左右两个孩子，前序应为：a, 左根, ..., 右根, ...
- 后序应为：..., 左根, ..., 右根, a
- 当前后序中e在a前一个位置，前序中e在a后一个位置
- 说明**只有一个子树，根为e**

### 第三步：验证树结构

构建树：
```
      a
      |
      e
     / \
    b   d
        |
        c
```

验证：
- 前序（根左右）：a → e → b → d → c ✓
- 后序（左右根）：b → c → d → e → a ✓

**a的孩子：只有e**

### 第四步：逐项分析

**A. 只有e ✅**
- 正确，a只有一个孩子e

**B. 有e, b ❌**
- b是e的孩子，不是a的孩子

**C. 有e, c ❌**
- c是d的孩子，不是a的孩子

**D. 无法确定 ❌**
- 可以唯一确定

## 易错点分析

### 易错点1：误认为前序第二个就是左孩子
❌ 错误理解：前序a后面是e，所以e是左孩子，可能还有右孩子
✅ 正确理解：需结合后序判断。如果a有右孩子，后序中a前面应该是右孩子的根，但这里是e，说明只有一个子树

### 易错点2：混淆孩子和子孙
❌ 错误理解：b、d、c都在a的子树中，都是a的孩子
✅ 正确理解：孩子是直接连接的下一层节点，b、c、d是a的子孙，但只有e是a的直接孩子

### 易错点3：仅凭前序或后序判断
❌ 错误理解：只看前序，认为无法确定
✅ 正确理解：前序+后序可以确定树结构（虽然不如中序+前序/后序准确，但本题可以确定）

## 知识点对比

| 遍历组合 | 能否唯一确定树 | 本题应用 |
|---------|--------------|---------|
| **前序+中序** | ✓ 可以 | 标准方法 |
| **后序+中序** | ✓ 可以 | 标准方法 |
| **前序+后序** | △ 部分可以 | 本题恰好可以确定 |

| 判断依据 | 前序特征 | 后序特征 | 结论 |
|---------|---------|---------|------|
| **只有左孩子** | 根后紧跟左孩子 | 左孩子在根前 | 本题情况 |
| **只有右孩子** | 根后紧跟右孩子 | 右孩子在根前 | - |
| **左右都有** | 根后先左后右 | 先左后右最后根 | - |

**前序+后序的局限性**：无法区分某些只有单孩子的情况（左or右），但本题通过序列可以确定。
