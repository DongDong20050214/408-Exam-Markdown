# 完全二叉树真题汇总

## 2009年第05题

**元数据**
- 年份：[../../01-真题原卷/2009年408真题#05|2009年第5题](../../01-真题原卷/2009年408真题#05|2009年第5题.md)
- 科目：#数据结构
- 知识点：#数据结构/树/完全二叉树
- 难度：⭐⭐⭐
- 关联知识：[知识库/数据结构/二叉树](知识库/数据结构/二叉树.md)

**题干**

已知一棵完全二叉树的第 6 层（设根为第 1 层）有 8 个叶结点，则该完全二叉树的结点个数最多是（）。

A. 39  B. 52  C. 111  D. 119

**答案**：C

**考点分析**
- 第6层8个叶节点，第5层满
- 前5层：2^5-1=31
- 第6层最多：64
- 总数：31+64=95？需重算

## 解题思路（逐项分析法）

### 第一步：明确题目条件

- 完全二叉树，**根为第1层**
- 第6层有 **8个叶结点**
- 求：该完全二叉树的结点个数**最多**是多少
- 选项：A.39  B.52  C.111  D.119

**关键前提——完全二叉树的层结点上限**（根为第1层）：
```
第1层：最多 2^(1-1) = 2^0 = 1 个
第2层：最多 2^(2-1) = 2^1 = 2 个
第3层：最多 2^(3-1) = 2^2 = 4 个
第4层：最多 2^(4-1) = 2^3 = 8 个
第5层：最多 2^(5-1) = 2^4 = 16 个
第6层：最多 2^(6-1) = 2^5 = 32 个
第7层：最多 2^(7-1) = 2^6 = 64 个
```

### 第二步：分析"最多"的含义

**完全二叉树的叶结点分布规则**：叶结点只能出现在最后两层（最深两层）。如果第k层有叶结点，则树最多延伸到第k+1层。

本题第6层已有8个叶结点 → 树**可能**延伸到第7层。

要获得**最多**结点数，需要：
1. 第6层尽可能满（所有位置都有结点）→ 32个
2. 第6层中非叶结点尽可能多（让它们在第7层生儿育女）→ 第6层8个叶结点，其余 32−8=24 个是非叶结点
3. 第7层每个非叶结点的孩子尽可能多 → 每个有2个孩子，共 24×2=48 个
4. 第7层所有结点都是叶结点（没有第8层）

### 第三步：逐项判断

**A. 39 ❌**

```
计算：39 ≈ 31(前5层满) + 8(第6层8个)
这种理解：认为第6层只有那8个叶结点，没有其余结点。
但完全二叉树的定义：如果第6层有8个叶结点，
则第6层必然是从左到右连续填充的。8个叶结点可能
只占第6层的一部分（前8个位置），也可能分散。
但无论如何，39远小于最大可能值。

验证：若总共39个，则第6层最多 39−31=8 个结点。
这相当于第6层仅有8个结点（全为叶结点），树到第6层为止。
这是"最少"情况的候选，不是"最多"。

结论：❌ 是最少情况，不是最多
```

**B. 52 ❌**

```
计算：52 = 31(前5层) + 21(第6层部分?)
没有明确的推导路径。52 ≈ 31+21，21不是2的幂，也不符合完全二叉树的填充规律。

若第6层部分填充，则总节点在32~63之间（6层完全二叉树）。
但题目要求"最多"，应延伸到第7层。

结论：❌ 无合理推导，且不是最大值
```

**C. 111 ✅**

```
推导过程：
步骤1：前5层满 → 2^5−1 = 31 个结点
步骤2：第6层满（32个），其中8个是叶结点 → 非叶结点 = 32−8 = 24 个
步骤3：这24个非叶结点各生2个孩子 → 第7层最多 24×2 = 48 个结点
步骤4：总数 = 31(前5层) + 32(第6层) + 48(第7层) = 111

验证合理性：
- 第6层32个结点从左到右排列
- 8个叶结点必然在最右侧（完全二叉树叶结点在最后两层的最右侧）
- 左侧24个非叶结点各有2个孩子在第7层
- 第7层48个结点全部为叶结点
- 满足完全二叉树定义 ✅

结论：✅ 最大结点数 = 111
```

**D. 119 ❌**

```
常见错误推导：
119 = 2^7 − 1 − 8 = 127 − 8

错误逻辑：认为完全二叉树共7层满（127个结点），
然后减去第7层缺少的8个结点（因为第6层有8个叶结点）。

问题分析：
- 第6层有8个叶结点 → 这8个结点没有孩子
- 第7层缺少的结点数 = 8个叶结点的孩子数 = 8×2 = 16个（如果第6层全满）
- 满二叉树到第7层 = 127，减去16 = 111，不是119！
- 119的错误在于：减8而不是减16（忘记每个叶结点意味着失去2个孩子）

正确逻辑：
- 第6层满(32)，8个叶结点失去 8×2=16 个第7层结点
- 满7层=127，127−16=111

结论：❌ 错误地将"8个叶结点"理解为"第7层缺8个结点"，忽略了一对二关系
```

### 第四步：确认答案

最大结点数 = 31（1-5层满）+ 32（第6层满）+ 48（第7层：24个非叶父结点×2）= **111**

答案：**C**

## 易错点分析

### 陷阱1：混淆"最多"和"最少"的叶结点含义

❌ **错误理解**：第6层只有8个结点（全部是叶结点），直接计算 31+8=39。

✅ **正确理解**：「第6层有8个叶结点」≠「第6层只有8个结点」。第6层可以有32个结点（满的），其中8个恰好是叶结点（位于最右侧），其余24个有孩子。为了求**最大值**，应让第6层尽可能满，非叶结点尽可能多。

| 理解方式 | 第6层结点数 | 第7层结点数 | 总数 | 对应选项 |
|----------|-----------|-----------|------|---------|
| ❌ 错误（最少倾向） | 8个（全是叶） | 0 | 39 | A |
| ✅ 正确（最大化） | 32个（8叶+24非叶） | 48 | 111 | C |

### 陷阱2：忘记"每个叶结点失去两个后代"

❌ **错误计算**：满7层127个，减去8个缺失 = 119 → 选D。

✅ **正确计算**：第6层8个叶结点，每个缺少2个第7层孩子 = 缺16个第7层结点。满7层127 − 16 = 111。

关键关系：
```
1个第k层叶结点 → 失去 2 个第(k+1)层结点（该结点在满二叉树中应有的2个孩子）
n个第k层叶结点 → 失去 2n 个第(k+1)层结点
```

### 陷阱3：根结点层号混淆

❌ **错误理解**：默认根为第0层，则第6层最多 2^6=64 个结点，推导出 31+64+... → 混乱。

✅ **正确理解**：题目明确「设根为第1层」，则第k层最多 2^(k−1) 个结点。
- 根=第0层时：第6层最多 64
- 根=第1层时：第6层最多 32

**验证**：若按根=第0层计算 → 前6层满=63，第6层64个（8叶+56非叶），第7层112个 → 总数 63+64+112=239，不在选项中。所以必然根=第1层。

## 知识点对比

| 对比项 | 满二叉树（Full） | 完全二叉树（Complete） | 普通二叉树 |
|--------|-----------------|----------------------|-----------|
| 定义 | 每层都满，所有叶结点在同一层 | 除最后一层外每层满，最后一层从左到右连续填充 | 无特殊限制 |
| 第k层结点数 | 2^(k−1) | ≤2^(k−1) | ≤2^(k−1) |
| 叶结点分布 | 全在最深层 | 仅分布在最后1~2层 | 任意层均可 |
| 结点数范围（h层） | 精确 2^h−1 | [2^(h−1), 2^h−1] | [h, 2^h−1] |
| 本题适用 | ❌ 第6层有叶则不是满二叉树 | ✅ 正是完全二叉树 | — |

| 对比项 | 求最大结点数 | 求最小结点数 |
|--------|------------|------------|
| 策略 | 树尽可能延伸，每层尽可能满 | 树尽早终止 |
| 本题场景 | 延伸到第7层，第6层满 | 止于第6层，第6层仅8个 |
| 第6层处理 | 32个全满（含8叶） | 仅8个（全为叶） |
| 第7层 | 48个（全部为叶） | 不存在 |
| 结果 | 111（选项C） | 39（选项A） |

**记忆技巧**：完全二叉树求「最多」结点的口诀——「满到不能再满」：前几层全满、当前层满中让非叶结点最多、下一层能生尽生。求「最少」则相反——「省到不能再省」：当前层刚好满足叶结点数、后面一层也不要。
