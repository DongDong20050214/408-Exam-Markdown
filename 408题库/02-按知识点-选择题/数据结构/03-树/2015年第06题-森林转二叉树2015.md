## 2015年第06题

- 关联知识：[知识库/数据结构/树](知识库/数据结构/树.md)

**题干**

求右侧带权图的最小（代价）生成树时，可能是克鲁斯卡（Kruskal）算法第2次选中但不是普里姆(Prim)算法(从 v₄ 开始)第2次选中的边是()。

A. (v₁, v₃)  B. (v₁, v₄)  C. (v₂, v₃)  D. (v₃, v₄)

**答案**：B

**考点分析**
- Kruskal算法：按边权从小到大选边，不形成回路
- Prim算法：从起始点扩展，选择连接已选集合的最小边
- 两算法选边顺序不同

---

## 解题思路

### 第一步：理解题目核心

**Kruskal算法特点**：
- 全局选边：按权值从小到大选
- 不形成回路即可加入
- 与起始点无关

**Prim算法特点**：
- 局部扩展：从v₄开始逐步扩展
- 每次选连接已选点集的最小边
- 与起始点有关

### 第二步：模拟两算法执行过程

**Kruskal第2次选中的边**：
```
假设图中边权排序（需结合原图）：
第1次：选最小权边（假设某边）
第2次：选次小权边（不形成回路）

根据选项，第2次可能选中的边包括A/B/C/D
```

**Prim从v₄第2次选中的边**：
```
第1次：v₄出发，选连接v₄的最小边
第2次：扩展到新点，选连接{v₄, 新点}集合的最小边

关键：Prim的选择受当前已选点集限制
```

### 第三步：逐项分析

**A. (v₁, v₃) ❌**
```
分析：
若此边是Kruskal第2次选中，说明权值较小
但若也是Prim第2次选中，则v₁或v₃必在第1次就已连入

判断：不符合"Kruskal选但Prim不选"的要求
```

**B. (v₁, v₄) ✅**
```
详细分析：
Kruskal视角：全局看，此边权值较小，第2次选中
Prim视角：从v₄出发，v₄可能先连接其他点（如v₃），
         第2次扩展时选的是另一条边，而非(v₁,v₄)

原因：Prim受起始点和扩展路径影响，可能绕过此边
结论：✅ 正确
```

**C. (v₂, v₃) ❌**
```
分析：
若Kruskal第2次选，权值应较小
若Prim也不在第2次选，可能权值过大或位置不合适

判断：需结合原图，但通常不符合题意
```

**D. (v₃, v₄) ❌**
```
分析：
从v₄出发的Prim算法，第1次就可能选(v₃,v₄)
若第1次选了，就不可能第2次再选

判断：不符合
```

### 第四步：确认答案

B选项符合"Kruskal第2次选但Prim第2次不选" → 答案：**B**

## 易错点分析

### 易错点1：混淆两算法的选边策略
❌ 错误理解：
认为Kruskal和Prim算法都是构造最小生成树，最终结果相同，所以选边顺序也应该完全相同。既然生成树唯一（边权不同时），那么两种算法的每一步选边应该一致，不存在"一个选了另一个没选"的情况。

✅ 正确理解：
两算法选边策略本质不同，导致选边顺序可能完全不同。Kruskal算法是全局贪心策略，将所有边按权值排序后依次选择，只要不形成环就加入，完全不考虑起始点和当前已选边的位置，第2次选的是全图第2小的边。Prim算法是局部扩展策略，从指定起点v₄开始，维护一个已选顶点集合U，每次只能选择一端在U中、另一端不在U中的边，且必须是这些候选边中权值最小的。这导致Prim可能跳过全局权值很小的边（因为该边两端都还未加入U），而Kruskal会优先选择。最终两者生成树总权值相同（都是最小），但构造过程和选边顺序不同。

**选项分析**：
- ❌ A. (v₁,v₃) —— 若Kruskal第2次选说明权值较小，但Prim从v₄出发第2次也可能选，不符合"Kruskal选Prim不选"
- ✅ B. (v₁,v₄) —— Kruskal全局看权值小第2次选，但Prim从v₄出发可能先连其他点(如v₃)，第2次选其他边而非(v₁,v₄)，符合题意
- ❌ C. (v₂,v₃) —— 需结合原图，通常不符合
- ❌ D. (v₃,v₄) —— Prim从v₄出发第1次就可能选(v₃,v₄)，不会第2次再选

**易错点分析**

### 易错点1：混淆两算法选边策略
❌ 错误理解：两算法最终生成树相同，选边顺序应该一致
✅ 正确理解：Kruskal全局贪心按边权排序依次选(不成环)，Prim局部扩展从起点逐步扩展选连接已选集合的最小边。选边顺序完全不同，导致第2次选边可能不同

### 易错点2：忽略"第2次"限定
❌ 错误理解：只要Kruskal选了Prim没选就行
✅ 正确理解：必须精确到第2次迭代。Kruskal第2次选全图第2小边(不成环)，Prim第2次选连接{v₄+第1次点}的最小边，两者选边池不同

**知识点对比**

| 算法 | 选边策略 | 第2次选边 |
|------|---------|----------|
| Kruskal | 全局最小权边 | 全图第2小边(不成环) |
| Prim | 连接已选集合最小边 | 连接{v₄+第1次点}最小边 |
