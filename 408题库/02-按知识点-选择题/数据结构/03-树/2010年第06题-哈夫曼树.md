# 2010年第6题

**元数据**
- 年份：2010年第6题
- 科目：#数据结构
- 难度：⭐⭐

**题干**

对 $n (n \geqslant 2)$ 个权值均不相同的字符构造哈夫曼树。下列关于该哈夫曼树的叙述中，错误的是（）。

A. 该树一定是一棵完全二叉树  
B. 树中一定没有度为 1 的结点  
C. 树中两个权值最小的结点一定是兄弟结点  
D. 树中任一非叶结点的权值一定不小于下一层任一结点的权值

**答案**：A

**考点分析**

本题考查哈夫曼树（最优二叉树）的性质。

各选项分析：
- A错误：哈夫曼树不一定是完全二叉树。完全二叉树要求节点从上到下、从左到右连续排列，但哈夫曼树的构造只保证带权路径长度最小，不保证形态是完全二叉树。
- B正确：哈夫曼树的构造过程每次合并两个节点，因此所有非叶节点度数都是2，不存在度为1的节点。
- C正确：构造过程中，每次选择权值最小的两个节点合并为兄弟节点。
- D正确：由于每次选最小权值合并，父节点权值是子节点权值之和，因此上层节点权值不小于下层节点。

该题考查哈夫曼树的多个重要性质，需要深入理解其构造过程和结构特征。

## 解题思路（逐项分析法）

### 第一步：明确题目条件
- n≥2个权值均不相同的字符
- 构造哈夫曼树（最优二叉树）
- 找出**错误**的叙述

### 第二步：回顾哈夫曼树的核心性质
1. 构造过程：每次选两个最小权值合并，新结点权值为两者之和
2. 所有非叶结点度为2，没有度为1的结点
3. 两个最小权值结点最先合并，必为兄弟
4. 父结点权值 = 子结点权值之和 ≥ 子结点权值
5. 哈夫曼树**不一定是完全二叉树**

### 第三步：逐项判断

**A. 该树一定是一棵完全二叉树 ❌（错误）**
```
完全二叉树要求：除最后一层外每层满，最后一层从左到右连续。
哈夫曼树的形态由权值分布决定，不一定满足此要求。

反例：权值 {1, 2, 3, 4}
构造：合并1+2=3 → {3,3,4} → 合并3+3=6 → {4,6} → 合并4+6=10
树形：根(10)左=6右=4；6的左=3右=3；3的左=1右=2
最后一层：1,2,4 不连续（中间缺位）
结论：不是完全二叉树 ❌ 这就是错误选项
```

**B. 树中一定没有度为1的结点 ✅（正确）**
```
哈夫曼树每次合并两个结点产生一个新结点
新结点有两个孩子 → 度为2
原始结点要么是叶子（度0），要么参与合并成为某结点的孩子
不存在只有一个孩子的结点 → 没有度为1的结点
结论：正确 ✅
```

**C. 树中两个权值最小的结点一定是兄弟结点 ✅（正确）**
```
构造第一步：选两个最小权值合并
它们成为同一个父结点的左右孩子 → 兄弟关系
结论：正确 ✅
```

**D. 树中任一非叶结点的权值一定不小于下一层任一结点的权值 ✅（正确）**
```
因为每次选最小权值合并，越先被合并的结点在越底层
父结点权值 = 左子 + 右子 ≥ 每个子结点
所以非叶结点权值 ≥ 其子树中任意结点权值
包括下一层的所有结点
结论：正确 ✅
```

### 第四步：确认答案
A是唯一错误的叙述，哈夫曼树不一定是完全二叉树。

答案：**A**

## 易错点分析

### 陷阱1：混淆哈夫曼树与完全二叉树
❌ **错误理解**：哈夫曼树要求带权路径长度最小，看起来"紧凑"，所以是完全二叉树。
✅ **正确理解**：哈夫曼树只保证WPL最小，不约束形态。实际上哈夫曼树形态完全由权值序列决定，可以是极不平衡的。

### 陷阱2：认为"任一非叶结点权值≥下一层"是错的
❌ **错误想法**：子结点权值之和可能大于某一层的其他结点
✅ **正确理解**：父结点权值=左右子之和≥每个子结点。但"下一层任一个"不是指同一父结点的兄弟，而是指树中同一深度的任意结点。由于构造的贪心性质，上层结点的组成元素（原始权值）不会小于下层结点，所以此性质成立。

### 陷阱3：误以为最小权值结点在最后一层
❌ 初学哈夫曼树时容易以为越小的权值越深
✅ 权值最小的结点确实深度最大（这是哈夫曼编码的性质），但"一定是兄弟"是因为它们在第一步就被合并

## 知识点对比

| 对比项 | 哈夫曼树 | 完全二叉树 | 二叉排序树 |
|--------|---------|-----------|-----------|
| 构造依据 | 权值与WPL最小 | 层序连续填充 | 关键字大小 |
| 度为1的结点 | 无 | 可能（最后一层） | 可能 |
| 叶结点分布 | 任意位置 | 最后两层 | 任意位置 |
| 形态确定性 | 不唯一（等权值） | 结点数确定则唯一 | 插入顺序决定 |
| 是否一定是完全二叉树 | ❌ 不是 | ✅ 是 | ❌ 不是 |

**记忆口诀**：哈夫曼树三必有——无度为1、最小权兄弟、父权≥子权；哈夫曼树一不保证——不一定完全二叉树。
