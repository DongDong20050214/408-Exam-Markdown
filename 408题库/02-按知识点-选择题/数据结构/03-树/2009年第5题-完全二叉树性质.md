# 2009年第5题

**元数据**
- 年份：[../../01-真题原卷/2009年408真题#5|2009年第5题](../../01-真题原卷/2009年408真题#5)
- 科目：#数据结构
- 难度：⭐⭐

**题干**

已知一棵完全二叉树的第 6 层（设根为第 1 层）有 8 个叶结点，则该完全二叉树的结点个数最多是（）。

A. 39 

B. 52 

C. 111 

D. 119

**答案**：C

**考点分析**
- 本题考查完全二叉树的性质和结点个数计算。完全二叉树除最后一层外，其他层都是满的，最后一层从左到右连续排列。
- 第6层有8个叶结点，说明这8个结点没有子结点。要使总结点数最多，需要让第6层尽可能多的结点有子结点（即第7层尽可能满）。
- 第6层最多有$2^5=32$个结点，其中8个是叶结点，则有24个结点可以有子结点。前5层共有$2^0+2^1+...+2^4=31$个结点，第6层32个，第7层最多48个（24×2），总计$31+32+48=111$个。
- 需要理解完全二叉树的性质和层次结构的计算方法。

---

## 解题思路

### 第一步：理解题目核心

**题目条件**：
- 完全二叉树，根为第1层
- 第6层有8个叶结点
- 求结点个数**最多**是多少

**完全二叉树性质**：
- 除最后一层外，每层都满
- 最后一层从左到右连续填充
- 叶结点只能出现在最后两层

### 第二步：逐项分析

**A. 39 ❌**
```
详细分析：
39 = 31(前5层满) + 8(第6层)
这种理解认为第6层只有8个结点（全是叶子），树止于第6层

判断依据：
前5层满：2^0 + 2^1 + ... + 2^4 = 2^5 - 1 = 31
第6层：8个结点（全为叶子）
总数：31 + 8 = 39

问题：这是"最少"情况（树尽早终止），不是"最多"
题目要求最多，应让树延伸到第7层

结论：❌ 是最少结点数，不是最多
```

**B. 52 ❌**
```
详细分析：
52 无法通过合理的完全二叉树结构推导出来
52 = 31 + 21，但21既不是2的幂，也不符合完全二叉树的填充规律

可能的错误推导：
- 误将第6层部分填充理解为21个结点
- 但完全二叉树第6层要么少于32个止于此层，要么满32个延伸到第7层

判断依据：无法通过完全二叉树的性质推导出52
结论：❌ 不合理
```

**C. 111 ✅**
```
详细分析：
要使结点数最多，策略是：
1. 第6层尽可能满 → 32个结点
2. 第6层中非叶结点尽可能多 → 32 - 8 = 24个非叶结点
3. 这24个非叶结点在第7层各生2个孩子 → 24 × 2 = 48个
4. 第7层所有结点都是叶结点

计算过程：
前5层满：2^5 - 1 = 31个
第6层满：32个（其中8个叶子，24个非叶）
第7层：24 × 2 = 48个（全为叶子）
总数：31 + 32 + 48 = 111

验证完全二叉树性质：
- 第6层满（从左到右连续）✓
- 8个叶子在第6层最右侧（完全二叉树叶子在最后两层）✓
- 24个非叶结点在左侧，各有2个孩子在第7层 ✓
- 第7层从左到右连续填充48个结点 ✓

判断依据：符合完全二叉树定义，且结点数最大化
结论：✅ 最多111个结点
```

**D. 119 ❌**
```
详细分析：
常见错误推导：
119 = 127 - 8 = 2^7 - 1 - 8

错误逻辑：
认为满二叉树到第7层有127个结点
第6层有8个叶结点，意味着第7层"缺少"8个结点
所以总数 = 127 - 8 = 119

错误原因：
第6层的1个叶结点 → 失去2个第7层孩子（不是1个）
8个叶结点 → 失去 8 × 2 = 16个第7层结点
正确计算：127 - 16 = 111（不是119）

判断依据：混淆了"叶结点数"和"缺失的下层结点数"的对应关系（1:2）
结论：❌ 计算错误
```

### 第三步：确认答案

最大结点数 = 31 + 32 + 48 = **111** → 答案：**C**

## 易错点分析

### 易错点1：混淆"最多"和"最少"
❌ 错误理解：第6层有8个叶结点 = 第6层只有8个结点
✅ 正确理解：第6层可以有最多32个结点，其中8个恰好是叶子，其余24个是非叶结点，可以在第7层有孩子

| 理解方式 | 第6层 | 第7层 | 总数 | 类型 |
|---------|------|------|------|------|
| 最少（树尽早终止） | 8个（全叶） | 0 | 39 | 选项A |
| 最多（树延伸最远） | 32个（8叶+24非叶） | 48 | 111 | 选项C ✅ |

### 易错点2：忘记"1个叶结点失去2个后代"
❌ 错误计算：满7层127个，减去8个叶结点 = 119
✅ 正确计算：1个叶结点 → 失去2个孩子，8个叶结点 → 失去16个第7层结点

```
关键关系：
叶结点 = 没有孩子的结点
在满二叉树中，1个第k层结点本应有2个第(k+1)层孩子
若该结点是叶子 → 失去2个孩子
n个叶结点 → 失去2n个下层结点
```

### 易错点3：根结点层号混淆
❌ 错误：默认根为第0层，则第6层最多64个结点
✅ 正确：题目明确"根为第1层"，第k层最多2^(k-1)个结点

| 编号方式 | 第1层 | 第6层最多 | 前5层总数 | 前6层总数 |
|---------|------|----------|----------|----------|
| 根=第0层 | 2^0=1 | 2^6=64 | 2^6-1=63 | 2^7-1=127 |
| 根=第1层 ✅ | 2^0=1 | 2^5=32 | 2^5-1=31 | 2^6-1=63 |

## 知识点对比

| 树类型 | 叶结点分布 | 第k层结点数 | 高度h的结点总数 |
|-------|-----------|------------|----------------|
| 满二叉树 | 全在最深层 | 2^(k-1) | 2^h - 1 |
| 完全二叉树 | 最后1~2层 | ≤2^(k-1) | [2^(h-1), 2^h-1] |
| 普通二叉树 | 任意层 | ≤2^(k-1) | [h, 2^h-1] |

| 策略 | 树的延伸 | 当前层处理 | 下一层 | 结果 |
|------|---------|-----------|--------|------|
| 求最多 | 尽可能延伸 | 尽可能满 | 非叶结点各生2孩子 | 111 |
| 求最少 | 尽早终止 | 刚好满足叶结点数 | 不存在 | 39 |
