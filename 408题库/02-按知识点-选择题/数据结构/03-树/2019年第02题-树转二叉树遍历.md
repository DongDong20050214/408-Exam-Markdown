# 树转二叉树真题汇总

## 2019年第02题

**元数据**
- 年份：[[../../01-真题原卷/2019年408真题#02|2019年第2题]]
- 科目：#数据结构
- 知识点：#数据结构/树/树转二叉树
- 难度：⭐⭐
- 关联知识：[[知识库/数据结构/树]]

**题干**

若将一棵树 T 转化为对应的二叉树 BT，则下列对 BT 的遍历中，其遍历序列与 T 的后根遍历序列相同的是（）。

A. 先序遍历  B. 中序遍历  C. 后序遍历  D. 按层遍历

**答案**：B

**解题思路**

**树与二叉树的转换规则**（左孩子右兄弟表示法）：
- 树的第一个孩子 → 二叉树的左孩子
- 树的下一个兄弟 → 二叉树的右孩子

**遍历对应关系**：

| 树的遍历 | 对应二叉树的遍历 |
|---------|----------------|
| **先根遍历** | 先序遍历 |
| **后根遍历** | 中序遍历 |

**选项分析**：
- ❌ A. 先序遍历 —— 对应树的先根遍历
- ✅ B. 中序遍历 —— 对应树的后根遍历
- ❌ C. 后序遍历 —— 无直接对应
- ❌ D. 按层遍历 —— 无直接对应

**易错点分析**

❌ **错误1：选择A选项**
- 混淆先根遍历和后根遍历的对应关系
- ✅ 正确：树先根=二叉树先序，树后根=二叉树中序

❌ **错误2：认为后根对应后序**
- 按名称联想，认为"后根"对应"后序"
- ✅ 正确：遍历对应关系由转换规则决定，不是名称对应

❌ 错误2：按名称类比，认为"后根"对应"后序"
按"后"字类比，直觉认为树的"后根遍历"应该对应二叉树的"后序遍历"，因为名字里都有"后"字。这种基于名称相似性的联想忽略了算法本质。

✅ 正确理解：
遍历对应关系由"左孩子右兄弟"转换规则决定，不是名称对应。树的后根遍历是"先访问所有子树，最后访问根"，转换后的二叉树中，第一个孩子变成左子树，兄弟节点变成右子树，所以访问顺序变成"左子树（第一个孩子及其后代）→根→右子树（兄弟节点）"，这正是中序遍历的定义。名称相似不代表算法对应，必须从结构转换规则推导遍历对应关系。推导方法：画出具体例子，标注转换前后的访问顺序，观察模式。记忆口诀：树后根=二叉中序（都是"根在中间"），树先根=二叉先序（都是"根在最先"）。

❌ 错误3：不理解"左孩子右兄弟"转换的深层含义
死记硬背"树后根对应二叉中序"，不理解为什么是这样的对应关系。遇到变式题目（如森林转二叉树、反向推导）就无法应对。

✅ 正确理解：
左孩子右兄弟表示法的本质是用二叉树结构编码多叉树的"纵向关系"（父子）和"横向关系"（兄弟）。左指针存储第一个孩子（纵向关系），右指针存储下一个兄弟（横向关系）。推导遍历对应：树的后根遍历"子树1→子树2→...→根"，转换后第一个孩子（子树1）变成左子树，其余兄弟（子树2、3...）通过右指针串联成右侧链，所以遍历变成"左子树（子树1）→根→右子树（子树2的链）"即中序。树的先根遍历"根→子树1→子树2→..."，对应"根→左子树→右子树"即先序。关键理解：原树的"所有子树"被拆成"左子树（第一个孩子）+右子树（兄弟链）"两部分。

**知识点对比**

| 概念 | 树 | 对应二叉树 |
|------|----|---------  |
| **先根遍历** | 根→子树1→子树2→... | 先序遍历（根→左→右） |
| **后根遍历** | 子树1→子树2→...→根 | 中序遍历（左→根→右） |

**示例验证**

树T：
```
    A
   /|\
  B C D
  |
  E
```

**树的先根遍历**：A, B, E, C, D
**树的后根遍历**：E, B, C, D, A

**转换为二叉树BT**（左孩子右兄弟）：
```
    A
   /
  B
 / \
E   C
     \
      D
```

**二叉树BT的遍历**：
- 先序：A, B, E, C, D ✓ 与树的先根相同
- 中序：E, B, C, D, A ✓ 与树的后根相同
- 后序：E, D, C, B, A
- 层次：A, B, C, E, D

**对应关系推导**

树的后根遍历：先访问所有子树，最后访问根
- 对应二叉树：先访问左子树（对应第一个孩子及其子树），再访问根，再访问右子树（对应兄弟节点）
- 这正是中序遍历：左→根→右

**转换规则可视化**

```
树：          二叉树：
  A            A
 /|\          /
B C D   →    B
             / \
            E   C
                 \
                  D

B的孩子E → B的左孩子
B的兄弟C → B的右孩子  
C的兄弟D → C的右孩子
```

**关键理解**：
- 左指针：指向第一个孩子（纵向关系）
- 右指针：指向下一个兄弟（横向关系）
- 后根遍历：子树遍历完→根
- 中序遍历：左子树→根→右子树，与后根顺序一致

## 易错点补充

### 易错点4：不理解左孩子右兄弟的含义
❌ 错误理解：随意转换，不遵循"左孩子右兄弟"规则
✅ 正确理解：
- 左指针：指向该结点的第一个孩子
- 右指针：指向该结点的下一个兄弟

### 易错点5：混淆树的度和二叉树的度
❌ 错误理解：认为树的度大于2就不能转为二叉树
✅ 正确理解：任何树都可以通过左孩子右兄弟法转为二叉树
