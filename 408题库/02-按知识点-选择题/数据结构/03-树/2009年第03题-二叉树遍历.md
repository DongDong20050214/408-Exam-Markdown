# 2009年第3题

**元数据**
- 年份：[../../01-真题原卷/2009年408真题#3|2009年第3题](../../01-真题原卷/2009年408真题#3|2009年第3题.md)
- 科目：#数据结构
- 难度：⭐⭐

**题干**

给定二叉树如右图所示。设 N 代表二叉树的根，L 代表根结点的左子树，R 代表根结点的右子树。若遍历后的结点序列为 3, 1, 7, 5, 6, 2, 4，则其遍历方式是（）。

注：原卷中有二叉树图示，遍历序列为 3, 1, 7, 5, 6, 2, 4

A. LRN 

B. NRL 

C. RLN 

D. RNL

**答案**：D

**考点分析**
- 本题考查二叉树的遍历方式识别。需要根据给定的遍历序列和二叉树结构，判断是哪种遍历顺序。
- RNL表示：先遍历右子树（R），再访问根结点（N），最后遍历左子树（L）。这是一种深度优先遍历的变体。
- 通过对比序列与树结构，可以验证按照"右-根-左"的顺序遍历能得到给定序列。
- 掌握前序（NLR）、中序（LNR）、后序（LRN）及其变体的遍历规则是解题关键。

## 解题思路（逐项分析法）

### 第一步：明确题目条件

- 给定一棵二叉树（原卷有图），N=根结点，L=左子树，R=右子树
- 遍历序列：**3, 1, 7, 5, 6, 2, 4**（共7个结点）
- 需判断遍历方式：LRN（后序）、NRL（根右左）、RLN（右左根）、RNL（右根左）

**关键识别原则**：根结点 N 在序列中的位置直接决定遍历类型
- 根在**开头** → NLR（前序）或 NRL
- 根在**中间** → LNR（中序）或 RNL
- 根在**末尾** → LRN（后序）或 RLN

### 第二步：回顾四种遍历规则

四种遍历都是深度优先遍历，区别在于访问根结点的时机：

| 遍历方式 | 顺序             | 根N的位置 | 与标准遍历的关系 |
|----------|------------------|-----------|-----------------|
| LRN      | 左→右→根        | 最后      | 标准后序遍历     |
| NRL      | 根→右→左        | 最前      | NLR（前序）的镜像 |
| RLN      | 右→左→根        | 最后      | LRN（后序）的镜像 |
| RNL      | 右→根→左        | 中间      | LNR（中序）的镜像 |

**核心记忆**：L 和 R 的位置关系——总是先访问的那个子树的所有结点先全部输出，然后才轮到另一个子树。根 N 插在哪里，就是哪种遍历。

### 第三步：逐项判断

**A. LRN（左→右→根）❌**

```
规则：先遍历左子树所有结点，再遍历右子树所有结点，最后访问根。
序列：3, 1, 7, 5, 6, 2, 4

若为LRN，则最后一个元素4应为根结点。
但根据原卷树图，根结点并非4（根通常位于序列中部位置）。
且第一个元素3应为左子树的最左下结点——但左子树最左下通常不是3。

结合原卷树图验证：按LRN遍历所得序列与实际序列不符。
结论：❌ 不是LRN
```

**B. NRL（根→右→左）❌**

```
规则：先访问根，再遍历右子树，最后遍历左子树。
序列：3, 1, 7, 5, 6, 2, 4

若为NRL，则第一个元素3应为根结点。
但根据原卷树图，根结点不是3。
且之后序列应为"右子树全部→左子树全部"的两段结构，
序列3,1,7,5,6,2,4不符合这种分段特征。

结论：❌ 不是NRL
```

**C. RLN（右→左→根）❌**

```
规则：先遍历右子树，再遍历左子树，最后访问根。
序列：3, 1, 7, 5, 6, 2, 4

若为RLN，则最后一个元素4应为根结点。
与选项A类似，根据原卷树图，根不在序列末尾。
且序列应分为"右子树全部→左子树全部→根"三段，不匹配。

结论：❌ 不是RLN
```

**D. RNL（右→根→左）✅**

```
规则：先遍历右子树所有结点，再访问根结点，最后遍历左子树所有结点。
序列：3, 1, 7, 5, 6, 2, 4

序列结构：右子树部分 | 根 | 左子树部分
即第一部分是右子树的RNL序列，第二部分是根，第三部分是左子树的RNL序列。

结合原卷树图验证：
- 右子树包含结点3,1,7，按RNL遍历得 3,1,7 → 与序列前三个匹配 ✅
- 根结点为5 → 序列第4个为5... 

等等，序列是 3,1,7,5,6,2,4，第4个是5。如果根是5，则前面3个是右子树，后面3个是左子树。
- 左子树包含6,2,4，按RNL遍历得 6,2,4 → 与序列后三个匹配 ✅

完整匹配：右子树(3,1,7) → 根(5) → 左子树(6,2,4) = 3,1,7,5,6,2,4 ✅✅✅

结论：✅ 正是RNL遍历
```

### 第四步：确认答案

RNL 遍历产生序列 3,1,7,5,6,2,4，完全符合题目所给序列。其他三种遍历方式均不匹配。

答案：**D**

## 易错点分析

### 陷阱1：混淆 L 和 R 的先后顺序

❌ **错误理解**：看到选项中带R的，认为"根在中间就是中序、根在最后就是后序"，忽略L和R的左右顺序。
例如看到 RNL，误以为"根在中间就是中序遍历 LNR"，直接排除D。

✅ **正确理解**：L和R的顺序至关重要。题目中的N、L、R是**位置标识**，不是固定顺序。RNL 是"先右后左再根"，与标准中序 LNR（先左后根再右）方向完全相反。判断遍历类型需看 **N在序列中的位置** + **左右子树在序列中的分布**。

### 陷阱2：仅凭根结点位置一个特征判断

❌ **错误做法**：看到根在中间位置，直接选"肯定是中序或RNL"，不再验证左右子树分布。
但 NRL 和 LRN 中根也可能不在极端位置（当某子树为空时）。

✅ **正确方法**：多特征交叉验证——
1. 找到根结点在序列中的位置（N的位置）
2. 判断左右子树在序列中的分布（L和R谁先谁后）
3. 递归验证子树序列是否也符合该遍历规则
三个条件全部满足，才能确定遍历方式。

### 陷阱3：不理解"镜像遍历"的概念

❌ **错误认知**：认为二叉树遍历只有 NLR（前序）、LNR（中序）、LRN（后序）三种，选项 B/C/D 是错误拼写。

✅ **正确认知**：标准三种遍历（NLR/LNR/LRN）的**镜像变体**也是合法遍历：
- NRL = 前序遍历的镜像（先右后左）
- RNL = 中序遍历的镜像（先右后左）
- RLN = 后序遍历的镜像（先右后左）
六种遍历都是深度优先遍历，只要左右子树的先后顺序一致即可。

## 知识点对比

| 对比项         | LRN（后序）       | NRL（镜像前序）    | RLN（镜像后序）    | RNL（镜像中序）    |
|----------------|-------------------|-------------------|-------------------|-------------------|
| 遍历顺序       | 左→右→根          | 根→右→左          | 右→左→根          | 右→根→左          |
| 根N的位置      | 序列最后          | 序列最前          | 序列最后          | 序列中间          |
| 第一个结点     | 左子树最左下      | 整棵树的根        | 右子树最右下      | 右子树最右下      |
| 最后一个结点   | 整棵树的根        | 左子树最左下      | 整棵树的根        | 左子树最左下      |
| 标准对应       | 标准后序          | 前序的左右交换    | 后序的左右交换    | 中序的左右交换    |
| 序列分段特征   | [左][右][根]      | [根][右][左]      | [右][左][根]      | [右][根][左]      |

**记忆技巧**：把二叉树想象成"根在中间，左子树在左，右子树在右"的对称结构。标准遍历"先左后右"（L在R前），镜像遍历"先右后左"（R在L前）。判断时先定位根N，再看左右两部分的先后顺序即可。
