**题干**

下列函数的时间复杂度是（）。A. O(log n)  B. O(n^1/2)  C. O(n)  D. O(nlog n)

**答案**：B

**解题思路**

**代码分析**（推测）：
```c
int sum = 0;
for (int i = 1; sum <= n; i++) {
    sum += i;
}
```

**循环条件**：sum ≤ n

**计算循环次数**：
- 第i次循环后：sum = 1+2+3+...+i = $\frac{i(i+1)}{2}$
- 循环终止条件：$\frac{i(i+1)}{2} > n$
- 即：$i^2 + i > 2n$
- 忽略低阶项：$i^2 \approx 2n$
- 解得：$i \approx \sqrt{2n} = O(\sqrt{n})$

**选项分析**：
- ❌ A. O(log n) —— 太小
- ✅ B. O(n^{1/2}) —— 即O(√n)，正确
- ❌ C. O(n) —— 太大
- ❌ D. O(nlog n) —— 太大

**易错点分析**

❌ **错误1：认为循环n次，选C**
- 误以为i从1到n
- ✅ 正确：sum累加到n，不是i到n

❌ **错误2：认为是O(log n)**
- 混淆几何级数与算术级数
- ✅ 正确：算术级数求和，复杂度O(√n)

**知识点对比**

| 循环条件 | 复杂度 | 示例 |
|---------|--------|------|
| i<n | O(n) | `for(i=0;i<n;i++)` |
| sum<n, sum+=i | O(√n) | 本题 |
| i*=2 | O(log n) | `for(i=1;i<n;i*=2)` |
| i*i<n | O(√n) | `for(i=1;i*i<n;i++)` |

**算术级数求和**

$$\sum_{i=1}^{k} i = \frac{k(k+1)}{2} \approx \frac{k^2}{2}$$

当和达到n时：
$$\frac{k^2}{2} = n \Rightarrow k = \sqrt{2n} = O(\sqrt{n})$$

**关键理解**：
- 循环变量i，但判断条件是sum
- sum是i的二次函数
- 要达到n需要√n次循环

## 解题思路

### 第一步：理解题目核心
考查累加循环的时间复杂度计算，关键在于识别循环变量与终止条件的关系（sum累加到n，而非i递增到n）。

### 第二步：逐项分析
- ❌ A. O(log n)：对数复杂度适用于每次循环规模减半（如二分查找），本题是等差数列累加
- ✅ B. O(n^1/2)：等差数列前k项和≈k²/2，当和达到n时k≈√n
- ❌ C. O(n)：若i直接递增到n才是O(n)，本题sum更快达到n
- ❌ D. O(nlog n)：典型的归并排序复杂度，本题无嵌套对数结构

### 第三步：确认答案
通过等差数列求和公式推导：k(k+1)/2 > n → k ≈ √(2n)，时间复杂度为O(√n) = O(n^1/2)。

## 易错点分析

### 易错点1：误判循环次数
❌ 错误理解：看到循环变量i就认为执行n次，选C
✅ 正确理解：判断条件是sum≤n而非i<n，需分析sum与i的增长关系（二次vs线性）

### 易错点2：复杂度类型混淆
| 复杂度 | 典型特征 | 代码示例 |
|-------|----------|---------|
| O(√n) | 二次累加到n / 平方条件 | sum+=i直到sum>n / i*i<n |
| O(log n) | 指数增长变量 | i*=2 / i=i<<1 |
| O(n) | 线性递增到n | for(i=0;i<n;i++) |
