# 顺序表真题汇总

## 2023年第01题

**元数据**
- 年份：[../../01-真题原卷/2023年408真题#01|2023年第1题](../../01-真题原卷/2023年408真题#01|2023年第1题.md)
- 科目：#数据结构
- 知识点：#数据结构/线性表/顺序表 #时间复杂度 #有序表 #随机访问
- 难度：⭐⭐ (中等)
- 关联知识：[知识库/数据结构/顺序表基础](知识库/数据结构/顺序表基础.md) [2019年第01题-时间复杂度分析](..\00-算法基础\2019年第01题-时间复杂度分析.md)

**题干**

下列对顺序存储的有序表（长度为 $n$ ）实现给定操作的算法中，平均时间复杂度为 $O(1)$ 的是（）。

A. 查找包含指定值元素的算法

B. 插入包含指定值元素的算法

C. 删除第 $i$ ( $1 \leqslant i \leqslant n$ ) 个元素的算法

D. 获取第 $i$ ( $1 \leqslant i \leqslant n$ ) 个元素的算法

**答案**：D

**考点分析**
- 考察点1：顺序表的随机访问特性（支持O(1)时间的按位置访问）
- 考察点2：各种基本操作的时间复杂度分析（查找、插入、删除、访问）
- 考察点3：有序表查找可使用折半查找优化到O(log n)，但仍不是O(1)
- 考察点4：插入和删除操作需要移动元素，平均时间复杂度为O(n)

**知识点详解**

顺序表采用连续存储空间存储数据元素，支持随机访问。各操作时间复杂度分析：

**A选项（错误）**：查找包含指定值的元素
- 无序表：需要顺序查找，平均比较 n/2 次，时间复杂度 O(n)
- 有序表：可使用折半查找，时间复杂度 O(log n)
- 无论哪种情况都不是 O(1)

**B选项（错误）**：插入包含指定值的元素
- 有序表插入需先找到插入位置（折半查找 O(log n)）
- 插入后需移动后续元素保持有序性，平均移动 n/2 个元素
- 总时间复杂度为 O(n)

**C选项（错误）**：删除第 i 个元素
- 删除位置已知，无需查找
- 但需将第 i+1 到第 n 个元素前移一位
- 平均移动 (n-1)/2 个元素，时间复杂度 O(n)

**D选项（正确）**：获取第 i 个元素
- 顺序表采用连续存储，首地址为 base，每个元素大小为 sizeof(ElemType)
- 第 i 个元素地址为：base + (i-1) × sizeof(ElemType)
- 通过地址直接访问，仅需一次运算，时间复杂度 O(1)

**解题关键**：理解顺序表的"随机存取"特性，即支持按位置直接访问，这是顺序表相比链表的主要优势。

## 解题思路（逐项分析法）

### 第一步：明确题目条件
- 数据结构：顺序存储的**有序表**
- 长度：n
- 要求：平均时间复杂度为 O(1) 的操作

### 第二步：回顾顺序表特性
顺序表核心特性：
1. ✅ **随机访问**：支持 O(1) 按位置访问
2. ❌ **插入删除**：需要移动元素，O(n)
3. ❌ **查找**：即使有序也需 O(log n)（折半查找）

### 第三步：逐项判断

**A. 查找包含指定值元素 ❌**
```
无序表：顺序查找 O(n)
有序表：折半查找 O(log n)
结论：无论如何都不是 O(1)
```

**B. 插入包含指定值元素 ❌**
```
步骤1：找位置 O(log n)（折半查找）
步骤2：移动元素 O(n)（保持有序）
结论：O(n)
```

**C. 删除第 i 个元素 ❌**
```
步骤1：定位元素 O(1)
步骤2：移动后续元素 O(n)
结论：O(n)
```

**D. 获取第 i 个元素 ✅**
```
地址计算：base + (i-1) × sizeof(ElemType)
直接访问：O(1)
结论：O(1) ✓
```

### 第四步：确认答案
只有 D 选项是 O(1)，答案：**D**

## 易错点分析

### 陷阱1：误以为有序表查找是O(1)
❌ **错误理解**：有序表可以折半查找，很快，是O(1)
✅ **正确理解**：折半查找是O(log n)，不是O(1)

### 陷阱2：混淆"按位置访问"和"按值查找"
- **按位置访问**（给定下标i）：O(1)，直接计算地址
- **按值查找**（给定value）：需要遍历或折半，O(n)或O(log n)

### 陷阱3：忽略"平均"时间复杂度
删除操作虽然定位是O(1)，但移动元素的平均次数是(n-1)/2，总体仍是O(n)

## 知识点对比

| 操作 | 顺序表 | 单链表 | 记忆技巧 |
|------|--------|--------|----------|
| 按位置访问 | O(1) ✅ | O(n) | 顺序表"随机访问" |
| 按值查找 | O(n) | O(n) | 都需要遍历 |
| 插入（已知位置） | O(n) | O(1) | 链表"指针修改" |
| 删除（已知位置） | O(n) | O(1) | 链表不移动元素 |

**记忆口诀**：顺序表擅长"读"（访问），链表擅长"改"（插入删除）
