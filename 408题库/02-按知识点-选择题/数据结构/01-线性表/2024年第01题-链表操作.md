# 2024年第1题

**元数据**
- 年份：2024年第1题
- 科目：#数据结构
- 难度：⭐⭐

**题干**：已知带头结点的非空单链表L的头指针为h，结点结构为data next，其中next是指向直接后继结点的指针。现有指针p和q，若p指向L中非首且非尾的任意一个结点，则执行语句序列"q = p->next; p->next = q->next; q->next = h->next; h->next = q;"的结果是（）。

A. 在p所指结点后插入q所指结点

B. 在q所指结点后插入p所指结点

C. 将p所指结点移动到L的头结点之后

D. 将q所指结点移动到L的头结点之后

**答案**：D

**考点分析**：本题考查单链表的操作。分析语句序列：第1句让q指向p的后继结点；第2句将p的next指针指向q的后继（即删除q）；第3句将q的next指向头结点后的第一个结点；第4句将头结点的next指向q。整个过程是将p的后继结点q从原位置删除，插入到头结点之后。

---

## 解题思路（标准化分析）

### 第一步：理解题目核心
执行一系列链表操作语句，分析最终结果。

**已知条件**：
- 带头结点的非空单链表L，头指针h
- p指向非首且非尾的任意结点
- q是临时指针

**语句序列**：
```c
q = p->next;           // 语句1
p->next = q->next;     // 语句2
q->next = h->next;     // 语句3
h->next = q;           // 语句4
```

### 第二步：逐句追踪指针变化

**初始状态**（假设链表为h→A→B→C→D，p指向B）：
```
h → A → B → C → D → NULL
        ↑
        p
```

**语句1：q = p->next;**
```
作用：q指向p的后继结点
结果：q指向C
链表：h → A → B → C → D → NULL
            ↑   ↑
            p   q
```

**语句2：p->next = q->next;**
```
作用：p的next跳过q，指向q的后继
结果：C从链表中断开
链表：h → A → B ─┐
                 └→ D → NULL
            ↑
            p
独立：        C → D
            ↑
            q
```

**语句3：q->next = h->next;**
```
作用：q的next指向头结点的后继（第一个元素A）
结果：q指向A
链表：h → A → B → D → NULL
      ↑
      C ──┘
      ↑
      q
```

**语句4：h->next = q;**
```
作用：头结点的next指向q
结果：C成为新的第一个元素
最终：h → C → A → B → D → NULL
          ↑
          q
```

**操作本质**：
1. 将p的后继结点q从原位置删除
2. 将q插入到头结点之后（头插法）

### 第三步：逐项判断

**A. 在p所指结点后插入q所指结点 ❌**
```
分析：q本来就在p后面
如果是插入，q应该还在p后，但实际q移到了链表开头
结论：错误
```

**B. 在q所指结点后插入p所指结点 ❌**
```
分析：p没有移动，仍在原位置（只是后继改变）
结论：错误
```

**C. 将p所指结点移动到L的头结点之后 ❌**
```
分析：p指向B，B没有移动
最终链表h→C→A→B→D，B不在头结点之后
结论：错误
```

**D. 将q所指结点移动到L的头结点之后 ✅**
```
分析：q指向C（p的后继）
操作：
1. 语句1-2：将C从原位置删除
2. 语句3-4：将C插入到头结点之后
最终：h→C→...，C确实在头结点之后
结论：正确 ✅
```

### 第四步：确认答案
将p的后继结点q移到链表开头 → 答案：**D**

## 易错点分析

### 易错点1：混淆p和q的角色
❌ 错误理解：操作的是p所指结点，p是主角
✅ 正确理解：p是定位指针用于找位置，q才是被移动的结点。语句1"q=p->next"让q指向p的后继，之后所有操作围绕q展开。语句2"p->next=q->next"将q摘除，语句3-4"q->next=h->next; h->next=q"将q插入头部。整个过程p指针始终不动，只是p的next被修改。移动的是q(p的后继)不是p。常见套路：给定p，通过q=p->next定位要操作的结点再对q删除/插入

### 易错点2：未完整追踪每条语句
❌ 错误做法：只看最后两句，以为是插入操作
✅ 正确方法：
- 语句1-2：删除操作（将q从原位置移除）
- 语句3-4：插入操作（将q插入头部）
- 必须逐句追踪才能理解完整逻辑

### 易错点3：不理解头插法
❌ 错误理解：头插法只是简单的插入操作
✅ 正确理解：头插法是链表基础插入技巧，在头部插入新结点，核心两步：①新结点的next指向原首元素(newNode->next=h->next)，建立连接；②头结点的next指向新结点(h->next=newNode)，变成新首元素。顺序不可颠倒否则丢失原首元素指针。本题语句3-4是标准头插法。时间O(1)无需遍历，常用于链表逆置、头部快速插入。口诀：新指旧头指新

头插法的标准操作：
```c
newNode->next = h->next;  // 新结点指向原首元素
h->next = newNode;        // 头结点指向新结点
```

本题语句3-4就是标准头插法：
```c
q->next = h->next;  // 语句3
h->next = q;        // 语句4
```

## 知识点对比

### 链表操作组合

| 操作 | 步骤 | 效果 |
|------|------|------|
| **删除+头插** | ①断开 ②插开头 | 移动到开头（本题） |
| **删除+尾插** | ①断开 ②插结尾 | 移动到结尾 |
| **简单删除** | 跳过结点 | 仅删除 |
| **简单插入** | 修改指针 | 仅插入 |

### 头插法vs尾插法

| 方法 | 代码 | 结果位置 | 时间复杂度 |
|------|------|---------|-----------|
| **头插法** | new→next=h→next<br>h→next=new | 第一个位置 | O(1) |
| **尾插法** | tail→next=new<br>tail=new | 最后位置 | O(1)需尾指针 |

### 本题操作分解

| 阶段 | 语句 | 作用 | 操作类型 |
|------|------|------|---------|
| 准备 | q=p→next | 定位q | 赋值 |
| 删除 | p→next=q→next | 移除q | 删除 |
| 插入 | q→next=h→next<br>h→next=q | q插到头部 | 头插法 |

**记忆口诀**：先删后插，头插法移到开头
