# 2018年第18题

**元数据**
- 年份：2018年第18题
- 科目：#计算机组成原理
- 知识点：#计算机组成原理/寻址方式/变址寻址 #数组访问
- 难度：⭐⭐⭐
- 关联知识：[知识库/计算机组成原理/寻址方式](知识库/计算机组成原理/寻址方式.md)

**题干**

按字节编址的计算机中，某 double 型数组 A 的首地址为 2000H，使用变址寻址和循环结构访问数组 A，保存数组下标的变址寄存器初值为 0，每次循环取一个数组元素，其偏移地址为变址值乘以 sizeof(double)，取完后变址寄存器内容自动加 1。若某次循环所取元素的地址为 2100H，则进入该次循环时变址寄存器的内容是（）。

A. 25  
B. 32  
C. 64  
D. 100

**答案**：B

**考点分析**

本题考查变址寻址方式和数组地址计算：
- double类型占8字节，即sizeof(double) = 8
- 数组元素地址 = 首地址 + 变址值 × sizeof(double)
- 2100H - 2000H = 100H = 256（十进制）
- 变址值 = 256 ÷ 8 = 32
- 这是第33个元素（下标从0开始），变址寄存器内容为32

---

## 解题思路

### 第一步：理解题目核心
变址寻址方式访问数组，求某次循环时变址寄存器的值。

**关键信息**：
- 数组A首地址：2000H
- 数据类型：double（8字节）
- 某次取元素地址：2100H
- 偏移地址 = 变址值 × sizeof(double)
- 求：进入该次循环时变址寄存器的内容

**变址寻址公式**：
```
元素地址 = 首地址 + 变址寄存器值 × sizeof(数据类型)
```

### 第二步：计算偏移量
```
偏移量 = 元素地址 - 首地址
       = 2100H - 2000H
       = 100H
```

**十六进制转十进制**：
```
100H = 1×16² + 0×16¹ + 0×16⁰
     = 256(十进制)
```

### 第三步：计算变址寄存器值
```
变址寄存器值 = 偏移量 ÷ sizeof(double)
             = 256 ÷ 8
             = 32
```

这表示访问的是A[32]（数组下标从0开始）。

### 第四步：逐项判断

**A. 25 ❌**
```
验证：2000H + 25×8 = 2000H + 200(十进制) = 2000H + C8H = 20C8H
结果：20C8H ≠ 2100H
结论：错误
```

**B. 32 ✅**
```
验证：2000H + 32×8 = 2000H + 256(十进制) = 2000H + 100H = 2100H
结果：2100H = 2100H ✓
结论：正确 ✅
```

**C. 64 ❌**
```
验证：2000H + 64×8 = 2000H + 512(十进制) = 2000H + 200H = 2200H
结果：2200H ≠ 2100H
结论：错误
```

**D. 100 ❌**
```
分析：这是十六进制偏移量100H的十进制值256
但变址寄存器存的是数组下标，不是字节偏移量
如果是100：2000H + 100×8 = 2000H + 800(十进制) = 2000H + 320H = 2320H
结果：2320H ≠ 2100H
结论：错误
```

### 第五步：确认答案
偏移256字节 ÷ 8字节/元素 = 32 → 答案：**B**

## 易错点分析

### 易错点1：混淆字节偏移和数组下标
❌ 错误理解：偏移量是100H，所以变址寄存器是100
✅ 正确理解：偏移量100H=256字节是字节偏移量，但变址寄存器存储的是数组逻辑下标。数组下标=字节偏移÷元素大小=256÷8=32。变址寻址的本质是将逻辑下标自动转换为物理地址，硬件公式为地址=基址+下标×元素大小

| 概念 | 含义 | 本题值 | 单位 |
|------|------|-------|------|
| 字节偏移 | 目标地址-首地址 | 256 | 字节 |
| 元素大小 | sizeof(double) | 8 | 字节 |
| 数组下标 | 变址寄存器值 | 32 | 无 |

### 易错点2：忘记进制转换
❌ 错误理解：100H就是100
✅ 正确理解：计算机地址用十六进制表示（H后缀），必须转为十进制计算。100H=1×16²=256，不是100。常见错误是直接把100H当十进制100，导致256÷8=32被误算为100÷8=12.5。记忆：16²=256、16³=4096

**进制转换对照**：
| 十六进制 | 十进制 | 计算 |
|---------|-------|------|
| 100H | 256 | 16²=256 |
| C8H | 200 | 12×16+8=200 |
| 200H | 512 | 2×16²=512 |

### 易错点3：不理解变址寻址的工作原理
❌ 错误理解：变址寻址就是简单的数组访问
✅ 正确理解：变址寻址是硬件支持的寻址方式，专为数组优化。公式：地址=基址+变址寄存器×元素大小，乘法由硬件自动完成。变址寄存器存储逻辑下标(0,1,2...)，循环中只需简单递增(+1)，无需手动计算地址，比软件方式效率高。本题：2100H=2000H+IX×8，解得IX=32

变址寻址的硬件实现：

```c
// 伪代码
地址 = 基地址 + 变址寄存器 × 数据类型大小

// 本题
地址 = 2000H + IX × 8
2100H = 2000H + IX × 8
IX = (2100H - 2000H) / 8 = 100H / 8 = 256 / 8 = 32
```

**为什么用变址寻址**：
- 变址寄存器存储逻辑下标（0,1,2,3...）
- 硬件自动乘以元素大小，计算物理地址
- 循环中只需递增变址寄存器（+1），不需要程序员计算地址

## 知识点对比

### 寻址方式对比

| 寻址方式 | 有效地址计算 | 适用场景 | 寄存器作用 |
|---------|------------|---------|-----------|
| **立即寻址** | 无(数据在指令中) | 常量 | 无 |
| **直接寻址** | EA = A | 全局变量 | 无 |
| **间接寻址** | EA = (A) | 指针访问 | 存地址 |
| **寄存器寻址** | EA = Ri | 临时变量 | 存数据 |
| **变址寻址** | EA = 基址+IX×size | 数组访问 | 存下标 |
| **基址寻址** | EA = BR+A | 重定位 | 存基址 |

### 数组访问的两种方式

| 方式 | 地址计算 | 优点 | 缺点 |
|------|---------|------|------|
| **软件计算** | addr = base + i×size | 灵活 | 每次循环需要乘法 |
| **变址寻址** | 硬件自动计算 | 快速，减少指令 | 需要硬件支持 |

**本题体现的优化**：
```c
// 软件方式（需要乘法指令）
for(i=0; i<n; i++) {
    addr = base + i * sizeof(double);  // 每次都要算
    访问addr;
}

// 变址寻址（硬件自动乘）
IX = 0;
for(循环n次) {
    访问(base + IX×8);  // 硬件自动完成乘法
    IX++;  // 只需简单递增
}
```
