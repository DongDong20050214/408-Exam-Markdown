# 双向链表真题汇总

## 2023年第02题

**元数据**
- 年份：[../../01-真题原卷/2023年408真题#02|2023年第2题](../../01-真题原卷/2023年408真题#02|2023年第2题.md)
- 科目：#数据结构
- 知识点：#数据结构/线性表/双向链表 #指针操作 #链表插入
- 难度：⭐⭐ (中等)
- 关联知识：[知识库/数据结构/双向链表基础](知识库/数据结构/双向链表基础.md) [知识库/数据结构/链表插入删除](知识库/数据结构/链表插入删除.md)

**题干**

现有非空双向链表 L，其结点结构为 prev data next，prev 是指向直接前驱结点的指针，next 是指向直接后继结点的指针。若要在 L 中指针 p 所指向的结点（非尾结点）之后插入指针 s 指向的新结点，则在执行语句序列 "s->next = p->next; p->next = s;" 后，下列语句序列中还需要执行的是（）。

A. s->next->prev = p; s->prev = p; 

B. p->next->prev = s; s->prev = p; 

C. s->prev = s->next->prev; s->next->prev = s; 

D. p->next->prev = s->prev; s->next->prev = p; 

**答案**：C

**考点分析**
- 考察点1：双向链表插入操作需要修改4个指针（2个next指针，2个prev指针）
- 考察点2：指针变量的值在赋值后会改变，需要通过已保存的值访问原节点
- 考察点3：理解s->next保存的是原p->next的值，而p->next已被修改为s
- 考察点4：追踪指针操作顺序，分析各语句执行后的链表状态

**知识点详解**

双向链表在p结点后插入s结点，需要修改4个指针域。

**初始状态**：
```
... ←→ [p] ←→ [q] ←→ ...  (q是p的原后继)
         [s]（待插入）
```

**已执行的语句分析**：
```c
s->next = p->next;  // s->next指向q（保存了原p->next的值）
p->next = s;        // p->next指向s（p->next的值被修改了）
```

**执行后的状态**：
- s->next = q（保存了原p的后继地址）
- p->next = s（p的next已经改为s）
- s->prev = ?（未设置，待完成）
- q->prev = p（仍指向p，应该改为指向s，待完成）

**还需完成的操作**：
1. 设置s->prev = p
2. 设置q->prev = s

**关键理解**：
- **p->next** 此时已经指向s，所以 `p->next->prev` 访问的是 **s的prev字段**
- **s->next** 保存的是q的地址，所以 `s->next->prev` 访问的是 **q的prev字段**

**选项分析**：

**A选项（错误）**：`s->next->prev = p; s->prev = p;`
- 第一句：`s->next->prev = p` 即 `q->prev = p`，但q->prev本来就是p，没有改变，错误（应该改为s）
- 第二句：`s->prev = p` 正确

**B选项（错误）**：`p->next->prev = s; s->prev = p;`
- 第一句：`p->next->prev = s` 即 `s->prev = s`，让s的prev指向自己，逻辑错误
- 第二句：`s->prev = p` 正确，但会覆盖第一句的错误赋值

**C选项（正确）**：`s->prev = s->next->prev; s->next->prev = s;`
- 第一句：`s->prev = s->next->prev` 即 `s->prev = q->prev` 即 `s->prev = p`（正确）
- 第二句：`s->next->prev = s` 即 `q->prev = s`（正确）
- 两句都正确且必要

**D选项（错误）**：`p->next->prev = s->prev; s->next->prev = p;`
- 第一句：s->prev未初始化，使用其值是错误的
- 第二句：`q->prev = p`，没有修改q的prev（仍指向p），错误

**完整插入过程**：
```c
s->next = p->next;           // 步骤1（已执行）
p->next = s;                 // 步骤2（已执行）
s->prev = s->next->prev;     // 步骤3：s->prev = p
s->next->prev = s;           // 步骤4：q->prev = s
```

**最终状态**：
```
... ←→ [p] ←→ [s] ←→ [q] ←→ ...
```

**易错点分析**

❌ **错误1：认为B选项正确**
- 误以为 `p->next->prev = s` 是设置 `q->prev = s`
- ✅ 正确：此时 `p->next` 已经指向 `s`，所以 `p->next->prev` 是 `s->prev`，这句话让 `s->prev = s`（自己指向自己），逻辑错误

❌ **错误2：认为A选项第一句正确**
- 误以为 `s->next->prev = p` 是必要操作
- ✅ 正确：`q->prev` 本来就是 `p`，这句话没有改变任何状态，遗漏了必须的 `q->prev = s`

❌ **错误3：按逻辑直觉写代码**
- 直觉上想写 `q->prev = s; s->prev = p`，但此时已经没有变量指向q
- ✅ 正确：必须通过 `s->next`（保存了q的地址）来访问q，写成 `s->next->prev = s`

**知识点对比**

| 操作类型 | 单向链表 | 双向链表 |
|---------|---------|---------|
| **插入修改指针数** | 2个 | 4个 |
| **在p后插入s** | `s->next = p->next; p->next = s;` | 还需：`s->prev = p; s->next->prev = s;` |
| **删除p后继** | `p->next = p->next->next;` | 还需：`p->next->next->prev = p;` |
| **访问前驱** | 不能直接访问 | `p->prev` |

**指针操作顺序对比**

| 写法 | s->prev = p | s->next->prev = s | 是否正确 |
|------|-------------|-------------------|---------|
| **直接赋值** | ✅ 正确 | ✅ 正确 | 可以 |
| **C选项写法** | `s->prev = s->next->prev` | `s->next->prev = s` | ✅ 可以 |
| **交换顺序** | `s->next->prev = s` 先执行 | `s->prev = s->next->prev` 后执行 | ❌ 错误（q->prev已被修改） |

**关键理解**：C选项的顺序不能颠倒
- 正确：先执行 `s->prev = s->next->prev`（此时q->prev还是p）
- 错误：如果先执行 `s->next->prev = s`，则q->prev变成s，再执行第一句会得到 `s->prev = s`

## 解题思路（标准化分析）

### 第一步：理解题目核心
双向链表在p结点后插入s结点，已执行前两条语句，需要找出还需执行的语句。

**已执行的语句**：
```c
s->next = p->next;  // s的next指向原p的后继
p->next = s;        // p的next指向s
```

**初始状态**：
```
... ←→ [p] ←→ [q] ←→ ...
        [s]（待插入）
```

**执行已给语句后**：
```
    ┌─→ [s] ─→ [q] ←→ ...
... ←→ [p] ┘
```

**分析**：
- ✅ s->next = q（正确）
- ✅ p->next = s（正确）
- ❌ s->prev = ?（未设置）
- ❌ q->prev = p（仍指向p，应改为s）

### 第二步：确定还需完成的操作

双向链表插入需要修改**4个指针**：
1. s->next = p->next ✅（已完成）
2. p->next = s ✅（已完成）
3. s->prev = p ❌（待完成）
4. q->prev = s ❌（待完成）

**关键**：此时p->next已经指向s，所以：
- `p->next->prev` 实际是 `s->prev`
- `s->next->prev` 实际是 `q->prev`

### 第三步：逐项判断

**A. s->next->prev = p; s->prev = p; ❌**
```
第一句：s->next->prev = p
解析：q->prev = p
问题：q->prev本来就是p，没有改变！应该改为s
第二句：s->prev = p ✅ 正确

结论：第一句错误，遗漏了必需的q->prev = s操作
```

**B. p->next->prev = s; s->prev = p; ❌**
```
注意：此时p->next已经是s！

第一句：p->next->prev = s
解析：s->prev = s（让s的prev指向自己）
问题：逻辑错误，s->prev应该指向p，不是s

第二句：s->prev = p ✅ 正确
但会覆盖第一句的错误赋值

结论：第一句逻辑错误
```

**C. s->prev = s->next->prev; s->next->prev = s; ✅**
```
第一句：s->prev = s->next->prev
解析：s->prev = q->prev
      s->prev = p ✅ 正确（q->prev当前是p）

第二句：s->next->prev = s
解析：q->prev = s ✅ 正确

结论：两句都正确且完整 ✅
```

**D. p->next->prev = s->prev; s->next->prev = p; ❌**
```
第一句：p->next->prev = s->prev
解析：s->prev = s->prev（s->prev未初始化）
问题：使用未初始化的值

第二句：s->next->prev = p
解析：q->prev = p
问题：q->prev本来就是p，没有改变，应该是q->prev = s

结论：两句都错误
```

### 第四步：确认答案
只有C选项正确完成了s->prev = p和q->prev = s → 答案：**C**

**注意**：文件开头答案写的是C，但考点分析写的答案是B，这里纠正为C。

## 易错点分析

### 易错点1：忽略指针已被修改的事实
❌ 错误理解：p->next仍然指向q，所以p->next->prev就是q->prev
✅ 正确理解：指针是动态的，执行"p->next=s"后，p->next已从q改为s，此时"p->next->prev"访问的是s的prev域不是q的prev。要访问q必须通过s->next，因为s->next保存了原p->next的值(q的地址)。指针操作必须逐句追踪当前值，不能停留在初始状态假设

| 时刻 | p->next指向 | p->next->prev是谁的prev |
|------|-----------|----------------------|
| 修改前 | q | q->prev |
| 修改后 | s | s->prev ⚠️ |

### 易错点2：误选B选项
❌ 错误理解：p->next->prev=s就是让q->prev指向s
✅ 正确理解：执行"p->next=s"后，p->next已指向s，此时"p->next->prev"访问的是s->prev而非q->prev。所以"p->next->prev=s"实际是"s->prev=s"，让s前驱指向自己，逻辑错误。虽然第二句s->prev=p会覆盖，但第一句本身错误。正确做法是C选项，通过s->next(保存了q地址)访问q，用"s->next->prev=s"实现"q->prev=s"

### 易错点3：不理解C选项的巧妙之处
❌ 错误理解：C选项写法太绕，不如直接赋值
✅ 正确理解：C选项通过间接访问实现赋值，是双向链表插入经典技巧。第一句"s->prev=s->next->prev"利用q->prev(当前值p)给s->prev赋值，等价于s->prev=p。第二句"s->next->prev=s"通过s->next访问q，修改q->prev为s。优势是不需额外临时变量保存q地址。但顺序不可颠倒：先读q->prev旧值(p)，再改q->prev新值(s)。口诀：先读旧值再改新值

C选项通过间接访问实现赋值：
```c
s->prev = s->next->prev;  // 利用q->prev(当前是p)给s->prev赋值
s->next->prev = s;        // 通过s->next访问q，修改q->prev
```

**优势**：
- 不需要额外的临时变量保存q的地址
- 利用已经设置好的s->next（保存了q的地址）来访问q

**顺序不能颠倒**：
- ✅ 正确：先读q->prev(是p)，再改q->prev(为s)
- ❌ 错误：如果先改q->prev=s，再读q->prev就得到s而非p

## 知识点对比

### 单向链表vs双向链表插入

| 操作 | 单向链表 | 双向链表 |
|------|---------|---------|
| 修改指针数 | 2个 | 4个 |
| 在p后插入s | s->next=p->next<br>p->next=s | 还需：<br>s->prev=p<br>s->next->prev=s |
| 是否需要前驱 | 否 | 是 |
| 能否反向遍历 | 否 | 是 |

### 双向链表插入的标准步骤

| 步骤 | 语句 | 说明 | 本题状态 |
|------|------|------|---------|
| 1 | s->next = p->next | s指向原后继 | ✅ 已完成 |
| 2 | s->prev = p | s指向原前驱 | ❌ 待完成 |
| 3 | p->next = s | 前驱指向s | ✅ 已完成 |
| 4 | s->next->prev = s | 后继指向s | ❌ 待完成 |

**注意顺序**：步骤1必须在步骤3之前（保存原p->next的值）

### 指针操作的两种写法

| 目标 | 直接写法 | 间接写法(C选项) |
|------|---------|----------------|
| s->prev = p | `s->prev = p;` | `s->prev = s->next->prev;` |
| q->prev = s | `q->prev = s;` | `s->next->prev = s;` |

间接写法的必要性：当没有变量直接指向q时，必须通过s->next访问q。
