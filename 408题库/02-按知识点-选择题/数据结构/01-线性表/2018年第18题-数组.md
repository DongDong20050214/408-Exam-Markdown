# 题目

按字节编址的计算机中，某 double 型数组 A 的首地址为 2000H，使用变址寻址和循环结构访问数组 A，保存数组下标的变址寄存器初值为 0，每次循环取一个数组元素，其偏移地址为变址值乘以 sizeof(double)，取完后变址寄存器内容自动加 1。若某次循环所取元素的地址为 2100H，则进入该次循环时变址寄存器的内容是（）。

A. 25 

B. 32 

C. 64 

D. 100 

# 答案

B

# 考点分析

本题考查变址寻址方式和数组元素地址计算。关键知识点：
1. 变址寻址的有效地址计算：EA = 基地址 + 变址寄存器 × 数据类型大小
2. double类型在大多数系统中占8字节（64位）
3. 地址计算：2100H - 2000H = 100H = 256(十进制)，即偏移量为256字节
4. 变址寄存器值 = 偏移量 ÷ sizeof(double) = 256 ÷ 8 = 32，对应数组下标A[32]
5. 理解寻址过程：变址寻址通过变址寄存器存储数组下标，硬件自动完成与元素大小的乘法运算，实现高效的数组访问

## 解题思路（标准化分析）

### 第一步：理解题目核心
变址寻址方式访问double数组，求变址寄存器的值。

**注意**：本题与《2018年第18题-变址寻址.md》是同一题，详细分析参考该文件。

**核心公式**：
```
元素地址 = 首地址 + 变址寄存器值 × sizeof(double)
2100H = 2000H + 变址值 × 8
```

**计算过程**：
```
偏移量 = 2100H - 2000H = 100H = 256(十进制)
变址值 = 256 ÷ 8 = 32
```

### 第二步：逐项判断

**A. 25 ❌**
```
验证：2000H + 25×8 = 20C8H ≠ 2100H
结论：错误
```

**B. 32 ✅**
```
验证：2000H + 32×8 = 2100H ✓
结论：正确 ✅
```

**C. 64 ❌**
```
验证：2000H + 64×8 = 2200H ≠ 2100H
结论：错误
```

**D. 100 ❌**
```
分析：这是十六进制偏移量，不是数组下标
结论：错误
```

### 第三步：确认答案
偏移256字节 ÷ 8字节/元素 = 32 → 答案：**B**

## 易错点分析

### 易错点1：混淆字节偏移和数组下标
❌ 错误理解：偏移100H就是下标100
✅ 正确理解：下标 = 字节偏移 ÷ 元素大小 = 256 ÷ 8 = 32

### 易错点2：忘记进制转换
❌ 错误理解：100H = 100(十进制)
✅ 正确理解：100H = 256(十进制)

## 知识点对比

| 寻址方式 | 有效地址 | 适用场景 |
|---------|---------|---------|
| 变址寻址 | 基址+IX×size | 数组访问 |
| 直接寻址 | EA = A | 全局变量 |
| 间接寻址 | EA = (A) | 指针访问 |
