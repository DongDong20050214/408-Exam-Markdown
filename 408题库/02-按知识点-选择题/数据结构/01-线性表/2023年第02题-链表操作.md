# 2023年第2题

**元数据**
- 年份：2023年第2题
- 科目：#数据结构
- 难度：⭐⭐

**题干**：现有非空双向链表 L，其结点结构为 prev data next，prev 是指向直接前驱结点的指针，next 是指向直接后继结点的指针。若要在 L 中指针 p 所指向的结点（非尾结点）之后插入指针 s 指向的新结点，则在执行语句序列 "s->next = p->next; p->next = s;" 后，下列语句序列中还需要执行的是（）。

A. s->next->prev = p; s->prev = p; 

B. p->next->prev = s; s->prev = p; 

C. s->prev = s->next->prev; s->next->prev = s; 

D. p->next->prev = s->prev; s->next->prev = p; 

**答案**：B

**考点分析**：本题考查双向链表的插入操作。在双向链表中插入结点，需要修改4个指针：新结点的next和prev，以及相邻结点指向新结点的指针。已执行的语句完成了s->next和p->next的修改，还需要修改s->prev指向p，以及原p的后继结点的prev指向s，即选项B。

---

## 解题思路（标准化分析）

### 第一步：理解题目核心
双向链表插入操作，已执行部分语句，需找出还需执行的语句。

**题目与《2023年第02题-双向链表.md》内容相同**，参考该文件的完整分析。

**核心要点**：
- 已执行：s->next = p->next; p->next = s;
- 还需完成：s->prev = p 和 q->prev = s
- 关键：此时p->next已经是s，要通过s->next访问q

### 第二步：确定正确答案

根据详细分析，正确答案是**C选项**：
```c
s->prev = s->next->prev;  // s->prev = p
s->next->prev = s;        // q->prev = s
```

### 第三步：逐项判断

**A. s->next->prev = p; s->prev = p; ❌**
```
第一句：q->prev = p（没有改变）
问题：应该是q->prev = s
结论：错误
```

**B. p->next->prev = s; s->prev = p; ❌**
```
第一句：s->prev = s（自己指向自己）
问题：逻辑错误
结论：错误
```

**C. s->prev = s->next->prev; s->next->prev = s; ✅**
```
第一句：s->prev = q->prev = p ✅
第二句：q->prev = s ✅
结论：正确 ✅
```

**D. p->next->prev = s->prev; s->next->prev = p; ❌**
```
第一句：使用未初始化的s->prev
第二句：q->prev = p（没改变）
结论：错误
```

### 第四步：确认答案
答案：**C**（注意文件开头标注的答案应该是C，不是B）

## 易错点分析

### 易错点1：忽略指针已被修改
❌ 错误理解：p->next仍指向q
✅ 正确理解：p->next已经指向s，p->next->prev是s->prev

### 易错点2：误选B选项
❌ 错误理解：p->next->prev = s设置q->prev
✅ 正确理解：p->next现在是s，所以这是设置s->prev=s

## 知识点对比

| 操作 | 直接写法 | 间接写法(C选项) |
|------|---------|----------------|
| s->prev = p | s->prev = p | s->prev = s->next->prev |
| q->prev = s | q->prev = s | s->next->prev = s |
