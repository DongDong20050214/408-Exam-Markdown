# 2014年第3题

**元数据**
- 年份：2014年第3题
- 难度：⭐⭐

**题干**：循环队列放在一维数组 A[0...M-1]中，end1 指向队头元素，end2 指向队尾元素的后一个位置。假设队列两端均可进行入队和出队操作，队列中最多能容纳 M-1 个元素。初始时为空。下列判断队空和队满的条件中，正确的是（）。

**选项**：
- A. 队空: end1 == end2; 队满：end1 == (end2 + 1) mod M
- B. 队空: end1 == end2; 队满：end2 == (end1 + 1) mod (M-1)
- C. 队空: end2 == (end1 + 1) mod M; 队满：end1 == (end2 + 1) mod M
- D. 队空: end1 == (end2 + 1) mod M; 队满：end2 == (end1 + 1) mod (M - 1)

**答案**：A

**考点分析**：
本题考查循环队列的队空和队满判断条件。

关键点：
- end1 指向队头元素
- end2 指向队尾元素的后一个位置
- 队列最多容纳 M-1 个元素（牺牲一个空间来区分队空和队满）

队空条件：当 end1 == end2 时，队列为空。

队满条件：当队尾指针的下一个位置是队头时，队列满。即 (end2 + 1) mod M == end1，等价于 end1 == (end2 + 1) mod M。

因此答案是 A。

## 解题思路

### 第一步：理解题目核心条件
关键信息提取：
- 数组：A[0...M-1],容量M个单元
- end1指向队头元素
- end2指向队尾元素的**后一个位置**（注意不是队尾元素本身）
- 最多容纳M-1个元素（牺牲一个空间区分空满）
- 初始为空队列

### 第二步：推导队空和队满条件

**队空条件推导**：
```
当队列为空时,没有任何元素
end1和end2应该指向同一位置
因此: end1 == end2
```

**队满条件推导**：
```
最多容纳M-1个元素,意味着留一个空位
当队尾的下一个位置是队头时,队列满
end2指向队尾的下一个位置
再向后一步就是队头: (end2 + 1) mod M == end1
即: end1 == (end2 + 1) mod M
```

### 第三步：逐项分析

**A. 队空: end1 == end2; 队满: end1 == (end2 + 1) mod M ✅**
```
详细分析：
  队空：end1==end2,两指针重合,没有元素 ✓
  队满：(end2+1) mod M == end1
       意味着end2的下一个位置是end1
       此时队列有M-1个元素 ✓
判断依据：符合循环队列标准设计
结论：正确 ✅
```

**B. 队空: end1 == end2; 队满: end2 == (end1 + 1) mod (M-1) ❌**
```
详细分析：
  队空条件正确 ✓
  队满条件: mod (M-1)错误
错误原因：
  数组大小是M,应该mod M,不是mod (M-1)
  (end1+1) mod (M-1)可能越界
反例：M=5,end1=4时,(4+1) mod 4 = 1,但实际应该是0
结论：错误
```

**C. 队空: end2 == (end1 + 1) mod M; 队满: end1 == (end2 + 1) mod M ❌**
```
详细分析：
  队空条件: end2 == (end1+1) mod M
  意味着end2在end1后面一个位置,队列有1个元素
错误原因：这是有1个元素的状态,不是空队列
具体说明：
  空队列应该end1==end2
  这里end2在end1后面,说明有元素
结论：错误
```

**D. 队空: end1 == (end2 + 1) mod M; 队满: end2 == (end1 + 1) mod (M-1) ❌**
```
详细分析：
  队空条件: end1 == (end2+1) mod M
  意味着end1在end2后面一个位置
错误原因：
  这表示end1在end2前面,队列是满的状态
  与队空定义矛盾
  队满条件的mod (M-1)也错误
结论：错误
```

### 第四步：确认答案
标准循环队列设计：牺牲一个空间,用end1==end2判空,用(end2+1)%M==end1判满 → 答案：**A**

## 易错点分析

### 易错点1：混淆end2的两种定义方式
❌ 错误理解：end2指向队尾元素本身
✅ 正确理解：本题end2指向队尾元素的**后一个位置**

| 定义方式 | end2含义 | 队满条件 | 入队操作 |
|---------|---------|---------|---------|
| 方式1 | 队尾元素 | 需要额外标志 | 先赋值再移动 |
| 方式2(本题) | 队尾后一位 | (end2+1)%M==end1 | 先移动再赋值 |

### 易错点2：取模运算的底数错误
❌ 错误理解：数组容量M-1,所以mod (M-1)
✅ 正确理解：数组下标0到M-1,循环时必须mod M

**具体例子**：
```
数组A[0...4], M=5
end2=4时,下一个位置：
  正确：(4+1) mod 5 = 0 ✓
  错误：(4+1) mod 4 = 1 ✗ (跳过了A[0])
```

### 易错点3：队空和队满条件搞反
| 状态 | end1与end2关系 | 元素个数 | 判定条件 |
|------|---------------|---------|---------|
| 队空 | end1 == end2 | 0个 | 两指针重合 |
| 队满 | (end2+1)%M == end1 | M-1个 | 队尾下一个是队头 |
| 有1个元素 | end2 == (end1+1)%M | 1个 | 队尾紧跟队头 |
