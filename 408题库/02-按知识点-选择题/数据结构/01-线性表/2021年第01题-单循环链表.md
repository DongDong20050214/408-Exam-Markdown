# 单循环链表真题汇总

## 2021年第01题

**元数据**
- 年份：[../../01-真题原卷/2021年408真题#01|2021年第1题](../../01-真题原卷/2021年408真题#01|2021年第1题.md)
- 科目：#数据结构
- 知识点：#数据结构/线性表/单循环链表 #链表删除
- 难度：⭐⭐⭐
- 关联知识：[2021年第01题-单循环链表](2021年第01题-单循环链表.md)

**题干**

已知头指针 h 指向一个带头结点的非空单循环链表，结点结构为 data next，其中 next 是指向直接后继结点的指针，p 是尾指针，q 是临时指针。现要删除该链表的第一个元素，正确的语句序列是（）。

A. h -> next = h -> next -> next; q = h -> next; free(q);
B. q = h -> next; h -> next = h -> next -> next; free(q);
C. q = h -> next; h -> next = q -> next; if(p != q) p = h; free(q);
D. q = h -> next; h -> next = q -> next; if(p == q) p = h; free(q);

**答案**：D

**考点分析**
- 单循环链表删除首元素需要更新头指针的next
- 若删除的是唯一元素（p==q），需要更新尾指针p指向头结点
- 先保存待删除节点，再修改指针，最后释放

## 解题思路（逐项分析法）

### 第一步：明确题目条件
- 数据结构：**带头结点的非空单循环链表**
- h：头指针（指向头结点）
- p：尾指针（指向最后一个元素）
- q：临时指针
- 操作目标：删除第一个元素（头结点的直接后继）

### 第二步：回顾单循环链表特性
单循环链表删除首元素的关键点：
1. ✅ 保存待删除结点（h->next）
2. ✅ 修改头结点的next指针，跳过第一个元素
3. ✅ **特殊情况**：若只有一个元素（p==q），删除后链表为空，需更新尾指针p指向头结点h
4. ✅ 释放原首元素的内存

### 第三步：逐项判断

**A. h -> next = h -> next -> next; q = h -> next; free(q); ❌**
```
问题1：先修改h->next，再赋值给q
执行顺序：
  h->next = h->next->next  // h->next现在指向第二个元素
  q = h->next              // q指向第二个元素（不是待删除的首元素！）
  free(q)                  // 错误地删除了第二个元素

问题2：未处理尾指针p的更新
当只有一个元素时，p仍指向已删除的结点，产生悬空指针

结论：错误，删除了错误的结点
```

**B. q = h -> next; h -> next = h -> next -> next; free(q); ❌**
```
执行顺序分析：
  q = h->next              // ✅ 正确保存待删除结点
  h->next = h->next->next  // ✅ 跳过第一个元素
  free(q)                  // ✅ 释放正确的结点

但存在致命问题：
当只有一个元素时：
  - 删除前：h->next指向唯一元素，p也指向它，p->next指向h
  - 删除后：h->next指向第二个元素（即h自己，因为循环）
  - 尾指针p仍指向已释放的内存（悬空指针！）

结论：主逻辑正确，但缺少尾指针更新，存在内存安全问题
```

**C. q = h -> next; h -> next = q -> next; if(p != q) p = h; free(q); ❌**
```
执行顺序：
  q = h->next              // ✅ 保存待删除结点
  h->next = q->next        // ✅ 跳过第一个元素
  if(p != q) p = h         // ❌ 条件判断错误！
  free(q)

条件判断错误分析：
当只有一个元素时：p == q（都指向唯一元素）
  - 条件 p != q 为假，不执行 p = h
  - 删除后p仍指向已释放内存（悬空指针）

当有多个元素时：p != q（p指向尾，q指向首）
  - 条件为真，执行 p = h（错误地把尾指针指向头结点！）
  - 但实际尾结点并未改变

结论：条件判断逻辑完全相反
```

**D. q = h -> next; h -> next = q -> next; if(p == q) p = h; free(q); ✅**
```
执行顺序：
  q = h->next              // ✅ 保存待删除结点
  h->next = q->next        // ✅ 头结点跳过首元素
  if(p == q) p = h         // ✅ 正确的条件判断
  free(q)                  // ✅ 释放内存

情况1：只有一个元素（p == q）
  - p和q都指向唯一元素
  - 删除后链表为空（只剩头结点）
  - 执行 p = h，尾指针指向头结点 ✅

情况2：多个元素（p != q）
  - 删除首元素不影响尾指针
  - 不执行更新，p保持指向原尾结点 ✅

结论：所有情况处理正确
```

### 第四步：确认答案
删除单循环链表首元素的完整逻辑：
1. 保存待删除结点
2. 修改头指针
3. **关键：当删除的是唯一元素时（p==q），更新尾指针**
4. 释放内存

答案：**D**

## 易错点分析

### 陷阱1：先修改指针再保存结点
❌ **错误做法**：`h->next = h->next->next; q = h->next;`
✅ **正确理解**：必须先用临时指针保存待删除结点，否则修改后无法访问原首元素

### 陷阱2：尾指针更新条件判断反了
❌ **错误理解**：`if(p != q) p = h;` 认为"不相等时才需要更新"
✅ **正确理解**：`if(p == q) p = h;` 只有当删除的是唯一元素（p和q指向同一个）时，才需要更新尾指针

| 情况 | p和q关系 | 是否更新p | 原因 |
|------|----------|-----------|------|
| 只有1个元素 | p == q | ✅ 需要 | 删除后链表为空，p应指向h |
| 多个元素 | p != q | ❌ 不需要 | 删除首元素不影响尾结点 |

### 陷阱3：忽略单循环链表的"循环"特性
单循环链表与单链表的区别：
- 单链表：尾结点next为NULL
- 单循环链表：尾结点next指向头结点，**没有NULL**

删除唯一元素后：
- 单链表：h->next = NULL
- 单循环链表：h->next指向自己（h），且p = h

## 知识点对比

| 对比项 | 单链表 | 单循环链表 | 关键区别 |
|--------|--------|------------|----------|
| 尾结点next | NULL | 指向头结点 | 循环链表无NULL |
| 删除唯一元素后 | h->next=NULL | h->next=h, p=h | 需更新尾指针 |
| 判空条件 | h->next==NULL | h->next==h 或 p==h | 不能用NULL判断 |
| 遍历终止条件 | p==NULL | p->next==h | 回到头结点停止 |

**记忆技巧**：单循环链表的"循环"体现在尾结点指回头结点，所以删除唯一元素后，尾指针p也要指向头结点h，保持循环结构。
