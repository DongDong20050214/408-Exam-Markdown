# 2016年第2题

**元数据**
- 年份：2016年第2题
- 难度：⭐⭐

**题干**：已知一个带有表头结点的双向循环链表 $L$，结点结构为 prev data next，其中，prev 和 next 分别是指向其直接前驱和直接后继结点的指针。现要删除指针 p 所指的结点，正确的语句序列是（）。

A. p->next->prev = p->prev; p->prev->next = p->prev; free(p);

B. p->next->prev = p->next; p->prev->next = p->next; free(p);

C. p->next->prev = p->next; p->prev->next = p->prev; free(p);

D. p->next->prev = p->prev; p->prev->next = p->next; free(p);

**答案**：D

**考点分析**：

本题考查双向循环链表的删除操作。删除结点 p 的核心是调整前驱和后继结点的指针：
1. **前驱结点的 next 指针调整**：p 的前驱结点（p->prev）的 next 应指向 p 的后继结点（p->next），即 p->prev->next = p->next
2. **后继结点的 prev 指针调整**：p 的后继结点（p->next）的 prev 应指向 p 的前驱结点（p->prev），即 p->next->prev = p->prev
3. **释放结点**：调整指针后释放 p 占用的内存空间，即 free(p)。这三步缺一不可，且顺序可以交换但不影响正确性

## 解题思路

### 第一步：理解双向循环链表删除操作
删除节点p需要完成的工作：
```
删除前：...← prev ← p → next →...
删除后：...← prev ← → next →...
           ↑_____________↓
```

需要修改两个指针：
1. p的前驱节点的next指针：指向p的后继
2. p的后继节点的prev指针：指向p的前驱

### 第二步：推导正确语句

**p的前驱节点**：p->prev
- 它的next指针应该跳过p，指向p->next
- 语句：`p->prev->next = p->next`

**p的后继节点**：p->next
- 它的prev指针应该跳过p，指向p->prev
- 语句：`p->next->prev = p->prev`

**释放内存**：
- 语句：`free(p)`

### 第三步：逐项分析

**A. p->next->prev = p->prev; p->prev->next = p->prev; free(p); ❌**
```
详细分析：
  第1句：p->next->prev = p->prev ✓ (后继的prev指向前驱)
  第2句：p->prev->next = p->prev ✗ (前驱的next指向自己)
错误原因：
  p->prev->next应该指向p->next,而不是p->prev
  这会让前驱节点指向自己,形成错误链接
示意图：prev.next = prev (自环)
结论：错误
```

**B. p->next->prev = p->next; p->prev->next = p->next; free(p); ❌**
```
详细分析：
  第1句：p->next->prev = p->next ✗ (后继的prev指向自己)
  第2句：p->prev->next = p->next ✓ (前驱的next指向后继)
错误原因：
  p->next->prev应该指向p->prev,而不是p->next
  这会让后继节点的prev指向自己,形成自环
示意图：next.prev = next (自环)
结论：错误
```

**C. p->next->prev = p->next; p->prev->next = p->prev; free(p); ❌**
```
详细分析：
  第1句：p->next->prev = p->next ✗ (后继的prev指向自己)
  第2句：p->prev->next = p->prev ✗ (前驱的next指向自己)
错误原因：
  两句都错误,让两个节点都形成自环
  完全没有建立prev和next之间的连接
结论：错误
```

**D. p->next->prev = p->prev; p->prev->next = p->next; free(p); ✅**
```
详细分析：
  第1句：p->next->prev = p->prev ✓
        后继节点的prev指针指向p的前驱
  第2句：p->prev->next = p->next ✓
        前驱节点的next指针指向p的后继
  第3句：free(p) ✓
        释放p的内存空间
判断依据：
  完成了双向链接：prev ←→ next
  p被正确移除
验证：
  操作前：prev ← p → next
  操作后：prev ←→ next
结论：正确 ✅
```

### 第四步：确认答案
删除操作需要让前驱的next指向后继，后继的prev指向前驱 → 答案：**D**

## 易错点分析

### 易错点1：混淆前驱和后继的指向
❌ 错误理解：p->prev->next = p->prev (让前驱指向自己)
✅ 正确理解：p->prev->next = p->next (让前驱跨过p指向后继)

| 操作 | 错误写法 | 正确写法 | 结果 |
|------|---------|---------|------|
| 修改前驱的next | p->prev->next = p->prev | p->prev->next = p->next | 前驱指向后继 |
| 修改后继的prev | p->next->prev = p->next | p->next->prev = p->prev | 后继指向前驱 |

### 易错点2：不理解双向链表需要修改两个方向
❌ 错误理解：只修改一个方向的指针就够了
✅ 正确理解：双向链表必须同时修改prev和next两个方向

**单向链表 vs 双向链表**：
```
单向链表删除：
  只需：p->prev->next = p->next (一句)

双向链表删除：
  需要：p->prev->next = p->next (修改next方向)
       p->next->prev = p->prev (修改prev方向)
```

### 易错点3：指针修改的记忆技巧
| 需要修改的指针 | 完整表达式 | 记忆口诀 |
|--------------|-----------|---------|
| 前驱的next | p->prev->next = p->next | 前驱跳过p指向后继 |
| 后继的prev | p->next->prev = p->prev | 后继跳过p指向前驱 |

**对称性**：注意两句话的对称结构
- 左边：前驱的next、后继的prev
- 右边：p的后继、p的前驱
