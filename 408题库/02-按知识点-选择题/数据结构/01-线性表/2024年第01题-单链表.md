# 单链表真题汇总

## 2024年第01题

**元数据**
- 年份：[[../../01-真题原卷/2024年408真题#01|2024年第1题]]
- 科目：#数据结构
- 知识点：#数据结构/线性表/单链表 #指针操作 #链表删除 #链表插入
- 难度：⭐⭐ (中等)
- 关联知识：[[知识库/数据结构/单链表基础]] [[知识库/数据结构/链表插入删除]]

**题干**

已知带头结点的非空单链表L的头指针为h，结点结构为data next，其中next是指向直接后继结点的指针。现有指针p和q，若p指向L中非首且非尾的任意一个结点，则执行语句序列"q = p->next; p->next = q->next; q->next = h->next; h->next = q;"的结果是（）。

A. 在p所指结点后插入q所指结点

B. 在q所指结点后插入p所指结点

C. 将p所指结点移动到L的头结点之后

D. 将q所指结点移动到L的头结点之后

**答案**：D

**考点分析**
- 考察点1：单链表的结点删除操作（将q从原位置删除）
- 考察点2：链表头插法（将q插入到头结点之后）
- 考察点3：指针变量的理解与追踪
- 相似题型：[[2022年408真题#03]] [[2019年408真题#01]]

## 解题思路（逐项分析法）

### 第一步：明确题目条件
- 数据结构：**带头结点的非空单链表L**
- h：头指针（指向头结点）
- p：指向非首且非尾的任意结点
- q：临时指针
- 操作语句：
  1. `q = p->next;`
  2. `p->next = q->next;`
  3. `q->next = h->next;`
  4. `h->next = q;`

### 第二步：逐条追踪指针变化

初始状态假设链表为：`h → A → B → C → D → NULL`，p指向B

**语句1：`q = p->next;`**
```
操作：让q指向p的后继结点
结果：q指向C
链表：h → A → B → C → D → NULL
           ↑   ↑
           p   q
```

**语句2：`p->next = q->next;`**
```
操作：让p的next指向q的next（跳过C）
结果：C从链表中断开
链表：h → A → B ─┐
                 └→ D → NULL
           ↑
           p
独立：     C → D → NULL
           ↑
           q
```

**语句3：`q->next = h->next;`**
```
操作：让q的next指向头结点的后继（第一个元素A）
结果：q指向A
链表：h → A → B → D → NULL
      ↑    ↑
      |    p
      C ────┘
      ↑
      q
```

**语句4：`h->next = q;`**
```
操作：头结点的next指向q
结果：C成为新的第一个元素（头插法）
最终链表：h → C → A → B → D → NULL
               ↑
               q
```

**操作总结**：
1. 将q所指结点（C）从原位置删除
2. 将C插入到头结点之后（头插法）

### 第三步：逐项判断

**A. 在p所指结点后插入q所指结点 ❌**
```
初始：p指向B，q=p->next指向C
如果是"在B后插入C"，C本来就在B后面，无意义
实际操作：C被移到了链表开头，不是B后面
结论：错误
```

**B. 在q所指结点后插入p所指结点 ❌**
```
初始：p指向B，q指向C
如果是"在C后插入B"，结果应该是：h→A→C→B→D
实际结果：h→C→A→B→D（C在最前，不是B在C后）
结论：错误
```

**C. 将p所指结点移动到L的头结点之后 ❌**
```
初始：p指向B
如果是"将B移到开头"，结果应该是：h→B→A→C→D
实际结果：h→C→A→B→D（是C移到开头，不是B）
结论：错误
```

**D. 将q所指结点移动到L的头结点之后 ✅**
```
初始：q=p->next指向C
操作：
  1. 将C从原位置(B和D之间)删除
  2. 将C插入到头结点后（头插法）
结果：h→C→A→B→D
      ↑
      q正好指向C，且C在头结点之后

结论：完全正确 ✅
```

### 第四步：确认答案
操作本质：**头插法** - 将p的后继结点q移到链表开头

答案：**D**

## 易错点分析

### 陷阱1：混淆p和q的角色
❌ **错误理解**：以为操作的是p所指结点
✅ **正确理解**：
- p是定位指针，用于找到要移动的结点
- q才是被移动的结点（q = p->next）
- **移动的是q，不是p**

### 陷阱2：未完整追踪指针变化
❌ **错误做法**：只看最后一句`h->next = q`，以为q插到开头
✅ **正确方法**：必须逐句追踪
- 语句1-2：删除操作（将q从原位置断开）
- 语句3-4：插入操作（将q插到开头）

### 陷阱3：不理解头插法的典型特征
头插法的标准操作：
```c
// 1. 新结点next指向原首元素
new->next = h->next;
// 2. 头结点next指向新结点
h->next = new;
```

本题语句3-4就是标准头插法：
```c
q->next = h->next;  // q指向原首元素
h->next = q;        // 头结点指向q
```

## 知识点对比

| 链表操作 | 代码特征 | 结果 |
|----------|----------|------|
| **头插法** | `new->next=h->next; h->next=new;` | 插到第一个位置 |
| **尾插法** | `tail->next=new; tail=new;` | 插到最后位置 |
| **删除结点** | `p->next=p->next->next;` | 跳过后继结点 |
| **本题操作** | 先删除+再头插 | 移动结点到开头 |

**链表基本操作组合**：

| 组合 | 步骤 | 效果 |
|------|------|------|
| 删除+头插 | ①断开 ②插开头 | 移动到开头（本题） |
| 删除+尾插 | ①断开 ②插结尾 | 移动到结尾 |
| 删除+中间插 | ①断开 ②插指定位置 | 移动到中间 |

**记忆技巧**：
- 看到`q = p->next`：q是要操作的目标
- 看到`p->next = q->next`：删除q
- 看到`q->next = h->next; h->next = q`：头插法插入q
