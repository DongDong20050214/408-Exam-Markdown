# 二维数组地址计算真题汇总

## 2021年第03题

**元数据**
- 年份：[[../../01-真题原卷/2021年408真题#03|2021年第3题]]
- 科目：#数据结构
- 知识点：#数据结构/数组/地址计算
- 难度：⭐⭐
- 关联知识：[[知识库/数据结构/数组]]

**题干**

已知二维数组 A 按行优先方式存储，每个元素占用 1 个存储单元。若元素 A[0][0] 的存储地址是 100，A[3][3] 的地址是 220，则元素 A[5][5] 的存储地址是（）。

A. 295  B. 300  C. 301  D. 306

**答案**：B

**解题思路**

**二维数组按行优先存储地址公式**：
$$Addr(A[i][j]) = Base + (i \times n + j) \times size$$

其中：
- Base：起始地址
- n：数组列数
- size：每个元素占用的存储单元数

**已知条件**：
- $Addr(A[0][0]) = 100$
- $Addr(A[3][3]) = 220$
- size = 1

**步骤1：求列数n**

由公式：
$$Addr(A[3][3]) = 100 + (3 \times n + 3) \times 1$$
$$220 = 100 + 3n + 3$$
$$120 = 3n + 3$$
$$3n = 117$$
$$n = 39$$

**验证**：数组列数为39

**步骤2：计算A[5][5]的地址**

$$Addr(A[5][5]) = 100 + (5 \times 39 + 5) \times 1$$
$$= 100 + (195 + 5)$$
$$= 100 + 200$$
$$= 300$$

**选项分析**：
- ❌ A. 295 —— 计算错误
- ✅ B. 300 —— 正确答案
- ❌ C. 301 —— 可能是下标理解错误（从1开始）
- ❌ D. 306 —— 计算错误

**易错点分析**

❌ **错误1：列数计算错误**
- 误以为 $3 \times n + 3 = 120$ 中，$3 \times n = 120$，得 $n = 40$
- ✅ 正确：$3n + 3 = 120$，$3n = 117$，$n = 39$

❌ **错误2：使用行数代入公式**
- 误用"行数m"计算，认为A[3][3]是第4行第4列，有4行
- ✅ 正确：A[3][3]表示下标为3,3（从0开始），存储时需要前3行+前3列+当前元素 = 3×n+3个元素

❌ **错误3：忘记A[0][0]已占用位置**
- 误认为A[3][3]到A[0][0]之间有3×3=9个元素
- ✅ 正确：从A[0][0]到A[3][3]包含 3×n+3+1 个元素，但地址差是 3×n+3（因为A[0][0]是起点）

**知识点对比**

| 存储方式 | 地址计算公式 | 示例（下标从0开始） |
|---------|------------|------------------|
| **行优先** | $Base + (i \times n + j) \times size$ | A[2][3] = Base + (2×n+3)×size |
| **列优先** | $Base + (j \times m + i) \times size$ | A[2][3] = Base + (3×m+2)×size |

**下标起始对比**

| 下标起始 | A[3][3]的存储位置 | 前面有多少个元素 |
|---------|-----------------|----------------|
| **从0开始** | 第 3×n+3 个偏移 | 3×n+3 个 |
| **从1开始** | 第 (3-1)×n+(3-1) = 2×n+2 个偏移 | 2×n+2 个 |

**本题特殊性**：下标从0开始（常规定义）

**计算步骤可视化**

以n=39为例，按行优先存储：

| 元素 | 计算 | 偏移量 | 地址 |
|------|------|--------|------|
| A[0][0] | 0×39+0 | 0 | 100 |
| A[0][1] | 0×39+1 | 1 | 101 |
| A[1][0] | 1×39+0 | 39 | 139 |
| A[3][3] | 3×39+3 | 120 | 220 ✓ |
| A[5][5] | 5×39+5 | 200 | 300 ✓ |

**行优先存储的物理布局**

```
内存地址：100  101  102  ...  138  139  140  ...
元素：    A[0][0] A[0][1] ... A[0][38] A[1][0] A[1][1] ...
```

每一行有n=39个元素，A[3][3]在第4行第4列（下标从0），需要跨过3整行再加3个元素。

**相关公式总结**

| 数组类型 | 地址公式 | 说明 |
|---------|---------|------|
| **一维数组** | $Base + i \times size$ | i为下标 |
| **二维行优先** | $Base + (i \times n + j) \times size$ | n为列数 |
| **二维列优先** | $Base + (j \times m + i) \times size$ | m为行数 |
| **三维行优先** | $Base + ((i \times n_2 + j) \times n_3 + k) \times size$ | $n_2, n_3$为第2、3维大小 |

## 解题思路（标准化分析）

### 第一步：理解题目核心
二维数组按行优先存储，已知两个元素的地址，求第三个元素的地址。

**已知条件**：
- A[0][0]地址 = 100
- A[3][3]地址 = 220
- 每个元素1个存储单元
- 按行优先存储
- 求A[5][5]的地址

**二维数组行优先地址公式**：
$$Addr(A[i][j]) = Base + (i \times n + j) \times size$$

其中n是列数，size是元素大小。

### 第二步：求列数n

**建立方程**：
```
Addr(A[0][0]) = Base = 100
Addr(A[3][3]) = Base + (3×n + 3)×1 = 220
```

代入：
```
100 + 3n + 3 = 220
3n + 3 = 120
3n = 117
n = 39
```

**验证**：数组列数为39列。

### 第三步：计算A[5][5]地址

```
Addr(A[5][5]) = 100 + (5×39 + 5)×1
              = 100 + (195 + 5)
              = 100 + 200
              = 300
```

### 第四步：逐项判断

**A. 295 ❌**
```
可能错误：计算3n时出错
或者误算5×39的值
结论：错误
```

**B. 300 ✅**
```
计算：100 + 5×39 + 5 = 100 + 200 = 300
验证：正确 ✅
结论：正确答案
```

**C. 301 ❌**
```
可能错误：误以为下标从1开始
或者多加了1
结论：错误
```

**D. 306 ❌**
```
可能错误：列数算成40
5×40+5 = 205，100+205 = 305（接近但不等）
或其他计算错误
结论：错误
```

### 第五步：确认答案
列数39，A[5][5]地址300 → 答案：**B**

## 易错点分析

### 易错点1：求列数时方程列错
❌ 错误理解：地址差220-100=120，误认为3n=120
✅ 正确理解：按行优先存储，A[i][j]相对A[0][0]的偏移量是i×n+j(前i整行的所有元素+第i行前j列)。A[3][3]偏移量=3×n+3，地址差=220-100=120，所以3×n+3=120，解得3n=117，n=39。关键理解i×n+j不是行列相乘，而是"前i行列数总和(i×n)+当前行列偏移(j)"

| 步骤 | 错误做法 | 正确做法 |
|------|---------|---------|
| 建方程 | 3n = 120 | 3n + 3 = 120 |
| 求n | n = 40 ❌ | n = 39 ✅ |

### 易错点2：混淆元素个数和地址偏移
❌ 错误理解：从A[0][0]到A[3][3]有4×4=16个元素
✅ 正确理解：按行优先，A[i][j]相对A[0][0]的偏移量是i×n+j(前i行所有元素i×n个+第i行前j列j个)。A[3][3]偏移量=3×n+3不是3×3=9。易混淆点：A[3][3]是下标(3,3)即第4行第4列，但偏移不是4×4，而是前3整行(每行n个)+第4行前3列，共3n+3个。地址差=偏移量×元素大小

### 易错点3：行优先公式记忆错误
❌ 错误公式：Addr(A[i][j])=Base+(i+j×n)×size(列优先)
✅ 正确公式：Addr(A[i][j])=Base+(i×n+j)×size(行优先)。行优先是按行逐个存储，先存完第0行所有n个元素再存第1行，A[i][j]前面有i整行(i×n个)+第i行前j列(j个)，共i×n+j个。列优先是按列存储，A[i][j]前面有j整列(j×m个)+第j列前i行(i个)，共j×m+i个。记忆：行优先行在前(i×n)，列优先列在前(j×m)

| 存储方式 | 公式 | 含义 |
|---------|------|------|
| **行优先** | i×n + j | 前i整行 + 第i行前j列 |
| **列优先** | j×m + i | 前j整列 + 第j列前i行 |

## 知识点对比

### 行优先vs列优先地址计算

| 方式 | 公式 | 例：A[3][3] (n=39,m=?) |
|------|------|---------------------|
| 行优先 | Base + (i×n+j)×size | 100 + (3×39+3)×1 = 220 |
| 列优先 | Base + (j×m+i)×size | 100 + (3×m+3)×1 = 220 |

### 二维数组存储可视化（n=39）

```
行优先存储顺序：
地址: 100  101  102  ...  138  139  140  ...  220  ...  300
元素: [0][0][0][1]...[0][38][1][0][1][1]...[3][3]...[5][5]
      ←────第0行────→←────第1行────→
```

**每行39个元素**：
- 第0行：地址100-138
- 第1行：地址139-177
- 第2行：地址178-216
- 第3行：地址217-255，A[3][3]在220
- 第4行：地址256-294
- 第5行：地址295-333，A[5][5]在300

### 地址计算通用步骤

| 步骤 | 操作 | 本题示例 |
|------|------|---------|
| 1 | 确定存储方式 | 行优先 |
| 2 | 列出地址公式 | Base + (i×n+j)×size |
| 3 | 利用已知条件求参数 | 由A[3][3]求n=39 |
| 4 | 代入目标元素坐标 | A[5][5]: 100+200=300 |

### 常见参数求解

| 已知 | 未知 | 求解方法 |
|------|------|---------|
| 两个元素地址 | 列数n | 列方程求解 |
| 首尾元素地址 | 数组大小 | 地址差÷元素大小 |
| 地址和坐标 | 存储方式 | 验证行优先/列优先 |

**本题关键公式**：
$$n = \frac{Addr(A[3][3]) - Addr(A[0][0]) - 3}{3} = \frac{120-3}{3} = 39$$
