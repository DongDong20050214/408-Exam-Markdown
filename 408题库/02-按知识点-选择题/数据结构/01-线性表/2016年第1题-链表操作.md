# 2016年第1题

**元数据**
- 年份：2016年第1题
- 难度：⭐⭐

**题干**：已知表头元素为 $c$ 的单链表在内存中的存储状态如右表所示。

现将 f 存放于 1014H 处并插入到单链表中，若 f 在逻辑上位于 a 和 e 之间，则 a, e, f 的 "链接地址" 依次是（）。

| 地址 | 元素 | 链接地址 |
|------|------|----------|
| 1000H | a | 1010H |
| 1004H | b | 100CH |
| 1008H | c | 1000H |
| 100CH | d | NULL |
| 1010H | e | 1004H |
| 1014H | f | ? |

A. $1010\mathrm{~H}, 1014\mathrm{~H}, 1004\mathrm{H}$

B. 1010H, 1004H, 1014H

C. $1014\mathrm{~H}, 1010\mathrm{~H}, 1004\mathrm{H}$

D. 1014H, 1004H, 1010H

**答案**：A

**考点分析**：

本题考查单链表的插入操作和指针理解。关键点：
1. **链表逻辑顺序分析**：从表头 c (1008H) 开始，c->next = 1000H (a)，a->next = 1010H (e)，e->next = 1004H (b)，b->next = 100CH (d)，d->next = NULL。原链表逻辑顺序为：c → a → e → b → d
2. **插入操作**：在 a 和 e 之间插入 f，逻辑顺序变为 c → a → f → e → b → d。需要修改：a 的 next 指向 f (1014H)，f 的 next 指向 e (1010H)，e 的 next 保持不变 (1004H)
3. **答案验证**：a 的链接地址改为 1014H (指向f)，e 的链接地址保持 1004H (指向b)，f 的链接地址为 1010H (指向e)，即选项 A

---

## 解题思路

### 第一步：理解题目核心
单链表通过指针链接，物理位置不连续。题目给定f的物理地址1014H，要求将f插入到a和e之间，需要修改相关节点的链接地址（next指针）。

### 第二步：分析原链表逻辑顺序
从表头c (1008H) 开始追踪：
- c->next = 1000H → a
- a->next = 1010H → e
- e->next = 1004H → b
- b->next = 100CH → d
- d->next = NULL

**原链表逻辑顺序：** c → a → e → b → d

### 第三步：执行插入操作
**目标：** 在a和e之间插入f，链表变为 c → a → f → e → b → d

**需要修改的链接地址：**
1. a的next：从1010H（指向e）改为 **1014H**（指向f）
2. f的next：设为 **1010H**（指向e）
3. e的next：保持 **1004H**（指向b，不变）

### 第四步：逐项分析

- ❌ A. 1010H, 1014H, 1004H：a的链接应为1014H（指向f），不是1010H
- ❌ B. 1010H, 1004H, 1014H：a的链接错误，且f应指向e(1010H)不是1014H
- ❌ C. 1014H, 1010H, 1004H：f应指向e(1010H)，不是b(1004H)
- ✅ D. 1014H, 1004H, 1010H：a→f(1014H), e→b(1004H不变), f→e(1010H)完全正确

### 第五步：确认答案
插入f到a和e之间，修改链接：a→1014H, e→1004H(不变), f→1010H → 答案：**D**

**注：若原答案标注为A，应为标注错误。根据链表插入逻辑，D为正确答案。**

## 易错点分析

### 易错点1：混淆"物理地址"与"链接地址"
❌ 错误理解：链接地址就是节点自身的物理地址，f存储在1014H所以f的链接地址是1014H
✅ 正确理解：物理地址(存储地址)是节点存储的内存位置，如f在1014H；链接地址(next指针值)是节点的next域存储的值，指向下一个节点的物理地址。节点a物理地址1000H，a的链接地址1014H(a->next指向f)；节点f物理地址1014H，f的链接地址1010H(f->next指向e)。链接地址不是"自己在哪"而是"自己指向谁"

### 易错点2：单链表插入操作的指针修改顺序
❌ 错误理解：先修改前驱指向新节点，再让新节点指向后继
✅ 正确理解：单链表插入必须按正确顺序，否则链断裂。正确顺序：①先保存后继(q=p->next)，②新节点指向后继(newNode->next=q)，③前驱指向新节点(p->next=newNode)。若颠倒顺序先p->next=newNode，原后继地址丢失导致链断裂。本题：先f->next=1010H(f指向e)，再a->next=1014H(a指向f)。口诀：先存后继，新指旧，前指新

| 操作步骤 | 代码 | 注意事项 |
|---------|------|---------|
| ①保存后继 | q = p->next | 必须先保存，否则链断裂 |
| ②新节点指向后继 | newNode->next = q | f指向e |
| ③前驱指向新节点 | p->next = newNode | a指向f |

**本题操作分解：**
```c
// 初始：a->next = 1010H (e)
// 目标：a → f → e

// 步骤1：f指向e
f->next = 1010H;  // f (1014H) 指向 e (1010H)

// 步骤2：a指向f  
a->next = 1014H;  // a (1000H) 指向 f (1014H)

// 结果：a的链接1014H，e的链接1004H，f的链接1010H
```
