# 链表合并真题汇总

## 2013年第01题

**元数据**
- 年份：[../../01-真题原卷/2013年408真题#01|2013年第1题](../../01-真题原卷/2013年408真题#01|2013年第1题.md)
- 科目：#数据结构
- 知识点：#数据结构/链表/合并
- 难度：⭐⭐
- 关联知识：[知识库/数据结构/链表](知识库/数据结构/链表.md)

**题干**

已知两个长度分别为 m 和 n 的升序链表, 若将它们合并为一个长度为 m + n 的降序链表, 则最坏情况下的时间复杂度是 ( )。A. O(n)  B. O(mn)  C. O(min(m,n))  D. O(max(m,n))

**答案**：D

**解题思路**

**合并过程**：
1. 双指针遍历两个升序链表
2. 比较节点值，较小的插入结果链表头部（降序）
3. 直到两个链表都遍历完

**时间复杂度分析**：
- 需要访问所有节点：m+n个
- 每个节点访问1次
- 总时间：O(m+n)

**化简O(m+n)**：
- 当m≈n时：O(m+n)=O(2m)=O(m)=O(n)
- 当m>>n或n>>m时：O(m+n)=O(max(m,n))
- 因此O(m+n)=O(max(m,n))

**选项分析**：
- ❌ A. O(n) —— 忽略了m
- ❌ B. O(mn) —— 错误，不是嵌套循环
- ❌ C. O(min(m,n)) —— 只遍历短链表不够
- ✅ D. O(max(m,n)) —— 等价于O(m+n)

**易错点分析**

❌ **错误1：选B，认为是O(mn)**
- 误以为是嵌套遍历
- ✅ 正确：是并行遍历，O(m+n)

❌ **错误2：选C，认为短链表遍历完就结束**
- 忽略长链表剩余部分
- ✅ 正确：必须处理完所有节点

**知识点对比**

| 操作 | 时间复杂度 |
|------|-----------|
| **合并两个有序链表** | O(m+n) |
| **查找链表元素** | O(n) |
| **插入链表头部** | O(1) |

**复杂度等价关系**

$$O(m+n) = O(\max(m,n))$$

因为：
$$m+n \leq 2\max(m,n)$$

**关键理解**：
- 每个节点访问1次，共m+n个节点
- O(m+n)在选项中等价于O(max(m,n))

## 解题思路

### 第一步：理解合并过程
两个升序链表合并成降序链表的核心操作：
- 使用双指针分别遍历两个链表
- 每次比较两个指针指向的节点值
- 将较小的节点插入结果链表的头部（实现降序）
- 继续遍历直到两个链表都处理完毕

### 第二步：逐项分析

**A. O(n) ❌**
```
详细分析：该选项只考虑了其中一个链表的长度n
判断依据：合并操作需要遍历两个链表的所有节点
错误原因：忽略了长度为m的链表,当m>>n时明显不对
结论：错误
```

**B. O(mn) ❌**
```
详细分析：认为合并操作是嵌套循环
判断依据：实际上是两个链表的并行遍历,不是嵌套
错误原因：混淆了双指针遍历和嵌套循环
具体对比：
  嵌套循环：对m中每个元素,遍历n中所有元素 → O(mn)
  双指针：两个指针分别向后移动,总共移动m+n次 → O(m+n)
结论：错误
```

**C. O(min(m,n)) ❌**
```
详细分析：认为短链表遍历完就结束了
判断依据：合并操作必须处理所有m+n个节点
错误原因：忽略了长链表的剩余部分
反例：m=100,n=1,短链表结束后还有99个节点要处理
结论：错误
```

**D. O(max(m,n)) ✅**
```
详细分析：合并需要访问所有m+n个节点,每个节点访问1次
数学等价性：O(m+n) = O(max(m,n))
证明：
  因为 max(m,n) ≤ m+n ≤ 2·max(m,n)
  所以 O(m+n) = O(max(m,n))
判断依据：这是标准的时间复杂度表示方式
结论：正确 ✅
```

### 第三步：确认答案
合并操作需要遍历两个链表的所有节点,时间复杂度为O(m+n),在选项中等价表示为O(max(m,n)) → 答案：**D**

## 易错点分析

### 易错点1：混淆双指针遍历和嵌套循环
❌ 错误理解：认为双指针是嵌套关系,时间复杂度O(mn)
✅ 正确理解：双指针是并行遍历,两个指针独立移动,时间复杂度O(m+n)

| 遍历方式 | 代码特征 | 时间复杂度 | 典型场景 |
|---------|---------|-----------|---------|
| 嵌套循环 | for i in m: for j in n | O(mn) | 矩阵遍历 |
| 双指针并行 | while i<m or j<n | O(m+n) | 合并有序序列 |

### 易错点2：忽略长链表剩余部分
❌ 错误理解：短链表遍历完就结束,复杂度O(min(m,n))
✅ 正确理解：必须处理完所有节点,包括长链表的剩余部分

**具体例子**：
```
链表1: 1→3→5 (m=3)
链表2: 2 (n=1)

短链表结束后:
- 已比较2次
- 链表1还剩5需要处理
- 总操作次数：m+n=4次
```

### 易错点3：不理解O(m+n)与O(max(m,n))的等价性
| 表达式 | 含义 | 关系 |
|-------|------|------|
| O(m+n) | 精确描述 | 实际复杂度 |
| O(max(m,n)) | 上界估计 | 数学等价 |
| O(min(m,n)) | 下界估计 | 不适用本题 |
