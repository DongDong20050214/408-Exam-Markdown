# 2011年第3题

**元数据**
- 年份：2011年第3题
- 科目：#数据结构
- 难度：⭐⭐

**题干**

已知循环队列存储在一维数组 A[0...n-1] 中，且队列非空时 front 和 rear 分别指向队头元素和队尾元素。若初始时队列为空，且要求第 1 个进入队列的元素存储在 A[0] 处，则初始时 front 和 rear 的值分别是（）。

A. 0,0 

B. $0, n - 1$ 

C. n-1,0 

D. $n - 1, n - 1$ 

**答案**：D

**考点分析**

本题考查循环队列的初始化问题。题目要求第1个元素存储在 A[0] 处，且 front 和 rear 分别指向队头元素和队尾元素（而非指向队尾元素的下一个位置）。当队列为空时，可以设置 front = rear = n-1，这样第一次入队时，rear = (rear+1) % n = 0，元素存入 A[0]，同时 front 也指向 A[0]。这种设计保证了第一个元素正确存储在 A[0] 位置。

## 解题思路（逐项分析法）

### 第一步：明确题目条件
- 数组：A[0...n-1]
- 非空时：front和rear指向队头和队尾元素
- 要求：第1个元素存A[0]
- 问：初始front和rear的值

### 第二步：分析入队操作
入队操作通常：`rear = (rear+1) % n`

第1个元素要存A[0]：
```
入队前：rear = ?
入队后：rear = (rear+1) % n = 0
```
反推：`(rear+1) % n = 0`
解得：`rear = n-1`

### 第三步：逐项判断

**A. 0,0 ❌**
```
front=0, rear=0
入队：rear = (0+1)%n = 1
第1个元素存A[1]，不是A[0]
结论：错误
```

**B. 0, n-1 ❌**
```
front=0, rear=n-1
入队：rear = (n-1+1)%n = 0
第1个元素存A[0] ✓
但front=0已占用，逻辑矛盾
结论：错误
```

**C. n-1, 0 ❌**
```
front=n-1, rear=0
入队：rear = (0+1)%n = 1
第1个元素存A[1]，不是A[0]
结论：错误
```

**D. n-1, n-1 ✅**
```
front=n-1, rear=n-1
入队：rear = (n-1+1)%n = 0
第1个元素存A[0] ✓
此时非空，front也要指向队头：
front = 0（更新为队头位置）
结论：正确 ✅
```

### 第四步：确认答案
答案：**D**

## 易错点分析

### 陷阱1：混淆两种循环队列定义
❌ 错误理解：rear指向队尾的下一个位置
✅ 正确理解：本题rear指向队尾元素本身

### 陷阱2：忽略空队列的特殊性
❌ 错误理解：空队列front=rear任意值
✅ 正确理解：需满足第1个元素存A[0]的约束

## 知识点对比

| 定义 | rear含义 | 空队列判定 | 满队列 |
|------|---------|-----------|--------|
| 方式1 | 队尾下一个 | front==rear | (rear+1)%n==front |
| 方式2 | 队尾元素 | front==rear | 需额外标志 |
| 本题 | 队尾元素 | 初始n-1,n-1 | 题目未涉及 |
