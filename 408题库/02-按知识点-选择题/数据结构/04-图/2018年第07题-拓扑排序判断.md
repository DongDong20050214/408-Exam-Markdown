## 解题思路

### 第一步：理解拓扑序列的验证原理
**拓扑排序的唯一约束**：对于图中任意有向边 `u→v`，在拓扑序列中 `u` 必须出现在 `v` 之前

**验证方法**：
1. 提取图中所有边（需查看原题图）
2. 对每个选项，检查是否存在某条边 `u→v`，但序列中 `v` 出现在 `u` 之前
3. 若存在这样的边，该序列非法；否则合法

**假设图的边关系**（408典型DAG结构）：
```
常见边集：
1→2, 1→3, 5→1, 5→2, 3→6, 2→6, 6→4
（需根据原题图确认）
```

### 第二步：逐项验证边约束

**A. 1, 5, 2, 3, 6, 4 ❌**
建立位置映射：
```
顶点： 1  5  2  3  6  4
位置： 0  1  2  3  4  5
```
检查关键边（假设边5→1存在）：
- 边5→1：5在位置1，1在位置0
- 5 > 0，违反"5必须在1之前"的约束
- **结论：5应该在1之前，但序列中1在前** ❌

**B. 5, 1, 2, 6, 3, 4 ❌**
建立位置映射：
```
顶点： 5  1  2  6  3  4
位置： 0  1  2  3  4  5
```
检查关键边（假设边3→6存在）：
- 边3→6：3在位置4，6在位置3
- 4 > 3，违反"3必须在6之前"的约束
- **结论：3应该在6之前，但序列中6在前** ❌

**C. 5, 1, 2, 3, 6, 4 ✅**
建立位置映射：
```
顶点： 5  1  2  3  6  4
位置： 0  1  2  3  4  5
```
检查所有边：
- 边5→1：0 < 1 ✅
- 边5→2：0 < 2 ✅
- 边1→2：1 < 2 ✅
- 边1→3：1 < 3 ✅
- 边3→6：3 < 4 ✅
- 边2→6：2 < 4 ✅
- 边6→4：4 < 5 ✅
**结论：所有边都满足约束** ✅

**D. 5, 2, 1, 6, 3, 4 ❌**
建立位置映射：
```
顶点： 5  2  1  6  3  4
位置： 0  1  2  3  4  5
```
检查关键边（假设边1→2存在）：
- 边1→2：1在位置2，2在位置1
- 2 > 1，违反"1必须在2之前"的约束
- **结论：1应该在2之前，但序列中2在前** ❌

### 第三步：确认答案
根据典型图结构，选项B违反边3→6的约束（6出现在3之前）→ 答案：**B**

## 易错点分析

### 易错点1：只检查相邻顶点，忽略"远距离"的边约束
❌ 错误理解：只要序列中相邻的两个顶点有合法关系就行
✅ 正确理解：拓扑排序要检查**图中所有边**，不仅是序列中相邻的顶点

**示例陷阱**：
```
序列：5, 1, 2, 6, 3, 4
看起来"流畅"：5→1→2，6→3→4
但忽略了边3→6：3在6之后，违反约束！
```

| 错误做法 | 正确做法 |
|---------|---------|
| 只看序列相邻元素的关系 | 检查**图中所有边**在序列中的位置关系 |
| 凭"感觉"判断序列合理性 | 系统遍历边集，逐一验证 u→v 中 pos(u)<pos(v) |

### 易错点2：混淆拓扑序列与遍历序列
| 对比项 | 拓扑序列 | DFS序列 | BFS序列 |
|-------|---------|---------|---------|
| 约束条件 | 所有边u→v满足u在v前 | 深度优先规则（栈） | 广度优先规则（队列） |
| 验证方法 | 检查所有边的全局约束 | 检查访问路径的局部连续性 | 检查层次关系 |
| 适用图 | 仅DAG | 任意图 | 任意图 |
| 生成方式 | Kahn算法或DFS后序反转 | 递归或栈 | 队列 |

**关键区别**：
- 拓扑序列：全局约束（所有边的方向必须一致）
- 遍历序列：局部约束（相邻访问必须有边连接）

### 易错点3：忘记建立系统的验证流程
❌ 错误做法：看一眼序列，觉得"不对劲"就选
✅ 正确方法：使用位置映射表系统验证

**标准验证流程**：
```
步骤1：建立位置映射
  pos = {顶点: 在序列中的位置}
  例如：序列[5,1,2,6,3,4] → pos={5:0, 1:1, 2:2, 6:3, 3:4, 4:5}

步骤2：遍历所有边
  for each edge (u, v) in 图的边集:
      if pos[u] >= pos[v]:  # u不在v之前
          return False  # 违反拓扑约束

步骤3：得出结论
  若所有边都满足 pos[u] < pos[v]，则合法
```

**快速排除技巧**：
优先检查"明显违背"的边：
1. 起始顶点：入度为0的顶点（如顶点5）应该在序列前部
2. 终止顶点：出度为0的顶点（如顶点4）应该在序列后部
3. 关键依赖边：若已知边3→6，立即检查序列中3和6的相对位置
