- 难度：⭐⭐⭐

**题干**

修改递归方式实现的图的深度优先搜索（DFS）算法，将输出（访问）顶点信息的语句移到退出递归前（即执行输出语句后立刻退出递归）。采用修改后的算法遍历有向无环图 $G$，若输出结果中包含 $G$ 中的全部顶点，则输出的顶点序列是 $G$ 的（）。

A. 拓扑有序序列

B. 逆拓扑有序序列

C. 广度优先搜索序列

D. 深度优先搜索序列

**答案**：B

**考点分析**

本题考查DFS算法变形与拓扑排序的关系。

标准DFS递归框架：
```c
void DFS(Graph G, int v) {
    visited[v] = true;
    visit(v);  // 标准位置：进入时访问
    for (每个邻接点w) {
        if (!visited[w])
            DFS(G, w);
    }
}
```

修改后的DFS：
```c
void DFS(Graph G, int v) {
    visited[v] = true;
    for (每个邻接点w) {
        if (!visited[w])
            DFS(G, w);
    }
    visit(v);  // 修改位置：退出前访问
}
```

**分析**：
- 将访问操作移到退出递归前，意味着先递归访问所有后继节点，最后才输出当前节点
- 对于有向无环图（DAG），这相当于"后序遍历"
- 在DAG中，如果存在边 $u \to v$，DFS从u出发必然先递归访问v，然后才输出u
- 因此v的输出在u之前，这与拓扑排序（u在v之前）相反
- 得到的是**逆拓扑有序序列**

**拓扑排序与DFS的关系**：
- 标准拓扑排序：入度为0的顶点在前
- DFS后序遍历：出度处理完的顶点在后
- DFS的退出顺序恰好是拓扑排序的逆序

---

## 解题思路

### 第一步：理解题目核心
修改后DFS将输出移到退出递归前，变为后序遍历。对于DAG，后序遍历使得后继先输出、前驱后输出，得到逆拓扑序。

### 第二步：逐项分析

**A. 拓扑有序序列 ❌**
```
详细分析：拓扑序要求u→v时u在v前
判断依据：后序遍历先递归访问所有后继，再输出当前节点，导致v在u之前输出，与拓扑序相反
结论：顺序颠倒
```

**B. 逆拓扑有序序列 ✅**
```
详细分析：后序遍历的输出特性
判断依据：
  1. 退出递归前输出=后序遍历
  2. 存在边u→v时，DFS(u)先递归DFS(v)
  3. v及其后继全部输出后，才输出u
  4. 结果：v在u前，是拓扑序的逆序
结论：正确答案
```

**C. 广度优先搜索序列 ❌**
```
详细分析：BFS是按层次遍历
判断依据：BFS使用队列按距离源点的跳数访问，与DFS递归深入再回溯的机制完全不同
结论：遍历策略不同
```

**D. 深度优先搜索序列 ❌**
```
详细分析：标准DFS在进入时输出
判断依据：标准DFS是前序遍历，修改后变为后序遍历，输出序列完全不同
结论：访问时机改变
```

### 第三步：确认答案
后序遍历DAG，后继先于前驱输出，得到逆拓扑序 → 答案：**B**

## 易错点分析

### 易错点1：混淆前序和后序遍历的结果
❌ 错误：认为只是改变输出位置，结果应该类似
✅ 正确：前序（进入时输出）和后序（退出前输出）得到的序列完全相反

### 易错点2：DFS访问时机对DAG的影响
| 访问时机 | 遍历类型 | 对DAG结果 | 原因 |
|---------|---------|----------|------|
| 进入递归时 | 前序遍历 | 普通DFS序列 | 先输出再递归 |
| 退出递归前 | 后序遍历 | 逆拓扑序 | 先递归后输出，后继先完成 |
