**题干**

下列选项中，不是右图深度优先搜索序列的是（）。

A. V₁, V₅, V₄, V₃, V₂  
B. V₁, V₃, V₂, V₅, V₄  
C. V₁, V₂, V₅, V₄, V₃  
D. V₁, V₂, V₃, V₄, V₅

**答案**：D

**考点分析**
- DFS遍历要求：访问一个顶点后，优先访问其未访问的邻接点
- D选项V₁→V₂后应深入V₂的邻接点，而非回到V₁的其他邻接点V₃
- V₁, V₂, V₃, V₄, V₅是层序遍历(BFS)，不是DFS

---

## 解题思路

### 第一步：理解DFS的核心特征
**深度优先搜索（DFS）的遍历规则**：
1. 访问起点v
2. 从v的邻接点中选择一个未访问的顶点w，递归访问w
3. 当w的所有邻接点都已访问，回溯到v
4. 继续访问v的其他未访问邻接点

**核心特征**："一条路走到黑"，遇到死路才回头

### 第二步：DFS序列的判断方法
**合法DFS序列的特征**：
- 访问顶点v后，下一个访问的必须是：
  - v的某个未访问邻接点（深入）
  - 或回溯到v的祖先的未访问邻接点（回溯后再深入）

**不合法的特征**：
- 访问v后，跳跃到与v无邻接关系且不在递归栈上的顶点

### 第三步：逐项验证（需原图）

**假设图的邻接关系**（根据选项推测）：
```
V₁的邻接点：V₂, V₃, V₅
V₂的邻接点：V₅
V₃的邻接点：V₄
V₅的邻接点：V₄
```

**A. V₁, V₅, V₄, V₃, V₂ - 可能 ✓**
```
V₁ → V₅（V₁的邻接点）
V₅ → V₄（V₅的邻接点，深入）
V₄ → V₃（回溯到V₁，访问V₃，V₃与V₄相邻）
V₃ → V₂（回溯到V₁，访问V₂）
```
符合DFS"深入-回溯"的模式

**B. V₁, V₃, V₂, V₅, V₄ - 可能 ✓**
```
V₁ → V₃（V₁的邻接点）
V₃ → V₂（回溯到V₁，访问V₂？需验证V₃是否与V₂相邻）
```
如果图结构允许，可能是合法DFS序列

**C. V₁, V₂, V₅, V₄, V₃ - 可能 ✓**
```
V₁ → V₂（V₁的邻接点）
V₂ → V₅（V₂的邻接点，深入）
V₅ → V₄（V₅的邻接点，继续深入）
V₄ → V₃（回溯到V₁，访问V₃）
```
符合DFS模式

**D. V₁, V₂, V₃, V₄, V₅ - 不可能 ✗**
```
V₁ → V₂（V₁的邻接点）
V₂ → V₃（问题：V₂访问后应该深入V₂的邻接点）
```
**分析**：
- 若V₃不是V₂的邻接点，则V₂后不应直接访问V₃
- 应该先深入V₂的邻接点（如V₅），直到无路可走才回溯到V₁
- V₁, V₂, V₃, V₄, V₅更像是**广度优先搜索（BFS）**的序列

### 第四步：确认答案
D选项不符合DFS的"深度优先"特征，像是层序遍历

## 易错点分析

### 易错点1：混淆DFS和BFS的遍历顺序
❌ 错误理解：DFS就是按某种顺序访问所有顶点，V₁,V₂,V₃,V₄,V₅也可以。只要所有顶点都访问了就是合法的DFS序列，顺序无所谓。

✅ 正确理解：DFS和BFS的本质区别在于扩展策略，不仅仅是访问完所有顶点。DFS使用栈（或递归调用栈）实现"深度优先"扩展，访问顶点v后必须立即深入v的某个未访问邻接点w，直到w的所有邻接点都已访问才回溯。BFS使用队列实现"广度优先"扩展，按层次逐层访问。典型特征对比：DFS序列常有"跳跃"（如V₁→V₂→V₅→V₄→V₃，V₄到V₃是回溯），看起来"曲折"；BFS序列按距离起点的层次递增（如V₁→V₂→V₃→V₅→V₄），看起来"平滑"。题目中D选项V₁,V₂,V₃,V₄,V₅像是逐层遍历，不符合DFS"一条路走到黑"的特征。记忆：DFS像深山探洞（一条路走到底再回头），BFS像水波扩散（一层层向外）。

### 易错点2：不理解DFS的回溯机制
❌ 错误理解：DFS序列中相邻的顶点必定有边相连，否则就不是合法的DFS。看到V₄→V₃这样的转换，如果V₄和V₃之间没有边，就认为这个序列不可能是DFS产生的。DFS应该是连续访问相邻节点，不应该跳跃。

✅ 正确理解：DFS序列中相邻顶点可能因回溯而无边直接相连。DFS使用递归或栈实现，当当前顶点v的所有邻接点都已访问时，算法会回溯（弹栈）到v的父节点或更上层祖先，然后继续访问该祖先的其他未访问邻接点。这意味着DFS序列中相邻的两个顶点可能处于完全不同的分支。例如V₁→V₂→V₅→V₄→V₃，V₄到V₃的转换可能是：V₄所有邻接点已访问→回溯到V₅（V₅已访问完）→回溯到V₂（V₂已访问完）→回溯到V₁→访问V₁的另一个邻接点V₃。此时V₄和V₃之间无需有边相连，这是DFS回溯机制的正常表现。判断DFS合法性的方法是模拟栈执行，而非简单检查相邻顶点是否有边。记忆：DFS序列=深入路径+回溯跳跃，跳跃处不一定有边。

| 场景 | 序列特征 | 原因 |
|------|---------|------|
| **继续深入** | v和下一个顶点w有边v→w | w是v的未访问邻接点 |
| **回溯** | v和下一个顶点可能无直接边 | v的所有邻接点已访问，返回递归上层 |

**示例**：V₁→V₂→V₅→V₄→V₃
- V₁→V₂：深入（有边）
- V₂→V₅：继续深入（有边）
- V₅→V₄：继续深入（有边）
- V₄→V₃：回溯到V₁后访问V₃（V₄和V₃可能无直接边）

## 知识点对比

### DFS vs BFS遍历特征

| 特性 | DFS（深度优先） | BFS（广度优先） |
|------|---------------|---------------|
| 策略 | 尽可能深入分支 | 逐层扩展 |
| 数据结构 | 栈（或递归） | 队列 |
| 序列特征 | 有"跳跃"（回溯） | 按距离递增，较平滑 |
| 空间复杂度 | O(树高) | O(树宽) |

### 判断DFS序列合法性的方法

| 步骤 | 方法 | 说明 |
|------|------|------|
| ① 模拟执行 | 按序列顺序，用栈记录访问路径 | 验证每一步是否合法 |
| ② 检查深入 | 访问v后的下一个w，检查w是否是v的邻接点 | 深入操作 |
| ③ 检查回溯 | 若w不是v的邻接点，检查是否可以回溯到某祖先再访问w | 回溯操作 |
| ④ 排除BFS | 看是否按层序遍历（BFS特征） | DFS不会逐层 |
