**题干**

下列关于最小生成树的叙述中，正确的是（）。I. 最小生成树的代价唯一  II. 所有权值最小的边一定会出现在所有的最小生成树中  III. 使用普里姆（Prim）算法从不同顶点开始得到的最小生成树一定相同  IV. 使用普里姆算法和克鲁斯卡尔（Kruskal）算法得到的最小生成树总不相同

A. 仅 I  B. 仅 II  C. 仅 I、III  D. 仅 II、IV

**答案**：A

**考点分析**
- I正确：代价唯一

---

## 解题思路

### 第一步：理解最小生成树的基本性质
**最小生成树（MST）**：
- 包含图中所有顶点
- 恰好有n-1条边（n为顶点数）
- 边的权值总和最小
- 是一棵树（连通且无环）

**关键问题**：
- MST是否唯一？
- MST的代价（总权值）是否唯一？
- 最小权值边是否必选？

### 第二步：逐项分析

**I. 最小生成树的代价唯一 - 正确 ✓**

**证明**：
假设存在两个MST T₁和T₂，代价分别为W₁和W₂
- 若W₁ < W₂，则T₂不是最小生成树（矛盾）
- 若W₂ < W₁，则T₁不是最小生成树（矛盾）
- 因此 W₁ = W₂，代价唯一

**注意**：代价唯一 ≠ MST唯一
- 不同的边组合可能产生相同的代价

**示例**：
```
图：  A---2---B
      |       |
      2       2
      |       |
      C---2---D

两种MST：
T₁: A-B, A-C, C-D（代价6）
T₂: A-B, B-D, A-C（代价6）
代价相同，但树结构不同
```

**II. 所有权值最小的边一定会出现在所有的MST中 - 错误 ✗**

**反例**：
```
图：  A---1---B
      |       |
      1       1
      |       |
      C-------D

权值最小的边：(A,B)=1, (A,C)=1, (B,D)=1
但MST只需选3条边连接4个顶点
可能的MST：A-B, A-C, B-D（不包含C-D）
```

**更明显的反例**：若图中有多条权值相同的最小边形成环，则环中必有一条边不被选中

**III. 使用Prim算法从不同顶点开始得到的MST一定相同 - 错误 ✗**

**Prim算法**：从起点逐步扩展，每次加入连接已选集合和未选集合的最小边

**反例**：
```
图：  A---2---B
      |       |
      2       2
      |       |
      C---2---D

从A开始：A→B（或A→C），然后扩展，可能得到MST₁
从B开始：B→A（或B→D），然后扩展，可能得到MST₂
```

当有多条相同权值边可选时，不同起点可能产生不同MST（但代价相同）

**IV. 使用Prim算法和Kruskal算法得到的MST总不相同 - 错误 ✗**

**错误原因**：
- 当MST唯一时，两种算法得到相同的树
- 当有多个MST时，两种算法可能得到相同或不同的树

**反例**（MST唯一）：
```
图：  A---1---B---2---C

MST唯一：A-B, B-C（代价3）
Prim和Kruskal都得到这个唯一的MST
```

### 第三步：逐项分析选项
**A. 仅I ✅**
只有I正确，最小生成树的代价唯一

**B. 仅II ❌**
II错误，最小权值边不一定都出现在所有MST中

**C. 仅I、III ❌**
III错误，不同起点可能得到不同MST

**D. 仅II、IV ❌**
II和IV都错误

## 易错点分析

### 易错点1：混淆"代价唯一"和"树唯一"
❌ 错误理解：最小生成树的结构必须唯一，因为如果有多个不同的树，它们的代价肯定不同，那就不是"最小"生成树了。所以I"代价唯一"是对的，同时也意味着树的结构唯一。如果有两棵树结构不同但代价相同，那说明算法实现有问题。

✅ 正确理解：最小生成树的**代价（总权值）必定唯一**，但**树的结构可能不唯一**。用反证法：若存在两个MST代价不同，则代价大的那个不是"最小"生成树，矛盾。所以所有MST代价必然相同。但当图中有权值相同的边时（如3条边权值都为2），选择不同的边组合可以构造出结构不同的树，只要总权值相同即可。例如4个顶点的图若有多条权值为2的边，可能有多种连接方式，代价都是6，都是合法MST。

### 易错点2：认为最小边必选
❌ 错误理解：权值最小的边一定会出现在最小生成树中，因为不选最小边而选更大的边，怎么可能让总代价最小呢？所以II"所有权值最小的边一定会出现在所有的MST中"是正确的。贪心算法就是要优先选择权值最小的边。

✅ 正确理解：权值最小的边不一定全部出现在MST中，关键看是否形成环。生成树的定义：n个顶点恰好n-1条边且无环。若图中有k条权值相同的最小边，且这k条边在图中形成环路（如三角形3条边权值都是1），则MST只能选其中k-1条，必须舍弃至少1条以保持无环性。Kruskal算法按边权从小到大加边，遇到会形成环的边就跳过，即使该边权值最小。例如完全图K₃（3个顶点全连接，边权都为1），只能选2条边，必有1条最小边不被选。

| 最小边情况 | 是否必选 | 原因 |
|---------|---------|------|
| **唯一的最小边** | ✓ 必选 | 不选则代价更大 |
| **多条相同最小边不形成环** | ✓ 都选 | 都能降低代价 |
| **多条相同最小边形成环** | ❌ 不全选 | 选其中n-1条，有一条会形成环 |

**反例**：
```
三角形图：A-1-B, B-1-C, C-1-A（3条边权值都是1）
MST只能选2条边（如A-B, B-C）
有1条最小边（C-A）不被选中
```

## 知识点对比

### MST的唯一性判断

| 条件 | MST是否唯一 | 代价是否唯一 |
|------|-----------|------------|
| 所有边权值都不同 | ✓ 唯一 | ✓ 唯一 |
| 存在相同权值边 | ❌ 可能不唯一 | ✓ **一定唯一** |
| 关键结论 | 树结构可能多样 | **代价必然唯一** |

### Prim vs Kruskal算法

| 算法 | 策略 | 起点依赖 | 得到的MST |
|------|------|---------|----------|
| **Prim** | 顶点扩展，每次加最小边 | 依赖起点选择 | 可能不同（权值相同时） |
| **Kruskal** | 边排序，避免环 | 不依赖起点 | 可能不同（权值相同时） |
| 共同点 | 都是贪心算法 | - | **代价一定相同** |

### 最小边的选择规则

| 场景 | 选择规则 | 示例 |
|------|---------|------|
| 唯一最小边 | 必选 | 图中只有一条权值为1的边 |
| 多个最小边分散 | 都选（不形成环） | 图的不同区域各有权值为1的边 |
| 多个最小边成环 | 选其中部分 | 三角形3条边权值相同，选2条 |
