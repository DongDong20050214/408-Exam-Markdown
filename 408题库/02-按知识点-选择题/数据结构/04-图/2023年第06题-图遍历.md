# 2023年第6题

**元数据**
- 年份：2023年第6题
- 科目：#数据结构
- 难度：⭐⭐

**题干**：已知无向连通图 $G$ 中各边的权值均为 1。下列算法中，一定能够求出图 $G$ 中从某顶点到其余各顶点最短路径的是（）。

I. 普里姆（Prim）算法

II. 克鲁斯卡尔(Kruskal) 算法

III. 图的广度优先搜索算法

A. 仅I

B. 仅 III

C. 仅I、II

D. I、II、III 

**答案**：B

**考点分析**：本题考查图的最短路径算法。Prim算法和Kruskal算法都是求最小生成树的算法，不能求最短路径。广度优先搜索(BFS)按层次遍历，在边权均为1的无向连通图中，BFS能够找到从起点到各顶点的最短路径，因为第一次访问到某顶点时一定是最短路径。

---

## 解题思路

### 第一步：理解题目关键条件
**题目要求**：求从某顶点到其余各顶点的**最短路径**
**关键条件**：边权均为1

**最短路径问题分类**：
- **单源最短路径**：从一个源点到其他所有顶点
- **边权为1**：可以简化为层次遍历问题

### 第二步：逐个算法分析

**I. Prim算法 - 不能求最短路径 ✗**

**算法目标**：求最小生成树（Minimum Spanning Tree）
- 连接所有顶点，总权值最小
- 是一棵树，不关心从某点到其他点的路径长度

**示例**：
```
图：A--1--B--1--C
    |           |
    1           1
    |           |
    D-----2-----E

最小生成树可能选：A-B, B-C, A-D, C-E（总权值5）
但A到E的路径：A→B→C→E（长度3）
而最短路径：A→D→E（长度3，但也可能走A→...其他路径）
```

**结论**：Prim算法不保证任意两点间路径最短

**II. Kruskal算法 - 不能求最短路径 ✗**

**算法目标**：也是求最小生成树
- 按边权从小到大加边，不形成环
- 同样只关心总权值，不关心点对点路径

**结论**：Kruskal算法也不能求最短路径

**III. 广度优先搜索（BFS）- 能求最短路径 ✓**

**算法特点**：
- 按层次遍历，先访问距离源点1步的顶点，再访问2步的，以此类推
- **边权为1时**，第一次访问到某顶点时，必定是最短路径

**证明**：
```
设源点为S，第一次访问到顶点V时，经过k条边
假设存在更短路径（经过k-1条边），那么V应该在第k-1层被访问
矛盾！所以第一次访问即为最短路径
```

**示例**：
```
图：  A---B
      |\ /|
      | X |
      |/ \|
      C---D

BFS从A出发：
层0: A（距离0）
层1: B, C（距离1）- 直接相连
层2: D（距离2）- 通过B或C到达

每个顶点第一次访问时的距离即为最短距离
```

**注意**：若边权不为1，BFS不能保证最短路径（需要Dijkstra等算法）

### 第三步：逐项分析选项
**A. 仅I ❌**
Prim算法求最小生成树，不求最短路径

**B. 仅III ✅**
边权为1时，BFS能求最短路径

**C. 仅I、II ❌**
两个都是最小生成树算法，不求最短路径

**D. I、II、III ❌**
I和II都不能求最短路径

## 易错点分析

### 易错点1：混淆最小生成树和最短路径
❌ 错误理解：最小生成树用最少的边（n-1条）连接所有顶点，而且总权值最小，所以树中任意两点的路径必然是它们之间的最短路径。因此Prim和Kruskal算法既能求最小生成树，也能求最短路径，答案应该选D（三个算法都能）。

✅ 正确理解：最小生成树（MST）和最短路径是两个完全不同的优化目标。MST优化的是**全局代价**（所有边权值之和最小），最短路径优化的是**点对点距离**（源点到某顶点的路径长度最小）。MST为了降低总权值，可能会选择一些局部绕远的连接方式。例如图中A-B权值10，A-C权值1，C-B权值1，MST选A-C-B总权2（虽然A到B在树中走2条边），而不选A-B直连权值10。MST树中A到B路径长度是2，但直连边A-B才是最短路径。

**反例**：
```
A--1--B--1--C
|           |
2           2
|           |
D-----1-----E

最小生成树：A-B, B-C, D-E, A-D（总权5，包含边权2）
A到E最短路径：A→D→E（长度3，走边权2+1）
但如果走A→B→C→E，长度也是3

最小生成树可能选A-B-C路径而不选A-D路径，
导致树中A到E要绕路
```

### 易错点2：认为BFS对所有图都能求最短路径
❌ 错误理解：BFS（广度优先搜索）按层次遍历，每层距离起点的距离相同，所以BFS总是能找到最短路径，无论边权是否相等。即使边权不同，BFS第一次到达某顶点时也是经过最少的边数，因此就是最短路径。所以III在任何图中都能求最短路径。

✅ 正确理解：BFS只在**边权为1（或所有边权相等）**时能保证找到最短路径。BFS按层次扩展，第k层顶点表示从源点经过k条边到达。边权为1时，k条边的路径长度=k，第一次访问到某顶点时就是最短距离。但边权不同时，"边数最少"≠"路径长度最短"。例如A到C有两条路径：A-B-C经过2条边但总权值为2+1=3，A-D-C经过2条边但总权值为5+1=6，BFS同时到达但实际最短路径是前者。需要Dijkstra等算法处理带权图。

| 图的边权 | BFS能否求最短路径 | 原因 | 应用算法 |
|---------|------------------|------|---------|
| **边权均为1** | ✓ 能 | 层次=距离，第一次访问即最短 | **BFS** |
| **边权相等但≠1** | ✓ 能 | 本质同上，层次×权值=距离 | BFS |
| **边权不同** | ❌ 不能 | 后访问的可能更短 | **Dijkstra** |
| **有负权边** | ❌ 不能 | BFS无法处理 | Bellman-Ford |

**反例（边权不同）**：
```
图：A--5--B--1--C
    |          
    2          
    |          
    D          

BFS从A：
层1: B(权5), D(权2)  - BFS认为B和D距离相同
层2: C(经B到达，总权6)

但实际：A→D→...若D连接C，可能更短
真实最短路径可能不是BFS第一次到达的路径
```

## 知识点对比

### 最小生成树 vs 最短路径

| 问题类型 | 目标 | 算法 | 结果结构 |
|---------|------|------|---------|
| **最小生成树** | 连接所有顶点，总权值最小 | Prim, Kruskal | 树（n-1条边） |
| **最短路径** | 源点到各点路径长度最小 | BFS(无权), Dijkstra | 最短路径树 |
| **关键区别** | 不保证点对点最短 | 保证源点到各点最短 | 目标不同 |

### 各类最短路径算法对比

| 算法 | 适用场景 | 时间复杂度 | 能否处理负权 |
|------|---------|-----------|-------------|
| **BFS** | 边权为1（无权图） | O(n+e) | N/A（无权） |
| **Dijkstra** | 非负权图 | O(n²)或O(elogn) | ❌ 不能 |
| Bellman-Ford | 有负权边 | O(ne) | ✓ 能 |
| Floyd | 全源最短路径 | O(n³) | ✓ 能 |
| SPFA | 有负权边（优化版） | 平均O(ke) | ✓ 能 |

### 本题关键辨析

| 算法 | 目标问题 | 能否求最短路径 | 原因 |
|------|---------|---------------|------|
| **Prim** | 最小生成树 | ❌ 否 | 树中路径不保证最短 |
| **Kruskal** | 最小生成树 | ❌ 否 | 同上 |
| **BFS** | 层次遍历 | ✓ 能（**边权为1时**） | 第一次访问=最短距离 |
