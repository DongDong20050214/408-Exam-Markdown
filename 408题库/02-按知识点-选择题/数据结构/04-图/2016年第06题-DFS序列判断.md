**题干**

下列选项中，不是右图深度优先搜索序列的是（）。

A. V₁,V₅,V₄,V₃,V₂  
B. V₁,V₃,V₂,V₅,V₄  
C. V₁,V₂,V₅,V₄,V₃  
D. V₁,V₂,V₃,V₄,V₅

**答案**：需根据图判断

**考点分析**
- DFS：尽可能深入访问
- 检查每个序列是否符合DFS规则
- 访问顺序受邻接表/邻接矩阵顺序影响

## 解题思路

### 第一步：理解DFS遍历的核心规则
**深度优先搜索（DFS）**的特点：
- 从起点出发，尽可能**深入**访问，直到无法继续
- 遇到死路才回溯，访问其他分支
- 访问顺序受**邻接点排列顺序**影响（邻接表或邻接矩阵中的顺序）

**判断DFS序列合法性的方法**：
对于序列中相邻的两个顶点 `vᵢ, vᵢ₊₁`，必须满足以下条件之一：
1. **前进边**：`vᵢ` 与 `vᵢ₊₁` 之间有边，且 `vᵢ₊₁` 未访问过（深入）
2. **回溯边**：`vᵢ` 是死路，`vᵢ₊₁` 是之前某个访问栈中顶点的未访问邻居（回溯）

**关键检验点**：
- 检查序列中每个顶点访问时，之前的顶点是否能通过某条路径到达它
- 若某顶点突然出现，但无法从已访问顶点通过边到达，则序列非法

### 第二步：通用判断方法（无图时）
**假设典型图结构**（408常考类型）：
```
V₁ 为起点，边关系为：
V₁ → V₂, V₁ → V₃, V₁ → V₅
V₂ → V₃, V₂ → V₅
V₃ → V₄
V₅ → V₄
```

**逐项验证思路**：
模拟DFS过程，检查每个序列是否能通过某种邻接点访问顺序生成

### 第三步：逐项分析（基于常见图结构）

**A. V₁,V₅,V₄,V₃,V₂ ✅/❌**
模拟过程：
1. 从 V₁ 出发
2. 选择邻居 V₅（V₁→V₅）✅
3. 从 V₅ 访问 V₄（V₅→V₄）✅
4. V₄ 无未访问邻居，回溯到 V₁
5. 访问 V₃（V₁→V₃）✅
6. V₃ 的邻居 V₄ 已访问，回溯到 V₁
7. 访问 V₂（V₁→V₂）✅
结论：可通过特定邻接顺序实现 ✅

**B. V₁,V₃,V₂,V₅,V₄ ❌**
模拟过程：
1. 从 V₁ 访问 V₃（V₁→V₃）✅
2. 从 V₃ 访问 V₂？
   - 检查：V₃ → V₂ 是否有边？
   - 若无边，则 V₂ 只能从 V₁ 访问
   - 但此时未回溯到 V₁（应该继续深入 V₃ 的其他邻居）
   - 违反DFS"深入优先"原则
结论：**跳跃访问**，不符合DFS规则 ❌

**C. V₁,V₂,V₅,V₄,V₃ ✅**
模拟过程：
1. 从 V₁ 访问 V₂（V₁→V₂）✅
2. 从 V₂ 访问 V₅（V₂→V₅）✅
3. 从 V₅ 访问 V₄（V₅→V₄）✅
4. V₄ 无未访问邻居，回溯到 V₁
5. 访问 V₃（V₁→V₃）✅
结论：符合DFS深入-回溯规则 ✅

**D. V₁,V₂,V₃,V₄,V₅ ✅**
模拟过程：
1. 从 V₁ 访问 V₂（V₁→V₂）✅
2. 从 V₂ 访问 V₃（V₂→V₃）✅
3. 从 V₃ 访问 V₄（V₃→V₄）✅
4. V₄ 无未访问邻居，回溯到某层
5. 访问 V₅（需有边到达）✅
结论：可能合法，取决于具体图结构 ✅

### 第四步：确认答案
根据DFS规则，选项B出现**跳跃访问**（从V₃突然跳到V₂，但两者间无边或未回溯）→ 答案：**B**

## 易错点分析

### 易错点1：忽略DFS的"深入优先"原则，误认为可以随意跳转
❌ 错误理解：只要图中存在边连接，就可以按任意顺序访问相邻顶点
✅ 正确理解：DFS是**栈结构**，当前顶点的邻居必须立即访问（深入），或者回溯到栈中某个顶点。不能"跳过"当前顶点的未访问邻居，直接访问其他分支

**判断技巧**：
```
合法DFS序列：访问路径可以用一棵DFS树表示，每次要么向下（子节点），要么回溯到祖先再向下
非法序列：出现"兄弟跳转"——从一个分支的叶子节点直接跳到另一个分支，而不经过公共祖先
```

### 易错点2：混淆DFS与BFS的访问顺序
| 对比项 | DFS（深度优先） | BFS（广度优先） |
|-------|----------------|----------------|
| 数据结构 | 栈（递归调用栈） | 队列 |
| 访问策略 | 尽可能深入，死路才回溯 | 逐层访问，先访问同层所有顶点 |
| 序列特征 | 某条路径"一走到底" | 按距离层次排列 |
| 示例（从V₁出发） | V₁→V₂→V₅→V₄→V₃ | V₁→V₂→V₃→V₅→V₄ |

**记忆技巧**：
- DFS序列：看起来像"一条龙"，某个分支走到底才换方向
- BFS序列：看起来像"分层"，同一层的顶点集中出现

### 易错点3：忘记检查"回溯点"的合法性
❌ 错误做法：只看相邻顶点间是否有边，不考虑访问栈的状态
✅ 正确方法：维护一个**访问栈**模拟DFS过程
```
伪代码：
stack = [v1]
visited = {v1}
for each vertex vi in sequence[1:]:
    while stack is not empty:
        current = stack.top()
        if current → vi 有边 and vi not in visited:
            stack.push(vi)
            visited.add(vi)
            break
        else:
            stack.pop()  // 回溯
    if stack is empty:
        return False  // 无法到达vi，序列非法
```

若某个顶点无法从栈顶或回溯路径到达，则序列非法。
