**题干**

若将 n 个顶点 e 条弧的有向图采用邻接表存储, 则拓扑排序算法的时间复杂度是 ( )。

A. O(n)  B. O(n + e)  C. O(n²)  D. O(n × e)

**答案**：B

**考点分析**
- 遍历所有顶点和边
- 时间复杂度O(n+e)

---

## 解题思路

### 第一步：理解拓扑排序算法流程
**拓扑排序**：对有向无环图（DAG）的顶点进行线性排序，使得对于每条边(u,v)，u在v之前

**基于邻接表的拓扑排序算法**（Kahn算法）：
```
1. 计算所有顶点的入度 - O(n+e)
2. 将入度为0的顶点入队 - O(n)
3. while 队列非空:
     出队一个顶点v - O(1)
     输出v
     for v的每个邻接顶点w:  - 总共遍历所有边
         w的入度减1
         if w的入度变为0:
             w入队
```

### 第二步：时间复杂度分析

**步骤1：计算入度**
- 遍历邻接表的所有边：O(e)
- 初始化n个顶点的入度数组：O(n)
- 小计：O(n+e)

**步骤2：初始化队列**
- 扫描n个顶点找入度为0的：O(n)

**步骤3：主循环**
- 每个顶点入队、出队各一次：O(n)
- 遍历所有顶点的邻接表：每条边被访问一次，O(e)
- 小计：O(n+e)

**总时间复杂度**：O(n+e) + O(n) + O(n+e) = O(n+e)

### 第三步：为什么是O(n+e)而不是O(n²)

**关键**：邻接表存储下，遍历所有边的时间是O(e)，不是O(n²)

**邻接表特点**：
- 顶点v的邻接表长度 = v的出度
- 所有邻接表长度之和 = 边数e
- 遍历所有邻接表 = 遍历所有边 = O(e)

**对比邻接矩阵**：
- 若用邻接矩阵存储，需要扫描n×n的矩阵找边，时间O(n²)
- 但题目明确说是**邻接表**存储

### 第四步：逐项分析
**A. O(n) ❌**
只考虑了顶点数，忽略了边的遍历
拓扑排序必须遍历所有边来更新入度

**B. O(n+e) ✅**
正确：遍历n个顶点 + 遍历e条边

**C. O(n²) ❌**
这是邻接矩阵存储时的复杂度
邻接表存储不需要O(n²)

**D. O(n×e) ❌**
高估了复杂度，每条边只被访问一次，不是n次

## 易错点分析

### 易错点1：混淆邻接表和邻接矩阵的复杂度
❌ 错误理解：拓扑排序需要遍历所有顶点并检查它们之间的关系，涉及到n个顶点和所有可能的n(n-1)个顶点对，所以时间复杂度应该是O(n²)。虽然题目说用邻接表存储，但遍历所有边还是需要检查每对顶点的关系，复杂度还是O(n²)，答案应该选C。

✅ 正确理解：存储结构直接决定遍历复杂度。**邻接表**是链式结构，每个顶点只存储其实际邻接的顶点，所有邻接表长度之和等于边数e，遍历所有邻接表时间为O(e)。**邻接矩阵**是n×n的二维数组，存储所有可能的顶点对关系（包括不存在的边），扫描整个矩阵需要O(n²)时间。拓扑排序需要遍历n个顶点（O(n)）和处理e条边更新入度（邻接表下O(e)），总时间O(n+e)。题目明确"邻接表存储"，不能用邻接矩阵的O(n²)。

**示例**：
```
4个顶点，3条边：1→2, 2→3, 3→4

邻接表：
1: [2]
2: [3]
3: [4]
4: []
遍历时间：O(4+3) = O(n+e)

邻接矩阵：
  1 2 3 4
1 0 1 0 0
2 0 0 1 0
3 0 0 0 1
4 0 0 0 0
扫描时间：O(16) = O(n²)
```

### 易错点2：认为每个顶点都要遍历所有边
❌ 错误理解：拓扑排序要处理n个顶点，每个顶点都需要查看图中的所有e条边来确定哪些边与它相关，所以总共需要n×e次操作，时间复杂度应该是O(n×e)，答案应该选D。这样才能保证每个顶点都检查了所有可能的依赖关系。

✅ 正确理解：邻接表的关键特性是**每个顶点只存储自己的出边**，处理顶点v时只需遍历v的邻接表（包含v的所有出边），不需要检查图中其他无关的边。拓扑排序中，每个顶点出队一次，遍历其邻接表更新后继顶点的入度。所有n个顶点的邻接表长度之和∑degree(v)=e（每条边恰好出现在其起点的邻接表中一次），因此遍历所有邻接表总时间O(e)，而非O(n×e)。每条边只被访问一次，不会因为有n个顶点就重复访问n次。

| 错误理解 | 正确理解 | 区别 |
|---------|---------|------|
| 每个顶点扫描所有e条边 | 每个顶点只扫描自己的邻接表 | 邻接表避免无关边 |
| n×e次操作 | 所有邻接表总长度=e次操作 | 每条边只访问1次 |
| O(n×e) | O(n+e) | 复杂度差异巨大 |

**关键理解**：
```
示例：4个顶点，边1→2, 2→3, 3→4

邻接表：
v1: [2]        - 长度1
v2: [3]        - 长度1  
v3: [4]        - 长度1
v4: []         - 长度0
总长度 = 1+1+1+0 = 3 = e

遍历所有邻接表 = 遍历3条边 = O(e)
不是4×3 = O(n×e)
```

## 知识点对比

### 邻接表vs邻接矩阵的时间复杂度

| 操作 | 邻接表 | 邻接矩阵 | 适用场景 |
|------|--------|---------|---------|
| 拓扑排序 | O(n+e) | O(n²) | 邻接表优于矩阵 |
| 判断边(u,v)是否存在 | O(degree(u)) | O(1) | 矩阵查询快 |
| 遍历v的邻接点 | O(degree(v)) | O(n) | 邻接表遍历快 |
| 空间复杂度 | O(n+e) | O(n²) | 邻接表省空间 |

### 图算法时间复杂度总结

| 算法 | 邻接表 | 邻接矩阵 | 备注 |
|------|--------|---------|------|
| **拓扑排序** | O(n+e) | O(n²) | 需遍历所有边 |
| DFS | O(n+e) | O(n²) | 需遍历所有边 |
| BFS | O(n+e) | O(n²) | 需遍历所有边 |
| Prim（最小生成树） | O(n²)或O(elogn) | O(n²) | 取决于实现 |
| Dijkstra（最短路径） | O(n²)或O(elogn) | O(n²) | 取决于实现 |

**记忆规律**：需要遍历边的算法，邻接表都是O(n+e)，邻接矩阵都是O(n²)
