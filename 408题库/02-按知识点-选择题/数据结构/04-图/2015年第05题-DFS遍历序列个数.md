# 图遍历真题汇总

## 2015年第05题

**元数据**
- 年份：[2015年第5题](../../../01-真题原卷/2015年408真题.md#05)
- 科目：#数据结构
- 知识点：#数据结构/图/遍历
- 难度：⭐⭐⭐
- 关联知识：[图的遍历](../../../../AI-知识点原子库/图的遍历.md)

**题干**

设有向图 G = (V, E), 顶点集 V = {v0, v1, v2, v3}, 边集 E = {<v0, v1>, <v0, v2>, <v0, v3>, <v1, v3>}。若从顶点 V0 开始对图进行深度优先遍历, 则可能得到的不同遍历序列个数是 ( )。A. 2  B. 3  C. 4  D. 5

**答案**：D

**考点分析**
- 从v0出发有3个邻接点，计算排列组合

---

## 解题思路

### 第一步：理解图结构和DFS序列生成规则
**题目给定信息**：
- 顶点集：V = {v0, v1, v2, v3}
- 边集：E = {<v0, v1>, <v0, v2>, <v0, v3>, <v1, v3>}
- 起点：v0

**边关系图示**：
```
    v0 ─→ v1
    │      │
    │      ↓
    ↓─→ v2 v3
```
即：v0指向v1, v2, v3；v1指向v3

### 第二步：分析DFS遍历的分支点
**DFS序列个数**取决于**每个顶点的邻接点选择顺序**

**关键分支点分析**：
1. **起点v0**有3个邻接点：v1, v2, v3 → 可选3个方向
2. 每个方向继续深入，检查是否还有未访问邻接点

### 第三步：枚举所有可能的DFS序列

**策略**：从v0出发，按不同顺序访问邻接点，用决策树展开

**分支1：先访问v1**
- 序列开头：`v0, v1, ...`
- v1的邻接点：v3（未访问）
- 继续：`v0, v1, v3, ...`
- v3无未访问邻接点，回溯到v0
- v0剩余邻接点：v2
- 完整序列：`v0, v1, v3, v2` ✅

**分支2：先访问v2**
- 序列开头：`v0, v2, ...`
- v2无邻接点（度为0），回溯到v0
- v0剩余邻接点：v1, v3（任选其一）
  - **子分支2.1**：选v1
    - `v0, v2, v1, ...`
    - v1的邻接点：v3
    - 完整序列：`v0, v2, v1, v3` ✅
  - **子分支2.2**：选v3
    - `v0, v2, v3, ...`
    - v3无未访问邻接点，回溯到v0
    - v0剩余邻接点：v1
    - 完整序列：`v0, v2, v3, v1` ✅

**分支3：先访问v3**
- 序列开头：`v0, v3, ...`
- v3无邻接点，回溯到v0
- v0剩余邻接点：v1, v2（任选其一）
  - **子分支3.1**：选v1
    - `v0, v3, v1, ...`
    - v1的邻接点v3已访问，回溯到v0
    - v0剩余邻接点：v2
    - 完整序列：`v0, v3, v1, v2` ✅
  - **子分支3.2**：选v2
    - `v0, v3, v2, ...`
    - v2无邻接点，回溯到v0
    - v0剩余邻接点：v1
    - 完整序列：`v0, v3, v2, v1` ✅

### 第四步：统计并逐项分析

**所有可能的DFS序列**（共5种）：
1. `v0, v1, v3, v2`
2. `v0, v2, v1, v3`
3. `v0, v2, v3, v1`
4. `v0, v3, v1, v2`
5. `v0, v3, v2, v1`

**A. 2 ❌**
序列个数远超2种
结论：计数错误 ❌

**B. 3 ❌**
序列个数超过3种
结论：计数不足 ❌

**C. 4 ❌**
序列个数超过4种
结论：仍然少算 ❌

**D. 5 ✅**
恰好等于完整枚举的结果
结论：正确 ✅

### 第五步：确认答案
通过决策树完整枚举，共5种DFS序列 → 答案：**D**

## 易错点分析

### 易错点1：遗漏某些分支，导致计数不完整
❌ 错误做法：只考虑v0的3个邻接点，认为答案是3
✅ 正确理解：DFS序列数不仅取决于第一步的选择，还要考虑**回溯后的再选择**

**计数公式（复杂图）**：
```
不能简单用排列组合A(n,m)，因为DFS有约束：
- 必须沿着边访问（不能跳跃）
- 访问顺序受边的拓扑结构限制

正确方法：决策树枚举，每个分支点考虑所有未访问邻接点的选择
```

### 易错点2：混淆DFS序列数与拓扑序列数的计算方法
| 对比项 | DFS序列数 | 拓扑序列数 |
|-------|-----------|-----------|
| 适用图类型 | 任意有向图/无向图 | 仅有向无环图（DAG） |
| 计数依据 | 每个顶点的邻接点访问顺序 | 每层入度为0的顶点选择 |
| 回溯影响 | 回溯后可再选择其他分支 | 不涉及回溯概念 |
| 边的影响 | 只能沿边访问 | 必须满足所有边的偏序 |

**本题特点**：
- v0有3个邻接点 → 第一步3种选择
- v1访问后会影响v3的状态 → 产生更多分支
- 总数 = 各分支的累加（不是简单乘法）

### 易错点3：忘记考虑边的方向性，误将有向图当作无向图
❌ 错误理解：v1和v3相互连通，可以从v3访问v1
✅ 正确理解：边<v1, v3>是**有向边**，只能从v1到v3，不能反向

**有向图DFS规则**：
```
从顶点u出发，只能访问满足以下条件的顶点v：
1. 存在有向边<u, v>（不是<v, u>）
2. v尚未被访问

示例：
- 边<v1, v3>：可以从v1访问v3
- 无边<v3, v1>：不能从v3访问v1
```

**本题验证**：
- v3的出度为0（无指向其他顶点的边）
- 从v3出发必须立即回溯
- 这限制了某些序列的生成（如不能出现`v0, v3, v1, v3`这种重复）
