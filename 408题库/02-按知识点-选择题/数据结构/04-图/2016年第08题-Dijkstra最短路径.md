# 2016年第8题

**元数据**
- 年份：2016年第8题
- 难度：⭐⭐⭐

**题干**：使用迪杰斯特拉（Dijkstra）算法求下图中从顶点1到其他各顶点的最短路径，依次得到的各最短路径的目标顶点是（）。

```
带权图（见2016_Q08_0.jpeg）：
顶点1为起点，连接多个顶点
各边有不同权值
```

A. 5, 2, 3, 4, 6

B. 5, 2, 3, 6, 4

C. 5, 2, 4, 3, 6

D. 5, 2, 6, 3, 4

**答案**：C

**考点分析**：

本题考查Dijkstra算法的执行过程。该算法用于求单源最短路径（非负权图）：
1. **算法原理**：维护已确定最短路径的顶点集合S和未确定的集合U。每次从U中选择距离源点最近的顶点加入S，并更新U中其他顶点的距离。贪心策略保证每次选出的顶点到源点的路径确实最短
2. **执行过程**：初始S={1}，逐步扩展。每轮选择当前dist值最小的未访问顶点。根据图的权值分布，依次确定的顶点顺序为：5（距离1最近）→ 2 → 4 → 3 → 6
3. **关键点**：需要根据具体图的边权值进行模拟计算。Dijkstra是贪心算法，每次选择的都是当前已知最短距离的顶点，这保证了算法的正确性

---

## 解题思路

### 第一步：理解题目核心
Dijkstra算法每次选择dist值最小的未访问顶点，需要根据图的边权值逐轮模拟计算，记录顶点加入S集合的顺序。

### 第二步：逐项分析

**A. 5, 2, 3, 4, 6 ❌**
```
详细分析：第三个顶点为3
判断依据：根据图的边权，从1出发，先确定5和2后，下一个最近顶点应为4而非3
结论：顶点3、4的顺序错误
```

**B. 5, 2, 3, 6, 4 ❌**
```
详细分析：第三、第五个顶点错误
判断依据：在确定5、2后，通过已确定顶点到达4的距离小于到达3的距离
结论：3和4位置颠倒，且6在4之前不符合贪心选择
```

**C. 5, 2, 4, 3, 6 ✅**
```
详细分析：模拟Dijkstra过程
判断依据：
  初始：S={1}, dist[5]=最小 → 加入5
  第2轮：dist[2]=次小 → 加入2
  第3轮：通过1,5,2更新后，dist[4]最小 → 加入4
  第4轮：dist[3]最小 → 加入3
  第5轮：dist[6]最小 → 加入6
结论：正确答案
```

**D. 5, 2, 6, 3, 4 ❌**
```
详细分析：第三个顶点为6
判断依据：顶点6的最短路径长度大于顶点4，不应在4之前确定
结论：违反贪心策略
```

### 第三步：确认答案
按边权逐轮选择dist最小顶点：5→2→4→3→6 → 答案：**C**

## 易错点分析

### 易错点1：混淆BFS和Dijkstra
❌ 错误：按层次遍历顺序误以为离起点"跳数"少就先确定
✅ 正确：Dijkstra按**路径权值和**选择，不是按边数，需要实际计算dist值

### 易错点2：贪心选择的依据
| 算法 | 选择依据 | 是否考虑边权 |
|------|---------|------------|
| BFS | 离起点跳数 | 否（仅适用于无权图） |
| Dijkstra | 路径总权值 | 是（每次选dist最小） |
