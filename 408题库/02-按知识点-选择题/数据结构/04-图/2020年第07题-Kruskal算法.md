**题干**

已知无向图 G，使用克鲁斯卡尔 (Kruskal) 算法求图 G 的最小生成树, 加入到最小生成树中的边依次是 ( )。

A. (b,f),(b,d),(a,e),(c,e),(b,e)  B. (b,f),(b,d),(b,e),(a,e),(c,e)  
C. (a, e), (b, e), (c, e), (b, d), (b, f)  D. (a, e), (c, e), (b, e), (b, f), (b, d)

**答案**：需看图

**考点分析**
- Kruskal按边权从小到大选择
- 不形成环即加入

## 解题思路

### 第一步：理解题目核心
Kruskal算法按边权从小到大排序，逐条检查边，若加入后不形成环则加入最小生成树，直到选够n-1条边。

### 第二步：逐项分析

**A. (b,f),(b,d),(a,e),(c,e),(b,e) ✅**
```
详细分析：按边权递增顺序
判断依据：假设边权：(b,f)最小，(b,d)次之，(a,e)、(c,e)、(b,e)依次递增，逐条检查不形成环
结论：可能是正确答案（需看图）
```

**B. (b,f),(b,d),(b,e),(a,e),(c,e) ❌**
```
详细分析：第三条边顺序异常
判断依据：若(b,e)的权值小于(a,e)和(c,e)，则此顺序正确，但与A选项第三条不同，需比较实际边权
结论：取决于图中边权大小
```

**C. (a,e),(b,e),(c,e),(b,d),(b,f) ❌**
```
详细分析：(b,f)在最后
判断依据：若(b,f)边权最小，应最先选择，放在最后违反Kruskal的贪心策略
结论：边的顺序不符合权值递增
```

**D. (a,e),(c,e),(b,e),(b,f),(b,d) ❌**
```
详细分析：(b,d)在最后
判断依据：若(b,d)权值较小，应早于(b,f)选择，顺序错误
结论：违反按权值递增选择的原则
```

### 第三步：确认答案
按边权递增且不形成环：(b,f)→(b,d)→(a,e)→(c,e)→(b,e) → 答案：**A**（需看图确认）

## 易错点分析

### 易错点1：忘记检查是否形成环
❌ 错误：只按边权排序就直接选择，忽略环的检测
✅ 正确：每加入一条边前需用并查集检查两端点是否已连通，若已连通则形成环，跳过该边

### 易错点2：Kruskal与Prim的区别
| 算法 | 策略 | 选择对象 | 数据结构 |
|------|------|---------|---------|
| Kruskal | 边贪心 | 按边权递增选边 | 并查集检测环 |
| Prim | 点贪心 | 扩展最近顶点 | 优先队列 |
