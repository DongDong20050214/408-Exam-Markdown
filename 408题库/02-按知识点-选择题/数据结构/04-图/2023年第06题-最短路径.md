# 图的最短路径算法真题汇总

## 2023年第06题

**元数据**
- 年份：[[../../01-真题原卷/2023年408真题#06|2023年第6题]]
- 科目：#数据结构
- 知识点：#数据结构/图/最短路径 #BFS #最小生成树
- 难度：⭐⭐
- 关联知识：[[知识库/数据结构/图的最短路径]] [[知识库/数据结构/图的遍历]]

**题干**

已知无向连通图 $G$ 中各边的权值均为 1。下列算法中，一定能够求出图 $G$ 中从某顶点到其余各顶点最短路径的是（）。

I. 普里姆（Prim）算法
II. 克鲁斯卡尔(Kruskal) 算法
III. 图的广度优先搜索算法

A. 仅I  B. 仅 III  C. 仅I、II  D. I、II、III 

**答案**：B

**解题思路**

逐项分析各算法的功能：

**I. Prim算法**
- ❌ 功能：求**最小生成树**（MST）
- 目标：连接所有顶点，使得边权之和最小
- 不保证：从某顶点到其余顶点的路径最短
- 反例：
```
    A ---1--- B
    |         |
    1         1
    |         |
    C ---1--- D
```
MST可能选：A-B, B-D, D-C，此时A到C的路径长度为3
但A到C的最短路径是A-C，长度为1

**II. Kruskal算法**
- ❌ 功能：求**最小生成树**（MST）
- 目标：按边权从小到大选边，避免形成环
- 不保证：从某顶点到其余顶点的路径最短
- 与Prim一样，生成树不等于最短路径树

**III. 广度优先搜索（BFS）**
- ✅ 功能：按层次遍历，第一次到达某顶点的路径即为最短路径
- 前提：**各边权值相同**（本题权值均为1）
- 原理：BFS按距离递增顺序访问顶点，距离为d的顶点一定在距离为d+1的顶点之前被访问

**选项分析**：
- ❌ A. 仅I —— Prim不求最短路径
- ✅ B. 仅III —— BFS在等权图中可求最短路径
- ❌ C. 仅I、II —— Prim和Kruskal都是MST算法
- ❌ D. I、II、III —— I和II错误

**易错点分析**

❌ **错误1：混淆MST和最短路径**
- 误以为最小生成树上的路径就是最短路径
- ✅ 正确：MST保证**总边权最小**，不保证任意两点间路径最短
- 示例：
```
      2
  A ----- B
  |     / |
 1|   1/  |1
  |  /    |
  C ----- D
      2
```
MST选边：A-C(1), B-C(1), B-D(1)，A到D路径：A→C→B→D，长度3
最短路径：A→B→D，长度2+1=3 或 A→C→D，长度1+2=3（如果存在A-D边更优）

❌ **错误2：认为BFS只能用于无权图**
- 误以为有权图不能用BFS
- ✅ 正确：BFS适用于**等权图**（所有边权相同），本题权值均为1，可以用BFS

❌ **错误3：DFS也能求最短路径**
- 误以为深度优先搜索也能求最短路径
- ✅ 正确：DFS不按距离顺序访问，第一次到达的路径不一定最短，需要遍历所有路径才能确定

**知识点对比**

| 算法 | 解决问题 | 适用图 | 时间复杂度 | 能否求最短路径 |
|------|---------|--------|-----------|--------------|
| **Prim** | 最小生成树 | 无向连通图 | O(V²) 或 O(ElogV) | ❌ 否 |
| **Kruskal** | 最小生成树 | 无向连通图 | O(ElogE) | ❌ 否 |
| **BFS** | 单源最短路径 | **等权图** | O(V+E) | ✅ 是（等权） |
| **Dijkstra** | 单源最短路径 | 非负权图 | O(V²) 或 O(ElogV) | ✅ 是 |
| **Floyd** | 所有顶点对最短路径 | 任意图 | O(V³) | ✅ 是 |

**最小生成树 vs 最短路径树**

| 特性 | 最小生成树（MST） | 最短路径树（SPT） |
|------|-----------------|------------------|
| **目标** | 连接所有顶点，总边权最小 | 从源点到各顶点的路径最短 |
| **边数** | V-1条边 | V-1条边 |
| **唯一性** | 可能不唯一 | 可能不唯一 |
| **算法** | Prim, Kruskal | BFS（等权）, Dijkstra, Bellman-Ford |
| **是否保证最短路径** | ❌ 否 | ✅ 是 |

**BFS求最短路径的条件**

| 图的类型 | BFS能否求最短路径 | 原因 |
|---------|-----------------|------|
| **无权图** | ✅ 可以 | 边权视为1，等价于等权图 |
| **等权图** | ✅ 可以 | BFS按层次遍历，保证按距离递增访问 |
| **非等权正权图** | ❌ 不可以 | 需要用Dijkstra |
| **负权图** | ❌ 不可以 | 需要用Bellman-Ford |

## 解题思路

### 第一步：理解题目条件与各算法功能
**题目关键条件**：
- 图类型：无向连通图G
- 边权特点：**各边权值均为1**（等权图）
- 目标：求从某顶点到其余各顶点的最短路径

**核心分析**：边权为1的图，距离=经过的边数

### 第二步：逐项分析各算法

**I. Prim算法 ❌**
功能：求**最小生成树**（MST）
- 目标：用最小总边权连接所有顶点
- 保证：生成树边权总和最小
- 不保证：任意两点间路径最短

反例：
```
图：  A ---1--- B
      |         |
      1         1
      |         |
      C ---1--- D ---1--- E
```
Prim MST可能选：A-B, B-D, D-C, D-E
此时A到C的MST路径：A→B→D→C（长度3）
但A到C的最短路径：A→C（长度1）

**核心问题**：MST追求**全局最优**（总边权最小），不保证**局部最优**（单源最短路径）

结论：Prim不能求最短路径 ❌

**II. Kruskal算法 ❌**
功能：求**最小生成树**（MST）
- 策略：按边权从小到大选边，避免成环
- 结果：与Prim相同，得到MST

**与Prim的区别仅在于构造方式**：
- Prim：从点出发扩展
- Kruskal：从边出发选择

但两者本质相同：都是求MST，不保证最短路径

结论：Kruskal不能求最短路径 ❌

**III. 广度优先搜索（BFS）✅**
功能：按层次遍历，在**等权图**中可求单源最短路径

**原理**：
BFS使用队列，按距离递增顺序访问顶点：
1. 距离为0：源点s
2. 距离为1：s的所有邻接点
3. 距离为2：距离1的顶点的所有未访问邻接点
4. 依此类推...

**关键性质**：
- 第一次访问到某顶点v时，经过的路径长度就是s到v的最短路径长度
- 因为BFS保证：距离d的顶点一定在距离d+1的顶点之前被访问

**前提条件**：所有边权相同（本题满足：权值均为1）

**BFS vs Dijkstra**：
- BFS：等权图，O(V+E)，使用队列
- Dijkstra：非负权图，O(V²) 或 O(ElogV)，使用优先队列

本题边权为1，BFS是最优选择

结论：BFS可以求最短路径 ✅

### 第三步：逐项分析选项

**A. 仅I ❌**
Prim求MST，不求最短路径
结论：错误

**B. 仅III ✅**
BFS在等权图中求单源最短路径
结论：正确

**C. 仅I、II ❌**
Prim和Kruskal都求MST，都不求最短路径
结论：错误

**D. I、II、III ❌**
I和II错误，只有III正确
结论：错误

### 第四步：确认答案
边权均为1的等权图，只有BFS能求最短路径 → 答案：**B**

## 易错点分析

### 易错点1：混淆最小生成树（MST）与最短路径树（SPT）
❌ 错误理解：最小生成树上的路径就是最短路径
✅ 正确理解：两者目标不同，结果通常不同

| 对比项 | 最小生成树（MST） | 最短路径树（SPT） |
|-------|-----------------|------------------|
| 目标 | **总边权最小**连接所有顶点 | 从源点到各顶点**路径最短** |
| 优化对象 | 全局（整棵树的边权和） | 局部（每条源点到叶子的路径） |
| 算法 | Prim, Kruskal | BFS（等权）, Dijkstra, Bellman-Ford |
| 边数 | V-1条 | V-1条 |
| 保证 | 总权值最小 | 各路径最短 |

**反例说明**：
```
图：  A ---1--- B ---1--- C
      |                   |
      +-------3-----------+

MST选边：A-B(1), B-C(1)，总权2
  A到C路径：A→B→C，长度2

SPT选边：A-B(1), A-C(3)，总权4
  A到C路径：A→C，长度3（若直连更近则选直连）

实际最短路径：A→B→C，长度2
MST恰好包含最短路径（巧合）

但若图为：
  A ---1--- B
  |         |
 2|        3|
  |         |
  C ---1--- D

MST：A-B(1), B-C或A-C+C-D（不唯一）
A到D最短路径：取决于具体MST构造，可能不是最短
```

### 易错点2：误认为BFS只能用于无权图
❌ 错误理解：BFS只适用于无权图，有权图必须用Dijkstra
✅ 正确理解：BFS适用于**等权图**（所有边权相同）

| 图类型 | BFS是否适用 | 原因 |
|-------|------------|------|
| 无权图 | ✅ 是 | 可视为所有边权=1的等权图 |
| 等权图（如本题权值均为1） | ✅ 是 | BFS按层次遍历=按距离递增遍历 |
| 非等权正权图 | ❌ 否 | BFS无法处理不同权值，需Dijkstra |
| 负权图 | ❌ 否 | 需Bellman-Ford或SPFA |

**BFS能求最短路径的本质**：
等权图中，经过k条边的路径长度=k×边权。BFS第k层顶点距离源点恰好k条边，因此第一次到达时即为最短。

### 易错点3：不理解"一定能够求出"的严格性
题目问"一定能够求出"，要求算法的**充分性**（100%保证）

| 算法 | 能否一定求出最短路径 | 条件 |
|------|-------------------|------|
| Prim/Kruskal | ❌ 不能 | MST≠最短路径树 |
| BFS | ✅ 能（等权图） | 本题满足边权均为1 |
| Dijkstra | ✅ 能（非负权图） | 本题满足（权值1>0） |
| Bellman-Ford | ✅ 能（任意图） | 本题满足 |
| Floyd | ✅ 能（任意图） | 本题满足 |

**本题特殊性**：边权均为1
- BFS最简单高效：O(V+E)
- Dijkstra也能用但复杂度高：O(V²) 或 O(ElogV)
- 题目选项只有BFS，选它！
