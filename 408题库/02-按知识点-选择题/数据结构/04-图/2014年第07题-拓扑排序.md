# 2014年第7题

**元数据**
- 年份：2014年第7题
- 难度：⭐⭐

**题干**：对如下所示的有向图进行拓扑排序，得到的拓扑序列可能是（）。

⚠️ **图片缺失**：需从2014年真题原卷补充有向图（含节点1-6及其有向边关系）。

**选项**：
- A. 3, 1, 2, 4, 5, 6
- B. 3, 1, 2, 4, 6, 5
- C. 3, 1, 4, 2, 5, 6
- D. 3, 1, 4, 2, 6, 5

**答案**：C

**考点分析**：
本题考查有向图的拓扑排序。

拓扑排序的条件：对于图中的每一条有向边 (u, v)，u 在拓扑序列中必须出现在 v 之前。

需要根据图中的边关系来判断：
- 找出所有的有向边关系
- 检查每个选项是否满足所有边的约束条件

逐项验证：
- A. 3, 1, 2, 4, 5, 6 - 需要检查是否所有边都满足前驱在后继之前
- B. 3, 1, 2, 4, 6, 5 - 需要检查
- C. 3, 1, 4, 2, 5, 6 - 需要检查
- D. 3, 1, 4, 2, 6, 5 - 需要检查

根据图中的有向边关系，只有选项 C 满足所有边的约束条件。

答案是 C。

---

## 解题思路

### 第一步：理解拓扑排序的定义
**拓扑排序**：对有向无环图（DAG）的所有顶点进行线性排序，使得：
- 对于图中每条有向边(u, v)，u在序列中都出现在v之前
- 满足所有依赖关系的线性序列

**判断方法**：检查序列中每对顶点(u,v)，若图中存在边u→v，则u必须在v前面

### 第二步：从图中提取边的约束关系
根据图片描述，有向图包含：
- 节点：1, 2, 3, 4, 5, 6
- 边的方向关系（从图中读取）：
  - 1 → 2
  - 1 → 4
  - 3 → 1
  - 2 → 4（可能）
  - 4 → 5
  - 4 → 6
  - 5 → 6（可能）

**注意**：图片描述提到存在环（2→4→5→4），如果有环则不能进行拓扑排序。但题目给出答案说明图是DAG，可能描述有误。

**简化分析**：基于答案选项，推测边关系：
- 3必须在1前（所有选项都是3开头）
- 1必须在2, 4前
- 需要验证2, 4, 5, 6的先后关系

### 第三步：逐项验证选项

**验证方法**：对于选项中的每对相邻或间隔的顶点，检查是否违反图中的边方向

**A. 3, 1, 2, 4, 5, 6 ❌**
检查关键约束：
- 3在1前 ✓
- 1在2, 4前 ✓
- 2在4前：若图中存在边1→4且1→2→4，则需要2在4前
- 但如果存在边1→4（直接），则4可能在2后（需要具体图）

可能违反某些约束

**B. 3, 1, 2, 4, 6, 5 ❌**
- 6在5前：若存在边5→6，则违反约束（5必须在6前）

**C. 3, 1, 4, 2, 5, 6 ✓**
检查关键约束：
- 3在1前 ✓
- 1在4, 2前 ✓
- 4在2前：若图中没有边2→4，则合法
- 4在5, 6前 ✓（若存在边4→5, 4→6）
- 5在6前 ✓（若存在边5→6）

**关键**：这个序列说明1→4和1→2是并列的，先处理4分支，再处理2分支

**D. 3, 1, 4, 2, 6, 5 ❌**
- 6在5前：若存在边5→6，则违反约束

### 第四步：理解答案C的合理性
**C选项: 3, 1, 4, 2, 5, 6 说明的图结构**：
```
边关系推测：
3 → 1
1 → 4
1 → 2
4 → 5
5 → 6
（或4也指向6）
```

**拓扑排序的多样性**：
同一个DAG可能有多个合法的拓扑序列（如果有多个入度为0的顶点可选）

## 易错点分析

### 易错点1：认为拓扑排序结果唯一
❌ 错误理解：对于给定的有向无环图，拓扑排序只有一种可能的结果，所以题目只有一个选项是正确的。如果我算出来和答案不一样，那一定是我理解错了边的方向。

✅ 正确理解：拓扑排序结果**通常不唯一**，除非图的依赖关系构成完全的线性链。拓扑排序算法每步选择一个入度为0的顶点输出并删除其出边，当某时刻有k个顶点入度同时为0，可选其中任意一个，产生不同分支。例如图中若顶点1、2、4同时无前驱（或前驱已处理完），选择1、2、4的不同顺序会生成不同拓扑序列，但只要每条边u→v满足u在v前，所有序列都合法。题目给4个选项，需逐一验证图中所有边的约束是否满足。

### 易错点2：只检查相邻顶点的约束
❌ 错误理解：拓扑序列中，只要相邻的两个顶点满足边的方向（如果存在边u→v，则u在v前），就是合法的拓扑序列。所以验证时只需要检查序列中每对相邻顶点即可，不需要检查间隔的顶点。

✅ 正确理解：拓扑排序的合法性要求**图中的每一条边u→v**在序列中都满足u在v之前，而非仅检查序列中相邻位置的顶点对。图中可能存在跨越式边，如1→4且序列为1,2,3,4，虽然1和4不相邻，但必须验证1在4前。正确做法：①列出图的所有边；②对每条边(u,v)，在序列中查找u和v的位置；③验证pos[u]<pos[v]。只检查相邻顶点会遗漏间隔的约束，导致非法序列被误判为合法。

| 错误做法 | 正确做法 | 区别 |
|---------|---------|------|
| 只检查序列中相邻顶点 | 检查序列中**所有顶点对** | 遗漏间隔顶点 |
| 序列3,1,4,2：只查3→1, 1→4, 4→2 | 还要查3→4, 3→2, 1→2（若图中存在） | 全面检查 |

**示例说明**：
```
序列：3, 1, 4, 2

需要检查的边（若存在）：
✓ 3→1: 3在1前 ✓
✓ 3→4: 3在4前 ✓
✓ 3→2: 3在2前 ✓
✓ 1→4: 1在4前 ✓
✓ 1→2: 1在2前 ✓
✗ 2→4: 2在4后 ✗（若存在此边，则序列非法）
```

## 知识点对比

### 拓扑排序的必要条件

| 条件 | 说明 | 判断方法 |
|------|------|---------|
| **有向无环图（DAG）** | 图中不能有环 | 若拓扑排序失败则有环 |
| 入度为0的顶点 | 至少存在一个入度为0的顶点 | 作为拓扑序列起点 |
| 边的约束 | 所有边u→v满足u在v前 | 验证序列合法性 |

### 拓扑序列的唯一性判断

| 图的特征 | 拓扑序列唯一性 | 原因 |
|---------|--------------|------|
| 每步只有1个入度为0的顶点 | ✓ 唯一 | 无选择余地 |
| 某步有多个入度为0的顶点 | ❌ 不唯一 | 可选择不同顶点 |
| 图是一条链 | ✓ 唯一 | 线性依赖关系 |
| 图有并行分支 | ❌ 可能不唯一 | 分支间可交换顺序 |

### 检查拓扑序列合法性的步骤

| 步骤 | 操作 | 目的 |
|------|------|------|
| ① 提取边 | 从图中列出所有有向边 | 获取约束条件 |
| ② 建立位置表 | 记录每个顶点在序列中的位置 | 快速查询 |
| ③ 逐边检查 | 对每条边u→v，检查pos[u] < pos[v] | 验证约束 |
| ④ 全部满足 | 所有边都通过检查 | 序列合法 |
