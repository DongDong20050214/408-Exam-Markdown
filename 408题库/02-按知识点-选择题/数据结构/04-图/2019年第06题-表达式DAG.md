# 表达式DAG真题汇总

## 2019年第06题

**元数据**
- 年份：[2019年第6题](../../../01-真题原卷/2019年408真题.md#06)
- 科目：#数据结构
- 知识点：#数据结构/图/DAG #表达式
- 难度：⭐⭐
- 关联知识：[知识库/数据结构/DAG](知识库/数据结构/DAG.md)

**题干**

用有向无环图描述表达式 (x + y)((x + y) / x)，需要的顶点个数至少是（ ）。

A. 5  B. 6  C. 8  D. 9

**答案**：A

**解题思路**

**表达式分析**：$(x + y) \times ((x + y) / x)$

**关键**：DAG（有向无环图）可以**共享公共子表达式**

**识别子表达式**：
1. $x$ —— 叶子节点
2. $y$ —— 叶子节点
3. $x + y$ —— 公共子表达式（出现2次，只需1个节点）
4. $(x + y) / x$ —— 中间结果
5. $(x + y) \times ((x + y) / x)$ —— 根节点

**构造DAG**：
```
        *（根）
       / \
      /   \
    x+y    /
    / \   / \
   x   y x+y x
         (复用)
```

等等，这样x+y被复用了，但画图不太对。重新画：

```
        *
       / \
     +    /
    / \  / \
   x  y +  x
       / \
      x   y
```

不对，加号节点应该复用。正确的DAG：

```
节点编号：
1. x（叶子）
2. y（叶子）
3. x+y（计算节点）
4. /（计算节点）：(x+y)/x
5. *（根节点）：(x+y)*((x+y)/x)

DAG结构：
       ⑤*
      / \
    ③+  ④/
    /\  /\
   ①x ②y ③ ①x
         (复用③)
```

**节点清单**：
1. x（变量）
2. y（变量）
3. x+y（操作）
4. (x+y)/x（操作）
5. 最终结果（操作）

共**5个节点**

**如果不复用公共子表达式**（用树表示）：
```
          *
        /   \
       +     /
      / \   / \
     x   y +   x
          / \
         x   y
```
需要9个节点：x(3个), y(2个), +(2个), /(1个), *(1个)

**选项分析**：
- ✅ A. 5 —— 正确，复用x+y
- ❌ B. 6 —— 多算了1个
- ❌ C. 8 —— 部分复用
- ❌ D. 9 —— 完全不复用（树结构）

**易错点分析**

❌ **错误1：选择D(9)，不复用公共子表达式**
- 用树而非DAG表示，每次出现x都算一个节点
- ✅ 正确：DAG可以复用，相同子表达式只需一个节点

❌ **错误2：重复计算x节点**
- 认为x出现3次需要3个节点
- ✅ 正确：变量x只需1个节点，可以被多次引用

❌ **错误3：忘记复用(x+y)**
- 只复用变量，不复用中间结果
- ✅ 正确：(x+y)出现2次，DAG中只需1个节点

**知识点对比**

| 表示方式 | 结构 | 节点数（本题） | 特点 |
|---------|------|--------------|------|
| **表达式树** | 树形结构 | 9 | 不复用，每个操作一个节点 |
| **DAG** | 有向无环图 | 5 | 复用公共子表达式 |

**树vs DAG对比**

| 特性 | 表达式树 | 表达式DAG |
|------|---------|-----------|
| **节点数** | 更多 | 更少 |
| **公共子表达式** | 重复计算 | 复用节点 |
| **适用场景** | 简单表达式 | 优化编译、公共子表达式消除 |

**本题详细计数**

**方法1：列举所有节点**
1. 变量x：1个
2. 变量y：1个
3. 运算x+y：1个（复用）
4. 运算(x+y)/x：1个
5. 运算(x+y)*((x+y)/x)：1个

总计：5个

**方法2：按层次构建**
- 第1层（叶子）：x, y —— 2个
- 第2层：x+y —— 1个
- 第3层：(x+y)/x —— 1个（使用第2层的x+y和第1层的x）
- 第4层：最终结果 —— 1个（使用第2层的x+y和第3层的结果）

总计：5个

**DAG可视化**

```
层4:        [*]
           /   \
层3:      /     [/]
         /     /  \
层2:   [x+y]---+   |
       /  \        |
层1: [x]  [y]------+

箭头表示依赖关系
[x+y]被*和/共同使用（复用）
```

**关键理解**：
- DAG = Directed Acyclic Graph（有向无环图）
- 允许一个节点有多个父节点（被多次引用）
- 表达式优化的基础：公共子表达式消除（CSE）
- 编译器优化中常用技术

## 解题思路

### 第一步：理解题目核心
DAG可复用公共子表达式，表达式(x+y)((x+y)/x)中(x+y)出现2次但只需1个节点，关键是识别所有独立子表达式并复用。

### 第二步：逐项分析

**A. 5 ✅**
```
详细分析：最少节点数=变量+不重复运算
判断依据：
  1. 变量：x、y（2个）
  2. 运算：x+y（1个，复用）
  3. 运算：(x+y)/x（1个）
  4. 运算：最终乘法（1个）
  总计：2+3=5个节点
结论：正确答案
```

**B. 6 ❌**
```
详细分析：多算1个节点
判断依据：可能误认为x或y需要额外节点，或某个运算未复用
结论：计数错误
```

**C. 8 ❌**
```
详细分析：部分复用
判断依据：可能只复用变量x、y，而(x+y)重复计算2次，导致节点数增加
结论：未充分利用DAG的复用特性
```

**D. 9 ❌**
```
详细分析：用树而非DAG表示
判断依据：不复用任何子表达式，x出现3次算3个节点，y出现2次算2个节点，(x+y)出现2次算2个节点，加上/和*共9个
结论：忽略了DAG的复用机制
```

### 第三步：确认答案
节点清单：x、y、x+y（复用）、(x+y)/x、最终结果 → 答案：**A**

## 易错点分析

### 易错点1：混淆树和DAG的节点计数
❌ 错误：用表达式树计数，每个操作数出现都算一个新节点
✅ 正确：DAG允许节点被多次引用，相同子表达式只需1个节点

### 易错点2：DAG与树的区别
| 特性 | 表达式树 | 表达式DAG |
|------|---------|-----------|
| 公共子表达式 | 重复存储 | 共享节点（复用） |
| 节点数（本题） | 9 | 5 |
| 应用场景 | 简单表示 | 编译器优化（CSE） |
