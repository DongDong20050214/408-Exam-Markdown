**题干**

拓扑排序的结果可能是（根据图判断）。

A. 3,1,2,4,5,6  
B. 3,1,2,4,6,5  
C. 3,1,4,2,5,6  
D. 3,1,4,2,6,5

**答案**：需根据图的边关系判断

**考点分析**
- 拓扑排序必须满足所有边的方向：u→v则u在v前
- 检查每个选项是否满足所有边的约束

---

## 解题思路

### 第一步：理解拓扑排序的验证方法
拓扑排序的**核心约束**：对于图中任意边 `u→v`，在拓扑序列中 `u` 必须出现在 `v` 之前

**验证步骤**：
1. 提取图中所有边的方向关系（边集）
2. 对于每个选项序列，逐一检查所有边是否满足"起点在终点之前"
3. 若所有边都满足，则该序列是合法拓扑序列；否则非法

**假设图的边关系**（408典型结构）：
```
边集示例（需根据原题图确定）：
3→1, 1→2, 1→4, 2→5, 4→5, 4→6, 5→6
```

### 第二步：逐项验证边约束

**A. 3,1,2,4,5,6 ✅/❌**
检查每条边的约束：
- 3→1：3在位置0，1在位置1 → 3 < 1 ✅
- 1→2：1在位置1，2在位置2 → 1 < 2 ✅
- 1→4：1在位置1，4在位置3 → 1 < 3 ✅
- 2→5：2在位置2，5在位置4 → 2 < 4 ✅
- 4→5：4在位置3，5在位置4 → 3 < 4 ✅
- 4→6：4在位置3，6在位置5 → 3 < 5 ✅
- 5→6：5在位置4，6在位置5 → 4 < 5 ✅
结论：所有边都满足约束 ✅

**B. 3,1,2,4,6,5 ❌**
检查关键边：
- 前面边同A，均满足
- **关键检查**：5→6
  - 5在位置5，6在位置4
  - 5 > 4，违反"5必须在6之前"的约束
结论：违反边约束 ❌

**C. 3,1,4,2,5,6 ✅**
检查每条边：
- 3→1：0 < 1 ✅
- 1→2：1 < 3 ✅（1在位置1，2在位置3）
- 1→4：1 < 2 ✅（4在位置2）
- 2→5：3 < 4 ✅
- 4→5：2 < 4 ✅
- 4→6：2 < 5 ✅
- 5→6：4 < 5 ✅
结论：所有边都满足 ✅

**D. 3,1,4,2,6,5 ❌**
检查关键边：
- 5→6：5在位置5，6在位置4
- 违反"5在6之前"的约束
结论：与B选项相同错误 ❌

### 第三步：确认答案
根据典型图结构，选项A和C满足所有边的拓扑约束；选项B和D违反了5→6的边约束。
若题目只要求选一个合法序列，通常选**最先满足的**或**字典序最小**的 → 答案：**A**（或C，需查看原题图）

## 易错点分析

### 易错点1：只检查相邻顶点关系，忽略"跨越"的边约束
❌ 错误理解：只要序列中相邻的两个顶点有边连接，就认为合法
✅ 正确理解：拓扑排序要求**所有边**的起点都在终点之前，不仅是相邻顶点。例如边1→6，即使1和6在序列中不相邻（中间隔着2,4,5），也必须保证1出现在6之前

**对比示例**：
| 序列 | 边1→6的检查 | 结果 |
|------|------------|------|
| 3,1,2,4,5,6 | 1在位置1，6在位置5 → 1 < 5 ✅ | 合法 |
| 3,6,1,2,4,5 | 1在位置2，6在位置1 → 1 > 2 ❌ | 非法 |

### 易错点2：混淆拓扑排序与DFS/BFS遍历序列
| 对比项 | 拓扑排序 | DFS序列 | BFS序列 |
|-------|---------|---------|---------|
| 约束条件 | 所有边u→v，u在v前 | 深度优先规则（栈） | 广度优先规则（队列） |
| 适用图 | 有向无环图（DAG） | 任意图 | 任意图 |
| 唯一性 | 可能不唯一 | 取决于邻接顺序 | 取决于邻接顺序 |
| 验证方法 | 检查所有边的方向 | 检查访问路径连续性 | 检查层次关系 |

**记忆技巧**：
- 拓扑排序：全局约束（所有边的方向）
- DFS/BFS：局部约束（相邻访问的连通性）

### 易错点3：忘记检查某些"不明显"的边
❌ 错误做法：凭直觉觉得序列"看起来合理"就选择
✅ 正确方法：**系统化检查所有边**

**检查技巧**：
```
建立位置映射表：
顶点  3  1  2  4  5  6
位置  0  1  2  3  4  5

对于每条边u→v：
1. 查找u的位置pos_u
2. 查找v的位置pos_v
3. 判断pos_u < pos_v是否成立
4. 若任何一条边不满足，立即排除该选项
```

**常见陷阱边**：
- 起点和终点在序列中**距离较远**的边（如1→6）
- **反向排列**的关键边（如4,2而不是2,4，需检查是否有边2→4或4→2）
