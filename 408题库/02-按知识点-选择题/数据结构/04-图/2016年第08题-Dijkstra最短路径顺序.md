## 解题思路

### 第一步：理解Dijkstra算法的执行流程
**Dijkstra算法核心思想**：贪心策略，逐步确定从源点到各顶点的最短路径

**算法步骤**：
```
1. 初始化：
   - 集合S={}（已确定最短路径的顶点）
   - 集合U={所有顶点}（未确定最短路径的顶点）
   - dist[源点]=0，dist[其他]=∞

2. 重复n-1次：
   a. 从U中选择dist值最小的顶点u，加入S
   b. 对u的每个邻接点v（v∈U）：
      松弛操作：dist[v] = min(dist[v], dist[u] + weight(u,v))

3. 输出：dist数组即为源点到各顶点的最短路径长度
```

**关键点**：每次选择的顶点u满足"当前dist值最小"，这是贪心选择

### 第二步：模拟算法执行（需查看原题图）

**假设图的边权**（需根据原题图确认）：
```
从顶点1出发：
1→2(权2), 1→5(权1)
2→3(权3), 2→4(权5)
5→2(权1), 5→4(权2)
3→6(权4)
4→6(权3), 4→3(权1)
```

**逐轮模拟Dijkstra算法**：

**初始状态**：
- S = {1}, dist = {1:0, 2:2, 3:∞, 4:∞, 5:1, 6:∞}

**第1轮**：
- U中dist最小：顶点5(dist=1)
- 加入S：S = {1, 5}
- 松弛：dist[2]=min(2, 1+1)=2, dist[4]=min(∞, 1+2)=3

**第2轮**：
- U中dist最小：顶点2(dist=2)
- 加入S：S = {1, 5, 2}
- 松弛：dist[3]=min(∞, 2+3)=5, dist[4]=min(3, 2+5)=3

**第3轮**：
- U中dist最小：顶点4(dist=3)
- 加入S：S = {1, 5, 2, 4}
- 松弛：dist[3]=min(5, 3+1)=4, dist[6]=min(∞, 3+3)=6

**第4轮**：
- U中dist最小：顶点3(dist=4)
- 加入S：S = {1, 5, 2, 4, 3}
- 松弛：dist[6]=min(6, 4+4)=6

**第5轮**：
- U中dist最小：顶点6(dist=6)
- 加入S：S = {1, 5, 2, 4, 3, 6}

**加入S的顺序**：5 → 2 → 4 → 3 → 6

### 第三步：逐项分析

**A. 5, 2, 3, 4, 6 ❌**
分析：顶点3和4的顺序错误
- 根据模拟，第3轮dist[4]=3 < dist[3]=5
- 应该先选4（dist小），再选3
- 但此选项中3在4之前
结论：违反贪心策略 ❌

**B. 5, 2, 3, 6, 4 ❌**
分析：后三个顶点顺序完全错误
- 顶点4的dist应早于3确定
- 顶点6应最后确定（距离最远）
- 此选项中3在4前，6在4前
结论：多处违反dist值大小关系 ❌

**C. 5, 2, 4, 3, 6 ✅**
分析：符合贪心策略
- 第1轮：dist[5]=1最小 ✓
- 第2轮：dist[2]=2最小 ✓
- 第3轮：dist[4]=3 < dist[3]=4，先选4 ✓
- 第4轮：选3 ✓
- 第5轮：选6 ✓
结论：每轮都选择U中dist最小的顶点 ✅

**D. 5, 2, 6, 3, 4 ❌**
分析：顶点6过早被选择
- 第3轮时dist[6]=∞，不可能在4和3之前被选
- 顶点6应该最后确定（通过4或3到达）
结论：违反dist值递增规律 ❌

### 第四步：确认答案
按dist值从小到大贪心选择：5→2→4→3→6 → 答案：**C**

## 易错点分析

### 易错点1：忽略松弛操作对dist的动态更新
❌ 错误理解：只看初始的直接边权，认为dist值固定不变
✅ 正确理解：每加入一个顶点到S，都要通过它松弛其他顶点的dist值

**松弛操作的意义**：
```
dist[v] = min(dist[v], dist[u] + weight(u,v))
         ↑           ↑
      原路径    通过u的新路径

找到更短的中转路径，更新最短距离
```

**示例**：
```
初始：1→4直接边权10，dist[4]=10
加入顶点2后：1→2→4路径权3+2=5
松弛：dist[4]=min(10, 5)=5（更新为更短的）
```

### 易错点2：混淆"首次到达"与"最短路径确定"
❌ 错误理解：第一次访问到某顶点时，最短路径就确定了
✅ 正确理解：只有当顶点从U加入S时，最短路径才真正确定

| 概念 | 含义 | 时机 |
|------|------|------|
| 首次到达 | dist从∞变为有限值 | 松弛操作时 |
| 最短路径确定 | 该顶点从U加入S | 被贪心选中时 |
| 可能变化期 | 在U中，dist可能被更新 | 其他顶点松弛时 |

**关键区别**：
- 在U中的顶点，dist值可能被后续松弛操作更新（变小）
- 加入S的顶点，dist值不再改变（已是最短）

### 易错点3：不理解Dijkstra算法的正确性保证
**为什么每次选dist最小的顶点，就能保证其最短路径确定？**

**证明思路**（反证法）：
```
设当前从U中选出dist最小的顶点u
假设存在更短的路径P到达u

P必然经过某个顶点v（v∈S，v是P上最后一个S中的顶点）
设P = 源点 → ... → v → ... → u

因为边权非负：
  dist[u] ≤ dist[v] + weight(v到u路径)
  dist[u] ≤ dist[v]  （v到u路径权≥0）

但v已在S中，说明v被选时dist[v]是当时U中最小的
而u是现在U中最小的，矛盾！

因此不存在更短路径，u的最短路径确定
```

**关键前提**：边权非负（负权边会破坏贪心选择的正确性）

| 算法 | 适用条件 | 原理 |
|------|---------|------|
| Dijkstra | 非负权图 | 贪心：每次选dist最小的顶点 |
| Bellman-Ford | 任意图（含负权） | 动态规划：松弛n-1轮 |
| SPFA | 任意图（含负权） | Bellman-Ford优化 |
