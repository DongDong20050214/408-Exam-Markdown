**题干**

若将 n 个顶点 e 条弧的有向图采用邻接表存储, 则拓扑排序算法的时间复杂度是 ( )。A. O(n)  B. O(n + e)  C. O(n²)  D. O(n × e)

**答案**：B

**考点分析**
- 拓扑排序O(n+e)

---

## 解题思路

### 第一步：理解拓扑排序算法的基本步骤
**Kahn算法（BFS拓扑排序）- 邻接表实现**：
```
1. 计算所有顶点的入度：遍历所有边，O(e)
2. 将入度为0的顶点入队：遍历所有顶点，O(n)
3. 主循环：
   while 队列非空：
       取出队首顶点v，输出到拓扑序列
       遍历v的所有邻接点w：
           将w的入度减1
           若w的入度变为0，将w入队
```

**时间复杂度分析**：
- 步骤1（计算入度）：O(e) - 遍历邻接表中所有边
- 步骤2（初始化队列）：O(n) - 扫描所有顶点
- 步骤3（主循环）：
  - 每个顶点入队、出队各一次：O(n)
  - 每条边被访问一次（邻接表遍历）：O(e)
  - 合计：O(n + e)

**总时间复杂度**：O(e) + O(n) + O(n+e) = O(n+e)

### 第二步：逐项分析

**A. O(n) ❌**
分析：只考虑顶点数量，忽略边的遍历
- 拓扑排序必须遍历所有边来更新入度
- 若图有很多边（如接近完全图，e≈n²），O(n)明显不足
反例：
```
完全图：n=100, e=4950
仅O(n)=100，但实际需访问4950条边
```
结论：遗漏了边的遍历开销 ❌

**B. O(n + e) ✅**
分析：邻接表存储结构的标准复杂度
- **邻接表特点**：查找某顶点的所有邻接点只需O(出度)
- 主循环中：
  - n个顶点各入队出队一次：O(n)
  - 遍历邻接表时，每条边恰好访问一次：O(e)
- 总计：O(n + e)

**为什么是O(n+e)而非O(n×e)**：
- 每条边在邻接表中只存储一次
- 遍历所有顶点的邻接表，边的访问次数总和=e
- 不存在"对每个顶点遍历所有边"的嵌套

结论：正确答案 ✅

**C. O(n²) ❌**
分析：这是**邻接矩阵**存储时的复杂度
- 邻接矩阵：查找某顶点的所有邻接点需扫描一整行，O(n)
- n个顶点各扫描一次：O(n²)
- 但题目明确是**邻接表**存储

**存储结构对比**：
| 存储结构 | 查找顶点v的邻接点 | 拓扑排序复杂度 |
|---------|-----------------|---------------|
| 邻接表 | O(出度(v)) | O(n+e) ✅ |
| 邻接矩阵 | O(n) | O(n²) |

结论：混淆了存储结构 ❌

**D. O(n × e) ❌**
分析：这是严重高估
- 若为O(n×e)，意味着对每个顶点都遍历所有边
- 但拓扑排序中，每条边只在其起点出队时访问一次
- 不存在嵌套的"顶点循环×边循环"

反例验证：
```
设n=10, e=20
O(n×e) = 200
实际O(n+e) = 30
差距悬殊
```
结论：不符合算法逻辑 ❌

### 第三步：确认答案
邻接表存储，拓扑排序遍历n个顶点和e条边各一次 → 答案：**B**

## 易错点分析

### 易错点1：混淆不同存储结构的时间复杂度
❌ 错误理解：认为所有图算法的复杂度都相同，不考虑存储结构
✅ 正确理解：存储结构直接影响算法效率

| 存储结构 | 空间复杂度 | 查找邻接点 | 拓扑排序 | 适用场景 |
|---------|-----------|-----------|---------|---------|
| **邻接表** | O(n+e) | O(出度) | **O(n+e)** | 稀疏图 |
| **邻接矩阵** | O(n²) | O(n) | O(n²) | 稠密图、需频繁判断边是否存在 |

**记忆技巧**：
- 邻接表：**链式存储**，只存实际存在的边 → 复杂度与边数e相关
- 邻接矩阵：**二维数组**，必须为所有可能的边预留空间 → 复杂度与n²相关

### 易错点2：误以为每条边会被多次访问
❌ 错误推理：n个顶点，每个顶点可能访问e条边，所以是O(n×e)
✅ 正确分析：邻接表中每条边只存储在其起点的邻接表中

**访问次数验证**：
```
假设图：v1→v2, v1→v3, v2→v3

邻接表表示：
v1: [v2, v3]
v2: [v3]
v3: []

拓扑排序过程：
1. v1出队，访问v1的邻接表：v2, v3（访问2条边）
2. v2出队，访问v2的邻接表：v3（访问1条边）
3. v3出队，邻接表为空（访问0条边）

总共访问：2+1+0 = 3 = e（边数）
```

**关键理解**：
- 每条边在邻接表中只出现一次（在起点顶点的链表中）
- 遍历所有顶点的邻接表，边的访问总次数=e

### 易错点3：复杂度分析时忽略常数项和低阶项
**完整分析**：
```
拓扑排序各步骤：
1. 初始化入度数组：O(n)
2. 遍历邻接表计算入度：O(e)
3. 扫描入度数组，初始化队列：O(n)
4. 主循环：
   - 出队n次：O(n)
   - 遍历邻接表：O(e)
   - 入度减1和判断：O(e)
5. 合计：O(n) + O(e) + O(n) + O(n) + O(e) + O(e)
        = O(3n + 3e)
        = O(n + e)  【省略常数】
```

**大O记号规则**：
- 省略常数系数：3n → n
- 保留最高阶项：n + e（两项均为一次项，都保留）
- 若e >> n：O(n+e) ≈ O(e)
- 若n ≈ e：O(n+e) ≈ O(n)
