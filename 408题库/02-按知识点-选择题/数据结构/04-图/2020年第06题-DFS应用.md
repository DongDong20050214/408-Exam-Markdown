# DFS真题汇总

## 2020年第06题

**元数据**
- 年份：[[../../01-真题原卷/2020年408真题#06|2020年第6题]]
- 科目：#数据结构
- 知识点：#数据结构/图/DFS #拓扑排序
- 难度：⭐⭐⭐
- 关联知识：[[知识库/数据结构/深度优先搜索]]

**题干**

修改递归方式实现的图的深度优先搜索（DFS）算法，将输出顶点信息的语句移到退出递归前。采用修改后的算法遍历有向无环图 G，若输出结果中包含 G 中的全部顶点，则输出的顶点序列是 G 的（）。

A. 拓扑有序序列  B. 逆拓扑有序序列  C. 广度优先搜索序列  D. 深度优先搜索序列

**答案**：B

**解题思路**

**标准DFS算法**：
```c
void DFS(Graph G, int v) {
    visit(v);              // 访问顶点v（输出）
    visited[v] = true;
    for (每个邻接点w) {
        if (!visited[w])
            DFS(G, w);     // 递归访问
    }
}
```

**修改后的DFS算法**（输出语句移到退出递归前）：
```c
void DFS_Modified(Graph G, int v) {
    visited[v] = true;
    for (每个邻接点w) {
        if (!visited[w])
            DFS_Modified(G, w);  // 递归访问
    }
    visit(v);              // 退出递归前输出（后序）
}
```

**关键理解**：
- **标准DFS**：进入时访问 → **前序遍历**
- **修改后DFS**：退出时访问 → **后序遍历**

**后序遍历的特点**：
- 先访问所有后代节点，再访问当前节点
- 对于有向无环图（DAG），后序遍历得到的序列是**逆拓扑序**

**为什么是逆拓扑序？**

拓扑排序要求：如果有边u→v，则u在v之前。

后序DFS：
1. 访问v时，先递归访问v的所有后继
2. v的所有后继访问完后，才访问v
3. 因此，如果有边u→v，则v会比u**先被访问**（先输出）
4. 这正是拓扑序的**逆序**

**示例**：

有向无环图：
```
A → B → D
↓   ↓
C → E
```

边：A→B, A→C, B→D, B→E, C→E

**从A开始的DFS后序遍历**：
1. 访问A，标记已访问
2. 递归B：
   - 递归D：D无后继，输出D
   - 递归E：E无后继，输出E
   - B的后继都访问完，输出B
3. 递归C：
   - E已访问，跳过
   - C的后继都访问完，输出C
4. A的后继都访问完，输出A

**输出序列**：D, E, B, C, A

**一个拓扑序**：A, B, C, D, E 或 A, C, B, E, D 等
**逆拓扑序**：E, D, C, B, A 或 D, E, B, C, A 等

输出序列D,E,B,C,A确实是逆拓扑序的一种。

**选项分析**：
- ❌ A. 拓扑有序序列 —— 正序应该是A在最前
- ✅ B. 逆拓扑有序序列 —— 正确，后继在前驱之前
- ❌ C. 广度优先搜索序列 —— BFS是按层次遍历
- ❌ D. 深度优先搜索序列 —— 标准DFS是前序，这里是后序

**易错点分析**

❌ **错误1：选择D选项**
- 误认为修改后的DFS仍然是标准DFS序列
- ✅ 正确：输出位置改变后，序列完全不同（前序变后序）

❌ **错误2：选择A选项**
- 混淆拓扑序和逆拓扑序
- ✅ 正确：后序遍历使得后继先输出，前驱后输出，是逆序

❌ **错误3：认为是某种特定的拓扑序**
- 误以为只有一种拓扑序列
- ✅ 正确：拓扑序可能有多种，但后序DFS得到的一定是逆拓扑序的一种

**知识点对比**

| DFS访问时机 | 遍历类型 | 对DAG的结果 | 应用 |
|-----------|---------|-----------|------|
| **进入时访问** | 前序遍历 | 一般DFS序列 | 图的连通性、路径查找 |
| **退出时访问** | 后序遍历 | 逆拓扑序 | 拓扑排序（反向）、强连通分量 |

**拓扑排序方法对比**

| 方法 | 原理 | 得到序列 | 时间复杂度 |
|------|------|---------|-----------|
| **Kahn算法（BFS）** | 不断删除入度为0的顶点 | 拓扑序 | O(V+E) |
| **DFS前序** | 按访问顺序输出 | 非拓扑序 | O(V+E) |
| **DFS后序反转** | 后序遍历再反转 | 拓扑序 | O(V+E) |
| **DFS后序直接** | 后序遍历直接输出 | 逆拓扑序 | O(V+E) |

**前序vs后序示例对比**

图：A→B→C

| 遍历方式 | 访问时机 | 输出序列 | 是否拓扑序 |
|---------|---------|---------|-----------|
| **前序DFS** | 进入时 | A, B, C | ✅ 是拓扑序 |
| **后序DFS** | 退出时 | C, B, A | ✅ 是逆拓扑序 |

**应用场景**

| 场景 | 使用算法 | 原因 |
|------|---------|------|
| **任务调度** | 拓扑排序 | 需要先完成依赖任务 |
| **编译顺序** | 拓扑排序 | 先编译被依赖的模块 |
| **强连通分量** | DFS后序 | Kosaraju算法第一步 |
| **检测环** | DFS | 回边检测 |

**关键理解**：
- 拓扑序：前驱在前，后继在后（符合依赖关系）
- 逆拓扑序：后继在前，前驱在后
- DFS后序自然产生逆拓扑序，因为递归"深入到底"再返回
- 要得到正拓扑序，需要将DFS后序结果反转，或使用Kahn算法

## 解题思路

### 第一步：理解题目核心
修改后的DFS在退出递归前输出，相当于后序遍历，对于DAG后序遍历输出的是逆拓扑序（后继先输出，前驱后输出）。

### 第二步：逐项分析

**A. 拓扑有序序列 ❌**
```
详细分析：拓扑序要求前驱在后继之前
判断依据：后序遍历是先递归访问所有后继再输出当前节点，导致后继先输出，前驱后输出，与拓扑序相反
结论：顺序颠倒
```

**B. 逆拓扑有序序列 ✅**
```
详细分析：后序遍历的特性
判断依据：
  1. 修改后DFS在退出递归前输出，是后序遍历
  2. 后序遍历：先访问所有后继，再访问当前节点
  3. 若有边u→v，v的后继先输出，v输出，u最后输出
  4. 结果：v在u之前，正好是拓扑序的逆序
结论：正确答案
```

**C. 广度优先搜索序列 ❌**
```
详细分析：BFS按层次遍历
判断依据：BFS使用队列按层次访问，与修改后的DFS递归回溯机制完全不同
结论：遍历策略不同
```

**D. 深度优先搜索序列 ❌**
```
详细分析：标准DFS是前序遍历
判断依据：标准DFS在进入递归时输出（前序），修改后在退出递归前输出（后序），序列完全不同
结论：访问时机改变导致结果不同
```

### 第三步：确认答案
后序遍历DAG：后继先于前驱输出，得到逆拓扑序 → 答案：**B**

## 易错点分析

### 易错点1：混淆前序和后序遍历
❌ 错误：认为修改后仍是标准DFS序列
✅ 正确：输出位置从进入递归时改为退出递归前，从前序变为后序，序列完全不同

### 易错点2：DFS访问时机与结果的关系
| 访问时机 | 遍历类型 | 对DAG的结果 | 特点 |
|---------|---------|-----------|------|
| 进入递归时 | 前序遍历 | 普通DFS序列 | 前驱先输出 |
| 退出递归前 | 后序遍历 | 逆拓扑序 | 后继先输出 |
