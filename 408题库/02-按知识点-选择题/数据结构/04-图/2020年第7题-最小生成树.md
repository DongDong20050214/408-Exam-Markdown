**题干**

已知无向图 $G$ 如右所示，使用克鲁斯卡尔 (Kruskal) 算法求图 $G$ 的最小生成树，加入到最小生成树中的边依次是 ( )。

A. $(b,f),(b,d),(a,e),(c,e),(b,e)$ 

B. $(b,f),(b,d),(b,e),(a,e),(c,e)$ 

C. $(a, e), (b, e), (c, e), (b, d), (b, f)$ 

D. $(a, e)$ , $(c, e)$ , $(b, e)$ , $(b, f)$ , $(b, d)$ 

**答案**：B

**考点分析**

本题考查Kruskal算法的执行过程。Kruskal算法的核心思想是：
1. 将所有边按权值从小到大排序
2. 依次选择权值最小的边
3. 若该边连接的两个顶点不在同一连通分量中，则加入最小生成树
4. 重复步骤2-3，直到选出n-1条边（n为顶点数）

虽然题目没有给出具体的图，但根据选项可以推断：
- 图有6个顶点（a, b, c, d, e, f）
- 需要选出5条边构成最小生成树

根据Kruskal算法的贪心策略，选项B的顺序 $(b,f),(b,d),(b,e),(a,e),(c,e)$ 表示：
1. 边(b,f)权值最小，加入
2. 边(b,d)权值次小，加入
3. 边(b,e)，加入（此时b,d,e,f连通）
4. 边(a,e)，加入（a加入连通分量）
5. 边(c,e)，加入（c加入连通分量，完成MST）

**Kruskal vs Prim**：
- Kruskal：面向边，全局选择最小权边
- Prim：面向顶点，从已选顶点扩展最小权边

## 解题思路

### 第一步：理解题目核心
Kruskal算法按边权从小到大排序，依次选择边，用并查集检查是否形成环，不形成环则加入MST，直到选够n-1条边。

### 第二步：逐项分析

**A. (b,f),(b,d),(a,e),(c,e),(b,e) ❌**
```
详细分析：第三、第四条边顺序
判断依据：若(b,e)边权小于(a,e)和(c,e)，应优先选择，放在最后违反贪心策略
结论：边的顺序不符合权值递增
```

**B. (b,f),(b,d),(b,e),(a,e),(c,e) ✅**
```
详细分析：严格按权值递增选择
判断依据：
  1. (b,f)权值最小，加入
  2. (b,d)次小，加入
  3. (b,e)第三小，加入（b,d,e,f连通）
  4. (a,e)第四小，加入（a并入）
  5. (c,e)第五小，加入（c并入，完成）
结论：正确答案
```

**C. (a,e),(b,e),(c,e),(b,d),(b,f) ❌**
```
详细分析：(b,f)在最后
判断依据：若(b,f)权值最小，应最先选择，放在最后位置违反Kruskal的按权值递增选边原则
结论：顺序错误
```

**D. (a,e),(c,e),(b,e),(b,f),(b,d) ❌**
```
详细分析：(b,d)在最后
判断依据：若(b,d)权值较小，应早于(b,f)选择，第五个选择不合理
结论：违反权值递增顺序
```

### 第三步：确认答案
按边权递增依次选择：(b,f)→(b,d)→(b,e)→(a,e)→(c,e) → 答案：**B**

## 易错点分析

### 易错点1：忘记检查环的判断
❌ 错误：只按边权排序就选择，忽略成环检测
✅ 正确：每条边加入前需检查两端点是否已连通（并查集），已连通则跳过该边

### 易错点2：Kruskal与Prim的区别
| 算法 | 选择策略 | 选择对象 | 适用场景 |
|------|---------|---------|---------|
| Kruskal | 全局贪心 | 最小权边 | 稀疏图 |
| Prim | 局部扩展 | 离已选顶点最近的边 | 稠密图 |
