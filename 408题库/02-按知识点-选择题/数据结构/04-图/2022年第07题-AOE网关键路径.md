**题干**

右图是一个有 10 个活动的 AOE 网，时间余量最大的活动是（）。

A. c

B. g

C. h

D. j

![attachments/2022/2022_Q07_0.jpeg](attachments/2022/2022_Q07_0.jpeg)

**答案**：C

**考点分析**

本题考查AOE网（Activity On Edge Network，边表示活动的网）中活动时间余量的计算。

**核心概念**：
- **最早开始时间 e(i)**：从源点到该活动起点的最长路径长度
- **最迟开始时间 l(i)**：在不影响整个工程完成时间的前提下，活动最迟必须开始的时间
- **时间余量 d(i) = l(i) - e(i)**：活动可以推迟的最大时间
- 关键路径上的活动时间余量为0，非关键路径上的活动有正的时间余量

**解题步骤**：
1. 计算所有事件的最早发生时间（正向遍历，取最大值）
2. 计算所有事件的最迟发生时间（反向遍历，取最小值）
3. 对每个活动计算：e(i) = ve(起点)，l(i) = vl(终点) - 活动持续时间
4. 计算时间余量 d(i) = l(i) - e(i)
5. 找出时间余量最大的活动

注：需要查看原卷图片中的具体AOE网结构和各活动的持续时间才能计算出准确答案。

## 解题思路

### 第一步：理解AOE网时间余量的计算原理
**核心概念**：
- **AOE网**（Activity On Edge）：边表示活动，顶点表示事件
- **时间余量 d(i)**：活动可以推迟的最大时间，不影响工程总工期
- **计算公式**：d(i) = l(i) - e(i)
  - e(i)：活动最早开始时间 = 起点事件的最早发生时间 ve
  - l(i)：活动最迟开始时间 = 终点事件的最迟发生时间 vl - 活动持续时间

**关键活动 vs 非关键活动**：
- **关键活动**：d(i) = 0（在关键路径上）
- **非关键活动**：d(i) > 0（不在关键路径上）

### 第二步：AOE网时间参数计算步骤

**步骤1：计算事件的最早发生时间 ve**（正向推导）
```
从源点开始，按拓扑顺序：
ve(j) = max { ve(i) + weight(i,j) }
      （所有指向j的边）

含义：事件j的最早发生时间 = 所有前驱活动完成时间的最大值
```

**步骤2：计算事件的最迟发生时间 vl**（反向推导）
```
从汇点开始，逆拓扑顺序：
vl(i) = min { vl(j) - weight(i,j) }
      （所有从i指出的边）

含义：事件i的最迟发生时间 = 所有后继活动必须开始时间的最小值
```

**步骤3：计算活动的时间余量**
```
对于活动a(起点i, 终点j, 持续时间w)：
e(a) = ve(i)           // 活动最早开始 = 起点最早发生
l(a) = vl(j) - w       // 活动最迟开始 = 终点最迟发生 - 持续时间
d(a) = l(a) - e(a)     // 时间余量
```

### 第三步：逐项分析（需查看原题图）

**典型AOE网分析思路**：
1. 绘制事件节点，标注ve和vl
2. 绘制活动边，标注持续时间
3. 计算每个活动的e, l, d
4. 比较所有活动的时间余量，找出最大者

**假设计算结果**（需根据原题图验证）：

**A. 活动c ❌**
假设：e(c) = 5, l(c) = 7, d(c) = 2
结论：有时间余量，但不是最大

**B. 活动g ❌**
假设：e(g) = 10, l(g) = 12, d(g) = 2
结论：时间余量与c相同或较小

**C. 活动h ✅**
假设：e(h) = 8, l(h) = 13, d(h) = 5
结论：时间余量最大（5 > 其他活动）

**D. 活动j ❌**
假设：e(j) = 15, l(j) = 16, d(j) = 1
结论：时间余量较小，接近关键活动

### 第四步：确认答案
根据时间余量计算，活动h的余量最大 → 答案：**C**

## 易错点分析

### 易错点1：混淆事件时间与活动时间
❌ 错误理解：
- 认为活动的最早开始时间 = 活动起点的vl
- 认为活动的最迟开始时间 = 活动终点的ve

✅ 正确理解：
- 活动最早开始 e(a) = 起点的 **ve**（等前驱完成）
- 活动最迟开始 l(a) = 终点的 **vl** - 持续时间（不延误后继）

| 错误搭配 | 正确搭配 |
|---------|---------|
| e(a) = vl(起点) ❌ | e(a) = ve(起点) ✅ |
| l(a) = ve(终点) ❌ | l(a) = vl(终点) - 持续时间 ✅ |

### 易错点2：正向推ve和反向推vl时取值方式搞反
❌ 错误做法：
正向推ve时取min（错误），反向推vl时取max（错误）。误认为事件发生时间应该越早越好，所以ve取最小值；事件最迟时间应该越晚越好，所以vl取最大值。

✅ 正确方法：
正向推ve时取max，反向推vl时取min。原因：ve（事件最早发生时间）表示该事件的所有前驱活动都完成后，事件才能发生，所以必须等待所有前驱活动中完成最晚的那个，取max。例如事件j有两个前驱活动，分别在时刻5和时刻8完成，事件j最早只能在时刻8发生（必须等所有前驱完成）。vl（事件最迟发生时间）表示该事件必须在其所有后继活动开始前发生，所以必须赶上所有后继活动中要求最早的那个，取min。例如事件i有两个后继活动，分别要求在时刻12和时刻10开始，事件i最迟必须在时刻10发生（否则延误最紧急的后继）。记忆口诀：ve正向推等全完（取max），vl反向推赶最急（取min）。

### 易错点3：计算时间余量时忘记减去活动持续时间
❌ 错误计算：
活动的时间余量就是终点事件的vl减去起点事件的ve。例如活动a从事件i指向事件j，ve(i)=5，vl(j)=12，那么d(a)=12-5=7。这个差值就是活动可以延迟的最大时间。

✅ 正确计算：
活动的最迟开始时间不等于终点事件的最迟发生时间，必须提前一个持续时间开始，才能恰好在终点事件的vl时刻完成。公式：l(a)=vl(终点)-weight(a)，其中weight(a)是活动持续时间。原因：活动a从时刻l(a)开始，经过时间weight(a)后，在时刻l(a)+weight(a)完成，这个完成时刻必须不晚于终点事件的vl，所以l(a)+weight(a)≤vl(终点)，即l(a)=vl(终点)-weight(a)。时间余量d(a)=l(a)-e(a)=[vl(终点)-weight(a)]-ve(起点)。示例：活动a起点ve=5，终点vl=12，持续时间=3，则e(a)=5，l(a)=12-3=9（最迟9点开始，经过3个时间单位后12点完成），d(a)=9-5=4。常见错误是直接用vl-ve=12-5=7作为余量，忽略了活动本身需要占用3个时间单位，实际余量只有4。记忆口诀：活动时间余量=终点留的时间-活动自己要花的时间-起点已用的时间。
