# 2021年408真题第29题

## 题目

在采用二级页表的分页系统中，CPU 页表基址寄存器中的内容是（）。

A. 当前进程的一级页表的起始虚拟地址

B. 当前进程的一级页表的起始物理地址

C. 当前进程的二级页表的起始虚拟地址

D. 当前进程的二级页表的起始物理地址

## 答案

B

## 考点分析

### 考点：二级页表的地址转换机制与页表基址寄存器

本题考查在采用二级页表的分页存储管理系统中，页表基址寄存器（PTBR）存储的内容及其作用。

### 解题思路：

1. **二级页表的结构**：
   - 一级页表（页目录）：索引二级页表
   - 二级页表（页表）：索引物理页框
   - 虚拟地址分为：一级页号 | 二级页号 | 页内偏移

2. **地址转换过程**：
   - 从 PTBR 获取一级页表的起始地址
   - 用一级页号索引一级页表，得到二级页表的地址
   - 用二级页号索引二级页表，得到物理页框号
   - 物理页框号 + 页内偏移 = 物理地址

3. **页表基址寄存器的内容分析**：
   
   **必须是物理地址而非虚拟地址**：
   - 地址转换过程由硬件 MMU 执行
   - MMU 需要直接访问物理内存中的页表
   - 如果存储虚拟地址，会陷入"鸡生蛋"问题（转换地址需要先转换页表地址）
   
   **必须是一级页表而非二级页表**：
   - 地址转换从一级页表开始
   - 一级页表是查找的起点
   - 二级页表的地址通过查询一级页表获得

4. **选项分析**：
   - A：虚拟地址 ✗（无法直接访问物理内存）
   - B：一级页表的物理地址 ✓（正确）
   - C：二级页表的虚拟地址 ✗（虚拟地址不可行）
   - D：二级页表的物理地址 ✗（应从一级页表开始）

### 关键点：

- 页表基址寄存器存储的是一级页表的起始物理地址
- 物理地址是必需的，因为 MMU 需要直接访问物理内存
- 地址转换过程：PTBR → 一级页表 → 二级页表 → 物理页框
- 页表本身也存储在物理内存中，需要用物理地址访问

### 易错点：

- 误认为存储虚拟地址（忽略了地址转换的"先有鸡还是先有蛋"问题）
- 混淆一级页表和二级页表的作用
- 不理解 MMU 硬件访问页表时必须使用物理地址
- 忘记页表基址寄存器是查找的起点，必须指向一级页表
