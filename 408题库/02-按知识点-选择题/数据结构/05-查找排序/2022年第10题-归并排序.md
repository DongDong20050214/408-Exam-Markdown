**题干**

使用二路归并排序对含 n 个元素的数组 M 进行排序时, 二路归并操作的功能是 ( )。

A. 将两个有序表合并为一个新的有序表
B. 将 M 划分为两部分，两部分的元素个数大致相等
C. 将 M 划分为 n 个部分，每个部分中仅含有一个元素
D. 将 M 划分为两部分, 一部分元素的值均小于另一部分元素的值

**答案**：A

**考点分析**
- 题目问的是"二路归并操作"的功能，不是归并排序的整体流程
- 归并操作（Merge）：合并两个有序子序列为一个有序序列

---

## 解题思路

### 第一步：理解归并排序的核心操作
**归并排序的两个阶段**：
1. **划分阶段**（Divide）：递归地将数组二分，直到每个子数组只有1个元素
2. **归并阶段**（Merge）：将相邻的有序子数组两两合并

**二路归并操作**特指：
- 输入：两个已排序的子序列
- 输出：一个合并后的有序序列
- 过程：逐个比较两序列的元素，依次取较小者

### 第二步：归并操作的具体过程
**示例**：
```
左子序列: [3, 7, 9]（有序）
右子序列: [2, 5, 8]（有序）

归并过程：
比较 3 vs 2 → 取2 → [2]
比较 3 vs 5 → 取3 → [2, 3]
比较 7 vs 5 → 取5 → [2, 3, 5]
比较 7 vs 8 → 取7 → [2, 3, 5, 7]
比较 9 vs 8 → 取8 → [2, 3, 5, 7, 8]
右序列空，左序列剩余直接追加 → [2, 3, 5, 7, 8, 9]
```

**关键**：归并操作的前提是两个输入序列已经有序

### 第三步：逐项分析选项
**A. 将两个有序表合并为一个新的有序表 ✅**
这正是二路归并操作的定义和功能

**B. 将M划分为两部分，两部分的元素个数大致相等 ❌**
这是归并排序的**划分阶段**，不是归并操作本身
划分是递归调用实现的，不是Merge函数的功能

**C. 将M划分为n个部分，每个部分中仅含有一个元素 ❌**
这是归并排序的**递归终止条件**（划分到最细粒度），不是归并操作
单个元素自然有序，是归并操作的起始输入

**D. 将M划分为两部分，一部分元素的值均小于另一部分元素的值 ❌**
这是**快速排序的划分操作**（Partition），不是归并操作
快排以枢轴为界划分，归并不做这种划分

### 第四步：对比易混淆概念
**归并排序 vs 归并操作**：
```
归并排序（Merge Sort）：
  function mergeSort(arr):
    if arr.length <= 1:
      return arr
    mid = arr.length / 2
    left = mergeSort(arr[0...mid])    // 递归划分
    right = mergeSort(arr[mid...n])   // 递归划分
    return merge(left, right)         // 归并操作 ← 题目问的是这个

归并操作（Merge）：
  function merge(left, right):
    result = []
    while left非空 and right非空:
      if left[0] <= right[0]:
        result.append(left[0])
        left移除首元素
      else:
        result.append(right[0])
        right移除首元素
    追加剩余元素
    return result
```

## 易错点分析

### 易错点1：混淆归并操作和划分操作
❌ 错误理解：归并排序要把数组分成两半，所以归并操作就是划分。题目问的是二路归并操作，应该选B或C这些划分操作。"二路"就是指分成两路，所以B选项"将M划分为两部分"正是二路归并操作的定义。

✅ 正确理解："归并"（Merge）的英文含义就是"合并"，不是"分割"或"划分"。二路归并中的"二路"指的是合并两个有序序列，而不是分成两部分。归并排序确实需要划分数组，但划分是通过递归调用mergeSort函数实现的（分解问题阶段），归并操作merge函数的职责是合并两个已排序的子数组（合并子问题的解阶段）。这是分治算法的经典模式：分解（Divide）→ 递归求解（Conquer）→ 合并（Merge）。题目明确问"二路归并操作"，指的就是Merge函数的功能，即将两个有序表合并为一个新的有序表，选项A完全正确。记忆：Merge=合并，Divide=划分，不要混淆。

### 易错点2：将快速排序的划分误认为是归并操作
❌ 错误理解：归并排序也要划分数组，所以划分就是归并操作的一部分。选项D描述的是快速排序的Partition操作，但归并排序也需要把数组分成两部分，所以D也可能正确。所有排序算法的核心操作都差不多，都是划分和重组，归并排序的"归并"应该包括划分过程。

✅ 正确理解：归并操作（Merge）专指"合并两个有序序列"，与任何形式的"划分"都无关。这是命名和定义上的严格区分。选项D"将M划分为两部分，一部分元素的值均小于另一部分元素的值"是快速排序的Partition操作，它通过选择枢轴（pivot）并将元素交换到枢轴两侧实现按值划分。归并排序确实需要划分数组，但那是递归调用mergeSort函数实现的简单物理二分（从中间位置切开，不比较元素值），不是Merge函数的职责。快排和归并的本质差异：快排是"先做有意义的划分（按值分大小），再递归处理"；归并是"先递归分解到最简单（单元素），再合并"。记忆：快排的核心在Partition（划分），归并的核心在Merge（合并）。

## 知识点对比

### 归并排序 vs 快速排序的核心操作

| 维度 | 归并排序 | 快速排序 |
|------|---------|----------|
| **核心操作** | Merge（合并） | Partition（划分） |
| **操作时机** | 递归返回时（后序） | 递归前（先序） |
| **操作对象** | 两个已排序子数组 | 一个未排序数组 |
| **空间需求** | O(n)（合并需要辅助数组） | O(1)（原地划分） |
| **稳定性** | ✓ 稳定 | ❌ 不稳定 |

### 归并排序的完整流程示意

| 阶段 | 操作 | 数组状态 | 说明 |
|------|------|---------|------|
| **划分** | 递归二分 | [8,3,5,1,9,2] → [8,3,5] [1,9,2] | 不是归并操作 |
| ↓ | 继续划分 | [8] [3,5] [1] [9,2] | 递归调用 |
| ↓ | 划分到底 | [8] [3] [5] [1] [9] [2] | 单元素自然有序 |
| **归并** | **Merge操作** | [3] [5] → [3,5] | **题目问的** |
| ↓ | 继续归并 | [8] [3,5] → [3,5,8] | 合并有序子序列 |
| ↓ | 最终归并 | [3,5,8] [1,2,9] → [1,2,3,5,8,9] | 完成排序 |

### 归并操作（Merge）的伪代码

```
function merge(left[], right[]):
    result = []
    i = 0, j = 0
    
    // 逐个比较，取较小者
    while i < left.length AND j < right.length:
        if left[i] <= right[j]:
            result.append(left[i])
            i++
        else:
            result.append(right[j])
            j++
    
    // 追加剩余元素
    追加 left[i...] 到 result
    追加 right[j...] 到 result
    
    return result
```

**关键特征**：
- 输入：两个有序数组
- 输出：一个有序数组
- 时间：O(n)，n为两数组长度之和
