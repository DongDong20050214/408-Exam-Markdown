# 直接插入排序真题汇总

## 2020年第11题

**元数据**
- 年份：[[../../01-真题原卷/2020年408真题#11|2020年第11题]]
- 科目：#数据结构
- 知识点：#数据结构/排序/插入排序
- 难度：⭐⭐
- 关联知识：[[知识库/数据结构/插入排序]]

**题干**

对大部分元素已有序的数组进行排序时，直接插入排序比简单选择排序效率更高，其原因是（）。

I. 直接插入排序过程中元素之间的比较次数更少  II. 直接插入排序过程中所需要的辅助空间更少  III. 直接插入排序过程中元素的移动次数更少

A. 仅I  B. 仅 III  C. 仅I、II  D. I、II和III

**答案**：A

**考点分析**
- I正确：已有序时比较次数少

---

## 解题思路

### 第一步：对比两种算法的基本特征

**直接插入排序**：
- 从第2个元素开始，逐个插入到前面已排序部分
- 已有序时：每个元素只需比较1次（与前一个比较即可确定位置）
- 比较次数：最好O(n)，平均O(n²)

**简单选择排序**：
- 每次从未排序部分选择最小元素，放到已排序部分末尾
- 无论数据是否有序，都要遍历剩余所有元素找最小值
- 比较次数：始终O(n²)，与初始状态无关

### 第二步：针对"大部分元素已有序"逐项分析

**I. 比较次数更少 - 正确 ✓**

**直接插入排序（已有序）**：
```
序列：1, 2, 3, 4, 5（已有序）
第2个元素2：与1比较 → 2≥1，停止 → 1次比较
第3个元素3：与2比较 → 3≥2，停止 → 1次比较
...
总比较：n-1次
```

**简单选择排序（已有序）**：
```
序列：1, 2, 3, 4, 5（已有序）
第1趟：在[1,2,3,4,5]中找最小 → 4次比较
第2趟：在[2,3,4,5]中找最小 → 3次比较
...
总比较：4+3+2+1 = n(n-1)/2次
```

**结论**：大部分有序时，直接插入比较次数接近O(n)，选择排序仍是O(n²)

**II. 辅助空间更少 - 错误 ✗**

两种算法的空间复杂度：
- **直接插入排序**：O(1)（只需一个临时变量存储待插入元素）
- **简单选择排序**：O(1)（只需一个临时变量存储最小值位置）

**结论**：两者辅助空间相同，都是O(1)，这不是效率差异的原因

**III. 移动次数更少 - 错误 ✗**

**直接插入排序（已有序）**：
```
每个元素已在正确位置，无需移动
移动次数：0
```

**简单选择排序（已有序）**：
```
每趟找到的最小值已在正确位置
交换次数：0（或n-1次自我交换，取决于实现）
```

但考虑**大部分有序**（少数逆序）：
```
序列：1, 3, 2, 4, 5（只有3,2逆序）

直接插入：
- 处理2时：3向后移1位，2插入 → 移动2次
总移动：约等于逆序对数×2

简单选择：
- 第2趟：交换2和3 → 移动3次（临时变量）
总移动：约等于逆序对数×3
```

**重新分析**：移动次数与逆序对数量相关，大部分有序时直接插入移动也较少，但题目问的是"效率更高的原因"，**主因是比较次数**，不是移动次数。

实际上，简单选择排序的移动次数是固定的（每趟1次交换），而直接插入的移动与逆序对成正比。**大部分有序时，直接插入的移动次数也可能更多**（因为要移动已排序部分的元素）。

**重新理解III**：简单选择排序每趟只交换1次（3次移动），直接插入可能要移动多个元素。所以III错误。

### 第三步：逐项分析选项

**A. 仅I ✅**
正确：比较次数是主要差异

**B. 仅III ❌**
移动次数不是主因，且不一定更少

**C. 仅I、II ❌**
II错误，辅助空间相同

**D. I、II和III ❌**
II和III都不对

## 易错点分析

### 易错点1：认为移动次数是主要原因
❌ 错误理解：直接插入排序在大部分有序时效率高，是因为元素移动次数少。简单选择排序每次只交换一次（3次移动），而直接插入需要移动整个有序区的元素，显然选择排序移动更少，所以III应该是错的，答案选A没问题。但实际上我觉得I和III都对，答案应该是D。

✅ 正确理解：**比较次数是主因**，而非移动次数。大部分有序时：①直接插入O(n)次比较（每个元素仅需与前一元素比较即确定已在正确位置），选择排序仍需O(n²)次比较（必须遍历剩余所有元素找最小值，与数据有序程度无关）。②移动次数：选择排序每趟1次交换（3次元素赋值），直接插入可能需移动多个元素为插入腾出空间。大部分有序时直接插入移动也少，但即便移动多些，比较操作节省的时间远大于移动增加的时间，故比较次数是效率差异的决定因素。

### 易错点2：混淆辅助空间和操作空间
❌ 错误理解：直接插入排序需要不断移动已排序区的元素，移动过程需要额外的空间来暂存元素，所以辅助空间比简单选择排序更大。因此II是对的，答案应该选C（I和II都对）。

✅ 正确理解：算法的**辅助空间（额外空间复杂度）**仅指算法运行时除输入数据外需要的额外变量空间，不包括输入数组本身占用的空间。直接插入排序：需1个临时变量存储待插入元素。简单选择排序：需1个临时变量存储最小值索引及交换用临时变量。两者辅助空间都是O(1)，完全相同。元素移动操作是在原数组上进行的覆盖赋值，不消耗额外空间，只是时间开销。II不是效率差异原因。

| 概念 | 直接插入排序 | 简单选择排序 | 说明 |
|------|------------|------------|------|
| **输入数组** | 在原数组上操作 | 在原数组上操作 | 不算辅助空间 |
| **辅助变量** | 1个（暂存待插入元素） | 1个（暂存最小值下标） | 都是O(1) |
| **辅助空间** | O(1) | O(1) | **完全相同** |
| **移动操作** | 可能多次移动 | 元素交换 | 这是时间开销，非空间 |

## 知识点对比

### 插入排序 vs 选择排序的特性

| 特性 | 直接插入排序 | 简单选择排序 | 优劣对比 |
|------|------------|------------|---------|
| **时间复杂度（最好）** | O(n) | O(n²) | **插入排序优** |
| **时间复杂度（平均）** | O(n²) | O(n²) | 相同 |
| **时间复杂度（最坏）** | O(n²) | O(n²) | 相同 |
| **空间复杂度** | O(1) | O(1) | 相同 |
| **稳定性** | ✓ 稳定 | ❌ 不稳定 | 插入排序优 |
| **适用场景** | **数据已部分有序** | 数据量小、无序度高 | 插入排序适应性强 |

### 已有序数据的性能对比

| 序列状态 | 直接插入 | 简单选择 | 差异原因 |
|---------|---------|---------|---------|
| **完全有序** | n-1次比较，0次移动 | n(n-1)/2次比较 | **插入O(n) vs 选择O(n²)** |
| **大部分有序** | 接近O(n)次比较 | 仍然O(n²)次比较 | **插入自适应** |
| **完全逆序** | O(n²)次比较和移动 | O(n²)次比较 | 两者接近 |

### 不同排序算法的适用场景

| 算法 | 时间复杂度 | 空间 | 稳定性 | 最佳场景 |
|------|-----------|------|--------|---------|
| **直接插入** | O(n)~O(n²) | O(1) | ✓ | **小规模、部分有序** |
| 简单选择 | O(n²) | O(1) | ❌ | 小规模、移动代价大 |
| 冒泡排序 | O(n)~O(n²) | O(1) | ✓ | 教学演示 |
| 快速排序 | O(nlogn) | O(logn) | ❌ | 大规模、随机数据 |
| 归并排序 | O(nlogn) | O(n) | ✓ | 需要稳定性 |
