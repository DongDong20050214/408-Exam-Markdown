# 2017年第1题

**元数据**
- 年份：2017年第1题
- 难度：⭐⭐⭐

**题干**：下列函数的时间复杂度是（）。

```c
int func(int n) {
    int i=0, sum=0;
    while (sum < n) sum += ++i;
    return i;
} 
```

**选项**：
A. $O(\log n)$ 
B. $O(n^{1 / 2})$ 
C. $O(n)$ 
D. $O(n\log n)$ 

**答案**：B

**考点分析**：
本题考查算法时间复杂度的分析能力。关键在于分析循环执行的次数。循环中 `sum` 累加了 `1+2+3+...+i`，当 `sum >= n` 时循环结束。根据等差数列求和公式，`sum = i(i+1)/2`，因此需要满足 `i(i+1)/2 >= n`，解得 `i ≈ √(2n)`，即循环执行了 $O(\sqrt{n})$ 次。这是一个典型的数学分析问题，要求考生能够从代码逻辑推导出循环次数与输入规模之间的关系。

## 解题思路（逐项分析法）

### 第一步：明确题目条件
```c
int i=0, sum=0;
while (sum < n) sum += ++i;
```

### 第二步：分析sum的增长
每次循环：
```
第1次：sum = 1
第2次：sum = 1+2 = 3
第3次：sum = 1+2+3 = 6
第k次：sum = 1+2+...+k = k(k+1)/2
```
终止：$\frac{k(k+1)}{2} \geq n$

### 第三步：求k
```
k(k+1)/2 ≥ n
k² + k ≥ 2n
k² ≈ 2n
k ≈ √(2n) = O(√n)
```

### 第四步：逐项判断

**A. $O(\log n)$ ❌**
```
对数增长：log n
本题：√n
√n >> log n
结论：错误
```

**B. $O(n^{1/2})$ ✅**
```
n^(1/2) = √n
符合分析结果
结论：正确 ✅
```

**C. $O(n)$ ❌**
```
线性复杂度
√n << n
结论：错误
```

**D. $O(n\log n)$ ❌**
```
常见于高效排序
远大于√n
结论：错误
```

### 第五步：确认答案
答案：**B**

## 易错点分析

### 陷阱1：看循环变量递增就认为O(n)
❌ 错误理解：i从0增到某值，复杂度O(n)
✅ 正确理解：sum增长速度快，i只需增到√n

### 陷阱2：忽略等差数列求和
❌ 错误做法：直接看i的范围
✅ 正确方法：分析sum = k(k+1)/2

## 知识点对比

| 终止条件 | 循环次数 | 复杂度 |
|---------|---------|--------|
| i < n | n | O(n) |
| sum < n (sum=i) | n | O(n) |
| sum < n (sum=1+..+i) | √n | O(√n) |
| i*i < n | √n | O(√n) |
