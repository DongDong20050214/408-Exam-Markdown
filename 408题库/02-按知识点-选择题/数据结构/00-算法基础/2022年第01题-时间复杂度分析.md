# 2022年第1题

**元数据**
- 年份：[2022年第1题](../../../01-真题原卷/2022年408真题.md#01)
- 科目：#数据结构
- 知识点：#数据结构/算法基础/时间复杂度分析
- 难度：⭐⭐

**题干**

下列程序段的时间复杂度是（）。

$$
\begin{array}{l} \text {int sum = 0;} \\ \text {for (int i = 1; i <   n; i *= 2)} \\ \text {for (int j = 0; j <   i; j + + )} \\ \text {sum + + ;} \end{array}
$$

A. $O(\log n)$ 

B. $O(n)$ 

C. $O(n\log n)$ 

D. $O(n^{2})$ 

**答案**：B

**考点分析**

本题考查嵌套循环的时间复杂度分析，特别是外层循环变量按指数增长的情况。

外层循环：`i` 从 1 开始，每次乘以 2，即 $i = 1, 2, 4, 8, \ldots, 2^k$，直到 $i < n$，共执行 $\log_2 n$ 次。

内层循环：对于外层的每个 $i$ 值，内层循环执行 $i$ 次。

总执行次数为：$1 + 2 + 4 + 8 + \cdots + 2^{\lfloor \log_2 n \rfloor}$

这是一个等比数列，首项为 1，公比为 2，项数为 $\log_2 n$。根据等比数列求和公式：

$$S = \frac{2^{\log_2 n + 1} - 1}{2 - 1} = 2^{\log_2 n + 1} - 1 = 2n - 1$$

因此时间复杂度为 $O(n)$。

**关键点**：当循环变量呈指数增长时，各次迭代次数之和等于最后一项的常数倍（等比数列性质），因此总复杂度由最后一项决定。

## 解题思路（逐项分析法）

### 第一步：明确题目条件
```c
int sum = 0;
for (int i = 1; i < n; i *= 2)   // 外层：i = 1, 2, 4, 8, ...
    for (int j = 0; j < i; j++)   // 内层：执行 i 次
        sum++;
```
- 外层循环变量 i 按指数增长（×2）
- 内层循环次数 = 当前 i 的值
- 关键：内层执行次数**依赖**外层变量，不是常数

### 第二步：计算总执行次数
外层迭代 k 次，满足 2^k < n ≤ 2^(k+1)，即 k ≈ ⌊log₂n⌋

| 外层第t次 | i的值 | 内层执行次数 |
|-----------|-------|-------------|
| 1 | 1 = 2⁰ | 1 |
| 2 | 2 = 2¹ | 2 |
| 3 | 4 = 2² | 4 |
| ... | ... | ... |
| k | 2^(k-1) | 2^(k-1) |

总执行次数：1 + 2 + 4 + ... + 2^(k-1) = **2^k - 1**

因为 2^k < 2n（由 2^(k-1) < n 可知），所以 2^k - 1 = **O(n)**

### 第三步：逐项判断

**A. O(log n) ❌**
```
如果内层循环执行常数次（如 j < 100），则总复杂度为 O(log n)。
但本题内层执行 i 次，远大于常数。
结论：严重低估，错误。
```

**B. O(n) ✅**
```
等比求和：1+2+4+...+2^(k-1) = 2^k - 1 ≈ 2n - 1
结论：正确 ✅
```

**C. O(n log n) ❌**
```
典型错误：外层 O(log n) × 内层"平均"O(n) = O(n log n)
但内层不独立 — 每次的 i 值不同，不能用"平均n"来乘
正确做法是求和而非乘积
结论：常见错误答案
```

**D. O(n²) ❌**
```
这是当两层循环都为 O(n) 时的复杂度
与实际计算结果差距很大
结论：明显错误
```

### 第四步：确认答案
等比数列 1+2+4+...+2^(k-1) = 2^k - 1 = O(2^k) = O(n)

答案：**B**

## 易错点分析

### 陷阱1：直接"外层×内层"相乘
❌ **错误做法**：外层 log n 次 × 内层"大约n/2次" = O(n log n)
✅ **正确做法**：内层执行次数随外层变量变化，必须**逐项求和**再判断量级
```
关键判断：内层 for(j=0; j<n; j++) → 可以相乘
          内层 for(j=0; j<i; j++) → 不能相乘，必须求和
```

### 陷阱2：等比数列求和后误判量级
❌ **错误推理**：等比数列求和 = 2^k - 1，而 k = log n，所以 2^(log n) - 1 = n - 1 → O(n) ✅ 其实这个是对的
❌ **另一种错误**：k = log n，项数约为 log n，首项1末项≈n，误用"项数×平均值"得出 O(n log n)
✅ **正确理解**：等比数列的和由**末项主导**，1+2+4+...+n ≈ 2n = O(n)

### 陷阱3：混淆"指数增长"与"对数复杂度"
❌ 看到 `i *= 2` 就认为整个算法是 O(log n)
✅ 外层循环次数是 O(log n)，但**总工作量**需要累加每次迭代的代价

## 知识点对比

| 嵌套循环模式 | 外层次数 | 内层次数 | 总复杂度 | 分析方法 |
|-------------|---------|---------|---------|---------|
| for(i=1;i<n;i*=2) for(j=0;j<n;j++) | O(log n) | O(n) 常数 | O(n log n) | 直接相乘 |
| for(i=1;i<n;i*=2) for(j=0;j<i;j++) | O(log n) | 依赖i | **O(n)** | 等比求和 |
| for(i=1;i<n;i++) for(j=0;j<i;j++) | O(n) | 依赖i | O(n²) | 等差求和 |
| for(i=1;i<n;i++) for(j=1;j<n;j*=2) | O(n) | O(log n) | O(n log n) | 直接相乘 |

**记忆口诀**：内层依赖外层的变量 → 必须求和；内层与外层独立 → 直接相乘。`i*=2` 的外层是 O(log n) 次，但等比数列的和仍是 O(n)。
