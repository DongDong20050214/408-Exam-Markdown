# 2012年第1题

**元数据**
- 年份：[../../01-真题原卷/2012年408真题#01|2012年第1题](../../01-真题原卷/2012年408真题#01|2012年第1题.md)

- 难度：⭐⭐

**题干**

求整数 $n$ ( $n \geqslant 0$ ) 阶乘的算法如下，其时间复杂度是（ ）。

```c
int fact(int n){
    if (n<=1) return 1;
    return n*fact(n-1);
} 
```

A. $O(\log_2n)$ 

B. $O(n)$ 

C. $O(n\log_2n)$ 

D. $O(n^{2})$ 

**答案**：B

**考点分析**

本题考查递归算法的时间复杂度分析。该递归函数每次调用自身时参数减1，递归深度为n，每层递归只执行常数时间的乘法运算，因此总时间复杂度为 $T(n) = T(n-1) + O(1) = O(n)$。这是典型的线性递归结构。

## 解题思路（逐项分析法）

### 第一步：明确题目条件
- 递归函数：计算n!
- 递归终止条件：n ≤ 1
- 递归调用：fact(n-1)

### 第二步：回顾递归复杂度分析方法
递归时间复杂度 = 递归深度 × 每层操作时间
- 递归深度：从n递减到1需要多少次调用
- 每层操作：除递归调用外的其他操作

### 第三步：分析递归过程
```
fact(n) 调用链：
fact(n) → fact(n-1) → fact(n-2) → ... → fact(1)

递归深度：n次调用
每层操作：1次乘法 = O(1)
总时间：n × O(1) = O(n)
```

递推式：T(n) = T(n-1) + O(1)
展开：T(n) = O(1) + O(1) + ... + O(1) = n × O(1) = O(n)

### 第四步：确认答案
线性递归，递归深度n → 答案：**B**

## 易错点分析

### 陷阱1：看到递归就认为是指数级
❌ **错误理解**：递归都是O(2ⁿ)，很慢
✅ **正确理解**：递归复杂度取决于递归树的结构
- 单分支递归（如阶乘）：O(n)
- 双分支递归（如斐波那契朴素实现）：O(2ⁿ)
- 二分递归（如归并排序）：O(n log n)

### 陷阱2：混淆递归深度和函数调用次数
❌ **错误理解**：每层递归要调用n次，所以是O(n²)
✅ **正确理解**：
- 递归深度：n层
- 每层调用次数：1次
- 总调用次数：n次 → O(n)

### 陷阱3：忽略常数时间操作
❌ **错误理解**：有乘法运算，应该是O(n²)
✅ **正确理解**：
- 每层只有1次乘法：O(1)
- 不是n个数相乘需要n²次运算
- 是n层递归，每层O(1)

## 知识点对比

| 递归类型 | 代码特征 | 递归树 | 时间复杂度 | 典型例子 |
|---------|---------|--------|-----------|----------|
| 线性递归 | 1次递归调用 | 单链 | O(n) | 阶乘、链表遍历 |
| 二分递归 | 1次递归，规模减半 | 单链，变短快 | O(log n) | 二分查找 |
| 双分支递归 | 2次递归调用 | 二叉树 | O(2ⁿ) | 斐波那契（朴素） |
| 分治递归 | 2次递归+合并 | 二叉树+合并 | O(n log n) | 归并排序 |
