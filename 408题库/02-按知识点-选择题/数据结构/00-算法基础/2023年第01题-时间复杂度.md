# 2023年第1题

**元数据**
- 年份：2023年第1题
- 科目：#数据结构
- 难度：⭐⭐

**题干**：下列对顺序存储的有序表（长度为 $n$ ）实现给定操作的算法中，平均时间复杂度为 $O(1)$ 的是（）。

A. 查找包含指定值元素的算法

B. 插入包含指定值元素的算法

C. 删除第 $i$ ( $1 \leqslant i \leqslant n$ ) 个元素的算法

D. 获取第 $i$ ( $1 \leqslant i \leqslant n$ ) 个元素的算法

**答案**：D

**考点分析**：本题考查顺序存储有序表的时间复杂度分析。对于顺序存储的有序表，随机访问特性使得获取第i个元素可以直接通过下标计算实现，时间复杂度为O(1)。而查找需要O(log n)或O(n)，插入和删除由于需要移动元素,平均时间复杂度为O(n)。

## 解题思路（逐项分析法）

### 第一步：明确题目条件
- 数据结构：**顺序存储的有序表**
- 长度：n
- 问题：哪个操作的平均时间复杂度为O(1)

### 第二步：回顾顺序表核心特性
顺序表的关键特性：
1. ✅ **随机访问**：支持O(1)时间按位置访问（通过下标直接计算地址）
2. ❌ **插入删除**：需要移动元素，平均O(n)
3. ❌ **按值查找**：即使有序也需要遍历或折半查找

时间复杂度回顾：
- O(1)：常数时间，不随数据规模变化
- O(log n)：对数时间，如折半查找
- O(n)：线性时间，需要遍历

### 第三步：逐项判断

**A. 查找包含指定值元素的算法 ❌**
```
无序表查找：顺序查找
  - 平均比较次数：(1+2+...+n)/n = (n+1)/2
  - 时间复杂度：O(n)

有序表查找：折半查找
  - 每次比较排除一半元素
  - 最多比较log₂n次
  - 时间复杂度：O(log n)

结论：无论哪种情况都不是O(1)
```

**B. 插入包含指定值元素的算法 ❌**
```
有序表插入步骤：
  步骤1：查找插入位置
    - 折半查找：O(log n)
  
  步骤2：移动元素（保持有序）
    - 平均需要移动n/2个元素
    - 每个元素移动一位：O(n)
  
  步骤3：插入元素
    - 直接赋值：O(1)

总时间复杂度：O(log n) + O(n) + O(1) = O(n)

结论：平均时间复杂度为O(n)
```

**C. 删除第i (1≤i≤n) 个元素的算法 ❌**
```
删除第i个元素步骤：
  步骤1：定位第i个元素
    - 顺序表支持随机访问：base + (i-1)×sizeof(ElemType)
    - 时间：O(1) ✅
  
  步骤2：移动后续元素
    - 需将第i+1到第n个元素前移一位
    - 移动元素数：n-i个
    - 平均移动数：(0+1+...+(n-1))/n = (n-1)/2
    - 时间：O(n) ❌

总时间复杂度：O(1) + O(n) = O(n)

陷阱：定位是O(1)，但移动是O(n)，取最大值

结论：平均时间复杂度为O(n)
```

**D. 获取第i (1≤i≤n) 个元素的算法 ✅**
```
顺序表存储原理：
  - 起始地址：base
  - 每个元素大小：sizeof(ElemType)
  - 第i个元素地址：base + (i-1) × sizeof(ElemType)

操作步骤：
  1. 计算地址：一次乘法 + 一次加法 = O(1)
  2. 访问内存：直接读取 = O(1)

关键：不需要遍历，直接通过下标计算出地址

结论：时间复杂度为O(1) ✅
```

### 第四步：确认答案
顺序表的核心优势：**随机访问特性**，即支持O(1)时间按位置（下标）直接访问元素。

答案：**D**

## 易错点分析

### 陷阱1：混淆"按位置访问"和"按值查找"
❌ **错误理解**：获取第i个元素需要从头遍历到第i个
✅ **正确理解**：
- **按位置访问**（给定下标i）：O(1)，直接计算地址
- **按值查找**（给定value）：O(n)或O(log n)，需要比较

| 操作 | 输入 | 方法 | 时间复杂度 |
|------|------|------|------------|
| 按位置访问 | 下标i | 地址计算 | O(1) |
| 按值查找 | 值value | 遍历/折半 | O(n)/O(log n) |

### 陷阱2：认为有序表查找是O(1)
❌ **错误理解**：有序表可以折半查找，很快，就是O(1)
✅ **正确理解**：
- 折半查找：O(log n)，不是O(1)
- O(log n)：数据量翻倍，时间只增加1次比较
- O(1)：数据量翻倍，时间不变

例如：n=1000时，log₂1000≈10次比较，仍远大于1

### 陷阱3：忽略"平均"时间复杂度
C选项删除第i个元素：
- **最好情况**：删除最后一个元素，不移动，O(1)
- **最坏情况**：删除第一个元素，移动n-1个，O(n)
- **平均情况**：移动(n-1)/2个，O(n)

题目问的是**平均时间复杂度**，所以C选项是O(n)

## 知识点对比

| 操作 | 顺序表 | 单链表 | 记忆技巧 |
|------|--------|--------|----------|
| 按位置访问（获取第i个） | **O(1)** ✅ | O(n) | 顺序表"随机访问" |
| 按值查找 | O(n) | O(n) | 都需要遍历 |
| 按值查找（有序） | O(log n) | O(n) | 顺序表可折半 |
| 插入（已知位置） | O(n) | **O(1)** ✅ | 链表"指针修改" |
| 删除（已知位置） | O(n) | **O(1)** ✅ | 链表不移动元素 |

**记忆口诀**：
- 顺序表擅长"读"（访问）→ 随机访问O(1)
- 链表擅长"改"（插入删除）→ 指针修改O(1)

**顺序表随机访问原理**：
```
数组：int a[100];
访问a[i]：地址 = &a[0] + i * sizeof(int)
          ↑        ↑       ↑
        起始地址  偏移量  元素大小
        
只需1次乘法+1次加法，与n无关 → O(1)
```
