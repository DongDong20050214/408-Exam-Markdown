# 时间复杂度真题汇总

## 2011年第01题

**元数据**
- 年份：[[../../01-真题原卷/2011年408真题#01|2011年第1题]]
- 科目：#数据结构
- 知识点：#数据结构/算法基础/时间复杂度
- 难度：⭐⭐
- 关联知识：[[知识库/数据结构/时间复杂度]]

**题干**

设 n 是描述问题规模的非负整数，下面程序片段的时间复杂度是（ ）。
x = 2;
while (x < n / 2)
  x = 2 * x;

A. O(log₂n)  B. O(n)  C. O(nlog₂n)  D. O(n²)

**答案**：A

**考点分析**
- x每次翻倍
- 循环log₂n次

## 解题思路（逐项分析法）

### 第一步：明确题目条件
- 初始值：x = 2
- 循环条件：x < n/2
- 每次操作：x = 2 * x（x翻倍）

### 第二步：回顾相关知识点
指数增长型循环的时间复杂度分析：
- 变量呈指数增长（每次×2、×3等）→ O(log n)
- 变量呈线性增长（每次+1、+2等）→ O(n)

### 第三步：推导循环次数
```
第1次循环后：x = 4 = 2²
第2次循环后：x = 8 = 2³
第3次循环后：x = 16 = 2⁴
...
第k次循环后：x = 2^(k+1)

循环终止条件：2^(k+1) ≥ n/2
即：2^(k+1) ≥ n/2
    k+1 ≥ log₂(n/2) = log₂n - 1
    k ≥ log₂n - 2
```
循环执行次数为 O(log₂n)

### 第四步：确认答案
循环次数正比于 log₂n → 答案：**A**

## 易错点分析

### 陷阱1：看到循环就认为是O(n)
❌ **错误理解**：有while循环，复杂度就是O(n)
✅ **正确理解**：要看循环变量的增长方式
- 线性增长（i++）→ O(n)
- 指数增长（i*=2）→ O(log n)

### 陷阱2：混淆循环条件中的n/2
❌ **错误理解**：条件是n/2，所以复杂度是O(n/2) = O(n)
✅ **正确理解**：n/2只影响常数项，不改变对数本质
- x从2增长到n/2需要log₂(n/2) = log₂n - 1次
- 时间复杂度仍是O(log₂n)

### 陷阱3：误算循环次数
❌ **错误理解**：x每次翻倍，从2到n需要n/2次
✅ **正确理解**：指数增长非常快
- 2 → 4 → 8 → 16 → 32 → ...
- 到达n只需log₂n次，而非n/2次

## 知识点对比

| 循环类型 | 代码示例 | 增长方式 | 时间复杂度 |
|---------|---------|---------|-----------|
| 线性增长 | `i++` | 等差数列 | O(n) |
| 指数增长 | `i*=2` | 等比数列 | O(log n) |
| 对数衰减 | `n/=2` | 等比递减 | O(log n) |
| 平方增长 | `i*=i` | 指数型 | O(log log n) |

**记忆口诀**：倍增倍减都是对数，加减才是线性
