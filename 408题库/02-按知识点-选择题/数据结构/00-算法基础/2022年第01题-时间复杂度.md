- 关联知识：[知识库/数据结构/时间复杂度](知识库/数据结构/时间复杂度.md)

**题干**

下列程序段的时间复杂度是（）。
```
int sum = 0;
for (int i = 1; i < n; i *= 2)
  for (int j = 0; j < i; j++)
    sum++;
```

A. O(log n)  B. O(n)  C. O(n log n)  D. O(n²)

**答案**：B

**考点分析**
- 外层循环：i = 1, 2, 4, 8, ..., 共log n次
- 内层执行次数：1 + 2 + 4 + ... + n/2 = n-1
- 总复杂度：O(n)

## 解题思路（逐项分析法）

### 第一步：明确题目条件
- 外层循环：i=1, i<n, i*=2（指数增长）
- 内层循环：j=0, j<i, j++（每次执行i次）
- 关键：内层次数依赖外层变量i

### 第二步：识别题目类型
这是**依赖型嵌套循环**，内层上限随外层变化：
- 不能简单相乘（外层×内层）
- 需要对每次外层循环的内层次数求和

### 第三步：逐次累加计算总次数
```
外层第1次：i=1，内层执行1次
外层第2次：i=2，内层执行2次
外层第3次：i=4，内层执行4次
外层第4次：i=8，内层执行8次
...
外层最后：i=n/2，内层执行n/2次

总次数 = 1 + 2 + 4 + 8 + ... + n/2
这是等比数列，首项a₁=1，公比q=2，项数k=log₂n

求和公式：S = a₁(q^k - 1)/(q-1)
       = 1×(2^(log₂n) - 1)/(2-1)
       = n - 1
       = O(n)
```

### 第四步：确认答案
等比数列求和为O(n) → 答案：**B**

## 易错点分析

### 陷阱1：错误地相乘两层循环次数
❌ **错误理解**：外层O(log n)，内层O(n)，所以是O(n log n)
✅ **正确理解**：内层不是每次都执行n次
- 不是：log n × n = O(n log n)
- 而是：1+2+4+...+n/2 = O(n)

### 陷阱2：忽略等比数列求和公式
❌ **错误理解**：1+2+4+...有log n项，每项平均n/2，所以是O(n log n)
✅ **正确理解**：等比数列求和≈最大项
- 1+2+4+...+n/2 ≈ 2×(n/2) = n
- 等比数列和接近最大项的2倍

### 陷阱3：混淆独立循环和依赖循环
**独立型**（相乘）：
```c
for(i=1; i<n; i*=2)     // log n
  for(j=0; j<n; j++)    // n
// 总次数：log n × n = O(n log n)
```

**依赖型**（求和）：
```c
for(i=1; i<n; i*=2)     // log n
  for(j=0; j<i; j++)    // i（变化）
// 总次数：1+2+4+...+n/2 = O(n)
```

## 知识点对比

| 循环结构 | 内层次数 | 计算方式 | 时间复杂度 |
|---------|---------|---------|-----------|
| 独立嵌套 | 固定为n | 外层×内层 | O(log n × n) = O(n log n) |
| 依赖嵌套（等比） | 随i倍增 | 等比数列求和 | O(n) **本题** |
| 依赖嵌套（等差） | 随i递增 | 等差数列求和 | O(n²) |
| 依赖递减 | 从n到i | 调和级数 | O(n log n) |

**等比数列求和记忆**：S ≈ 2×最大项（公比≥2时）
