# 2011年第32题

**元数据**
- 年份：2011年第32题
- 科目：#数据结构
- 难度：⭐⭐

**题干**

有两个并发执行的进程 $\mathrm{P}_{1}$ 和 $\mathrm{P}_{2}$ , 共享初值为 1 的变量 $\mathbf{x}$ 。 $\mathrm{P}_{1}$ 对 $\mathbf{x}$ 加 1, $\mathrm{P}_{2}$ 对 $\mathbf{x}$ 减 1。加 1 和减 1 操作的指令序列分别如下所示。

//加1操作

load R1,x //取 x 到寄存器 R1 中

inc R1 

store x,R1 //将R1的内容存入x

//减1操作

load R2, x 

dec R2 

store x,R2 

两个操作完成后，x 的值（）。

A. 可能为-1或3

B. 只能为1

C. 可能为 0、1 或 2

D. 可能为-1、0、1或2

**答案**：C

**考点分析**

本题考查进程并发执行的临界区问题。初始 x=1，两个进程并发执行可能产生不同的交错顺序。若完全串行执行，结果为1。但若交错执行：如P1的load(R1=1)→P2的load(R2=1)→P1的inc(R1=2)→P1的store(x=2)→P2的dec(R2=0)→P2的store(x=0)，结果为0。或P2先完成再P1执行，结果为1。或P1完成后P2读到新值，结果为1。分析所有可能的交错，x的值可能为0、1或2，但不可能为负数或大于2。

---

## 解题思路

### 第一步：分析指令序列
P1加1操作（3条指令）：
```
I1: load R1, x   // R1 = x
I2: inc R1       // R1 = R1 + 1
I3: store x, R1  // x = R1
```

P2减1操作（3条指令）：
```
I4: load R2, x   // R2 = x
I5: dec R2       // R2 = R2 - 1
I6: store x, R2  // x = R2
```

初始：x = 1

### 第二步：枚举所有可能的执行顺序

**情况1：P1完全先于P2执行**
```
I1: R1=1 → I2: R1=2 → I3: x=2 → I4: R2=2 → I5: R2=1 → I6: x=1
结果：x=1
```

**情况2：P2完全先于P1执行**
```
I4: R2=1 → I5: R2=0 → I6: x=0 → I1: R1=0 → I2: R1=1 → I3: x=1
结果：x=1
```

**情况3：P1的load先执行，P2的store后执行（P2覆盖P1）**
```
I1: R1=1 → I4: R2=1 → I2: R1=2 → I3: x=2 → I5: R2=0 → I6: x=0
结果：x=0
```

**情况4：P2的load先执行，P1的store后执行（P1覆盖P2）**
```
I4: R2=1 → I1: R1=1 → I5: R2=0 → I6: x=0 → I2: R1=2 → I3: x=2
结果：x=2
```

**情况5：P1先store，P2后load（P2看到P1结果）**
```
I1: R1=1 → I2: R1=2 → I3: x=2 → I4: R2=2 → I5: R2=1 → I6: x=1
结果：x=1（与情况1相同）
```

**情况6：P2先store，P1后load（P1看到P2结果）**
```
I4: R2=1 → I5: R2=0 → I6: x=0 → I1: R1=0 → I2: R1=1 → I3: x=1
结果：x=1（与情况2相同）
```

### 第三步：归纳所有可能值
通过穷举所有6种典型交错顺序：
- **x=0**：两个load都读到1，最后store的是dec结果
- **x=1**：串行执行或某个操作完整后另一个才开始
- **x=2**：两个load都读到1，最后store的是inc结果

**不可能出现-1或3**：
- x=-1需要x初值为0或执行两次减1，但x=1且只减1次
- x=3需要执行两次加1，但只加1次

### 第四步：逐项分析

#### ❌ A. 可能为-1或3
- x=-1：需初值为0或减两次，实际不可能
- x=3：需加两次，实际只加一次
**错误**

#### ❌ B. 只能为1
忽略了并发交错导致的读写冲突
情况3和情况4分别产生x=0和x=2
**错误**

#### ✅ C. 可能为0、1或2（正确答案）
- **x=0**：P1先load(R1=1)，P2先load(R2=1)，P2后store(x=0)
- **x=1**：任一进程完整执行后另一个才开始
- **x=2**：P1先load(R1=1)，P2先load(R2=1)，P1后store(x=2)
**正确**

#### ❌ D. 可能为-1、0、1或2
包含不可能的-1
**错误**

### 对比表格

| 执行顺序 | P1的R1值 | P2的R2值 | 最后写入x | 结果 |
|---------|---------|---------|----------|------|
| P1完全先于P2 | 1→2 | 2→1 | P2写入1 | x=1 |
| P2完全先于P1 | 0→1 | 1→0 | P1写入1 | x=1 |
| 都读1，P2后写 | 1→2 | 1→0 | P2写入0 | x=0 |
| 都读1，P1后写 | 1→2 | 1→0 | P1写入2 | x=2 |

### 临界区问题的本质

| 条件 | 本题情况 | 影响 |
|------|---------|------|
| 原子性 | load-inc-store非原子 | 可被打断 |
| 互斥访问 | 无互斥保护 | 两进程同时读写 |
| 读-改-写冲突 | 存在 | 导致更新丢失 |
| 可能结果数 | 3种 | 0, 1, 2 |

## 易错点分析

### 易错点1：认为并发等价于顺序执行
❌ 错误理解：P1加1再P2减1（或反之），结果只能是1
✅ 正确理解：并发执行时指令可交错，两个进程可能同时load到x=1，导致一个操作被覆盖，产生0或2

### 易错点2：误认为能出现-1或3
❌ 错误推理：既然并发不确定，什么结果都可能
✅ 正确分析：每个进程只执行一次操作，x初值为1，加1最多得2，减1最少得0，不可能超出[0,2]范围

### 易错点3：遗漏某些交错情况
❌ 错误做法：只分析完全串行的两种情况（P1→P2或P2→P1），得出只能为1
✅ 正确方法：必须枚举所有关键交错顺序，特别是"都先load再分别store"的情况，这会导致读-改-写冲突
