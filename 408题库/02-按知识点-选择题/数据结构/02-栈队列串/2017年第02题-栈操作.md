# 2017年第2题

**元数据**
- 年份：2017年第2题
- 难度：⭐⭐

**题干**：下列关于栈的叙述中，错误的是（）。

I. 采用非递归方式重写递归程序时必须使用栈

II. 函数调用时，系统要用栈保存必要的信息

III. 只要确定了入栈次序，就可确定出栈次序

IV. 栈是一种受限的线性表，允许在其两端进行操作

**选项**：
A. 仅I
B. 仅 I、II、III
C. 仅 I、III、IV
D. 仅II、III、IV

**答案**：C

**考点分析**：
本题考查栈的基本特性和应用场景。I错误：非递归改写递归程序不一定需要栈，某些尾递归可以用循环直接改写。II正确：函数调用需要栈保存返回地址、参数和局部变量。III错误：入栈次序确定后，出栈次序仍可能不同，例如1、2、3依次入栈，可以有多种出栈序列（如3、2、1或2、3、1等）。IV错误：栈只允许在一端（栈顶）进行插入和删除操作，而非两端。因此I、III、IV均错误。

---

## 解题思路

### 第一步：逐项分析各命题

#### 命题I：采用非递归方式重写递归程序时必须使用栈

**分析**：
- 大多数递归转非递归需要栈（模拟系统调用栈）
- 但**尾递归**可以直接用循环改写，无需显式栈

**反例**：
```c
// 递归版本
int factorial(int n, int acc) {
    if (n == 0) return acc;
    return factorial(n-1, n*acc);  // 尾递归
}

// 非递归版本（无需栈）
int factorial_iter(int n) {
    int acc = 1;
    while (n > 0) {
        acc *= n;
        n--;
    }
    return acc;
}
```

**结论**：命题I **错误**

#### 命题II：函数调用时，系统要用栈保存必要的信息

**分析**：
- 函数调用需要保存：返回地址、参数、局部变量、寄存器状态
- 系统调用栈（System Stack）是必需的
- 后调用先返回（LIFO），符合栈特性

**结论**：命题II **正确**

#### 命题III：只要确定了入栈次序，就可确定出栈次序

**分析**：
- 入栈次序固定，但出栈次序仍可不同
- 因为可以在入栈过程中穿插出栈操作

**反例**：元素1,2,3依次入栈
- 可能序列1：1入→1出→2入→2出→3入→3出 → 出栈序列：1,2,3
- 可能序列2：1入→2入→2出→3入→3出→1出 → 出栈序列：2,3,1
- 可能序列3：1入→2入→3入→3出→2出→1出 → 出栈序列：3,2,1

**结论**：命题III **错误**

#### 命题IV：栈是一种受限的线性表，允许在其两端进行操作

**分析**：
- 栈确实是受限的线性表
- 但栈**只允许在一端（栈顶）进行操作**
- 两端都能操作的是双端队列（Deque）

**结论**：命题IV **错误**

### 第二步：总结错误命题

| 命题 | 正误 | 关键错误 |
|------|------|---------|
| I | ✗ | 尾递归不需要栈 |
| II | ✓ | 系统调用栈必需 |
| III | ✗ | 入栈序列→多种出栈序列 |
| IV | ✗ | 栈只能在一端操作 |

**错误命题**：I、III、IV

### 第三步：逐项分析选项

#### ❌ A. 仅I
只包含I，漏掉III和IV
**错误**

#### ❌ B. 仅I、II、III
包含正确命题II
**错误**（题目问错误的命题）

#### ✅ C. 仅I、III、IV（正确答案）
准确包含所有错误命题：I、III、IV
**正确**

#### ❌ D. 仅II、III、IV
包含正确命题II
**错误**

### 对比表格：栈的核心特性

| 特性 | 说明 | 本题体现 |
|------|------|---------|
| LIFO | 后进先出 | 命题II：调用栈的返回顺序 |
| 单端操作 | 只在栈顶操作 | 命题IV：不是"两端" |
| 受限线性表 | 限制访问位置 | 命题IV前半句正确 |
| 出栈序列多样性 | 入栈序列不能唯一确定出栈序列 | 命题III错误 |

### 递归转非递归对比

| 递归类型 | 是否需要栈 | 示例 |
|---------|-----------|------|
| 一般递归 | 需要 | 二叉树遍历、回溯算法 |
| **尾递归** | **不需要** | **阶乘、斐波那契（尾递归版）** |
| 多路递归 | 需要 | 汉诺塔、全排列 |

## 易错点分析

### 易错点1：认为所有递归转非递归都需要栈
❌ **错误理解**：递归本质就是栈，所以转非递归必须用栈（误选A或不选I）
✅ **正确理解**：尾递归的特点是递归调用是函数的最后一步操作，可以复用当前栈帧，用循环直接改写无需显式栈

### 易错点2：认为入栈序列能确定出栈序列
❌ **错误理解**：入栈是1,2,3，出栈就只能是3,2,1（误选不含III）
✅ **正确理解**：入栈过程可穿插出栈，1,2,3入栈顺序可产生5种合法出栈序列（卡特兰数）

### 易错点3：混淆栈和双端队列
❌ **错误理解**：栈可以两端操作（误选不含IV）
✅ **正确理解**：栈（Stack）只能在栈顶操作，双端队列（Deque）才能两端操作
