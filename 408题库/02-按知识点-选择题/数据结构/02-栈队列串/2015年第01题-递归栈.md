# 递归栈真题汇总

## 2015年第01题

**元数据**
- 年份：[../../01-真题原卷/2015年408真题#01|2015年第1题](../../01-真题原卷/2015年408真题#01|2015年第1题.md)
- 科目：#数据结构
- 知识点：#数据结构/栈队列串/递归
- 难度：⭐⭐
- 关联知识：[知识库/数据结构/栈](知识库/数据结构/栈.md)

**题干**

程序运行时使用栈来保存调用过程的信息，自栈底到栈顶保存的信息依次对应的是（）。

A. main()→S(1)→S(0)  B. S(0)→S(1)→main()  C. main()→S(0)→S(1)  D. S(1)→S(0)→main()

**答案**：A

**解题思路**

**函数调用栈规则**：
- 调用时：压入调用栈
- 返回时：弹出调用栈
- 栈底→栈顶 = 先调用→后调用

**假设调用序列**：
```c
main() {
    S(1);
}
int S(int n) {
    if (n > 0) return S(n-1);
    return 0;
}
```

**执行过程**：
1. main()开始 → 栈：[main]
2. main调用S(1) → 栈：[main, S(1)]
3. S(1)调用S(0) → 栈：[main, S(1), S(0)]
4. S(0)执行并返回 → 栈：[main, S(1)]
5. S(1)返回 → 栈：[main]
6. main结束 → 栈：[]

**在S(0)执行时的栈状态**（栈底→栈顶）：
main → S(1) → S(0)

**选项分析**：
- ✅ A. main()→S(1)→S(0) —— 正确
- ❌ B. S(0)→S(1)→main() —— 颠倒了
- ❌ C. main()→S(0)→S(1) —— S调用顺序错
- ❌ D. S(1)→S(0)→main() —— main应在底部

**易错点分析**

❌ **错误1：选择B，栈顶栈底混淆**
- 误认为栈顶在左
- ✅ 正确：栈底在左，栈顶在右（或栈底在下，栈顶在上）

❌ **错误2：选择C，递归顺序错**
- 认为S(0)先于S(1)调用
- ✅ 正确：S(1)先调用，再调用S(0)

**知识点对比**

| 位置 | 调用时间 | 示例 |
|------|---------|------|
| **栈底** | 最先调用 | main() |
| **栈中** | 中间调用 | S(1) |
| **栈顶** | 当前执行 | S(0) |

**调用栈可视化**

```
时刻1: main执行
栈: [main]

时刻2: main调用S(1)
栈: [main][S(1)]
     ↑底  ↑顶

时刻3: S(1)调用S(0)
栈: [main][S(1)][S(0)]
     ↑底        ↑顶
```

**关键理解**：
- 栈底 = 最早调用尚未返回的函数
- 栈顶 = 当前正在执行的函数
- 先调用的在下（底），后调用的在上（顶）

## 解题思路

### 第一步：理解函数调用栈机制
- 函数调用时压栈（保存返回地址、参数、局部变量）
- 函数返回时出栈
- **栈底→栈顶** = **先调用→后调用**

### 第二步：模拟调用过程
假设递归函数：
```c
int S(int n) {
    if (n == 0) return 0;
    return S(n-1);
}
main() { S(1); }
```

执行流程：
1. main()启动 → 栈：[main]
2. main调用S(1) → 栈：[main, S(1)]
3. S(1)调用S(0) → 栈：[main, S(1), S(0)] ← 题目考查此时刻
4. S(0)返回 → 栈：[main, S(1)]
5. S(1)返回 → 栈：[main]

### 第三步：逐项分析

#### ✅ A. main()→S(1)→S(0)（正确答案）
栈底到栈顶顺序：先调用main，再调用S(1)，最后调用S(0)
**正确**

#### ❌ B. S(0)→S(1)→main()
顺序完全颠倒，栈顶和栈底混淆
**错误**

#### ❌ C. main()→S(0)→S(1)
main正确，但S(0)和S(1)顺序反了（应该先S(1)再S(0)）
**错误**

#### ❌ D. S(1)→S(0)→main()
main应在栈底不是栈顶
**错误**

### 对比表格

| 位置 | 调用时间 | 状态 | 本题示例 |
|------|---------|------|---------|
| 栈底 | 最早调用 | 未返回 | main() |
| 栈中 | 中间调用 | 未返回 | S(1) |
| 栈顶 | 当前执行 | 活跃 | S(0) |

### 调用栈演变

```
时刻1: [main]
时刻2: [main][S(1)]
时刻3: [main][S(1)][S(0)] ← 题目问此时刻
         ↑底          ↑顶
```

## 易错点分析

### 易错点1：栈底栈顶方向混淆
❌ 错误理解：认为栈顶在左边或栈顶是最先调用的函数，选B
✅ 正确理解：栈底（左/下）是最早调用的函数，栈顶（右/上）是当前执行的函数

### 易错点2：递归调用顺序错误
❌ 错误思路：看到S(0)和S(1)，误认为S(0)先调用，选C
✅ 正确分析：S(1)调用S(0)，所以S(1)在栈中更靠近栈底

### 易错点3：忽略main函数的位置
❌ 遗漏main函数或认为main在栈顶
✅ main是程序入口，最早调用，必定在栈底
