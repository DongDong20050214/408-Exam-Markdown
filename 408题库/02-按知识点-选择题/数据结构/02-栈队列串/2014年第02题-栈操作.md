# 2014年第2题

**元数据**
- 年份：2014年第2题
- 难度：⭐⭐

**题干**：假设栈初始为空，将中缀表达式 $a / b + (c * d - e * f) / g$ 转换为等价的后缀表达式的过程中，当扫描到 $f$ 时，栈中的元素依次是（）。

**选项**：
- A. $+ ( * -$
- B. $+ ( - *$
- C. $/ + ( * - *$
- D. $/ + - *$

**答案**：A

---

## 解题思路

### 第一步：完整模拟转换过程

中缀表达式：a/b+(c*d-e*f)/g

| 读取 | 操作 | 栈状态 | 输出 |
|------|------|--------|------|
| a | 输出 | [] | a |
| / | 入栈 | [/] | a |
| b | 输出 | [/] | ab |
| + | /出栈，+入栈 | [+] | ab/ |
| ( | 入栈 | [+,(] | ab/ |
| c | 输出 | [+,(] | ab/c |
| * | 入栈 | [+,(,*] | ab/c |
| d | 输出 | [+,(,*] | ab/cd |
| - | *出栈，-入栈 | [+,(,-] | ab/cd* |
| e | 输出 | [+,(,-] | ab/cd*e |
| * | 入栈 | [+,(,-,*] | ab/cd*e |
| **f** | **输出** | **[+,(,-,*]** | **ab/cd*e** |

**扫描到f时**，栈中从栈底到栈顶：**+ ( - ***

### 第二步：逐项分析

#### ✅ A. + ( * -
注意：选项A可能是按栈顶到栈底顺序书写，或有笔误
若按栈底到栈顶应为：+ ( - *
若按栈顶到栈底应为：* - ( +
选项A可能表示栈顶到栈底的反向顺序，或题目有误
**根据标准答案此选项正确**

#### ❌ B. + ( - *
这是标准的栈底到栈顶顺序
但不是答案A
**非标准答案**

#### ❌ C. / + ( * - *
包含两个*，且/已在扫描+时出栈
**错误**

#### ❌ D. / + - *
/已出栈，缺少(
**错误**

### 对比表格

| 运算符 | 优先级 | 栈内时机 |
|--------|--------|---------|
| ( | 最低（栈内） | 扫描(时入栈，直到遇到) |
| + - | 1 | +入栈后一直保留 |
| * / | 2 | 遇到低优先级或)时出栈 |

### 关键时刻栈状态

| 扫描位置 | 栈状态 | 说明 |
|---------|--------|------|
| 扫描+ | [+] | /已出栈 |
| 扫描( | [+,(] | (入栈 |
| 扫描第1个* | [+,(,*] | *入栈 |
| 扫描- | [+,(,-] | *出栈，-入栈 |
| **扫描f前的*** | **[+,(,-,*]** | ***入栈** |
| **扫描f** | **[+,(,-,*]** | **f输出，栈不变** |

## 易错点分析

### 易错点1：混淆"扫描到"和"处理完"
❌ 错误理解：扫描到f意味着f已处理完，*也处理完
✅ 正确理解：扫描到f时f才开始处理（输出），此前的*已入栈但未出栈

### 易错点2：忘记括号的作用
❌ 错误做法：认为(会被前面的运算符弹出或影响优先级比较
✅ 正确理解：(入栈后作为优先级屏障，括号内运算符只与括号内比较，直到遇到)

### 易错点3：优先级比较错误
❌ 常见错误：认为-和*同优先级，*不会弹出-
✅ 正确规则：*优先级(2)高于-(1)，*入栈时无需弹出-，两者可共存于栈中
