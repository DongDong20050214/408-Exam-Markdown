## 2017年第02题

**元数据**
- 年份：[../../01-真题原卷/2017年408真题#02|2017年第2题](../../01-真题原卷/2017年408真题#02|2017年第2题.md)

**题干**

下列关于栈的叙述中，错误的是（）。

I. 采用非递归方式重写递归程序时必须使用栈  II. 函数调用时，系统要用栈保存必要的信息  III. 只要确定了入栈次序，就可确定出栈次序  IV. 栈是一种受限的线性表，允许在其两端进行操作

A. 仅I  B. 仅 I、II、III  C. 仅 I、III、IV  D. 仅II、III、IV

**答案**：C

**解题思路**

逐项判断正误：

**I. 采用非递归方式重写递归程序时必须使用栈**
- ❌ 错误
- 虽然递归调用隐含使用系统栈，但改写为非递归不一定必须显式使用栈
- 反例：尾递归可以改写为循环，不需要栈；某些递归可以用迭代变量替代
- 示例：阶乘递归可以改写为简单循环
```c
// 递归
int fact(int n) { return n<=1 ? 1 : n*fact(n-1); }
// 非递归（不用栈）
int fact(int n) { int r=1; for(int i=2; i<=n; i++) r*=i; return r; }
```

**II. 函数调用时，系统要用栈保存必要的信息**
- ✅ 正确
- 函数调用使用**调用栈**保存：返回地址、参数、局部变量等
- 栈帧（stack frame）在函数调用时压栈，返回时出栈
- 这是函数调用的基本机制

**III. 只要确定了入栈次序，就可确定出栈次序**
- ❌ 错误
- 入栈次序确定，但出栈次序有**多种可能**
- 示例：入栈1,2,3，可能的出栈序列有5种：
  - 1,2,3（边入边出）
  - 1,3,2（1出，2,3入，3出，2出）
  - 2,3,1（1,2入，2出，3入出，1出）
  - 2,1,3（1,2入，2出，1出，3入出）
  - 3,2,1（全入后全出）
- 出栈序列数量为卡特兰数：$C_n = \frac{1}{n+1}C_{2n}^{n}$

**IV. 栈是一种受限的线性表，允许在其两端进行操作**
- ❌ 错误
- 栈只允许在**一端**（栈顶）进行操作
- 如果两端都能操作，那是**双端队列（deque）**，不是栈
- 栈的定义：只能在栈顶进行插入和删除（LIFO）

**错误项**：I、III、IV

**选项分析**：
- ❌ A. 仅I —— 遗漏III、IV
- ❌ B. 仅I、II、III —— II正确，不应包含
- ✅ C. 仅I、III、IV —— 正确答案
- ❌ D. 仅II、III、IV —— II正确，不应包含

**易错点分析**

❌ **错误1：认为II错误**
- 误以为函数调用不一定用栈
- ✅ 正确：现代语言的函数调用机制都使用栈（调用栈/栈帧）

❌ **错误2：认为III正确**
- 混淆"入栈次序确定"和"出栈次序唯一"
- ✅ 正确：入栈次序固定，出栈可以随时进行，次序不唯一

❌ **错误3：认为IV正确**
- 混淆栈和双端队列
- ✅ 正确：栈只能在一端操作（栈顶），不是两端

**知识点对比**

| 数据结构 | 操作端 | 操作限制 | 特点 |
|---------|--------|---------|------|
| **栈** | 一端（栈顶） | 只能在栈顶插入和删除 | LIFO |
| **队列** | 两端（但分工不同） | 一端插入，另一端删除 | FIFO |
| **双端队列** | 两端（功能相同） | 两端都可插入和删除 | 灵活 |
| **线性表** | 任意位置 | 无限制 | 通用 |

**递归与栈的关系**

| 场景 | 是否必须用栈 | 说明 |
|------|------------|------|
| **递归调用** | 隐式使用系统栈 | 自动的栈帧管理 |
| **改写为非递归（一般）** | 通常需要显式栈 | 模拟递归的调用栈 |
| **改写为非递归（特殊）** | 不一定需要 | 尾递归、简单累积等可用循环 |

**入栈出栈序列关系**

| 入栈序列 | 可能的出栈序列数 | 示例（n=3） |
|---------|----------------|-----------|
| 1,2,3 | $C_3=5$种 | 123, 132, 213, 231, 321 |
| 固定n个 | $C_n=\frac{1}{n+1}C_{2n}^n$ | 卡特兰数 |

**函数调用栈帧内容**

| 栈帧包含 | 说明 |
|---------|------|
| **返回地址** | 函数返回后继续执行的位置 |
| **参数** | 传递给函数的参数值 |
| **局部变量** | 函数内定义的变量 |
| **保存的寄存器** | 调用前的寄存器状态 |

**关键理解**：
- 栈是单端操作的受限线性表（IV错在"两端"）
- 入栈次序不决定出栈次序（III错在"确定"）
- 非递归改写不总是需要显式栈（I错在"必须"）
- 函数调用机制依赖栈（II正确）

## 解题思路

### 第一步：理解题目核心
判断关于栈的4个叙述中哪些是错误的。

**题目要求**：选出错误的叙述组合

**4个叙述**：
- I. 采用非递归方式重写递归程序时必须使用栈
- II. 函数调用时，系统要用栈保存必要的信息
- III. 只要确定了入栈次序，就可确定出栈次序
- IV. 栈是一种受限的线性表，允许在其两端进行操作

### 第二步：逐项判断

**I. 采用非递归方式重写递归程序时必须使用栈 ❌**

分析：
- 递归隐式使用系统栈
- 但改写为非递归**不总是**需要显式栈
- 反例：尾递归可改为循环

示例：
```c
// 递归阶乘
int fact(int n) {
    if (n <= 1) return 1;
    return n * fact(n-1);
}

// 非递归改写（不用栈）
int fact(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++)
        result *= i;
    return result;
}
```

结论：关键词"必须"过于绝对，**错误** ❌

**II. 函数调用时，系统要用栈保存必要的信息 ✅**

分析：
- 函数调用使用**调用栈**（call stack）
- 栈帧（stack frame）保存：
  - 返回地址
  - 函数参数
  - 局部变量
  - 保存的寄存器
- 函数调用时压栈，返回时出栈

结论：这是函数调用的基本机制，**正确** ✅

**III. 只要确定了入栈次序，就可确定出栈次序 ❌**

分析：
- 入栈次序确定，但出栈可随时进行
- 出栈次序有**多种可能**

示例：入栈1,2,3的5种出栈序列
```
1. [1,2,3]：1入出,2入出,3入出（边入边出）
2. [1,3,2]：1入出,2-3入,3出,2出
3. [2,1,3]：1-2入,2出,1出,3入出
4. [2,3,1]：1-2入,2出,3入出,1出
5. [3,2,1]：1-2-3全入,3-2-1全出
```

出栈序列数量：**卡特兰数** $C_n = \frac{1}{n+1}C_{2n}^{n}$

结论：出栈次序不唯一，**错误** ❌

**IV. 栈是一种受限的线性表，允许在其两端进行操作 ❌**

分析：
- 栈只允许在**一端**（栈顶）进行操作
- 两端都能操作的是**双端队列**（deque）

栈的定义：
- 后进先出（LIFO）
- 只能在栈顶插入和删除
- 不能在栈底操作

结论：说"两端"是错误的，**错误** ❌

### 第三步：汇总错误项

**错误的叙述**：I、III、IV
**正确的叙述**：II

### 第四步：选项判断

**A. 仅I ❌**
```
包含：I
遗漏：III、IV
结论：错误
```

**B. 仅I、II、III ❌**
```
包含：I(错)、II(对)、III(错)
问题：II是正确的，不应包含在"错误"中
结论：错误
```

**C. 仅I、III、IV ✅**
```
包含：I(错)、III(错)、IV(错)
排除：II(对)
结论：正确 ✅
```

**D. 仅II、III、IV ❌**
```
包含：II(对)、III(错)、IV(错)
问题：II是正确的，不应包含
结论：错误
```

### 第五步：确认答案
错误的叙述是I、III、IV → 答案：**C**

## 易错点分析

### 易错点1：误认为非递归改写必须用栈
❌ 错误理解：递归本质就是栈，所以非递归改写一定要用显式栈
✅ 正确理解：
- 递归确实隐式使用系统栈
- 但改写为非递归有多种方法：
  - **尾递归**：可直接改为循环，无需栈
  - **简单累积**：用循环变量替代
  - **复杂递归**：才需要显式栈模拟

| 递归类型 | 非递归改写 | 是否需要栈 |
|---------|-----------|-----------|
| 尾递归 | 循环 | ❌ 不需要 |
| 简单累积 | 循环变量 | ❌ 不需要 |
| 多分支递归 | 显式栈 | ✅ 需要 |
| 回溯算法 | 显式栈 | ✅ 需要 |

**关键**：叙述中的"必须"过于绝对

### 易错点2：混淆"确定入栈次序"和"确定出栈次序"
❌ 错误理解：入栈顺序固定，出栈顺序也就固定了
✅ 正确理解：
- 入栈次序：元素进栈的顺序（固定）
- 出栈时机：可以在任意时刻出栈（灵活）
- 出栈次序：取决于何时出栈（不唯一）

**示例分析**：入栈1,2,3
```
序列1: 1入→1出, 2入→2出, 3入→3出 → [1,2,3]
序列2: 1入→1出, 2入→3入→3出→2出 → [1,3,2]
序列3: 1入→2入→2出→1出, 3入→3出 → [2,1,3]
序列4: 1入→2入→2出, 3入→3出→1出 → [2,3,1]
序列5: 1入→2入→3入, 3出→2出→1出 → [3,2,1]
```

**不可能的出栈序列示例**：[3,1,2]
- 要先出3，必须1-2-3全入栈
- 此时栈内：1在底，2中间，3在顶
- 3出后，只能先出2再出1
- 不可能出现3-1-2的顺序

**卡特兰数公式**：
$$C_n = \frac{1}{n+1}C_{2n}^{n} = \frac{(2n)!}{(n+1)!n!}$$

n=3时：$C_3 = \frac{1}{4} \times \frac{6!}{3! \times 3!} = \frac{20}{4} = 5$种

### 易错点3：混淆栈和双端队列
❌ 错误理解：栈可以在两端操作，只是有限制
✅ 正确理解：

| 数据结构 | 操作端 | 操作限制 | 特点 |
|---------|--------|---------|------|
| **栈(Stack)** | 一端(栈顶) | 只能在栈顶插入和删除 | LIFO |
| **队列(Queue)** | 两端(分工不同) | 队尾插入，队头删除 | FIFO |
| **双端队列(Deque)** | 两端(功能相同) | 两端都可插入和删除 | 灵活 |
| **线性表(List)** | 任意位置 | 无限制 | 通用 |

**栈的严格定义**：
- 只允许在**栈顶**一端进行操作
- 插入：push（压栈）
- 删除：pop（出栈）
- 访问：top（查看栈顶）
- 不能访问或操作栈底和中间元素

**记忆口诀**：栈是单端，队列双端分工，双端队列双端同功

### 易错点4：误认为函数调用不一定用栈
❌ 错误理解：有些语言可能用其他方式实现函数调用
✅ 正确理解：
- 现代编程语言的函数调用机制都使用**调用栈**
- 这是计算机体系结构的基础机制
- 栈帧包含：
  - 返回地址（return address）
  - 函数参数（parameters）
  - 局部变量（local variables）
  - 保存的寄存器（saved registers）

**函数调用过程**：
```
1. 调用函数时：创建新栈帧，压入调用栈
2. 执行函数：在当前栈帧中操作
3. 函数返回：弹出栈帧，恢复上一层的执行环境
```

**示例**：
```c
void funcB() { int b = 2; }
void funcA() { int a = 1; funcB(); }
int main() { funcA(); }

调用栈变化：
[main]
[main][funcA]
[main][funcA][funcB]  ← funcB执行时
[main][funcA]          ← funcB返回
[main]                 ← funcA返回
[]                     ← main返回
```

## 知识点对比

### 递归与非递归改写

| 递归类型 | 特征 | 非递归方法 | 是否需要显式栈 |
|---------|------|-----------|--------------|
| **尾递归** | 递归调用是最后操作 | 改为循环 | ❌ |
| **线性递归** | 单次递归调用 | 循环+变量 | ❌ |
| **树形递归** | 多次递归调用 | 显式栈 | ✅ |
| **回溯** | 需要保存多个状态 | 显式栈 | ✅ |

**示例对比**：
```c
// 尾递归（无需栈）
int sum(int n, int acc) {
    if (n == 0) return acc;
    return sum(n-1, acc+n);
}
// 改为循环
int sum(int n) {
    int acc = 0;
    while (n > 0) { acc += n; n--; }
    return acc;
}

// 树形递归（需要栈）
int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2);
}
// 改为非递归需要栈或其他技巧
```

### 出栈序列判断

**合法性判断方法**：
对于入栈序列1,2,3,...,n，判断出栈序列是否合法：

**规则**：对于出栈序列中的任意元素k，在k之后出现的所有小于k的元素必须按**递减**顺序排列

**示例**：
- [3,2,1]：3后面是2,1递减 ✓
- [2,3,1]：2后面只有1 ✓；3后面只有1 ✓
- [3,1,2]：3后面是1,2不是递减 ✗
