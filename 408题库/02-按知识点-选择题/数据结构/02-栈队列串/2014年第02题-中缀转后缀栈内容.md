**题干**

假设栈初始为空，将中缀表达式 a / b + (c*d - e*f) / g 转换为等价的后缀表达式的过程中，当扫描到 f 时，栈中的元素依次是（）。

A. +(* -  B. +(-*  C. / + (* - *)  D. / + - *

**答案**：B

**考点分析**
- 扫描到f时，栈中：+(-*

---

## 解题思路

### 第一步：回顾中缀转后缀的栈操作规则

| 遇到 | 操作 |
|------|------|
| 操作数 | 直接输出 |
| 左括号'(' | 直接入栈 |
| 右括号')' | 弹出栈中操作符直到遇到'(' |
| 操作符 | 弹出栈中优先级≥当前操作符的所有操作符，然后当前操作符入栈 |

**优先级**：'(' < '+' = '-' < '*' = '/'

### 第二步：逐步模拟转换过程

中缀表达式：**a / b + (c*d - e*f) / g**

| 读取 | 操作 | 栈状态 | 输出 | 说明 |
|------|------|--------|------|------|
| a | 输出 | [] | a | 操作数直接输出 |
| / | 入栈 | [/] | a | 栈空，直接入栈 |
| b | 输出 | [/] | ab | 操作数输出 |
| + | /出栈，+入栈 | [+] | ab/ | /优先级高，先出栈 |
| ( | 入栈 | [+,(] | ab/ | 左括号直接入栈 |
| c | 输出 | [+,(] | ab/c | 操作数输出 |
| * | 入栈 | [+,(,*] | ab/c | 括号内，*入栈 |
| d | 输出 | [+,(,*] | ab/cd | 操作数输出 |
| - | *出栈，-入栈 | [+,(,-] | ab/cd* | *优先级高于-，先出栈 |
| e | 输出 | [+,(,-] | ab/cd*e | 操作数输出 |
| * | 入栈 | [+,(,-,*] | ab/cd*e | *优先级高于-，直接入栈 |
| **f** | **输出** | **[+,(,-,*]** | **ab/cd*ef** | **操作数输出（目标时刻）** |

**扫描到f时栈状态**：**[+, (, -, *]**（从栈底到栈顶）

### 第三步：逐项分析

#### ❌ A. + ( * -
栈顺序：+,(,*,-
**错误**：*和-的顺序颠倒了。实际应是-在栈中，*后入栈在-之上。

#### ✅ B. + ( - *（正确答案）
栈顺序：+,(,-,*（从栈底到栈顶）
- '+'：外层加法，最早入栈
- '('：括号开始，保护括号内运算
- '-'：括号内减法，*弹出后入栈
- '*'：当前e*f的乘法，最晚入栈
**正确**

#### ❌ C. / + ( * - *
栈顺序：/,+,(,*,-,*
**错误**：
1. '/'在遇到'+'时已出栈（优先级高的先出）
2. '*'重复出现
3. 栈中元素过多

#### ❌ D. / + - *
栈顺序：/,+,-,*
**错误**：
1. '/'已出栈
2. 缺少'('（左括号应保留在栈中）

### 对比表格：关键时刻的栈状态

| 扫描位置 | 扫描符号 | 栈状态 | 输出 | 栈深度 |
|---------|---------|--------|------|--------|
| a/b后 | + | [+] | ab/ | 1 |
| (后 | ( | [+,(] | ab/ | 2 |
| c*d后 | - | [+,(,-] | ab/cd* | 3 |
| e后 | * | [+,(,-,*] | ab/cd*e | 4 |
| **f后** | **(未变)** | **[+,(,-,*]** | **ab/cd*ef** | **4** |

### 关键知识点对比

| 运算符 | 优先级 | 栈中行为 | 本题体现 |
|--------|--------|---------|---------|
| ( | 最低（入栈后） | 入栈作为屏障 | (保留到栈中 |
| +、- | 1 | 弹出≥自己的再入栈 | -弹出*后入栈 |
| *、/ | 2 | 弹出≥自己的再入栈 | *直接入栈（优先级高于-） |
| ) | - | 触发弹栈到( | 本题未扫描到 |

## 易错点分析

### 易错点1：混淆*和-的栈顺序
❌ **错误理解**：认为*优先级高应该在栈底，-在上面（选A）
✅ **正确理解**：栈是后进先出，*后入栈所以在栈顶，-先入栈在下面。栈顺序=入栈时间顺序的倒序

### 易错点2：误认为/仍在栈中
❌ **错误推理**：/入栈了，所以扫描到f时还在栈中（选C或D）
✅ **正确分析**：/在遇到+时，因为优先级高于+被弹出，此时栈中已无/

### 易错点3：忽略括号的屏障作用
❌ **错误做法**：认为括号不算操作符，不计入栈中（选D）
✅ **正确理解**：左括号'('入栈后作为优先级屏障，直到遇到右括号')'才弹出，扫描到f时'('仍在栈中
