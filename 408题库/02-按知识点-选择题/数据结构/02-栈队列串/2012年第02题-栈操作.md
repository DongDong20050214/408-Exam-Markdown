# 2012年第2题

**元数据**
- 年份：2012年第2题
- 难度：⭐⭐

**题干**

已知操作符包括 '+'、'-'、'*'、'/'、'(' 和 ')'。将中缀表达式 $a + b - a^{*}((c + d) / e - f) + g$ 转换为等价的后缀表达式 $ab + acd + e / f - * - g+$ 时，用栈来存放暂时还不能确定运算次序的操作符。若栈初始时为空，则转换过程中同时保存在栈中的操作符的最大个数是（）。

A. 5 

B. 7 

C. 8 

D. 11 

**答案**：A

**考点分析**

本题考查中缀表达式转后缀表达式过程中的栈操作。在转换过程中，遇到操作数直接输出，遇到操作符则与栈顶操作符比较优先级后决定入栈或出栈。关键是分析表达式 $a + b - a * ((c + d) / e - f) + g$ 中括号嵌套最深处，此时栈中会保存：外层的 +、-、内层的 *、(、(、+，共5个操作符，这是栈深度的最大值。

## 解题思路（逐项分析法）

### 第一步：回顾中缀转后缀的栈操作规则

| 遇到 | 操作 |
|------|------|
| 操作数 | 直接输出 |
| 左括号'(' | 直接入栈 |
| 右括号')' | 弹出栈中操作符直到遇到'(' |
| 操作符 | 弹出栈中优先级≥当前操作符的所有操作符，然后当前操作符入栈 |

优先级：'(' < '+' = '-' < '*' = '/'

### 第二步：完整模拟转换过程

| 读取 | 操作 | 栈状态 | 输出 | 栈深度 |
|------|------|--------|------|--------|
| a | 输出 | [] | a | 0 |
| + | 入栈 | [+] | a | 1 |
| b | 输出 | [+] | ab | 1 |
| - | +出栈，-入栈 | [-] | ab+ | 1 |
| a | 输出 | [-] | ab+a | 1 |
| * | 入栈 | [-,*] | ab+a | 2 |
| ( | 入栈 | [-,*,(] | ab+a | 3 |
| ( | 入栈 | [-,*,(,(] | ab+a | 4 |
| c | 输出 | [-,*,(,(] | ab+ac | 4 |
| + | 入栈 | [-,*,(,(,+] | ab+ac | **5** ✓ |
| d | 输出 | [-,*,(,(,+] | ab+acd | **5** ✓ |
| ) | +出栈，(出栈 | [-,*,(] | ab+acd+ | 3 |
| / | 入栈 | [-,*,(,/] | ab+acd+ | 4 |
| e | 输出 | [-,*,(,/] | ab+acd+e | 4 |
| - | /出栈，-入栈 | [-,*,(,-] | ab+acd+e/ | 4 |
| f | 输出 | [-,*,(,-] | ab+acd+e/f | 4 |
| ) | -出栈，(出栈 | [-,*] | ab+acd+e/f- | 2 |
| + | *出栈，-出栈，+入栈 | [+] | ab+acd+e/f-*- | 1 |
| g | 输出 | [+] | ab+acd+e/f-*-g | 1 |
| 结束 | +出栈 | [] | ab+acd+e/f-*-g+ | 0 |

**最大栈深度：5**（出现在读取'+'和'd'时）

### 第三步：验证各选项

#### ✅ A. 5（正确答案）
栈最大深度为5，出现在扫描到内层括号中的'+'时
栈内容为：[-,*,(,(,+]
**正确**

#### ❌ B. 7
可能误算了操作符总数或重复计数
**错误**

#### ❌ C. 8
可能把所有操作符（6个）加上所有括号（4个）误算为同时在栈中
**错误**

#### ❌ D. 11
可能把表达式中所有符号（操作符+括号）都计入
**错误**

### 对比表格

| 转换阶段 | 栈中操作符 | 栈深度 | 说明 |
|---------|-----------|--------|------|
| 扫描到第一个+ | [+] | 1 | 外层加法 |
| 扫描到* | [-,*] | 2 | 外层减法、乘法 |
| 扫描到第一个( | [-,*,(] | 3 | 第一层括号 |
| 扫描到第二个( | [-,*,(,(] | 4 | 第二层括号 |
| **扫描到内层+** | **[-,*,(,(,+]** | **5** | **最大深度** |
| 遇到第一个) | [-,*,(] | 3 | 内层括号匹配 |
| 遇到第二个) | [-,*] | 2 | 外层括号匹配 |
| 最后的+ | [+] | 1 | 最终加法 |

### 易错点

1. **只统计括号数量**：误以为2层括号=2个'('=栈深2，忽略了同时存在的操作符（-、*、+），实际最大深度=2个括号+3个操作符=5
2. **累加所有操作符**：误以为表达式中所有6个操作符和4个括号都会同时在栈中（共10个），忽略了右括号')'会触发出栈操作
3. **忽略优先级导致的提前出栈**：没有注意到遇到低优先级操作符或右括号时会弹出栈中元素，导致高估栈深度
