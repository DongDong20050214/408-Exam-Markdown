# 2019年第6题

**元数据**
- 年份：2019年第6题
- 难度：⭐⭐

**题干**：用有向无环图描述表达式 $(x + y)((x + y) / x)$ ，需要的顶点个数至少是（ ）。

A. 5 
B. 6 
C. 8 
D. 9 

**答案**：A

**考点分析**：本题考查表达式的DAG表示。DAG可以共享公共子表达式。表达式中(x+y)出现两次，可以共享。需要的顶点：x（1个），y（1个），x+y（1个），(x+y)/x（1个），最终结果的乘法（1个），共5个顶点。

---

## 完整解题

### 第一步：理解DAG表示表达式

**DAG（有向无环图）**用于表示表达式时可以**共享公共子表达式**，减少顶点数量。

表达式：$(x + y) \times ((x + y) / x)$

### 第二步：识别公共子表达式

**结构分析**：
- 主运算：乘法 `*`
- 左操作数：`x + y`
- 右操作数：`(x + y) / x`
  - 分子：`x + y`（与左操作数相同！）
  - 分母：`x`

**公共子表达式**：`x + y` 出现2次，可以共享。

### 第三步：构建DAG并计数顶点

```
层次结构：
1. 叶子节点：x, y（2个顶点）
2. 第一层运算：x + y（1个顶点，共享）
3. 第二层运算：(x + y) / x（1个顶点）
4. 第三层运算：最终的乘法（1个顶点）
```

**顶点清单**：
1. 变量x
2. 变量y
3. 加法节点：x+y（只需1个，被左右共享）
4. 除法节点：(x+y)/x
5. 乘法节点：(x+y) * ((x+y)/x)

**总计**：5个顶点

### 第四步：逐项分析

**A. 5 ✅**
- 正确答案
- 利用了公共子表达式共享

**B. 6 ❌**
- 可能误认为x出现2次需要2个顶点
- 实际上变量节点也可共享

**C. 8 ❌**
- 可能没有共享公共子表达式
- 把(x+y)计算了2次

**D. 9 ❌**
- 完全没有共享，所有节点都独立
- 这是语法树的做法，不是DAG

## 易错点分析

### 易错点1：没有识别公共子表达式
❌ 错误理解：每次运算都创建新节点，(x+y)出现2次就创建2个节点
✅ 正确理解：DAG的核心优势是共享公共子表达式，相同的子表达式只需一个节点，多处引用

### 易错点2：混淆DAG和语法树
❌ 错误理解：认为DAG和语法树一样，不能共享节点
✅ 正确理解：语法树不共享节点（树形结构），DAG允许共享（图形结构），这是DAG减少冗余的关键

### 易错点3：忘记变量也可以共享
❌ 错误理解：变量x出现2次，需要2个顶点
✅ 正确理解：变量节点也只需一个，所有引用该变量的地方都指向同一个顶点

## 知识点对比

| 表示方式 | 公共子表达式 | 顶点数 | 适用场景 |
|---------|------------|-------|---------|
| **语法树** | 不共享，重复计算 | 多（本题9个） | 语法分析 |
| **DAG** | 共享，消除冗余 | 少（本题5个） | 代码优化 |
| **三地址码** | 显式保存中间结果 | 中等 | 中间代码 |

| 节点类型 | 本题数量 | 说明 |
|---------|---------|------|
| **变量节点** | 2个 | x, y各1个（共享） |
| **运算节点** | 3个 | +（1个共享）, /（1个）, *（1个） |
| **总计** | 5个 | 最少顶点数 |

**对比示例**：
```
语法树表示（不共享）：
        *
       / \
    (x+y) /
         / \
      (x+y) x
需要顶点：x(2), y(2), +(2), /(1), *(1) = 8个

DAG表示（共享）：
        *
       / \
     +    /
    / \  / \
   x  y +  x
       / \
      x  y (指向已有的x,y)
需要顶点：x(1), y(1), +(1), /(1), *(1) = 5个
```
