# 2021年408真题第25题

## 题目

下列内核的数据结构或程序中，分时系统实现时间片轮转调度需要使用的是（）。

I. 进程控制块

II. 时钟中断处理程序

III. 进程就绪队列

IV. 进程阻塞队列

A. 仅II、III

B. 仅 I、IV

C. 仅 I、II、III

D. 仅 I、II、IV

## 答案

C

## 考点分析

### 考点：时间片轮转调度算法的实现机制

本题考查分时系统中时间片轮转（Round Robin, RR）调度算法的实现，需要理解该算法依赖的内核数据结构和程序。

### 解题思路：

1. **时间片轮转调度的基本原理**：
   - 就绪队列中的进程按 FIFO 顺序获得 CPU
   - 每个进程分配一个时间片（如 10ms）
   - 时间片用完后，进程被强制让出 CPU，放到就绪队列末尾
   - 调度器选择下一个就绪进程执行

### 逐项分析

#### ✅ I. 进程控制块（PCB）
- 存储进程状态（运行、就绪、阻塞）
- 保存进程上下文（寄存器、PC、PSW等）
- 进程切换时必须通过PCB恢复现场
- **必需**：时间片轮转涉及频繁进程切换

#### ✅ II. 时钟中断处理程序
- 定时产生时钟中断（如每10ms）
- 中断处理程序递减当前进程的时间片计数
- 时间片归零时触发调度程序
- **必需**：是时间片计时和到期检测的唯一机制

#### ✅ III. 进程就绪队列
- 按FIFO顺序组织所有就绪进程
- 时间片用完的进程插入队列尾部
- 调度器从队列头取下一个进程
- **必需**：体现"轮转"的核心逻辑

#### ❌ IV. 进程阻塞队列
- 管理等待I/O或事件的阻塞进程
- 与时间片轮转调度实现**无直接关系**
- 是进程管理的通用机制，非RR特有
- **不必需**：RR调度只关注就绪进程的轮转

### 对比表格

| 数据结构/程序 | RR调度必需性 | 作用 | 缺失后果 |
|--------------|-------------|------|---------|
| PCB | ✓ | 进程切换 | 无法保存/恢复上下文 |
| 时钟中断 | ✓ | 时间片计时 | 无法检测时间片到期 |
| 就绪队列 | ✓ | 轮转顺序 | 无法实现轮转逻辑 |
| 阻塞队列 | ✗ | I/O等待管理 | 不影响RR调度实现 |

### 易错点

1. **混淆时钟中断和时钟设备**：认为只需要硬件时钟而不需要中断处理程序，实际上必须有中断处理程序检查时间片并触发调度
2. **误选阻塞队列**：把进程管理的通用需求（阻塞队列）误认为是RR调度的特定需求，实际上RR只操作就绪队列
3. **忽略PCB的作用**：只关注调度算法逻辑，忽略进程切换需要PCB保存恢复上下文
