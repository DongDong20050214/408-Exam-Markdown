# 2014年第6题

**元数据**
- 年份：2014年第6题
- 难度：⭐⭐

**题干**：5个字符有如下4种编码方案，不是前缀编码的是（）。

**选项**：
- A. 01, 0000, 0001, 001, 1
- B. 011, 000, 001, 010, 1
- C. 000, 001, 010, 011, 100
- D. 0, 100, 110, 1110, 1100

**答案**：D

**考点分析**：
本题考查前缀编码（prefix code）的概念。

前缀编码定义：在一个编码方案中，任何一个编码都不是其他编码的前缀。这样可以保证解码的唯一性。

逐项检查：

A. 01, 0000, 0001, 001, 1
- 没有任何一个编码是另一个的前缀，是前缀编码

B. 011, 000, 001, 010, 1
- 没有任何一个编码是另一个的前缀，是前缀编码

C. 000, 001, 010, 011, 100
- 没有任何一个编码是另一个的前缀，是前缀编码

D. 0, 100, 110, 1110, 1100
- "0" 是所有其他编码的前缀吗？不是
- 但 "110" 是 "1100" 的前缀
- 因此不是前缀编码

答案是 D。

---

## 解题思路

### 第一步：理解前缀编码

**前缀编码（Prefix Code）定义**：
- 在编码集合中，任何一个编码都不是其他编码的前缀
- 保证解码的唯一性（无歧义）

**示例**：
- 前缀编码：{0, 10, 110, 111} ✓（互不为前缀）
- 非前缀编码：{0, 01, 10} ✗（0是01的前缀）

### 第二步：逐项检查各选项

#### ✅ A. 01, 0000, 0001, 001, 1

**检查方法**：逐一比较每对编码

| 编码对 | 是否为前缀关系 |
|-------|--------------|
| 01 vs 0000 | ✗（01不是0000的前缀） |
| 01 vs 0001 | ✗ |
| 01 vs 001 | ✗ |
| 01 vs 1 | ✗ |
| 0000 vs 0001 | ✗ |
| 0000 vs 001 | ✗ |
| 0000 vs 1 | ✗ |
| 0001 vs 001 | ✗ |
| 0001 vs 1 | ✗ |
| 001 vs 1 | ✗ |

**结论**：所有编码互不为前缀 → **是前缀编码** ✓

#### ✅ B. 011, 000, 001, 010, 1

**检查**：
- 所有以0开头的编码长度都为3（011, 000, 001, 010）
- 以1开头的编码只有"1"（长度1）
- 3位编码互不相同，且不是"1"的前缀（"1"也不是它们的前缀）

**结论**：**是前缀编码** ✓

#### ✅ C. 000, 001, 010, 011, 100

**检查**：
- 所有编码长度相同（都是3位）
- 长度相同的编码不可能存在前缀关系

**结论**：**是前缀编码** ✓

#### ❌ D. 0, 100, 110, 1110, 1100

**检查关键对**：
- "110" vs "1100"：**110是1100的前缀** ✗
- "0" vs 其他：0不是任何编码的前缀（其他都以1开头）
- "100" vs "1100"：100不是1100的前缀（第二位不同）
- "110" vs "1110"：110不是1110的前缀（第三位不同）

**关键发现**：
```
"110"  = 1 1 0
"1100" = 1 1 0 0
```
"110"是"1100"的前缀！

**结论**：**不是前缀编码** ✗（违反定义）

### 第三步：逐项分析选项

#### ❌ A. 01, 0000, 0001, 001, 1
是前缀编码，不符合题意
**错误**

#### ❌ B. 011, 000, 001, 010, 1
是前缀编码，不符合题意
**错误**

#### ❌ C. 000, 001, 010, 011, 100
是前缀编码，不符合题意
**错误**

#### ✅ D. 0, 100, 110, 1110, 1100（正确答案）
不是前缀编码（110是1100的前缀）
**正确**

### 对比表格：前缀编码判断技巧

| 编码特征 | 是否一定是前缀编码 | 说明 |
|---------|------------------|------|
| 所有编码长度相同 | ✓ 是 | 长度相同不可能有前缀关系 |
| 存在不同长度编码 | 不一定 | 需逐一检查 |
| 短编码是长编码的前缀 | ✗ 不是 | 违反定义 |

### 前缀编码的应用

| 应用场景 | 作用 | 示例 |
|---------|------|------|
| 哈夫曼编码 | 数据压缩 | 常见字符短编码，罕见字符长编码 |
| 莫尔斯电码 | 通信 | 点划组合，间隔区分 |
| UTF-8编码 | 字符编码 | 变长编码，无歧义解码 |

### 检查D选项的详细验证

```
D: 0, 100, 110, 1110, 1100

构建前缀树验证：
        root
       /    \
      0      1
     (✓)    / \
           0   1
          /     \
         0       1
        (100)    0
                (110) ← 这里已有编码
                  \
                   0
                 (1100) ← 违反！110已是编码，不能再有子节点
```

**前缀树规律**：
- 所有编码必须是叶子节点
- 若某节点已是编码（叶子），则不能再有子节点
- D选项中"110"既是编码又有子节点"1100" → 违反前缀编码规则

## 易错点分析

### 易错点1：只检查明显的短前缀
❌ **错误做法**：只检查"0"是否是其他编码的前缀，忽略其他组合
✅ **正确方法**：逐一检查所有编码对，特别注意中等长度编码之间的关系（如110和1100）

### 易错点2：混淆"前缀"的定义
❌ **错误理解**："110"和"1100"有共同开头但不算前缀关系
✅ **正确理解**：A是B的前缀 ⇔ B以A开头且B≠A（110完全是1100的开头部分）

### 易错点3：认为所有选项都是前缀编码
❌ **错误审题**：误读题目为"是前缀编码的是"，导致选错
✅ **正确审题**：题目明确问"**不是**前缀编码的是"，要找违反规则的选项
