# 栈容量真题汇总

## 2009年第02题

**元数据**
- 年份：[[../../01-真题原卷/2009年408真题#02|2009年第2题]]
- 科目：#数据结构
- 知识点：#数据结构/栈队列串/栈 #容量
- 难度：⭐⭐
- 关联知识：[[知识库/数据结构/栈]]

**题干**

设栈 S 和队列 Q 的初始状态均为空, 元素 a, b, c, d, e, f, g 依次进入栈 S。若每个元素出栈后立即进入队列 Q, 且 7 个元素出队的顺序是 b, d, c, f, e, a, g, 则栈 S 的容量至少是 ( )。

A. 1  B. 2  C. 3  D. 4

**答案**：C

**考点分析**
- b出栈需a、b在栈
- d出栈需a、c、d在栈
- 最多3个元素

## 解题思路（逐项分析法）

### 第一步：完整模拟栈操作过程

根据出队顺序b,d,c,f,e,a,g反推栈操作：

| 操作 | 栈状态 | 队列 | 栈深度 |
|------|--------|------|--------|
| a入 | [a] | [] | 1 |
| b入 | [a,b] | [] | 2 |
| **b出队** → b必先出栈 | [a] | [b] | 1 |
| c入 | [a,c] | [b] | 2 |
| d入 | [a,c,d] | [b] | **3** ✓ |
| **d出队** → d出栈 | [a,c] | [b,d] | 2 |
| **c出队** → c出栈 | [a] | [b,d,c] | 1 |
| e入 | [a,e] | [b,d,c] | 2 |
| f入 | [a,e,f] | [b,d,c] | **3** ✓ |
| **f出队** → f出栈 | [a,e] | [b,d,c,f] | 2 |
| **e出队** → e出栈 | [a] | [b,d,c,f,e] | 1 |
| **a出队** → a出栈 | [] | [b,d,c,f,e,a] | 0 |
| g入 | [g] | [b,d,c,f,e,a] | 1 |
| **g出队** → g出栈 | [] | [b,d,c,f,e,a,g] | 0 |

**关键发现**：栈深度最大值为**3**（出现在d出栈前和f出栈前）

### 第二步：验证各选项

#### ❌ A. 1
容量为1时，a入后b无法入栈（a未出栈），无法实现b第一个出队
**不可行**

#### ❌ B. 2
容量为2时：
- a入b入（栈满）→ b出 → c入d入（栈满）→ d出
- 此时栈为[a,c]，下一个出队需要c，可以出栈 ✓
- 但到f出队时：栈[a,e]→f入需要3个空间
**容量不足**

#### ✅ C. 3（正确答案）
容量为3时，完整模拟过程：
- 最大栈深出现在[a,c,d]和[a,e,f]两个时刻
- 恰好为3，满足所有操作要求
**可行且最小**

#### ❌ D. 4
容量为4可行但不是最小值，题目要求"至少"
**非最优解**

### 对比表格

| 栈容量 | 能否实现序列 | 关键瓶颈 | 是否最小 |
|--------|-------------|---------|---------|
| 1 | ✗ | a未出时b无法入栈 | - |
| 2 | ✗ | [a,e]时f无法入栈 | - |
| **3** | ✓ | 恰好满足最大深度需求 | ✓ |
| 4 | ✓ | 过剩 | ✗ |

### 易错点

1. **仅看局部未完整模拟**：只分析前几个元素（b,d,c）得出容量2或3，未检查后续f入栈时也需要容量3，应完整模拟全过程
2. **混淆栈深度与队列长度**：统计队列中元素数（最多6个）而非栈中同时存在的最大元素数（3个）
3. **忽略元素必须按序入栈的约束**：误以为可以跳过某些元素先让后面元素入栈，实际上a,b,c,d,e,f,g必须按此顺序依次到达栈
