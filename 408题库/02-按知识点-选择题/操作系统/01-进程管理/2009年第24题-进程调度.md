# 2009年第24题

**元数据**
- 年份：[[../../01-真题原卷/2009年408真题#24|2009年第24题]]
- 科目：#408真题
- 难度：⭐⭐

**题干**

下列进程调度算法中，综合考虑进程等待时间和执行时间的是（）。

A. 时间片轮转调度算法

B. 短进程优先调度算法

C. 先来先服务调度算法

D. 高响应比优先调度算法

**答案**：D

**考点分析**

- 本题考查进程调度算法的特点。高响应比优先调度算法的响应比计算公式为：响应比 = (等待时间 + 要求服务时间) / 要求服务时间，同时考虑了等待时间和执行时间。
- A选项时间片轮转只考虑时间片大小，不考虑等待时间和执行时间；B选项短进程优先只考虑执行时间；C选项先来先服务只考虑到达顺序。
- 高响应比优先算法综合了FCFS和SJF的优点，既考虑等待时间（对长作业有利），又考虑执行时间（对短作业有利），是一种折中的调度策略。
- 掌握各种调度算法的特点和适用场景是操作系统的核心内容。

## 解题思路

### 第一步：理解题目关键
找出"综合考虑等待时间和执行时间"的算法

### 第二步：逐项分析
**A. 时间片轮转 ❌**
```
只按固定时间片轮流执行
不考虑进程等待时间
不考虑进程执行时间
结论：不符合
```

**B. 短进程优先SJF ❌**
```
只考虑执行时间（服务时间短的优先）
不考虑等待时间
短进程可能饿死长进程
结论：不符合
```

**C. 先来先服务FCFS ❌**
```
只考虑到达顺序
不考虑等待时间
不考虑执行时间
结论：不符合
```

**D. 高响应比优先HRRN ✅**
```
响应比 = (等待时间 + 服务时间) / 服务时间
同时考虑等待时间和执行时间
等待越久，响应比越高（照顾长作业）
执行时间短，响应比也高（照顾短作业）
结论：符合
```

### 第三步：确认答案
只有HRRN综合考虑等待和执行时间 → 答案：**D**

## 易错点分析

### 易错点1：认为RR考虑了等待时间
❌ 错误：时间片轮转让所有进程都执行，考虑了等待
✅ 正确：RR只按时间片轮转，不主动计算等待时间

### 易错点2：混淆各算法考虑的因素
| 算法 | 等待时间 | 执行时间 | 到达顺序 |
|------|----------|----------|----------|
| FCFS | ❌ | ❌ | ✅ |
| SJF | ❌ | ✅ | ❌ |
| RR | ❌ | ❌ | 部分（轮转） |
| HRRN | ✅ | ✅ | ❌ |

**HRRN公式解析**：
- 响应比 = (等待时间 + 服务时间) / 服务时间
- 等待时间长 → 响应比大 → 优先级高（避免饥饿）
- 服务时间短 → 响应比大 → 优先级高（照顾短作业）
