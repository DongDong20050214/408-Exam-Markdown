# 银行家算法真题汇总

## 2020年第27题

**元数据**
- 年份：2020年第27题
- 科目：#操作系统
- 知识点：#操作系统/进程管理/银行家算法 #安全序列
- 难度：⭐⭐⭐

**题干**

某系统中有 A、B 两类资源各 6 个，t时刻安全性检测结果是（）。

A. 存在安全序列 P1、P2、P3  B. 存在安全序列 P2、P1、P3  C. 存在安全序列 P2、P3、P1  D. 不存在安全序列

**答案**：C

**考点分析**
- 可用资源：A=6-5=1, B=6-6=0
- 需要逐一检查是否能满足进程需求

---


## 解题思路

### 第一步：整理数据（需要题目表格）
```
假设数据：
总资源：A=6, B=6
进程  Allocation  Need
      A    B     A   B
P1    2    2     1   1
P2    1    3     1   0
P3    2    1     2   2

Available = (6,6) - (5,6) = (1,0)
```

### 第二步：银行家算法安全性检测
```
初始 Available = (1, 0)

检查P1: Need(1,1) > Available(1,0) ❌ B不足
检查P2: Need(1,0) ≤ Available(1,0) ✅
执行P2，释放(1,3)
Available = (1,0) + (1,3) = (2,3)

检查P1: Need(1,1) ≤ Available(2,3) ✅
执行P1，释放(2,2)
Available = (2,3) + (2,2) = (4,5)

检查P3: Need(2,2) ≤ Available(4,5) ✅
执行P3，释放(2,1)
Available = (4,5) + (2,1) = (6,6)

安全序列：P2 → P3 → P1
```

### 第三步：验证答案选项
```
A. P1→P2→P3：P1无法首先执行 ❌
B. P2→P1→P3：可能，但需验证P3
C. P2→P3→P1：符合上述分析 ✅
D. 不存在：已找到安全序列 ❌
```

### 第四步：确认答案
安全序列P2,P3,P1 → 答案：**C**

## 易错点分析

### 易错点1：Available计算错误
❌ 错误：Available = 总资源
✅ 正确：Available = 总资源 - 已分配总量

### 易错点2：忽视资源的多维性
❌ 错误：只看一种资源是否满足
✅ 正确：所有资源类型都要满足Need

| 步骤 | 进程 | Need | Available | 可执行 |
|------|------|------|-----------|--------|
| 初始 | - | - | (1,0) | - |
| 1 | P1 | (1,1) | (1,0) | ❌ |
| 1 | P2 | (1,0) | (1,0) | ✅ |
| 2 | P3 | (2,2) | (2,3) | ✅ |
| 3 | P1 | (1,1) | (4,5) | ✅ |

**银行家算法步骤**：
1. 计算Available
2. 查找Need ≤ Available的进程
3. 分配资源，进程完成释放
4. 更新Available，重复2-3
5. 所有进程完成→安全，否则不安全
