**题干**

并发执行进程 P0 和 P1 时产生的情形是（ ）。A. 不能保证进程互斥进入临界区，会出现"饥饿"现象  B. 不能保证进程互斥进入临界区，不会出现"饥饿"现象  C. 能保证进程互斥进入临界区，会出现"饥饿"现象  D. 能保证进程互斥进入临界区，不会出现"饥饿"现象

**答案**：D

**考点分析**
- Peterson算法正确

---


## 解题思路

### 第一步：理解Peterson算法
Peterson算法是经典的软件互斥算法，满足互斥、有限等待

### 第二步：分析算法特性
Peterson算法保证：
1. 互斥性：两进程不会同时进入临界区
2. 有限等待：进程不会无限期等待（不饥饿）
3. 空闲让进：临界区空闲时可立即进入

### 第三步：逐项分析
**A. 不互斥，有饥饿 ❌**
```
Peterson算法保证互斥
矛盾
```

**B. 不互斥，无饥饿 ❌**
```
Peterson算法保证互斥
矛盾
```

**C. 互斥，有饥饿 ❌**
```
互斥正确
但Peterson算法不会饥饿
通过turn变量轮流进入
```

**D. 互斥，无饥饿 ✅**
```
Peterson算法保证互斥
通过flag和turn变量
保证有限等待，不会饥饿
结论：正确
```

### 第四步：确认答案
互斥且不饥饿 → 答案：**D**

## 易错点分析

### 易错点1：认为Peterson会饥饿
❌ 错误：软件算法可能有缺陷，会饥饿
✅ 正确：Peterson算法经过证明，保证有限等待

### 易错点2：混淆互斥算法
| 算法 | 互斥性 | 有限等待 | 说明 |
|------|--------|----------|------|
| Peterson | ✅ | ✅ | 经典正确算法 |
| Dekker | ✅ | ✅ | 最早的正确算法 |
| 单标志法 | ❌ | - | 违反空闲让进 |
| 双标志先检查 | ❌ | - | 不保证互斥 |
