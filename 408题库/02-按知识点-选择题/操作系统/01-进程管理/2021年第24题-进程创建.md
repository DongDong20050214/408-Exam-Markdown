# 进程创建真题汇总

## 2021年第24题

**元数据**
- 年份：2021年第24题
- 科目：#操作系统
- 知识点：#操作系统/进程管理/进程创建
- 难度：⭐⭐

**题干**

下列操作中，操作系统在创建新进程时，必须完成的是（）。

I. 申请空白的进程控制块  II. 初始化进程控制块  III. 设置进程状态为执行态

A. 仅I  B. 仅 I、II  C. 仅 I、III  D. 仅 II、III

**答案**：B

**考点分析**
- 创建进程：申请PCB + 初始化PCB
- 新创建的进程初始状态是就绪态，不是执行态

---


## 解题思路

### 第一步：理解进程创建过程
进程创建：申请PCB、初始化PCB、设置状态、分配资源

### 第二步：逐项分析
**I. 申请空白PCB ✅**
```
PCB是进程存在的标志
创建进程必须先申请PCB
从PCB池分配空白PCB
结论：必须完成
```

**II. 初始化PCB ✅**
```
填写PCB信息：
- 进程ID
- 进程状态
- 程序计数器
- 寄存器信息等
不初始化PCB无法使用
结论：必须完成
```

**III. 设置状态为执行态 ❌**
```
新创建进程初始状态：就绪态
需要等待调度才能进入执行态
不是立即执行
结论：不是必须
```

### 第三步：确认答案
I、II必须完成 → 答案：**B**

## 易错点分析

### 易错点1：认为新进程立即执行
❌ 错误：创建进程后立即设置为执行态
✅ 正确：新进程初始为就绪态，等待调度

### 易错点2：混淆进程状态
| 状态 | 何时设置 | 说明 |
|------|---------|------|
| 就绪态 | 创建时 | 新进程初始状态 |
| 执行态 | 被调度后 | 获得CPU |
| 阻塞态 | 等待事件 | I/O等待 |

**进程创建步骤**：
1. 申请空白PCB
2. 初始化PCB
   - 进程ID
   - 父进程ID
   - 初始状态（就绪态）
   - 程序计数器
   - 优先级等
3. 分配必要资源
4. 插入就绪队列

**状态转换**：
```
创建 → 就绪态（初始）
就绪态 → 执行态（被调度）
执行态 → 就绪态（时间片用完/被抢占）
执行态 → 阻塞态（等待事件）
```

**关键点**：
- 申请PCB：必须
- 初始化PCB：必须
- 设置执行态：错误，应设置就绪态
