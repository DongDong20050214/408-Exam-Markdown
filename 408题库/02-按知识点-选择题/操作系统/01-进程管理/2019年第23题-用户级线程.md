# 线程真题汇总

## 2019年第23题

**元数据**
- 年份：[[../../01-真题原卷/2019年408真题#23|2019年第23题]]
- 科目：#操作系统
- 知识点：#操作系统/进程管理/线程
- 难度：⭐⭐
- 关联知识：[[2019年第23题-线程]]

**题干**

下列关于线程的描述中，错误的是（）。A. 内核级线程的调度由操作系统完成  B. 操作系统为每个用户级线程建立一个线程控制块  C. 用户级线程间的切换比内核级线程间的切换效率高  D. 用户级线程可以在不支持内核级线程的操作系统上实现

**答案**：B

**考点分析**
- B错误：用户级线程由用户进程管理

---


## 解题思路

### 第一步：理解用户级和内核级线程
用户级：用户空间管理，OS不可见
内核级：内核管理，OS可见可调度

### 第二步：逐项分析
**A. 内核级线程由OS调度 ✅**
```
内核级线程：OS可见
OS调度器直接调度线程
切换需陷入内核
结论：正确
```

**B. OS为用户级线程建立TCB ❌**
```
用户级线程：OS不可见
线程库在用户空间维护TCB
OS只看到进程，不知道线程存在
结论：错误
```

**C. 用户级切换效率高 ✅**
```
用户级切换：用户空间完成，不陷入内核
内核级切换：需陷入内核，开销大
结论：正确
```

**D. 用户级可在不支持内核级的OS上实现 ✅**
```
用户级线程由线程库实现
不需要OS支持
老版本Unix就是如此
结论：正确
```

### 第三步：确认答案
B错误 → 答案：**B**

## 易错点分析

### 易错点1：认为所有线程OS都要管理
❌ 错误：线程都需要OS建立TCB
✅ 正确：用户级线程OS不可见，线程库管理

### 易错点2：混淆两种线程
| 特性 | 用户级线程 | 内核级线程 |
|------|-----------|-----------|
| OS可见 | ❌ | ✅ |
| TCB位置 | 用户空间 | 内核空间 |
| 切换开销 | 小 | 大 |
| 并行执行 | ❌ | ✅ |
| 需OS支持 | ❌ | ✅ |
