# 批处理作业完成时间真题汇总

## 2012年第29题

**元数据**
- 年份：[[../../01-真题原卷/2012年408真题#29|2012年第29题]]
- 科目：#操作系统
- 知识点：#操作系统/进程管理/作业调度 #流水并发
- 难度：⭐⭐⭐
- 关联知识：[[知识库/操作系统/作业调度]]

**题干**

一个多道批处理系统中仅有P₁和P₂两个作业，P₂比P₁晚5ms到达。
P₁：计算60ms，I/O 80ms，计算20ms
P₂：计算120ms，I/O 40ms，计算40ms
若不考虑调度和切换时间，则完成两个作业需要的时间最少是（）。

A. 240ms  
B. 260ms  
C. 340ms  
D. 360ms

**答案**：B

**考点分析**
- P₁: 0-60计算，60-140 I/O，140-160计算
- P₂: 5开始，5-60等待(P₁占CPU)，60-140计算(80ms)，140-180 I/O，180-220计算
- P₂计算：5等到60开始(55ms)+60ms=115ms，剩60ms
- 优化：P₁ I/O时P₂可计算
- P₁: 0-60计算，60-140 I/O，140-160计算
- P₂: 5-125计算，140-180 I/O，180-220计算
- 实际：P₂从60开始计算120ms到180，P₁ I/O结束140，P₁计算到160
- 正确：P₂ I/O从180到220，P₂计算到260
- 最少时间260ms

---


## 解题思路

### 第一步：分析作业流程
```
P₁：计算60ms → I/O 80ms → 计算20ms（共160ms）
P₂：计算120ms → I/O 40ms → 计算40ms（共200ms）
P₂延迟5ms到达
```

### 第二步：绘制时间轴（最优调度）
**关键：CPU和I/O并行工作**

```
时间轴分析：
0-5:    P₁计算(5ms)                P₂未到达
5-60:   P₁计算(55ms)               P₂到达等待
60-140: P₁ I/O(80ms)               P₂计算(80ms)
140-160: P₁计算(20ms) P₁完成      P₂计算(20ms)
160-180: 空闲                      P₂计算(20ms) 计算完成
180-220: 空闲                      P₂ I/O(40ms)
220-260: 空闲                      P₂计算(40ms) P₂完成
```

**详细过程**：
- 0-60: P₁计算60ms
- 60-140: P₁执行I/O，P₂抢占CPU计算（已等待55ms）
- 60-140: P₂计算80ms（120ms中的前80ms）
- 140-160: P₁ I/O完成，抢占CPU计算20ms完成
- 160-180: P₂继续计算40ms（120ms剩余部分）
- 180-220: P₂执行I/O 40ms
- 220-260: P₂计算40ms完成

### 第三步：确认答案
总时间260ms → 答案：**B**

## 易错点分析

### 易错点1：忽视CPU和I/O并行
❌ 错误：P₁全部执行完(160ms)，P₂再执行(200ms)，共360ms
✅ 正确：P₁ I/O时P₂可用CPU，并行执行

### 易错点2：忽视P₂延迟到达
❌ 错误：P₂从0时刻开始计算
✅ 正确：P₂从5ms才到达，需等待

**时间轴图**：
| 时间 | CPU | I/O设备 | 说明 |
|------|-----|---------|------|
| 0-60 | P₁计算 | 空闲 | P₂未到 |
| 60-140 | P₂计算 | P₁ I/O | 并行 |
| 140-160 | P₁计算 | 空闲 | P₁抢占 |
| 160-180 | P₂计算 | 空闲 | P₂继续 |
| 180-220 | 空闲 | P₂ I/O | P₂ I/O |
| 220-260 | P₂计算 | 空闲 | P₂完成 |

**计算关键**：
- P₁总时间：60+80+20=160ms
- P₂总时间：120+40+40=200ms
- 但由于并行，实际：max(160, 5+200-重叠) = 260ms
