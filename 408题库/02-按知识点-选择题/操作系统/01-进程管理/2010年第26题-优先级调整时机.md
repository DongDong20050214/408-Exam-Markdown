# 进程优先级调整真题汇总

## 2010年第26题

**元数据**
- 年份：[[../../01-真题原卷/2010年408真题#26|2010年第26题]]
- 科目：#操作系统
- 知识点：#操作系统/进程管理/优先级调整
- 难度：⭐⭐
- 关联知识：[[知识库/操作系统/进程调度]]

**题干**

下列选项中，降低进程优先级的合理时机是（）。

A. 进程的时间片用完  
B. 进程刚完成I/O，进入就绪队列  
C. 进程长期处于就绪队列中  
D. 进程从就绪状态转为运行状态

**答案**：A

**考点分析**
- A正确：时间片用完，降低优先级，防止独占CPU
- B错误：I/O完成应提高优先级(I/O密集型)
- C错误：长期等待应提高优先级，防止饥饿
- D错误：获得CPU不应降级

---


## 解题思路

### 第一步：理解优先级调整原则
降低优先级：CPU使用过多（时间片用完）
提高优先级：等待过久、I/O完成

### 第二步：逐项分析
**A. 时间片用完 ✅**
```
进程用完时间片说明CPU密集
降低优先级，给其他进程机会
多级反馈队列的典型策略
结论：应该降低
```

**B. I/O完成进入就绪 ❌**
```
I/O完成说明是I/O密集型
I/O密集型进程应提高优先级
让其快速执行再次等待I/O
结论：应该提高，不应降低
```

**C. 长期处于就绪队列 ❌**
```
长期等待说明可能饥饿
应提高优先级防止饥饿
老化(aging)机制
结论：应该提高，不应降低
```

**D. 就绪→运行 ❌**
```
刚获得CPU开始执行
不应立即降低优先级
让其正常执行
结论：不应降低
```

### 第三步：确认答案
只有时间片用完应降低优先级 → 答案：**A**

## 易错点分析

### 易错点1：认为I/O完成应降低优先级
❌ 错误：I/O完成了，应该降低优先级
✅ 正确：I/O密集型进程应该提高优先级，快速执行

### 易错点2：混淆优先级调整策略
| 情况 | 优先级调整 | 原因 |
|------|-----------|------|
| 时间片用完 | ⬇️降低 | CPU使用过多 |
| I/O完成 | ⬆️提高 | I/O密集型 |
| 长期等待 | ⬆️提高 | 防止饥饿 |
| 获得CPU | ➡️不变 | 正常执行 |
