# 死锁真题汇总

## 2016年第25题

**元数据**
- 年份：[2016年第25题](../../../01-真题原卷/2016年408真题.md#25)
- 科目：#操作系统
- 知识点：#操作系统/进程管理/死锁
- 难度：⭐⭐⭐
- 关联知识：[死锁](../../../../AI-知识点原子库/死锁.md)

**题干**

系统中有 3 个不同的临界资源 R1, R2 和 R3, 被 4 个进程共享。若系统出现死锁, 则处于死锁状态的进程数至少是 ( )。A. 1  B. 2  C. 3  D. 4

**答案**：B

**考点分析**
- 形成环路至少需要2个进程

---


## 解题思路

### 第一步：理解死锁必要条件
死锁需要满足：互斥、占有并等待、非抢占、循环等待
关键：循环等待至少需要形成环路

### 第二步：分析最少死锁进程数
```
死锁环路最小情况：
P1持有R1，等待R2
P2持有R2，等待R1
形成循环等待：P1→R2→P2→R1→P1

最少2个进程即可死锁
```

### 第三步：逐项分析
**A. 1个进程 ❌**
```
单个进程无法形成循环等待
自己等待自己持有的资源，不符合死锁定义
结论：不可能
```

**B. 2个进程 ✅**
```
最小死锁环路：
进程P1：持有R1，等待R2
进程P2：持有R2，等待R1
形成P1→P2→P1的环路
结论：至少2个
```

**C. 3个进程 ❌**
```
2个进程即可死锁
不需要3个
结论：不是最少
```

**D. 4个进程 ❌**
```
2个进程即可死锁
不需要4个
结论：不是最少
```

### 第四步：确认答案
至少2个进程 → 答案：**B**

## 易错点分析

### 易错点1：认为需要资源数等于进程数
❌ 错误：3个资源需要至少3个进程死锁
✅ 正确：2个进程即可形成死锁环路

### 易错点2：混淆死锁进程数和资源数
| 概念 | 数量 | 说明 |
|------|------|------|
| 资源数 | 3个 | R1、R2、R3 |
| 进程数 | 4个 | 共享资源 |
| 最少死锁进程 | 2个 | 形成环路 |

**死锁示例**：
```
场景1（2进程死锁）：
P1: 持有R1，等待R2
P2: 持有R2，等待R1
→ 形成死锁

场景2（3进程死锁）：
P1: 持有R1，等待R2
P2: 持有R2，等待R3
P3: 持有R3，等待R1
→ 形成死锁

最少情况：2个进程
```
