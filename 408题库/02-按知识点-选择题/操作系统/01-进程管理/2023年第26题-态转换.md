# 态转换真题汇总

## 2023年第26题

**元数据**
- 年份：2023年第26题
- 科目：#操作系统
- 知识点：#操作系统/进程管理/态转换 #内核态 #用户态
- 难度：⭐⭐

**题干**

下列操作完成时，导致 CPU 从内核态转为用户态的是（）。

A. 阻塞进程  B. 执行 CPU 调度  C. 唤醒进程  D. 执行系统调用

**答案**：B

**考点分析**
- CPU调度后返回用户进程执行，从内核态转为用户态
- 系统调用是用户态→内核态

---


## 解题思路

### 第一步：理解内核态→用户态转换
内核态→用户态：从OS返回用户程序执行

### 第二步：逐项分析
**A. 阻塞进程 ❌**
```
阻塞进程在内核态完成
阻塞后不会立即返回用户态
仍需选择其他进程执行
结论：不会转为用户态
```

**B. 执行CPU调度 ✅**
```
调度完成后选中新进程
返回用户进程执行
内核态→用户态
结论：会转为用户态
```

**C. 唤醒进程 ❌**
```
唤醒进程在内核态完成
唤醒后进程进入就绪队列
不会立即返回用户态
结论：不会转为用户态
```

**D. 执行系统调用 ❌**
```
系统调用：用户态→内核态
不是内核态→用户态
方向相反
结论：不会转为用户态
```

### 第三步：确认答案
CPU调度完成转用户态 → 答案：**B**

## 易错点分析

### 易错点1：认为系统调用会转用户态
❌ 错误：系统调用完成返回用户态
✅ 正确：
- 系统调用开始：用户态→内核态
- 系统调用结束：内核态→用户态（返回）
- 题目问"执行"，是进入阶段

### 易错点2：混淆态转换时机
| 操作 | 态转换 | 时机 |
|------|--------|------|
| 系统调用 | 用户→内核 | 执行trap |
| 调度完成 | 内核→用户 | 返回进程 |
| 中断发生 | 用户→内核 | 中断响应 |
| 中断返回 | 内核→用户 | 恢复执行 |

**关键点**：
- 内核态→用户态：返回用户进程执行
- 调度后必然返回用户态
- 阻塞/唤醒在内核态完成，不返回
