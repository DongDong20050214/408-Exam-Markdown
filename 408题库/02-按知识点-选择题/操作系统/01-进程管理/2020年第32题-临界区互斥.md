# 临界区互斥真题汇总

## 2020年第32题

**元数据**
- 年份：2020年第32题
- 科目：#操作系统
- 知识点：#操作系统/进程管理/临界区
- 难度：⭐⭐

**题干**

下列准则中, 实现临界区互斥机制必须遵循的是 ( )。

I. 两个进程不能同时进入临界区  II. 允许进程访问空闲的临界资源  III. 进程等待进入临界区的时间是有限的  IV. 不能进入临界区的执行态进程立即放弃 CPU

A. 仅 I、IV  B. 仅II、III  C. 仅I、II、III  D. 仅 I、III、IV

**答案**：C

**考点分析**
- I：互斥访问  II：空闲让进  III：有限等待
- IV不是必须的

---


## 解题思路

### 第一步：理解临界区互斥准则
经典三原则：互斥、空闲让进、有限等待

### 第二步：逐项分析
**I. 两进程不能同时进入 ✅**
```
互斥访问：临界区互斥的核心
同一时刻只有一个进程在临界区
这是必须遵循的基本原则
结论：必须遵循
```

**II. 允许访问空闲的临界资源 ✅**
```
空闲让进：临界区空闲时应允许进入
不能无故阻止进程访问空闲资源
提高系统效率
结论：必须遵循
```

**III. 等待时间有限 ✅**
```
有限等待（避免饥饿）
进程不能无限期等待
保证每个进程最终能进入临界区
结论：必须遵循
```

**IV. 不能进入立即放弃CPU ❌**
```
让权等待：建议遵循但非必须
等待时放弃CPU可提高效率
但不是互斥机制的必要条件
忙等待也能实现互斥（如自旋锁）
结论：不是必须
```

### 第三步：确认答案
I、II、III必须遵循 → 答案：**C**

## 易错点分析

### 易错点1：认为让权等待是必须的
❌ 错误：不能进入必须立即放弃CPU
✅ 正确：让权等待是优化，不是必须条件

### 易错点2：混淆必须准则和优化准则
| 准则 | 是否必须 | 说明 |
|------|---------|------|
| 互斥访问 | ✅ | 核心要求 |
| 空闲让进 | ✅ | 提高效率 |
| 有限等待 | ✅ | 避免饥饿 |
| 让权等待 | ❌ | 优化建议 |

**临界区互斥必须准则**：
1. **互斥访问**：同时只有一个进程
2. **空闲让进**：空闲时允许进入
3. **有限等待**：等待时间有限

**让权等待**：
- 等待时释放CPU
- 是优化策略，不是必须
- 忙等待(spin lock)不让权但也合法

**示例**：
- 信号量：让权等待 ✅
- TSL指令：忙等待，不让权 ✅
- 都能实现互斥，但效率不同
