**题干**

下列有关基于时间片的进程调度的叙述中，错误的是（）。A. 时间片越短, 进程切换的次数越多, 系统开销也越大  B. 当前进程的时间片用完后, 该进程状态由执行态变为阻塞态  C. 时钟中断发生后，系统会修改当前进程在时间片内的剩余时间  D. 影响时间片大小的主要因素包括响应时间、系统开销和进程数量等

**答案**：B

**考点分析**
- B错误：时间片用完变为就绪态，不是阻塞态

---


## 解题思路

### 第一步：理解时间片轮转调度
时间片轮转（RR）：每个进程执行固定时间片
时间片用完：进程回到就绪队列末尾

### 第二步：逐项分析
**A. 时间片越短，开销越大 ✅**
```
时间片短 → 切换频繁 → 开销大
时间片长 → 切换少 → 开销小
需要权衡响应时间和系统开销
结论：正确
```

**B. 时间片用完变阻塞态 ❌**
```
时间片用完：执行态 → 就绪态
进程仍可运行，只是暂时失去CPU
阻塞态：等待事件（I/O等），不能运行
结论：错误
```

**C. 时钟中断修改剩余时间 ✅**
```
时钟中断周期性发生
每次中断：剩余时间片-1
为0时进行进程切换
结论：正确
```

**D. 时间片大小受多因素影响 ✅**
```
响应时间：越短越快响应
系统开销：切换次数影响开销
进程数量：进程多需要更小时间片
需要综合考虑
结论：正确
```

### 第三步：确认答案
B错误，时间片用完变就绪态 → 答案：**B**

## 易错点分析

### 易错点1：混淆就绪态和阻塞态
❌ 错误：时间片用完进程不能继续运行，变阻塞
✅ 正确：时间片用完进程能运行，只是等CPU，是就绪态

### 易错点2：不理解时间片大小权衡
| 时间片 | 响应时间 | 切换开销 | 适用场景 |
|--------|----------|----------|----------|
| 很小 | 快 | 大 | 交互式系统 |
| 适中 | 中等 | 中等 | 通用系统 |
| 很大 | 慢 | 小 | 接近FCFS |

**状态转换关键**：
```
时间片用完：执行态 → 就绪态（能运行，等CPU）
I/O请求：执行态 → 阻塞态（不能运行，等事件）
I/O完成：阻塞态 → 就绪态（能运行，等CPU）
被调度：就绪态 → 执行态（获得CPU）
```

**时间片设置原则**：
- 太小：频繁切换，开销大
- 太大：响应慢，退化为FCFS
- 经验值：10-100ms
