**题干**

下列关于进程和线程的叙述中，正确的是（）。

A. 不管系统是否支持线程，进程都是资源分配的基本单位  
B. 线程是资源分配的基本单位，进程是调度的基本单位  
C. 系统级线程和用户级线程的切换都需要内核的支持  
D. 同一进程中的各个线程拥有各自不同的地址空间

**答案**：A

**考点分析**
- A正确：进程是资源分配单位
- B错误：相反，进程是资源分配，线程是调度单位
- C错误：用户级线程切换不需要内核支持
- D错误：同一进程的线程共享地址空间

---


## 解题思路

### 第一步：理解进程和线程
进程：资源分配单位
线程：调度单位（CPU执行单位）

### 第二步：逐项分析
**A. 进程都是资源分配单位 ✅**
```
无论是否支持线程
进程都是资源分配的基本单位
拥有独立地址空间、文件、内存等
结论：正确
```

**B. 线程是资源分配单位 ❌**
```
说反了
线程是调度单位
进程是资源分配单位
结论：错误
```

**C. 用户级线程切换需要内核 ❌**
```
系统级线程切换：需要内核
用户级线程切换：在用户态完成，不需内核
结论：错误
```

**D. 线程拥有不同地址空间 ❌**
```
同一进程的线程共享地址空间
共享代码段、数据段、堆
只有栈是独立的
结论：错误
```

### 第三步：确认答案
只有A正确 → 答案：**A**

## 易错点分析

### 易错点1：混淆资源分配和调度单位
❌ 错误：线程更轻量，所以线程是资源分配单位
✅ 正确：
- 进程：资源分配单位（拥有资源）
- 线程：调度单位（执行实体）

### 易错点2：混淆用户级和系统级线程
| 线程类型 | 切换位置 | 内核感知 |
|----------|----------|----------|
| 用户级线程 | 用户态 | ❌ 内核不知道 |
| 系统级线程 | 内核态 | ✅ 内核管理 |
