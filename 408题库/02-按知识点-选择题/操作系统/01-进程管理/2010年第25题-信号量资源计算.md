# 信号量资源计算真题汇总

## 2010年第25题

**元数据**
- 年份：[[../../01-真题原卷/2010年408真题#25|2010年第25题]]
- 科目：#操作系统
- 知识点：#操作系统/进程管理/信号量
- 难度：⭐⭐⭐
- 关联知识：[[知识库/操作系统/信号量]]

**题干**

信号量初值为3，当前值为1。若M表示可用个数，N表示等待进程数，则M,N分别是（）。

A. 0,1  
B. 1,0  
C. 1,2  
D. 2,0

**答案**：B

**考点分析**
- 信号量>0：表示可用资源数
- 当前值=1：可用资源1个，无等待进程
- M=1, N=0

---


## 解题思路

### 第一步：理解信号量含义
信号量S：
- S>0：可用资源数M=S，等待进程N=0
- S≤0：可用资源M=0，等待进程N=|S|

### 第二步：分析当前状态
```
初值=3，当前值=1
当前值S=1>0
可用资源M=S=1
等待进程N=0
```

### 第三步：逐项分析
**A. M=0,N=1 ❌**
```
信号量=1>0，可用资源应该>0
矛盾
```

**B. M=1,N=0 ✅**
```
S=1>0
M=S=1（1个可用资源）
N=0（无等待进程）
结论：正确
```

**C. M=1,N=2 ❌**
```
S=1>0时，不应有等待进程
矛盾
```

**D. M=2,N=0 ❌**
```
M应该等于S=1，不是2
矛盾
```

### 第四步：确认答案
M=1，N=0 → 答案：**B**

## 易错点分析

### 易错点1：混淆初值和当前值
❌ 错误：初值3，所以M=3
✅ 正确：看当前值，当前值=1，所以M=1

### 易错点2：不理解信号量含义
| 信号量S | 可用资源M | 等待进程N | 说明 |
|---------|----------|----------|------|
| S>0 | M=S | N=0 | 有可用资源 |
| S=0 | M=0 | N=0 | 资源刚好用完 |
| S<0 | M=0 | N=\|S\| | 有进程等待 |

**记忆口诀**：
- 正数看资源（M=S）
- 负数看队列（N=|S|）
