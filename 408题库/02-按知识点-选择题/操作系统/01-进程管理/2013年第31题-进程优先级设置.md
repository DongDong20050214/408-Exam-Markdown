# 进程优先级真题汇总

## 2013年第31题

**元数据**
- 年份：[[../../01-真题原卷/2013年408真题#31|2013年第31题]]
- 科目：#操作系统
- 知识点：#操作系统/进程管理/优先级
- 难度：⭐⭐⭐
- 关联知识：[[知识库/操作系统/进程调度]]

**题干**

为提高系统资源利用率，合理的进程优先级设置应为（）。A. P1 > P2 > P3  B. P3 > P2 > P1  C. P2 > P1 = P3  D. P1 > P2 = P3

**答案**：B

**考点分析**
- IO密集型进程优先级高，P3>P2>P1

---


## 解题思路

### 第一步：理解优先级设置原则
I/O密集型优先级高：快速执行释放CPU，提高I/O利用率
CPU密集型优先级低：减少对CPU的长期占用

### 第二步：分析进程特点
```
需要题目给出进程信息
通常：
P1: CPU密集型
P2: I/O和CPU均衡
P3: I/O密集型

优先级应该：I/O密集 > 均衡 > CPU密集
即：P3 > P2 > P1
```

### 第三步：逐项分析
**A. P1>P2>P3 ❌**
```
CPU密集型优先级最高
会长期占用CPU
I/O设备空闲
资源利用率低
结论：不合理
```

**B. P3>P2>P1 ✅**
```
I/O密集型优先级最高
快速执行后等待I/O
CPU可执行其他进程
I/O和CPU并行工作
资源利用率高
结论：合理
```

**C. P2>P1=P3 ❌**
```
不符合I/O优先原则
结论：不合理
```

**D. P1>P2=P3 ❌**
```
CPU密集型最高
不合理
结论：不合理
```

### 第四步：确认答案
P3>P2>P1 → 答案：**B**

## 易错点分析

### 易错点1：认为CPU密集型优先级高
❌ 错误：CPU密集型计算量大，应该优先执行
✅ 正确：I/O密集型快速执行释放CPU，提高利用率

### 易错点2：不理解优先级设置目的
| 进程类型 | 优先级 | 原因 |
|----------|--------|------|
| I/O密集型 | 高 | 快速执行，让出CPU |
| CPU密集型 | 低 | 长期占用，后台执行 |

**资源利用率**：
- I/O优先：I/O执行时CPU可处理其他进程
- CPU优先：CPU忙碌时I/O设备空闲
