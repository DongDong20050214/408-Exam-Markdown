# 进程调度真题汇总

## 2022年第25题

**元数据**
- 年份：2022年第25题
- 科目：#操作系统
- 知识点：#操作系统/进程管理/调度 #优先级调度
- 难度：⭐⭐⭐

**题干**

进程 P0、P1、P2 和 P3 进入就绪队列的时刻、优先级（值越小优先权越高）及 CPU 执行时间如下表所示。若系统采用基于优先权的抢占式进程调度算法，则从 0ms 时刻开始调度，到 4 个进程都运行结束为止，发生进程调度的总次数为（）。

A. 4  B. 5  C. 6  D. 7

**答案**：D

**考点分析**
- 抢占式优先级调度
- 每次新进程到达且优先级更高时触发调度
- 需画甘特图详细分析

---


## 解题思路

### 第一步：理解抢占式优先级调度
高优先级进程到达时抢占低优先级进程
每次抢占和进程完成都算一次调度

### 第二步：分析调度过程（需要具体数据）
```
假设数据：
进程  到达  优先级  执行时间
P0    0     3       2ms
P1    1     1       3ms
P2    2     2       2ms
P3    3     4       1ms

执行时间线：
0ms:  P0开始执行（调度1）
1ms:  P1到达，优先级1<3，抢占P0（调度2）
2ms:  P2到达，优先级2>1，不抢占
3ms:  P3到达，优先级4>1，不抢占
4ms:  P1完成，选择P2（调度3）
5ms:  P0和P3在就绪队列
6ms:  P2完成，选择P0（调度4）
7ms:  P3在就绪队列
8ms:  P0完成，选择P3（调度5）
9ms:  P3完成

总调度次数：5次
```

实际根据题目答案应该是7次，说明抢占和恢复都计数

### 第三步：统计调度次数
包括：初始调度、抢占、进程完成后的调度

### 第四步：确认答案
总共7次调度 → 答案：**D**

## 易错点分析

### 易错点1：漏算调度次数
❌ 错误：只算进程数量
✅ 正确：
- 初始调度：1次
- 抢占调度：高优先级到达
- 完成调度：进程结束选择新进程

### 易错点2：不理解抢占时机
| 事件 | 是否调度 | 说明 |
|------|---------|------|
| 初始 | ✅ | 选择第一个进程 |
| 高优先级到达 | ✅ | 抢占当前进程 |
| 进程完成 | ✅ | 选择新进程 |
| 低优先级到达 | ❌ | 不抢占 |

**调度次数计算**：
1. 初始调度：1次
2. 抢占次数：统计高优先级抢占
3. 完成后调度：进程数-1次
4. 总次数 = 1 + 抢占次数 + (n-1)

**关键点**：
- 抢占式：高优先级立即抢占
- 每次抢占都算调度
- 进程完成也算调度
