# 2022年第28题

**元数据**
- 年份：2022年第28题
- 科目：#操作系统
- 知识点：#操作系统/进程管理/进程状态
- 难度：⭐⭐⭐

**题干**

下列事件或操作中，可能导致进程 $\mathrm{P}$ 由执行态变为阻塞态的是（）。

I. 进程 P 读文件

II. 进程 $\mathbf{P}$ 的时间片用完

III. 进程 P 申请外设

IV. 进程 P 执行信号量的 wait() 操作

A. 仅 I、IV

B. 仅 II、III

C. 仅 III、IV

D. 仅 I、III、IV

**答案**：D

**考点分析**

本题考查进程状态转换，特别是执行态到阻塞态的转换条件。进程由执行态变为阻塞态的本质是：**进程因等待某个事件的发生而暂时无法继续执行**。

**各选项分析**：

**I. 进程 P 读文件**：**可能导致阻塞**。
- 读文件通常涉及I/O操作，需要等待磁盘读取数据
- 若文件数据不在缓冲区，进程需等待I/O完成，进入阻塞态
- 若数据已在缓冲区，则可能不阻塞（但题目问"可能"，所以成立）

**II. 进程 P 的时间片用完**：**不导致阻塞**。
- 时间片用完导致进程从执行态转为就绪态（可执行但等待CPU）
- 进程并未等待I/O或其他事件，仍具备运行条件
- 这是就绪态转换，不是阻塞态

**III. 进程 P 申请外设**：**可能导致阻塞**。
- 若外设空闲，可能立即分配（不阻塞）
- 若外设忙碌，进程需等待外设可用，进入阻塞态等待
- 题目问"可能"，所以成立

**IV. 进程 P 执行信号量的 wait() 操作**：**可能导致阻塞**。
- wait(S)：若 S > 0，则 S--，进程继续执行（不阻塞）
- 若 S ≤ 0，则 S--，进程进入阻塞态，等待资源释放
- 题目问"可能"，所以成立

**结论**：I、III、IV 都可能导致进程从执行态变为阻塞态，答案为D。

## 解题思路

### 第一步：理解执行态→阻塞态
阻塞：等待事件，不能继续执行

### 第二步：逐项分析
**I. 读文件 ✅**
```
读文件需要I/O操作
等待磁盘读取数据
执行态→阻塞态
结论：可能阻塞
```

**II. 时间片用完 ❌**
```
时间片用完失去CPU
进程能运行但等待CPU
执行态→就绪态（不是阻塞）
结论：不会阻塞
```

**III. 申请外设 ✅**
```
外设忙碌时需要等待
进程阻塞等待外设可用
执行态→阻塞态
结论：可能阻塞
```

**IV. wait()操作 ✅**
```
wait(S)：
- S>0：S--，继续执行
- S≤0：S--，进入阻塞
可能导致阻塞
结论：可能阻塞
```

### 第三步：确认答案
I、III、IV可能阻塞 → 答案：**D**

## 易错点分析

### 易错点1：认为时间片用完也阻塞
❌ 错误：失去CPU就是阻塞
✅ 正确：
- 阻塞：等事件，不能运行
- 就绪：等CPU，能运行

### 易错点2：混淆状态转换
| 事件 | 转换 | 原因 |
|------|------|------|
| 读文件 | 执行→阻塞 | 等I/O |
| 时间片用完 | 执行→就绪 | 等CPU |
| 申请外设 | 执行→阻塞 | 等外设 |
| wait()≤0 | 执行→阻塞 | 等资源 |

**关键区别**：
- 阻塞态：等待外部事件
- 就绪态：等待CPU调度
