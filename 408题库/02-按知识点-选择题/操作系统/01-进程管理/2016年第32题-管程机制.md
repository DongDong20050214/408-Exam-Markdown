**题干**

下列关于管程的叙述中，错误的是（）。

A. 管程只能用于实现进程的互斥  
B. 管程是由编程语言支持的进程同步机制  
C. 任何时候只能有一个进程在管程中执行  
D. 管程中定义的变量只能被管程内的过程访问

**答案**：A

**考点分析**
- A错误：管程既能实现互斥，也能实现同步(通过条件变量)
- B正确：管程是高级语言提供的同步机制
- C正确：管程保证互斥访问
- D正确：管程封装性，变量私有化
- 管程功能：互斥+同步

---


## 解题思路

### 第一步：理解管程机制
管程（Monitor）：高级同步机制
特点：互斥访问、条件变量实现同步、封装性

### 第二步：逐项分析
**A. 管程只能用于实现互斥 ❌**
```
管程功能：
1. 互斥：同一时刻只有一个进程在管程内
2. 同步：条件变量wait/signal实现进程同步
管程既能实现互斥，也能实现同步
结论：错误
```

**B. 管程是语言支持的同步机制 ✅**
```
管程由编程语言提供
如Java的synchronized、C++的monitor
编译器保证互斥性
高级同步原语
结论：正确
```

**C. 同一时刻只能一个进程执行 ✅**
```
管程保证互斥访问
任何时刻最多一个进程在管程内
编译器自动加锁解锁
结论：正确
```

**D. 管程变量只能内部访问 ✅**
```
管程封装共享变量
外部不能直接访问
只能通过管程提供的过程访问
数据隐藏和保护
结论：正确
```

### 第三步：确认答案
A错误，管程不只能实现互斥 → 答案：**A**

## 易错点分析

### 易错点1：认为管程只能实现互斥
❌ 错误：管程保证互斥访问，只能实现互斥
✅ 正确：管程通过条件变量也能实现同步

### 易错点2：混淆管程和信号量
| 机制 | 互斥 | 同步 | 实现层次 | 使用难度 |
|------|------|------|---------|---------|
| 信号量 | ✅ | ✅ | 操作系统 | 难（易出错） |
| 管程 | ✅ | ✅ | 编程语言 | 易（自动互斥） |

**管程结构**：
```
Monitor Name {
    // 共享变量
    int count;
    
    // 条件变量（用于同步）
    condition full, empty;
    
    // 过程（自动互斥）
    procedure insert() {
        if (count == N)
            wait(full);  // 同步
        ...
        signal(empty);
    }
    
    procedure remove() {
        if (count == 0)
            wait(empty); // 同步
        ...
        signal(full);
    }
}
```

**关键点**：
- 互斥：管程自动保证
- 同步：条件变量wait/signal实现
