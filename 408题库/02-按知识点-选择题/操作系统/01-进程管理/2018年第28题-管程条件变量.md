**题干**

若 x 是管程内的条件变量，则当进程执行 x.wait() 时所做的工作是（）。

A. 实现对变量 x 的互斥访问
B. 唤醒一个在 x 上阻塞的进程
C. 根据 x 的值判断该进程是否进入阻塞状态
D. 阻塞该进程，并将之插入 x 的阻塞队列中

**答案**：D

**考点分析**
- wait操作：阻塞进程并加入等待队列

---


## 解题思路

### 第一步：理解管程条件变量
条件变量：用于进程同步
wait()：阻塞等待
signal()：唤醒等待进程

### 第二步：逐项分析
**A. 实现对x的互斥访问 ❌**
```
互斥访问由管程自动保证
不是条件变量的作用
条件变量用于同步，不是互斥
结论：错误
```

**B. 唤醒一个阻塞进程 ❌**
```
唤醒操作是signal()
不是wait()
wait()是阻塞自己
结论：错误
```

**C. 根据x值判断是否阻塞 ❌**
```
条件变量x不是普通变量，没有"值"
x.wait()直接阻塞，不判断
结论：错误
```

**D. 阻塞进程并插入x的队列 ✅**
```
x.wait()的工作：
1. 阻塞当前进程
2. 将进程加入x的等待队列
3. 释放管程（让其他进程进入）
等待x.signal()唤醒
结论：正确
```

### 第三步：确认答案
wait阻塞并入队 → 答案：**D**

## 易错点分析

### 易错点1：混淆wait和signal
❌ 错误：wait唤醒进程
✅ 正确：
- wait()：阻塞当前进程
- signal()：唤醒等待进程

### 易错点2：认为条件变量有值
❌ 错误：根据x的值判断
✅ 正确：条件变量只是同步机制，不存储值

| 操作 | 作用 | 当前进程 | 等待队列 |
|------|------|----------|----------|
| x.wait() | 阻塞等待 | 阻塞 | 加入x队列 |
| x.signal() | 唤醒进程 | 继续 | 移出一个进程 |

**管程条件变量机制**：
```
Monitor {
    condition x;  // 条件变量
    
    procedure P1() {
        if (条件不满足)
            x.wait();  // 阻塞并加入x队列
        ...
    }
    
    procedure P2() {
        ...
        x.signal();  // 唤醒x队列中的进程
    }
}
```

**关键点**：
- 条件变量 ≠ 普通变量
- wait：阻塞+入队
- signal：出队+唤醒
