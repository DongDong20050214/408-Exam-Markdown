**题干**

下列选项中，会导致进程从执行态变为就绪态的事件是（）。A. 执行 P(wait)操作  B. 申请内存失败  C. 启动 I/O 设备  D. 被高优先级进程抢占

**答案**：D

**考点分析**
- D：被抢占从执行态变就绪态

---


## 解题思路

### 第一步：理解执行态→就绪态
执行态：占有CPU正在执行
就绪态：能运行但CPU被占用
转换：失去CPU但仍可运行

### 第二步：逐项分析
**A. 执行P(wait)操作 ❌**
```
P操作在信号量<=0时阻塞
执行态→阻塞态（不是就绪态）
结论：变为阻塞态
```

**B. 申请内存失败 ❌**
```
申请内存失败进程阻塞等待
执行态→阻塞态
结论：变为阻塞态
```

**C. 启动I/O设备 ❌**
```
启动I/O后进程等待完成
执行态→阻塞态
结论：变为阻塞态
```

**D. 被高优先级进程抢占 ✅**
```
高优先级进程抢占CPU
当前进程失去CPU但能继续运行
执行态→就绪态（重新排队等CPU）
结论：变为就绪态
```

### 第三步：确认答案
只有被抢占会变为就绪态 → 答案：**D**

## 易错点分析

### 易错点1：混淆就绪态和阻塞态
❌ 错误：失去CPU就是就绪态
✅ 正确：
- 就绪态：能运行，等CPU
- 阻塞态：不能运行，等事件

### 易错点2：不理解状态转换
| 事件 | 转换 | 原因 |
|------|------|------|
| 被抢占 | 执行→就绪 | 能运行，等CPU |
| P操作阻塞 | 执行→阻塞 | 等信号量 |
| I/O请求 | 执行→阻塞 | 等I/O完成 |
| 内存不足 | 执行→阻塞 | 等内存 |
