# 线程状态转换真题汇总

## 2023年第27题

**元数据**
- 年份：[../../01-真题原卷/2023年408真题#27|2023年第27题](../../01-真题原卷/2023年408真题#27|2023年第27题.md)

- 难度：⭐⭐
- 关联知识：[2023年第27题-线程状态](2023年第27题-线程状态.md)

**题干**

下列由当前线程引起的事件或执行的操作中，可能导致该线程由执行态变为就绪态的是（）。

A. 键盘输入  B. 缺页异常  C. 主动出让 CPU  D. 执行信号量的 wait()操作

**答案**：C

**考点分析**
- 主动出让CPU（yield）：执行态→就绪态
- 键盘输入、wait()阻塞：执行态→阻塞态
- 缺页异常：缺页处理后继续执行

## 解题思路（状态转换分析法）

### 第一步：回顾线程三态模型

```
        ┌──────────┐
        │   就绪   │
        └──────────┘
         ↑    ↓
    调度 │    │ 时间片/yield
         │    ↓
        ┌──────────┐
   ┌───→│   执行   │
   │    └──────────┘
   │         ↓
唤醒 │         │ 等待事件
   │         ↓
   │    ┌──────────┐
   └────│   阻塞   │
        └──────────┘
```

**关键转换**：
- 就绪 → 执行：进程调度
- 执行 → 就绪：时间片用完 / 高优先级抢占 / **主动出让CPU**
- 执行 → 阻塞：等待I/O / 等待资源 / 等待信号量
- 阻塞 → 就绪：I/O完成 / 资源可用 / 信号量释放

### 第二步：明确题目要求

**题目问**：导致线程从**执行态**变为**就绪态**的操作

**注意**：
- 起点：执行态（正在CPU上运行）
- 终点：就绪态（可运行但暂时不运行）
- 不是阻塞态！

### 第三步：逐项分析

**A. 键盘输入 ❌**
```
场景：线程等待用户输入
结果：执行态 → 阻塞态（不是就绪态）
原因：I/O未完成，线程无法继续执行
```

**B. 缺页异常 ❌**
```
场景：访问的页面不在内存
处理：暂停 → 调入页面 → 继续执行
结果：执行态 → 执行态（中间可能短暂阻塞，但最终继续执行）
注意：缺页处理后继续执行，不会变为就绪态
```

**C. 主动出让CPU ✅**
```
场景：调用yield()主动放弃CPU
结果：执行态 → 就绪态
原因：线程本身可运行，只是主动让出时间片
```

**D. 执行信号量的wait()操作 ❌**
```
场景：P(S)操作，S--后S<0
结果：执行态 → 阻塞态（不是就绪态）
原因：资源不足，线程被加入等待队列
```

### 第四步：确认答案

只有 C 选项会导致"执行态 → 就绪态"，答案：**C**

## 易错点分析

### 陷阱1：混淆"就绪态"和"阻塞态"

| 状态 | 含义 | 能否立即运行 |
|------|------|--------------|
| 就绪态 | 万事俱备，只欠CPU | ✅ 是（只要分配CPU就能运行）|
| 阻塞态 | 等待某个事件 | ❌ 否（即使给CPU也无法运行）|

**记忆**：
- 就绪 = "能跑但没轮到"
- 阻塞 = "跑不了，在等事情"

### 陷阱2：误以为键盘输入会进入就绪态

❌ **错误理解**：键盘输入只是暂停一下，应该是就绪态
✅ **正确理解**：等待I/O期间线程无法运行，是阻塞态

**判断方法**：问自己"现在给它CPU，它能跑吗？"
- 能跑 → 就绪态
- 不能跑（需要等某个事件） → 阻塞态

### 陷阱3：缺页异常的特殊性

缺页异常容易误判为阻塞，但实际上：
```
缺页异常处理流程：
1. 暂停当前指令
2. 调入所需页面（可能阻塞）
3. 继续执行原指令

结果：从哪里来回哪里去（执行态 → 执行态）
```

**关键**：缺页是异常处理，不是状态转换

### 陷阱4：wait()操作的易混淆

| 操作 | 结果状态 | 原因 |
|------|----------|------|
| yield() 主动出让 | 就绪态 | 线程仍可运行 |
| wait() 信号量操作 | 阻塞态 | 资源不足，需等待 |
| sleep() 睡眠 | 阻塞态 | 等待时间到 |
| 等待I/O | 阻塞态 | 等待I/O完成 |

**记忆**：只有yield()是就绪态，其他等待操作都是阻塞态

## 知识点总结

### 执行态的三种去向

```
执行态可以转到：
1. 就绪态：时间片用完、高优先级抢占、yield()
2. 阻塞态：等待I/O、等待资源、wait()、sleep()
3. 终止态：进程结束
```

**快速判断**：
- 主动/被动放弃CPU但能继续跑 → 就绪态
- 需要等待某个事件才能跑 → 阻塞态

### 状态转换口诀

```
就绪执行靠调度，
执行阻塞等事件，
阻塞就绪事件完，
执行就绪让CPU。
```
