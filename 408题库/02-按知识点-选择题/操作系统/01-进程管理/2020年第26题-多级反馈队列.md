# 多级反馈队列真题汇总

## 2020年第26题

**元数据**
- 年份：2020年第26题
- 科目：#操作系统
- 知识点：#操作系统/进程管理/多级反馈队列
- 难度：⭐⭐

**题干**

下列与进程调度有关的因素中，在设计多级反馈队列调度算法时需要考虑的是（）。

I. 就绪队列的数量  II. 就绪队列的优先级  III. 各就绪队列的调度算法  IV. 进程在就绪队列间的迁移条件

A. 仅I、II  B. 仅 III、IV  C. 仅Ⅱ、Ⅲ、Ⅳ  D. I、II、III 和 IV

**答案**：D

**考点分析**
- 多级反馈队列需要考虑所有因素

---


## 解题思路

### 第一步：理解多级反馈队列设计要素
多级反馈队列是复杂调度算法，需要设计多个方面

### 第二步：逐项分析
**I. 就绪队列的数量 ✅**
```
需要决定设置几个队列
队列太少：灵活性不足
队列太多：管理开销大
常见：2-5个队列
结论：需要考虑
```

**II. 就绪队列的优先级 ✅**
```
各队列优先级如何设置
通常：队列1优先级最高，依次递减
高优先级队列先调度
结论：需要考虑
```

**III. 各就绪队列的调度算法 ✅**
```
每个队列内部用什么算法
常见：高优先级队列用RR，低优先级队列用FCFS
或所有队列都用RR但时间片不同
结论：需要考虑
```

**IV. 进程在队列间的迁移条件 ✅**
```
进程何时从一个队列迁移到另一个队列
降级：用完时间片降到下一级
升级：等待时间过长升级（防饥饿）
结论：需要考虑
```

### 第三步：确认答案
I、II、III、IV都需要考虑 → 答案：**D**

## 易错点分析

### 易错点1：认为只需考虑部分因素
❌ 错误：多级反馈队列只需设置优先级和算法
✅ 正确：需要全面设计，包括队列数、优先级、算法、迁移

### 易错点2：不理解多级反馈队列的复杂性
| 设计要素 | 作用 | 示例 |
|---------|------|------|
| 队列数量 | 决定层次 | 3个队列 |
| 优先级 | 调度顺序 | 队列1>队列2>队列3 |
| 内部算法 | 队列内调度 | RR或FCFS |
| 迁移条件 | 队列间移动 | 用完时间片降级 |

**多级反馈队列特点**：
- 综合多种调度算法优点
- 动态调整进程优先级
- 兼顾短作业和长作业
- 设计复杂，需考虑多方面

**典型设计**：
```
队列1：RR(q=10ms)，优先级高
队列2：RR(q=20ms)，优先级中
队列3：FCFS，优先级低
迁移：用完时间片降级
```
