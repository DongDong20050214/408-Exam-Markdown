# 进程终止真题汇总

## 2024年第24题

**元数据**
- 年份：2024年第24题
- 科目：#操作系统
- 知识点：#操作系统/进程管理/进程终止
- 难度：⭐⭐

**题干**

下列选项中，操作系统在终止进程时不一定执行的是（）。

A. 终止子进程  B. 回收进程占用的设备  C. 释放进程控制块  D. 回收为进程分配的内存

**答案**：A

**考点分析**
- A不一定：子进程可能被init进程收养

---


## 解题思路

### 第一步：理解进程终止操作
进程终止：回收资源、释放PCB

### 第二步：逐项分析
**A. 终止子进程 ❌不一定**
```
不同系统策略不同：
- 级联终止：终止所有子进程
- 孤儿进程：子进程被init收养继续运行
Unix/Linux：子进程变为孤儿进程
结论：不一定终止
```

**B. 回收进程占用的设备 ✅一定**
```
进程终止必须释放占用的设备
否则设备被永久占用
其他进程无法使用
结论：一定执行
```

**C. 释放进程控制块 ✅一定**
```
PCB是进程存在的标志
进程终止必须释放PCB
回收到PCB池
结论：一定执行
```

**D. 回收为进程分配的内存 ✅一定**
```
进程终止必须回收内存
释放地址空间
否则造成内存泄漏
结论：一定执行
```

### 第三步：确认答案
只有A不一定 → 答案：**A**

## 易错点分析

### 易错点1：认为子进程一定被终止
❌ 错误：父进程终止，子进程必须终止
✅ 正确：子进程可能被init收养

### 易错点2：不理解孤儿进程
| 概念 | 含义 | 处理方式 |
|------|------|---------|
| 孤儿进程 | 父进程先终止 | init收养 |
| 僵尸进程 | 子进程终止未回收 | 父进程wait回收 |

**进程终止必做操作**：
1. 回收所有资源（内存、设备等）
2. 释放PCB
3. 通知父进程

**子进程处理**：
- 级联终止：终止所有子进程
- 孤儿收养：子进程被init收养（不终止）
