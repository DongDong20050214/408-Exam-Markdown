## 解题思路

### 第一步：理解进程同步与互斥
互斥：多进程不能同时访问临界资源（信号量初值=1）
同步：进程间执行顺序的协调（信号量初值=0）

### 第二步：分析P/V操作
**互斥问题**：
```
信号量mutex初值=1
临界区前：P(mutex)
临界区后：V(mutex)
```

**同步问题**：
```
信号量S初值=0
前驱操作后：V(S)
后继操作前：P(S)
```

### 第三步：典型场景分析
**生产者-消费者问题**：
```
信号量empty=n（空缓冲区数）
信号量full=0（满缓冲区数）
信号量mutex=1（互斥访问）

生产者：
P(empty)    // 申请空缓冲区
P(mutex)    // 进入临界区
生产
V(mutex)    // 离开临界区
V(full)     // 增加满缓冲区

消费者：
P(full)     // 申请满缓冲区
P(mutex)    // 进入临界区
消费
V(mutex)    // 离开临界区
V(empty)    // 增加空缓冲区
```

## 易错点分析

### 易错点1：混淆互斥和同步信号量
❌ 错误：所有信号量初值都设为1
✅ 正确：
- 互斥信号量：初值=1（资源可用）
- 同步信号量：初值=0（等待事件）

### 易错点2：P/V操作顺序错误
❌ 错误：先P(mutex)再P(empty)
✅ 正确：先P(empty)再P(mutex)（避免死锁）

| 操作 | 互斥信号量 | 同步信号量 |
|------|----------|----------|
| 初值 | 1 | 0或资源数 |
| 用途 | 保护临界区 | 协调执行顺序 |
| 位置 | 临界区前后 | 前驱后继之间 |

**P/V操作原则**：
- P操作：申请资源，信号量-1
- V操作：释放资源，信号量+1
- 互斥：P/V成对出现
- 同步：前驱V，后继P
