**题干**

下列关于父进程与子进程的叙述中，错误的是（）。

A. 父进程与子进程可以并发执行
B. 父进程与子进程共享虚拟地址空间
C. 父进程与子进程有不同的进程控制块
D. 父进程与子进程不能同时使用同一临界资源

**答案**：B

**考点分析**
- B错误：父子进程有独立的虚拟地址空间

---


## 解题思路

### 第一步：理解父子进程关系
子进程：父进程通过fork()创建
关系：独立的进程，各有资源

### 第二步：逐项分析
**A. 可以并发执行 ✅**
```
父子进程是独立进程
可以并发执行
父进程不必等待子进程
结论：正确
```

**B. 共享虚拟地址空间 ❌**
```
每个进程有独立的虚拟地址空间
子进程创建时复制父进程地址空间
之后各自独立，修改互不影响
结论：错误
```

**C. 有不同的PCB ✅**
```
每个进程有独立的PCB
PCB包含进程ID、状态等信息
父子进程ID不同
结论：正确
```

**D. 不能同时使用同一临界资源 ✅**
```
临界资源同一时刻只能被一个进程使用
父子进程也要遵守互斥规则
需要通过互斥机制协调
结论：正确
```

### 第三步：确认答案
B错误 → 答案：**B**

## 易错点分析

### 易错点1：混淆进程和线程
❌ 错误：父子进程共享地址空间
✅ 正确：
- 进程：独立地址空间
- 线程：共享地址空间

### 易错点2：不理解fork机制
❌ 错误：子进程和父进程共享内存
✅ 正确：fork时复制地址空间，写时复制(COW)

| 特性 | 父子进程 | 同进程线程 |
|------|---------|-----------|
| 地址空间 | 独立 | 共享 |
| PCB/TCB | 各有PCB | 各有TCB |
| 数据段 | 独立 | 共享 |
| 代码段 | 独立副本 | 共享 |
| 并发执行 | ✅ | ✅ |

**fork()机制**：
```c
pid = fork();
// 创建子进程
// 子进程复制父进程地址空间
// 父子进程从fork()后继续执行
// pid: 父进程中为子进程ID，子进程中为0
```

**写时复制(COW)**：
- fork时不立即复制内存
- 父子进程共享只读页面
- 写入时才复制该页面
