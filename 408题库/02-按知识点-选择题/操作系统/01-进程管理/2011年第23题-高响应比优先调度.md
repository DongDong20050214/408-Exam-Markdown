**题干**

下列选项中，满足短任务优先且不会发生饥饿现象的调度算法是（）。A. 先来先服务  B. 高响应比优先  C. 时间片轮转  D. 非抢占式短任务优先

**答案**：B

**考点分析**
- B：高响应比优先兼顾短任务和等待时间

---


## 解题思路

### 第一步：理解题目要求
找出既满足短任务优先，又不会饥饿的算法

### 第二步：逐项分析
**A. 先来先服务FCFS ❌**
```
按到达顺序执行
不考虑任务长短
不满足短任务优先
结论：不符合
```

**B. 高响应比优先HRRN ✅**
```
响应比 = (等待时间 + 服务时间) / 服务时间
短任务：服务时间小，响应比高，优先执行
长任务：等待时间长，响应比也会变高，不会饥饿
满足两个条件
结论：符合
```

**C. 时间片轮转RR ❌**
```
所有进程轮流执行
不区分任务长短
不满足短任务优先
结论：不符合
```

**D. 非抢占式短任务优先SJF ❌**
```
短任务优先：满足✅
但长任务可能一直等待：会饥饿❌
不满足"不会饥饿"
结论：不符合
```

### 第三步：确认答案
只有HRRN满足两个条件 → 答案：**B**

## 易错点分析

### 易错点1：认为SJF不会饥饿
❌ 错误：SJF让短任务先执行，效率高，不会饥饿
✅ 正确：SJF会导致长任务饥饿（短任务不断到来）

### 易错点2：不理解HRRN如何避免饥饿
| 算法 | 短任务优先 | 避免饥饿 | 机制 |
|------|----------|----------|------|
| SJF | ✅ | ❌ | 只看服务时间 |
| HRRN | ✅ | ✅ | 等待时间+服务时间 |
| FCFS | ❌ | ✅ | 先来先服务 |
| RR | ❌ | ✅ | 时间片轮转 |

**HRRN原理**：
- 初始：短任务响应比高 → 优先执行（满足短任务优先）
- 长期：长任务等待时间增加 → 响应比升高 → 最终被调度（避免饥饿）
- 公式：Rp = (W + S) / S = 1 + W/S
  - W等待时间越长，Rp越大
