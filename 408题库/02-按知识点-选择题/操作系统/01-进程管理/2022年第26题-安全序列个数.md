**题干**

系统中有三个进程 P0、P1、P2 及三类资源 A、B、C。则此时系统中存在的安全序列的个数为（）。

A. 1  B. 2  C. 3  D. 4

**答案**：需计算

**考点分析**
- 银行家算法判断安全序列

---


## 解题思路

### 第一步：理解安全序列个数
安全序列可能有多个
需要枚举所有可能的执行顺序

### 第二步：银行家算法枚举（需要数据）
```
假设数据：
Available = (某值)
各进程Need已知

枚举所有3!= 6种排列：
1. P0→P1→P2
2. P0→P2→P1
3. P1→P0→P2
4. P1→P2→P0
5. P2→P0→P1
6. P2→P1→P0

逐一检查每种序列是否安全
```

### 第三步：判断安全序列
```
对每种序列：
1. 模拟分配过程
2. 检查每步Available是否满足Need
3. 满足所有步骤→安全序列
4. 统计安全序列个数
```

### 第四步：确认答案
根据具体数据统计安全序列个数

## 易错点分析

### 易错点1：认为只有一个安全序列
❌ 错误：系统安全状态只有唯一安全序列
✅ 正确：可能有多个安全序列

### 易错点2：枚举不完整
❌ 错误：找到一个安全序列就停止
✅ 正确：需要枚举所有排列

| 进程数 | 可能排列 | 需要检查 |
|--------|---------|---------|
| 2 | 2! = 2 | 全部 |
| 3 | 3! = 6 | 全部 |
| 4 | 4! = 24 | 全部 |

**安全序列判断方法**：
```
foreach 排列 in 所有排列:
    Available_temp = Available
    is_safe = true
    
    foreach 进程 in 排列:
        if Need[进程] > Available_temp:
            is_safe = false
            break
        Available_temp += Allocation[进程]
    
    if is_safe:
        安全序列数 += 1
```

**关键点**：
- 一个状态可能有多个安全序列
- 需要完整枚举
- 安全序列个数 ≤ n!
