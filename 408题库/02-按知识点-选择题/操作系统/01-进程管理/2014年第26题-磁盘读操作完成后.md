**题干**

一个进程的读磁盘操作完成后，操作系统针对该进程必做的是（）。A. 修改进程状态为就绪态  B. 降低进程优先级  C. 为进程分配用户内存空间  D. 增加进程的时间片大小

**答案**：A

**考点分析**
- IO完成后进程从阻塞态变为就绪态

---


## 解题思路

### 第一步：理解进程状态转换
进程读磁盘时：执行态→阻塞态（等待I/O）
I/O完成后：阻塞态→就绪态（等待CPU）

### 第二步：逐项分析
**A. 修改进程状态为就绪态 ✅**
```
I/O完成后进程从阻塞态变就绪态
这是必须操作，否则进程永远等待
OS必须将进程移到就绪队列
结论：必做
```

**B. 降低进程优先级 ❌**
```
I/O完成通常应提高优先级
I/O密集型进程优先级高
不是必做操作
结论：不是必做
```

**C. 分配用户内存空间 ❌**
```
内存在进程创建时分配
I/O完成不需要分配内存
结论：不是必做
```

**D. 增加时间片大小 ❌**
```
时间片大小是调度策略
不因单次I/O改变
结论：不是必做
```

### 第三步：确认答案
只有修改状态为就绪态是必做 → 答案：**A**

## 易错点分析

### 易错点1：认为I/O完成直接进入执行态
❌ 错误：I/O完成后进程继续执行
✅ 正确：I/O完成后进入就绪态，等待调度

### 易错点2：混淆进程状态转换
| 转换 | 触发条件 | 是否经过就绪态 |
|------|---------|---------------|
| 执行→阻塞 | 等待I/O | - |
| 阻塞→就绪 | I/O完成 | ✅进入就绪 |
| 就绪→执行 | 被调度 | - |
| 执行→就绪 | 时间片用完/被抢占 | - |
