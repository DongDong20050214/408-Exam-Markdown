# 处理机调度时机真题汇总

## 2012年第30题

**元数据**
- 年份：[[../../01-真题原卷/2012年408真题#30|2012年第30题]]
- 科目：#操作系统
- 知识点：#操作系统/进程管理/调度时机
- 难度：⭐⭐⭐
- 关联知识：[[知识库/操作系统/进程调度]]

**题干**

若某单处理器多进程系统中有多个就绪态进程，则下列关于处理机调度的叙述中，错误的是（）。

A. 在进程结束时能进行处理机调度  
B. 创建新进程后能进行处理机调度  
C. 在进程处于临界区时不能进行处理机调度  
D. 在进程等待I/O时能进行处理机调度

**答案**：C

**考点分析**
- A正确：进程结束释放CPU
- B正确：新进程可能抢占
- C错误：临界区可以调度（取决于调度策略）
- D正确：等待I/O时释放CPU

---


## 解题思路

### 第一步：理解调度时机
可以调度：进程结束、阻塞、时间片用完、新进程创建
不能调度：处理原子操作、关中断期间

### 第二步：逐项分析
**A. 进程结束时能调度 ✅**
```
进程结束释放CPU
需要选择新进程执行
必须进行调度
结论：正确
```

**B. 创建新进程后能调度 ✅**
```
新进程可能优先级更高
可以抢占当前进程
能够进行调度
结论：正确
```

**C. 临界区不能调度 ❌**
```
临界区是用户代码区域
可以被时间片中断
可以进行抢占式调度
不能调度是关中断期间，不是临界区
结论：错误
```

**D. 等待I/O时能调度 ✅**
```
进程阻塞等待I/O
主动释放CPU
必须调度其他进程
结论：正确
```

### 第三步：确认答案
C错误 → 答案：**C**

## 易错点分析

### 易错点1：混淆临界区和关中断
❌ 错误：临界区不能被打断，所以不能调度
✅ 正确：
- 临界区：用户代码，可以被抢占调度
- 关中断区：内核代码，不能调度

### 易错点2：不理解不可调度时机
| 情况 | 能否调度 | 原因 |
|------|----------|------|
| 进程结束 | ✅ | 必须选新进程 |
| 等待I/O | ✅ | 释放CPU |
| 临界区 | ✅ | 可抢占 |
| 关中断 | ❌ | 原子操作 |
| 内核态 | 看情况 | 可抢占内核可以 |
