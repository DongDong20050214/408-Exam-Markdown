# 进程间操作互斥真题汇总

## 2016年第30题

**元数据**
- 年份：[2016年第30题](../../../01-真题原卷/2016年408真题.md#30)
- 科目：#操作系统
- 知识点：#操作系统/进程管理/进程同步 #临界资源
- 难度：⭐⭐
- 关联知识：[知识库/操作系统/进程同步](知识库/操作系统/进程同步.md)

**题干**

进程P₁和P₂均包含并发执行的线程，部分伪代码描述如下所示。

```c
//进程P₁              //进程P₂
int x = 0;            int x = 0;
Thread1() {           Thread3() {
  int a;                int a;
  a=1;                  a=x;
  x += 1;               x += 3;
}                     }
Thread2() {           Thread4() {
  int a;                int b;
  a=2;                  b=x;
  x += 2;               x += 4;
}                     }
```

下列选项中，需要互斥执行的操作是（）。

A. a=1 与 a=2  
B. a=x 与 b=x  
C. x+=1 与 x+=2  
D. x+=3 与 x+=3

**答案**：C

**考点分析**
- A选项：不同进程的局部变量a，地址空间独立，无需互斥
- B选项：读操作，无需互斥
- C选项：同一进程P₁内的共享变量x的写操作，需要互斥
- D选项：不同进程P₁和P₂的x各自独立，无需互斥
- 关键：同一进程内的共享变量写操作需要互斥

---


## 解题思路

### 第一步：理解进程和线程的变量作用域
进程间：各有独立地址空间，变量不共享
线程间：同一进程的线程共享全局变量

### 第二步：分析变量作用域
```
P1进程：
  全局变量 x（Thread1和Thread2共享）
  Thread1局部变量 a
  Thread2局部变量 a

P2进程：
  全局变量 x（Thread3和Thread4共享）
  Thread3局部变量 a
  Thread4局部变量 b

P1的x和P2的x是不同变量（不同地址空间）
```

### 第三步：逐项分析
**A. a=1 与 a=2 ❌**
```
Thread1的a和Thread2的a都是局部变量
存储在各自线程栈中
不共享，无需互斥
结论：不需要互斥
```

**B. a=x 与 b=x ✅需要但题目问的是写操作**
```
Thread3读x，Thread4读x
都是读操作，可以并发
读-读不需要互斥
结论：不需要互斥
```

**C. x+=1 与 x+=2 ✅**
```
Thread1和Thread2都在P1进程内
共享P1的全局变量x
x+=1和x+=2都是"读-改-写"操作
非原子操作，需要互斥
结论：需要互斥
```

**D. x+=3 与 x+=4 ❌**
```
Thread3在P2进程，操作P2的x
Thread4在P2进程，操作P2的x
它们确实需要互斥，但C选项也需要
题目选最明确的C
```

等等，重新分析D：
```
D选项写的是"x+=3 与 x+=3"（两个相同）
应该是Thread3的x+=3执行两次？
这不符合逻辑，可能是题目笔误
如果是P1和P2的x操作，不需要互斥（不同变量）
```

### 第四步：确认答案
同一进程内线程共享变量的写操作需要互斥 → 答案：**C**

## 易错点分析

### 易错点1：认为不同进程的同名变量需要互斥
❌ 错误：P1的x和P2的x需要互斥
✅ 正确：不同进程的x是独立变量，不需要互斥

### 易错点2：混淆局部变量和全局变量
| 变量类型 | 作用域 | 是否共享 | 需要互斥 |
|---------|--------|---------|---------|
| 局部变量a | 线程栈 | ❌ | ❌ |
| 全局变量x | 进程空间 | ✅同进程线程共享 | ✅写操作需要 |
| 不同进程的x | 各自空间 | ❌ | ❌ |

**互斥条件判断**：
1. 是否访问同一变量？
2. 是否至少有一个写操作？
3. 满足以上两点 → 需要互斥

**本题关键**：
- Thread1和Thread2在同一进程P1
- 都对P1的全局变量x进行写操作
- x+=操作非原子，需要互斥
