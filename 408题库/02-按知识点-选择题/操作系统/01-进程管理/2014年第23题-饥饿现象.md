**题干**

下列调度算法中, 不可能导致饥饿现象的是 ( )。

A. 时间片轮转  B. 静态优先数调度  C. 非抢占式短作业优先  D. 抢占式短作业优先

**答案**：A

**考点分析**
- 时间片轮转保证公平

---


## 解题思路

### 第一步：理解饥饿现象
饥饿：进程长期得不到资源，无限期等待

### 第二步：逐项分析
**A. 时间片轮转 ✅不会饥饿**
```
所有进程轮流执行
每个进程定期获得CPU
保证公平性
结论：不会饥饿
```

**B. 静态优先数调度 ❌会饥饿**
```
低优先级进程可能永远得不到CPU
高优先级进程不断到达
低优先级饥饿
结论：可能饥饿
```

**C. 非抢占式短作业优先 ❌会饥饿**
```
短作业不断到达
长作业一直等待
长作业饥饿
结论：可能饥饿
```

**D. 抢占式短作业优先 ❌会饥饿**
```
更容易饥饿
短作业可抢占长作业
长作业更难执行
结论：可能饥饿
```

### 第三步：确认答案
只有时间片轮转不会饥饿 → 答案：**A**

## 易错点分析

### 易错点1：认为FCFS不会饥饿
❌ 错误：FCFS也会饥饿
✅ 正确：FCFS确实不饥饿，但题目没有这个选项

### 易错点2：混淆调度算法
| 算法 | 会饥饿 | 原因 |
|------|--------|------|
| RR | ❌ | 轮流执行 |
| FCFS | ❌ | 先到先服务 |
| 优先级 | ✅ | 低优先级饥饿 |
| SJF | ✅ | 长作业饥饿 |
| HRRN | ❌ | 等待越久优先级越高 |
