# 死锁真题汇总

## 2019年第30题

**元数据**
- 年份：2019年第30题
- 科目：#操作系统
- 知识点：#操作系统/进程管理/死锁
- 难度：⭐⭐

**题干**

下列关于死锁的叙述中，正确的是（）。

I. 可以通过剥夺进程资源解除死锁
II. 死锁的预防方法能确保系统不发生死锁
III. 银行家算法可以判断系统是否处于死锁状态
IV. 当系统出现死锁时，必然有两个或两个以上的进程处于阻塞态

**答案**：需看选项

**考点分析**
- I、II、IV正确
- III错误：银行家算法用于死锁避免，不是判断

## 解题思路

### 第一步：理解死锁处理方法
预防、避免、检测、解除

### 第二步：逐项分析
**I. 剥夺资源可解除死锁 ✅**
```
死锁解除方法之一：资源剥夺
强制回收某进程持有的资源
分配给其他进程，打破死锁
结论：正确
```

**II. 死锁预防确保不死锁 ✅**
```
死锁预防：破坏四个必要条件之一
如资源一次性分配（破坏占有并等待）
资源有序分配（破坏循环等待）
确保不会发生死锁
结论：正确
```

**III. 银行家算法判断死锁状态 ❌**
```
银行家算法：死锁避免
通过安全性检查避免进入不安全状态
不是判断是否已死锁
判断死锁用资源分配图或死锁检测算法
结论：错误
```

**IV. 死锁时至少两个进程阻塞 ✅**
```
死锁：循环等待资源
至少需要2个进程形成环路
都在等待对方资源，处于阻塞态
结论：正确
```

### 第三步：确认答案
I、II、IV正确，III错误 → 答案：**I、II、IV正确**

## 易错点分析

### 易错点1：混淆死锁避免和死锁检测
❌ 错误：银行家算法判断死锁
✅ 正确：
- 银行家算法：避免死锁（事前）
- 死锁检测：判断死锁（事后）

### 易错点2：认为单个进程能死锁
❌ 错误：一个进程等待自己的资源也算死锁
✅ 正确：死锁至少需要2个进程形成环路

| 死锁处理 | 时机 | 方法 | 示例 |
|---------|------|------|------|
| 预防 | 事前 | 破坏必要条件 | 资源有序分配 |
| 避免 | 事前 | 安全性检查 | 银行家算法 |
| 检测 | 事后 | 资源分配图 | 查找环路 |
| 解除 | 事后 | 剥夺/终止 | 回收资源 |

**关键区别**：
- 银行家算法：判断分配后是否安全
- 死锁检测：判断当前是否死锁
- 前者预防，后者诊断
