# 磁盘存取时间真题汇总

## 2015年第20题

**元数据**
- 年份：[[../../01-真题原卷/2015年408真题#20|2015年第20题]]
- 科目：#操作系统
- 知识点：#操作系统/文件管理/磁盘
- 难度：⭐⭐⭐
- 关联知识：[[2018年第31题-磁盘管理]]

**题干**

若磁盘转速为 7200 rpm, 平均寻道时间为 8 ms, 每个磁道包含 1000 个扇区, 则访问一个扇区的平均存取时间大约是 ( )。

A. 8.1ms  B. 12.2ms  C. 16.3ms  D. 20.5ms

**答案**：B

**考点分析**
- 存取时间 = 寻道 + 旋转延迟 + 传输
- 8ms + 4.17ms + 0.008ms ≈ 12.2ms

---


## 解题思路

### 第一步：理解磁盘存取时间组成
```
磁盘存取时间 = 寻道时间 + 旋转延迟 + 传输时间
```

### 第二步：计算各部分时间

**寻道时间（已给出）**
```
平均寻道时间 = 8ms
```

**旋转延迟时间**
```
转速：7200 rpm (转/分钟)
每转时间 = 60秒 / 7200转 = 60000ms / 7200 = 8.33ms
平均旋转延迟 = 每转时间 / 2 = 8.33ms / 2 ≈ 4.17ms
```

**传输时间**
```
每个磁道1000个扇区
传输1个扇区的时间 = 每转时间 / 1000
                 = 8.33ms / 1000
                 = 0.00833ms
                 ≈ 0.008ms（可忽略）
```

### 第三步：计算总时间
```
平均存取时间 = 8ms + 4.17ms + 0.008ms
            ≈ 12.2ms
```

### 第四步：逐项分析

**A. 8.1ms ❌**
```
详细分析：仅包含寻道时间，漏掉旋转延迟
判断依据：8ms + 0.1ms，只能是寻道+极小误差
结论：错误
```

**B. 12.2ms ✅**
```
详细分析：完整计算三项时间之和
判断依据：
- 寻道时间：8ms
- 旋转延迟：4.17ms
- 传输时间：0.008ms（忽略）
- 总计：≈12.2ms
结论：正确
```

**C. 16.3ms ❌**
```
详细分析：计算错误，可能用了完整旋转时间而非平均值
判断依据：8 + 8.33 ≈ 16.3，但旋转延迟应取平均值
结论：错误
```

**D. 20.5ms ❌**
```
详细分析：数值过大，计算有误
判断依据：超出合理范围
结论：错误
```

### 第五步：确认答案
寻道8ms + 旋转延迟4.17ms ≈ 12.2ms → 答案：**B**

## 易错点分析

### 易错点1：旋转延迟用完整旋转时间
❌ 错误理解：旋转延迟 = 每转时间 = 8.33ms
✅ 正确理解：平均旋转延迟 = 每转时间 / 2 = 4.17ms。目标扇区在磁道上随机分布，最好情况刚好在磁头下（延迟0），最坏情况刚错过需要转一整圈（8.33ms），平均就是半圈。这是概率期望，千万别用完整旋转时间否则结果直接翻倍

### 易错点2：rpm单位换算错误

| 概念 | 计算公式 | 结果 |
|------|---------|------|
| 转速 | 7200 rpm | 每分钟7200转 |
| 每转时间 | 60s / 7200 = 60000ms / 7200 | 8.33ms |
| 平均旋转延迟 | 8.33ms / 2 | 4.17ms |
| 传输1扇区 | 8.33ms / 1000 | 0.008ms |

**磁盘存取时间公式**：
```
T总 = T寻道 + T旋转 + T传输

其中：
- T寻道：平均寻道时间（题目给出）
- T旋转：平均旋转延迟 = (60/转速rpm) / 2 秒
- T传输：传输时间 = (60/转速rpm) / 每磁道扇区数 秒
```

**关键点**：
- 平均旋转延迟取半圈：因为目标扇区随机分布，平均需要转半圈
- 传输时间通常很小：单扇区传输时间 << 寻道和旋转时间
- rpm转换：7200 rpm = 120 rps (转/秒) = 每转8.33ms
