**题干**

某文件系统的簇和磁盘扇区大小分别为 1KB 和 512B。若一个文件的大小为 1026B，则系统分配给该文件的磁盘空间大小是（）。A. 1026B  B. 1536B  C. 1538B  D. 2048B

**答案**：D

**考点分析**
- 按簇分配，需2个簇=2KB=2048B

---


## 解题思路

### 第一步：理解簇分配机制
文件系统以簇为最小分配单位，不足一簇也要分配整个簇。

### 第二步：计算需要的簇数
```
文件大小：1026B
簇大小：1KB = 1024B

第一个簇：1024B（满）
剩余：1026 - 1024 = 2B

第二个簇：需要分配完整的1024B来存储剩余的2B
```

### 第三步：逐项分析
**A. 1026B ❌**
```
详细分析：这是文件实际大小，不是分配空间
判断依据：系统按簇分配，不能按字节精确分配
结论：错误
```

**B. 1536B ❌**
```
详细分析：1536B = 1.5KB
判断依据：簇大小1KB，不存在0.5个簇的分配
结论：错误，计算有误
```

**C. 1538B ❌**
```
详细分析：1538 = 1026 + 512
判断依据：可能误认为按扇区分配，但题目明确按簇分配
结论：错误
```

**D. 2048B ✅**
```
详细分析：2048B = 2KB = 2个簇
判断依据：
- 文件1026B需要2个簇
- 1个簇1024B不够存储1026B
- 必须分配2个簇
- 2×1024B = 2048B
结论：正确
```

### 第四步：确认答案
需要2个簇，分配空间2048B → 答案：**D**

## 易错点分析

### 易错点1：混淆文件实际大小和分配空间
❌ 错误理解：文件1026B，系统就分配1026B
✅ 正确理解：文件系统以簇为最小分配单位（本题1KB），不足一簇按一簇分配。1026B需要跨越2个簇边界（第1个簇装1024B，第2个簇装剩余2B），必须分配2个完整簇共2048B。浪费的1022B（2048-1026）称为内部碎片，是簇分配机制的代价。扇区512B只是磁盘物理单位，不影响文件系统分配逻辑

### 易错点2：按扇区大小计算
❌ 错误计算：1026B需要3个扇区(512B×3=1536B)
✅ 正确计算：题目明确按簇分配，扇区大小是干扰项

| 概念 | 大小 | 作用 |
|------|------|------|
| 扇区 | 512B | 磁盘物理单位 |
| 簇 | 1KB | 文件系统分配单位 |
| 文件大小 | 1026B | 实际数据 |
| 分配空间 | 2048B | 占用磁盘空间 |

**关键点**：
- 簇 = 分配单位（可包含多个扇区）
- 内部碎片 = 分配空间 - 实际大小 = 2048 - 1026 = 1022B
- 扇区仅是物理单位，不影响分配计算
