# 文件权限位数真题汇总

## 2017年第30题

**元数据**
- 年份：[../../01-真题原卷/2017年408真题#30|2017年第30题](../../01-真题原卷/2017年408真题#30|2017年第30题.md)
- 科目：#操作系统
- 知识点：#操作系统/文件管理/访问权限
- 难度：⭐⭐⭐
- 关联知识：[知识库/操作系统/文件权限](知识库/操作系统/文件权限.md)

**题干**

用户类别分为4类，访问权限分为5种。若文件控制块中用二进制位串表示文件权限，为表示不同类别用户对一个文件的访问权限，则描述文件权限的位数至少应为（）。

A. 5  
B. 9  
C. 12  
D. 20

**答案**：D

**考点分析**
- 4类用户 × 5种权限 = 20位
- 每一位表示一个（用户类别，权限类型）组合
- 位为1表示有该权限，为0表示无权限

---


## 解题思路

### 第一步：理解权限表示方法
题目要求用二进制位串表示权限，每一位代表一个（用户类别，权限类型）的组合。

### 第二步：计算所需位数
```
用户类别：4类（如：所有者、同组用户、其他用户、特殊用户）
访问权限：5种（如：读、写、执行、删除、修改）

每个用户类别对每种权限都需要单独标记（有/无）
总位数 = 用户类别数 × 权限种类数
      = 4 × 5
      = 20位
```

### 第三步：逐项分析
**A. 5 ❌**
```
详细分析：5位只能表示5种权限或5个对象
判断依据：
- 若5位表示5种权限：无法区分4类用户
- 若5位表示4类用户+1位其他：无法表示5种权限
- 信息量不足：log2(4×5) ≈ 4.3位是理论最小值，但需要20个独立标志
结论：错误，位数不够
```

**B. 9 ❌**
```
详细分析：9位是Unix标准权限位数(3×3)
判断依据：
- Unix系统：3类用户(owner/group/others) × 3种权限(rwx) = 9位
- 本题：4类用户 × 5种权限 = 20位
- 9 < 20：无法表示所有组合
结论：错误，位数不够
```

**C. 12 ❌**
```
详细分析：12 = 3×4 或 2.4×5，无法对应本题
判断依据：
- 4类用户×3种权限 = 12位（权限不足）
- 3类用户×4种权限 = 12位（用户类别不足）
- 都无法满足4类用户和5种权限
结论：错误，位数不够
```

**D. 20 ✅**
```
详细分析：每个（用户，权限）组合占1位
判断依据：
- 用户1：[权限1][权限2][权限3][权限4][权限5] 5位
- 用户2：[权限1][权限2][权限3][权限4][权限5] 5位
- 用户3：[权限1][权限2][权限3][权限4][权限5] 5位
- 用户4：[权限1][权限2][权限3][权限4][权限5] 5位
- 总计：4×5 = 20位
- 每位含义：1表示有该权限，0表示无权限
结论：正确
```

### 第四步：确认答案
4类用户×5种权限=20位 → 答案：**D**

## 易错点分析

### 易错点1：使用编码压缩而非位标志
❌ 错误理解：用3位编码表示5种权限(2³=8>5)，4位编码表示4类用户(2⁴=16>4)，共7位足够
✅ 正确理解：题目要求"二进制位串表示权限"，指的是位标志（bit flag）而非编码。每个（用户类别，权限类型）组合需要独立的1位表示有无该权限（1或0）。4类用户×5种权限=20个独立组合，需要20位。编码虽然节省空间但需要解析，位标志直接判断更高效

### 易错点2：混淆Unix权限系统
| 系统 | 用户类别 | 权限种类 | 位数 |
|------|---------|---------|------|
| Unix | 3类(owner/group/others) | 3种(rwx) | 9位 |
| 本题 | 4类 | 5种 | 20位 |

**权限位串示例**：
```
假设4类用户：所有者、同组、其他、系统
假设5种权限：读(R)、写(W)、执行(X)、删除(D)、修改(M)

20位表示：
所有者：[R][W][X][D][M]  = 11111 (全部权限)
同  组：[R][W][X][D][M]  = 11000 (读写执行)
其  他：[R][W][X][D][M]  = 10000 (只读)
系  统：[R][W][X][D][M]  = 11111 (全部权限)

完整位串：11111 11000 10000 11111 (20位)
```

**关键点**：
- 位标志vs编码：位标志每个属性独立，编码需要解析
- 4×5=20：笛卡尔积，每个组合需要1位
- 题目明确"二进制位串"：意味着用位标志而非编码
