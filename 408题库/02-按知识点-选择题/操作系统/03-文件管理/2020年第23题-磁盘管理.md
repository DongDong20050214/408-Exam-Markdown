# 2020年第23题

**元数据**
- 年份：[[../../01-真题原卷/2020年408真题#23|2020年第23题]]
- 科目：#操作系统
- 知识点：#操作系统/文件管理/打开文件表
- 难度：⭐⭐⭐

**题干**

若多个进程共享同一个文件 F，则下列叙述中，正确的是（）。

A. 各进程只能用"读"方式打开文件 F

B. 在系统打开文件表中仅有一个表项包含 F 的属性

C. 各进程的用户打开文件表中关于 F 的表项内容相同

D. 进程关闭 F 时, 系统删除 F 在系统打开文件表中的表项

**答案**：B

**考点分析**

本题考查操作系统中文件共享机制和打开文件表的管理。

**文件打开的两级表结构**：
1. **用户打开文件表（进程级）**：每个进程独立维护
   - 记录该进程打开的文件
   - 包含：读写指针、访问模式、指向系统打开文件表的指针
   
2. **系统打开文件表（系统级）**：全局唯一
   - 记录系统中所有被打开的文件
   - 包含：文件属性、打开计数、文件在磁盘上的位置等

**选项分析**：

**A. 各进程只能用"读"方式打开文件 F** ✗
- 多个进程共享文件不代表只能读
- 可以有读写、只读等多种打开方式
- 需要通过文件锁等机制协调并发访问

**B. 在系统打开文件表中仅有一个表项包含 F 的属性** ✓
- 系统打开文件表是全局的，每个被打开的文件只有一个表项
- 多个进程打开同一文件时，系统打开文件表中该文件的打开计数增加
- 文件的属性信息（大小、权限、inode等）在系统表中只保存一份

**C. 各进程的用户打开文件表中关于 F 的表项内容相同** ✗
- 不同进程的读写指针位置可能不同
- 打开模式可能不同（只读、读写等）
- 每个进程维护自己的文件访问状态

**D. 进程关闭 F 时, 系统删除 F 在系统打开文件表中的表项** ✗
- 只有当所有打开该文件的进程都关闭时，系统才删除表项
- 关闭时只是将打开计数减1
- 计数降为0时才删除系统打开文件表项

---


### 第三步：逐项分析

**A. 各进程只能用"读"方式打开文件 F ❌**
```
详细分析：文件共享不限制打开模式
判断依据：
- 共享文件：可以以只读、只写、读写等方式打开
- 并发控制：通过文件锁机制协调，而非限制打开模式
- 反例：多个进程可同时以读写方式打开日志文件
结论：错误，打开模式不受限制
```

**B. 在系统打开文件表中仅有一个表项包含 F 的属性 ✅**
```
详细分析：系统打开文件表是全局唯一的
判断依据：
- 系统打开文件表：系统级，记录所有被打开的文件
- 文件F被多个进程打开：系统表中只有一个表项
- 表项内容：文件属性（inode指针）、打开计数等
- 打开计数：记录有多少进程打开了该文件
- 关闭时：打开计数-1，计数为0时删除表项
结论：正确
```

**C. 各进程的用户打开文件表中关于 F 的表项内容相同 ❌**
```
详细分析：用户打开文件表是进程独立的
判断依据：
- 用户打开文件表：进程级，每个进程独立维护
- 表项内容：读写指针、打开模式、指向系统表的指针
- 不同进程：
  - 读写指针位置可能不同
  - 打开模式可能不同（只读vs读写）
  - 文件描述符值可能不同
- 示例：进程A读到第100字节，进程B读到第50字节
结论：错误，表项内容不相同
```

**D. 进程关闭 F 时, 系统删除 F 在系统打开文件表中的表项 ❌**
```
详细分析：只有所有进程都关闭时才删除
判断依据：
- 系统打开文件表：维护打开计数
- 某进程关闭：打开计数-1
- 删除时机：打开计数为0时才删除表项
- 原因：其他进程可能仍在使用该文件
结论：错误，不是立即删除
```

### 第四步：确认答案
系统打开文件表中每个文件只有一个表项 → 答案：**B**

## 易错点分析

### 易错点1：混淆两级打开文件表
❌ 错误理解：系统打开文件表和用户打开文件表是同一个表
✅ 正确理解：用户打开文件表是进程级的，每个进程独立维护，记录该进程打开的文件和读写指针（进程私有）。系统打开文件表是系统级的全局表，记录所有被打开文件的公共信息（文件属性、打开计数、inode指针）。多个进程打开同一文件时，各有自己的用户表项（读写指针独立），但共享一个系统表项（通过打开计数管理）

### 易错点2：不理解打开计数机制

| 操作 | 打开计数变化 | 系统表项 | 用户表项 |
|------|-------------|---------|---------|
| 进程1打开F | 0→1 | 创建 | 进程1创建 |
| 进程2打开F | 1→2 | 保持 | 进程2创建 |
| 进程1关闭F | 2→1 | 保持 | 进程1删除 |
| 进程2关闭F | 1→0 | 删除 | 进程2删除 |

**文件共享结构示意**：
```
进程1文件表：         系统打开文件表：      内存inode：
┌──────────┐         ┌──────────────┐      ┌─────────┐
│fd=3      │ ──┐     │ 文件F        │      │ 文件F   │
│读写指针=0│   │  ┌→ │ 打开计数=2   │ ───→ │ inode   │
└──────────┘   │  │  │ inode指针    │      │ 属性    │
               └──┘  └──────────────┘      └─────────┘
进程2文件表：      ↑                      
┌──────────┐      │
│fd=4      │ ─────┘
│读写指针=50│      
└──────────┘
```

**关键点**：
- 系统表项唯一：无论多少进程打开，只有一个表项
- 用户表项独立：每个进程有自己的表项和读写指针
- 打开计数：系统表维护，用于判断何时删除表项
- fork()特殊情况：父子进程共享用户表项和读写指针
