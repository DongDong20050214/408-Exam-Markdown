**题干**

设置当前工作目录的主要目的是（）。A. 节省外存空间  B. 节省内存空间  C. 加快文件的检索速度  D. 加快文件的读/写速度

**答案**：C

**考点分析**
- 当前工作目录加快检索

---


## 解题思路

### 第一步：理解当前工作目录作用
当前工作目录是进程的工作路径，用于简化相对路径访问。

### 第二步：逐项分析
**A. 节省外存空间 ❌**
```
详细分析：当前目录只是一个路径字符串，不涉及外存存储
判断依据：目录本身占用空间极小，与节省外存无关
结论：错误
```

**B. 节省内存空间 ❌**
```
详细分析：当前目录信息占用PCB少量字段，不是设计目的
判断依据：内存中只存一个路径字符串，几十字节
结论：错误，不是主要目的
```

**C. 加快文件的检索速度 ✅**
```
详细分析：设置当前目录后，访问相对路径文件无需从根目录搜索
判断依据：
- 无当前目录：每次访问都从根目录遍历整个路径
- 有当前目录：直接从当前目录查找，减少遍历层数
结论：正确，这是设置当前目录的主要目的
```

**D. 加快文件的读/写速度 ❌**
```
详细分析：读写速度取决于磁盘IO和缓存，与目录无关
判断依据：当前目录只影响路径查找，不影响数据传输
结论：错误
```

### 第三步：确认答案
当前工作目录减少路径搜索长度 → 答案：**C**

## 易错点分析

### 易错点1：混淆检索和读写
❌ 错误理解：当前目录能加快文件读写速度
✅ 正确理解：当前工作目录（存储在PCB中的路径字符串如/home/user）只加快文件名到inode的检索（路径解析）。访问相对路径doc/file.txt时从当前目录开始查找，比从根目录/home/user/doc/file.txt少遍历2层目录。但找到inode后，数据读写速度由磁盘I/O性能和页缓存决定，与目录层次无关。检索是定位文件，读写是传输数据，两个阶段

### 易错点2：忽略目录层次遍历成本
| 场景 | 查找路径 | 遍历层数 |
|------|---------|---------|
| 无当前目录 | /home/user/doc/file.txt | 从根目录遍历4层 |
| 有当前目录(/home/user) | doc/file.txt | 从当前目录遍历2层 |

**关键点**：
- 当前目录本质：PCB中的路径字符串
- 核心作用：将相对路径转换为绝对路径时的起点
- 性能提升：减少目录树遍历深度
