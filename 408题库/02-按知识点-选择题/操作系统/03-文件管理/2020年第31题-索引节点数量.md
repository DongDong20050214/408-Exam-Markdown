# 索引节点真题汇总

## 2020年第31题

**元数据**
- 年份：[[../../01-真题原卷/2020年408真题#31|2020年第31题]]
- 科目：#操作系统
- 知识点：#操作系统/文件管理/索引节点
- 难度：⭐⭐
- 关联知识：[[知识库/操作系统/文件系统]]

**题干**

某文件系统的目录项由文件名和索引节点号构成。若每个目录项长度为 64 字节，其中 4 字节存放索引节点号，60 字节存放文件名。文件名由小写英文字母构成，则该文件系统能创建的文件数量的上限为（）。

A. 2²⁶  B. 2³²  C. 2⁶⁰  D. 2⁶⁴

**答案**：B

**考点分析**
- 文件数量由索引节点号位数决定
- 4字节=32位，最多2³²个文件

---


## 解题思路

### 第一步：理解文件数量限制因素
文件数量上限由索引节点号的表示范围决定。

### 第二步：分析题目信息
```
目录项长度：64字节
- 索引节点号：4字节
- 文件名：60字节

关键信息：索引节点号占4字节
```

### 第三步：计算文件数量上限
```
索引节点号大小：4字节 = 32位
能表示的数值范围：0 ~ 2³²-1
文件数量上限：2³² 个
```

### 第四步：逐项分析

**A. 2²⁶ ❌**
```
详细分析：2²⁶ = 64M，远小于实际上限
判断依据：
- 26位只能表示64M个不同值
- 题目索引节点号占32位，不是26位
结论：错误，位数计算错误
```

**B. 2³² ✅**
```
详细分析：4字节=32位能表示2³²个值
判断依据：
- 索引节点号：4字节 = 32位
- 32位无符号整数范围：0 ~ 2³²-1
- 每个文件对应一个唯一的索引节点号
- 文件数量上限 = 索引节点数量上限 = 2³²
结论：正确
```

**C. 2⁶⁰ ❌**
```
详细分析：误用文件名字段长度
判断依据：
- 60字节是文件名字段长度，不是索引节点号
- 文件名长度只限制名称长度，不限制文件数量
- 不同文件可用相同长度文件名，但索引节点号必须唯一
结论：错误，混淆了文件名和索引节点号
```

**D. 2⁶⁴ ❌**
```
详细分析：误用目录项总长度
判断依据：
- 64字节是整个目录项长度
- 但只有索引节点号字段限制文件数量
- 2⁶⁴是8字节的表示范围，而题目索引节点号只有4字节
结论：错误
```

### 第五步：确认答案
索引节点号4字节=32位，能表示2³²个文件 → 答案：**B**

## 易错点分析

### 易错点1：混淆不同字段的作用
❌ 错误理解：文件名60字节，所以能创建2⁶⁰个文件
✅ 正确理解：文件数量由索引节点号的位数决定，文件名长度只限制单个名称能包含多少字符。4字节索引节点号能表示2³²个唯一值，即最多2³²个inode，对应最多2³²个文件。文件名即使支持60字节（可存长达60个ASCII字符），但不同文件可以有相同名称（不同目录下），文件名不具备唯一性约束

### 易错点2：不理解索引节点号的唯一性约束

| 字段 | 长度 | 作用 | 对文件数量的影响 |
|------|------|------|-----------------|
| 文件名 | 60字节 | 标识文件名称 | 不限制（可重名于不同目录） |
| 索引节点号 | 4字节 | 唯一标识文件 | 决定上限（必须唯一） |

**索引节点号与文件数量**：
```
- 每个文件对应一个inode（索引节点）
- 每个inode有唯一的inode号
- inode号用4字节表示 → 最多2³²个不同的inode号
- 因此最多创建2³²个文件

注意：
- 硬链接：多个文件名指向同一inode（共享inode号）
- 实际文件数 ≤ inode数量
- 题目问"能创建的文件数量上限" = inode数量上限
```

**目录项结构示意**：
```
目录项（64字节）：
┌─────────────────┬─────────────────────────────┐
│ inode号(4字节)  │      文件名(60字节)         │
│   12345678      │   "document.txt\0..."       │
└─────────────────┴─────────────────────────────┘
     ↓
    决定文件数量上限
```

**关键点**：
- 4字节 = 32位 → 2³² 个不同值
- 文件系统通过inode号定位文件
- 文件名可以很长或很短，不影响文件数量上限
