**题干**

为支持 CD-ROM 中视频文件的快速随机播放, 播放性能最好的文件数据块组织方式是 ( )。

A. 连续结构  B. 链式结构  C. 直接索引结构  D. 多级索引结构

**答案**：A

**考点分析**
- 连续结构随机访问最快

---


## 解题思路

### 第一步：理解CD-ROM视频播放需求
快速随机播放要求：能直接跳转到任意位置，访问延迟最小。

### 第二步：逐项分析
**A. 连续结构 ✅**
```
详细分析：文件块在磁盘上连续存储
判断依据：
- 随机访问：已知起始位置+偏移量，直接计算目标块物理地址
- 访问延迟：只需一次寻道+旋转
- CD-ROM特性：只读介质，不存在扩展问题
结论：正确，连续结构随机访问性能最优
```

**B. 链式结构 ❌**
```
详细分析：块之间通过指针链接
判断依据：
- 随机访问：必须从头顺序遍历链表
- 访问第n块：需要n次磁盘读取
- 性能：O(n)时间复杂度，无法快速跳转
结论：错误，不支持快速随机访问
```

**C. 直接索引结构 ❌**
```
详细分析：索引表直接指向各数据块
判断依据：
- 随机访问：需先读索引表，再读数据块（两次IO）
- 比连续结构多一次索引表访问
- 适合场景：需要扩展的文件
结论：错误，性能不如连续结构
```

**D. 多级索引结构 ❌**
```
详细分析：多级索引表指向数据块
判断依据：
- 随机访问：需读取多级索引表+数据块（多次IO）
- 二级索引：至少3次IO（一级索引+二级索引+数据）
- 访问延迟最大
结论：错误，性能最差
```

### 第三步：确认答案
CD-ROM只读特性+连续结构随机访问最快 → 答案：**A**

## 易错点分析

### 易错点1：认为索引结构随机访问更快
❌ 错误理解：索引支持随机访问，所以性能最好
✅ 正确理解：索引支持随机访问，但每次需先读索引表，IO次数多于连续结构

### 易错点2：忽略CD-ROM只读特性
| 场景 | 最佳结构 | 原因 |
|------|---------|------|
| 可读写磁盘 | 索引结构 | 需要支持扩展 |
| 只读CD-ROM | 连续结构 | 无扩展需求，访问最快 |

**IO次数对比**（访问文件中间某块）：
- 连续结构：1次IO（直接计算物理地址）
- 直接索引：2次IO（读索引表+读数据块）
- 二级索引：3次IO（读一级+读二级+读数据）
- 链式结构：n次IO（从头遍历）

**关键点**：
- 连续结构缺点（易扩展差）在只读介质上不成立
- 视频文件通常较大且连续，连续存储是最优选择
