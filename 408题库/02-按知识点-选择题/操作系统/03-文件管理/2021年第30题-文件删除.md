# 文件删除真题汇总

## 2021年第30题

**元数据**
- 年份：[2021年第30题](../../../01-真题原卷/2021年408真题.md#30)
- 科目：#操作系统
- 知识点：#操作系统/文件管理/文件删除
- 难度：⭐⭐
- 关联知识：[2023年第31题-文件操作](2023年第31题-文件操作.md)

**题干**

若目录 dir 下有文件 file1，则为删除该文件内核不必完成的工作是（）。

A. 删除 file1 的快捷方式
B. 释放 file1 的文件控制块
C. 释放 file1 占用的磁盘空间
D. 删除目录 dir 中与 file1 对应的目录项

**答案**：A

**考点分析**
- A不必：快捷方式由用户管理，不是内核工作
- BCD都是删除文件时内核必须完成的

---


## 解题思路

### 第一步：理解文件删除的内核工作
删除文件是内核操作，涉及目录项、FCB和磁盘空间的清理。

### 第二步：逐项分析

**A. 删除 file1 的快捷方式 ✅（不必完成）**
```
详细分析：快捷方式不由内核管理
判断依据：
- 快捷方式（软链接）：是一个独立的文件，内容是目标文件路径
- 目标文件删除：快捷方式仍然存在，只是指向失效
- 不会自动删除：操作系统不负责清理失效的快捷方式
- 用户责任：用户需要手动删除无效的快捷方式
- 示例：Windows删除文件，桌面快捷方式不会自动消失
结论：正确，内核不必删除快捷方式
```

**B. 释放 file1 的文件控制块 ❌（必须完成）**
```
详细分析：删除文件必须释放FCB/inode
判断依据：
- FCB/inode：存储文件元数据（大小、权限、块地址）
- 引用计数为0：所有硬链接都删除后，释放FCB
- 磁盘资源：FCB占用的磁盘块需要回收
- 必须操作：否则inode泄漏，无法创建新文件
结论：错误，内核必须释放FCB
```

**C. 释放 file1 占用的磁盘空间 ❌（必须完成）**
```
详细分析：删除文件必须回收数据块
判断依据：
- 数据块：文件实际内容占用的磁盘空间
- 空闲块管理：将数据块标记为空闲（位图置0或加入空闲链）
- 资源回收：磁盘空间是有限资源，必须回收
- 必须操作：否则磁盘空间泄漏
结论：错误，内核必须释放磁盘空间
```

**D. 删除目录 dir 中与 file1 对应的目录项 ❌（必须完成）**
```
详细分析：删除文件首先删除目录项
判断依据：
- 目录项：记录文件名和inode号的对应关系
- 删除目录项：文件名从目录中消失，用户无法找到文件
- 必须操作：这是删除文件的第一步
- ls命令：删除后，ls dir不再显示file1
结论：错误，内核必须删除目录项
```

### 第三步：确认答案
快捷方式由用户管理，内核不必删除 → 答案：**A**

## 易错点分析

### 易错点1：认为内核负责删除所有相关文件
❌ 错误理解：删除文件时内核会删除所有快捷方式和符号链接
✅ 正确理解：快捷方式（软链接）是独立文件，内容存储的是目标路径字符串（如"/home/file1"）。目标文件删除后软链接本身仍然存在，只是访问时会返回"文件不存在"错误。内核不追踪哪些软链接指向被删除的文件，清理失效链接是用户责任。这与硬链接不同：硬链接共享inode，引用计数归零前不会释放文件内容

### 易错点2：混淆硬链接和软链接（快捷方式）

| 链接类型 | 实现方式 | 删除目标后 | 内核处理 |
|---------|---------|-----------|---------|
| 硬链接 | 共享inode | 硬链接仍可用 | 引用计数-1 |
| 软链接（快捷方式） | 存储路径字符串 | 软链接失效 | 不影响软链接 |

**文件删除流程（引用计数为0）**：
```
1. 从目录dir删除file1的目录项
   - 目录中移除"file1 → inode 123"这条记录
   
2. inode引用计数减1
   - 若计数变为0，继续下面步骤
   - 若计数>0，保留inode和数据（还有其他硬链接）
   
3. 释放数据块
   - 标记file1占用的磁盘块为空闲
   - 位图法：置0
   - 空闲链法：加入空闲链
   
4. 释放FCB/inode
   - 回收inode号
   - 释放inode占用的磁盘空间
   
5. 快捷方式不处理
   - 快捷方式文件仍存在
   - 内容仍是"/dir/file1"路径
   - 访问时会返回"文件不存在"错误
```

**快捷方式失效示例**：
```
创建文件和快捷方式：
$ echo "hello" > /home/file1
$ ln -s /home/file1 /desktop/shortcut  (软链接)

删除原文件：
$ rm /home/file1

快捷方式状态：
$ ls -l /desktop/shortcut
lrwxrwxrwx 1 user user 11 Jun 10 10:00 shortcut -> /home/file1
                                                    ↑目标已不存在

$ cat /desktop/shortcut
cat: shortcut: No such file or directory

$ ls -l /desktop/  
(shortcut仍然存在，只是失效了)
```

**关键点**：
- 内核删除：目录项、inode、数据块
- 内核不删除：软链接（快捷方式）
- 快捷方式本质：一个内容为路径字符串的独立文件
