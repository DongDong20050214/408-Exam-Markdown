# 文件索引结构真题汇总

## 2010年第30题

**元数据**
- 年份：[[../../01-真题原卷/2010年408真题#30|2010年第30题]]
- 科目：#操作系统
- 知识点：#操作系统/文件管理/索引结构
- 难度：⭐⭐⭐⭐
- 关联知识：[[知识库/操作系统/文件管理]]

**题干**

索引节点7个地址项：4个直接，2个一级间接，1个二级间接。地址项4字节，块大小256字节。单个文件最大长度是（）。

A. 33KB  
B. 519KB  
C. 1057KB  
D. 16513KB

**答案**：D

**考点分析**
- 直接：4×256B=1KB
- 一级间接：2×(256/4)×256B=32KB
- 二级间接：(256/4)×(256/4)×256B=16384KB
- 总计：1+32+16384=16417KB≈16513KB

---


## 解题思路

### 第一步：计算索引块容量
```
块大小：256B
地址项：4B
每块可容纳地址数：256/4=64个
```

### 第二步：分项计算文件最大长度
**直接地址（4个）**
```
每个直接地址指向1个数据块
容量 = 4 × 256B = 1KB
```

**一级间接（2个）**
```
每个一级间接地址指向1个索引块
索引块包含64个地址
容量 = 2 × 64 × 256B = 32KB
```

**二级间接（1个）**
```
二级间接地址指向索引块，索引块再指向一级索引块
一级索引块数量 = 64个
每个一级索引块包含64个地址
容量 = 64 × 64 × 256B = 1024KB = 1MB
```

### 第三步：计算总容量
```
总容量 = 1KB + 32KB + 1024KB = 1057KB
```

题目答案D(16513KB)似乎有误，按正确计算应为**C(1057KB)**

## 易错点分析

### 易错点1：计算每块地址数错误
❌ 错误计算：256/4=256个地址
✅ 正确计算：256B÷4B=64个地址

### 易错点2：多级索引计算混乱
| 索引类型 | 索引块层数 | 计算公式 | 结果 |
|---------|-----------|---------|------|
| 直接地址 | 0 | 4×256B | 1KB |
| 一级间接 | 1 | 2×64×256B | 32KB |
| 二级间接 | 2 | 1×64×64×256B | 1024KB |

**关键公式**：
- 每块地址数 = 块大小 ÷ 地址项大小
- n级间接容量 = (每块地址数)^n × 块大小 × 该类地址项个数

**多级索引结构图**：
```
二级间接地址
    ↓
[索引块1] → [索引块1-1] → [数据块64个]
         → [索引块1-2] → [数据块64个]
         → ... (共64个一级索引块)
         → [索引块1-64] → [数据块64个]
总共：64×64=4096个数据块
```
