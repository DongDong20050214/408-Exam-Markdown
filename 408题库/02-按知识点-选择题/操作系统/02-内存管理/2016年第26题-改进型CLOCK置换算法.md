**题干**

采用改进型 CLOCK 置换算法，则该算法淘汰页的次序为（）。A. (0,0),(0,1),(1,0),(1,1)  B. (0,0),(1,0),(0,1),(1,1)  C. (0,0),(0,1),(1,1),(1,0)  D. (0,0),(1,1),(0,1),(1,0)

**答案**：B

**考点分析**
- 优先淘汰未访问未修改，再未访问已修改，再已访问未修改，最后已访问已修改

---


## 解题思路

### 第一步：理解改进型CLOCK算法
(访问位,修改位)
淘汰优先级：(0,0) > (1,0) > (0,1) > (1,1)

### 第二步：分析淘汰顺序
```
优先级从高到低：
1. (0,0)：未访问、未修改 - 最优淘汰
2. (1,0)：已访问、未修改 - 次优
3. (0,1)：未访问、已修改 - 较差
4. (1,1)：已访问、已修改 - 最差

说明：
- 修改位=1需要写回，开销大
- 访问位=0表示最近未用
- (0,0)淘汰代价最小
```

### 第三步：逐项分析
**A. (0,0),(0,1),(1,0),(1,1) ❌**
```
顺序：(0,0) → (0,1) → (1,0)
(0,1)应在(1,0)之后
结论：错误
```

**B. (0,0),(1,0),(0,1),(1,1) ✅**
```
完全符合优先级顺序
结论：正确
```

**C. (0,0),(0,1),(1,1),(1,0) ❌**
```
(1,0)应在(0,1)之前
结论：错误
```

**D. (0,0),(1,1),(0,1),(1,0) ❌**
```
顺序错乱
结论：错误
```

### 第四步：确认答案
淘汰顺序(0,0),(1,0),(0,1),(1,1) → 答案：**B**

## 易错点分析

### 易错点1：认为修改位优先于访问位
❌ 错误：先看修改位再看访问位
✅ 正确：先看访问位，访问位相同再看修改位

### 易错点2：不理解淘汰代价
| 状态 | 访问位 | 修改位 | 代价 | 优先级 |
|------|--------|--------|------|--------|
| (0,0) | 0 | 0 | 最小 | 1 |
| (1,0) | 1 | 0 | 较小 | 2 |
| (0,1) | 0 | 1 | 较大 | 3 |
| (1,1) | 1 | 1 | 最大 | 4 |

**关键原则**：
- 优先淘汰未修改页（避免写回）
- 其次考虑未访问页（最近未用）
