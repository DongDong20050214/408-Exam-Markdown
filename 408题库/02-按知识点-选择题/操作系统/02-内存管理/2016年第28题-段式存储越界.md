# 段式存储地址转换真题汇总

## 2016年第28题

**元数据**
- 年份：[../../01-真题原卷/2016年408真题#28|2016年第28题](../../01-真题原卷/2016年408真题#28|2016年第28题.md)
- 科目：#操作系统
- 知识点：#操作系统/内存管理/段式存储 #地址转换 #越界异常
- 难度：⭐⭐
- 关联知识：[知识库/操作系统/段式存储](知识库/操作系统/段式存储.md)

**题干**

某进程的段表内容如下所示。当访问段号为2、段内地址为400的逻辑地址时，进行地址转换的结果是（）。

| 段号 | 段长 | 内存起始地址 | 权限 | 状态 |
|------|------|-------------|------|------|
| 0    | 100  | 6000        | 只读 | 在内存 |
| 1    | 200  | —           | 读写 | 不在内存 |
| 2    | 300  | 4000        | 读写 | 在内存 |

A. 段缺失异常  
B. 得到内存地址 4400  
C. 越权异常  
D. 越界异常

**答案**：D

**考点分析**
- 段内地址=400，段长=300
- 400 ≥ 300，超出段长范围
- 触发越界异常(段内地址越界)
- 若段内地址<300，则物理地址=4000+段内地址

---


## 解题思路

### 第一步：分析段表信息
```
访问：段号=2，段内地址=400
段2信息：
- 段长：300
- 起始地址：4000
- 权限：读写
- 状态：在内存
```

### 第二步：检查越界
```
段内地址 ≥ 段长？
400 ≥ 300 → 是
触发越界异常
```

### 第三步：逐项分析
**A. 段缺失异常 ❌**
```
段2状态：在内存
不是缺失
结论：错误
```

**B. 得到内存地址4400 ❌**
```
如果不越界：物理地址=4000+400=4400
但已越界，不能计算地址
结论：错误
```

**C. 越权异常 ❌**
```
权限：读写
访问未指定读写类型
不涉及权限问题
结论：错误
```

**D. 越界异常 ✅**
```
段内地址400 ≥ 段长300
超出段范围
触发越界异常
结论：正确
```

### 第四步：确认答案
段内地址超出段长 → 答案：**D**

## 易错点分析

### 易错点1：直接计算物理地址
❌ 错误：4000+400=4400
✅ 正确：先检查越界，400≥300已越界

### 易错点2：混淆异常类型
| 异常 | 条件 | 本题 |
|------|------|------|
| 越界 | 段内地址≥段长 | ✅400≥300 |
| 缺失 | 段不在内存 | ❌在内存 |
| 越权 | 违反权限 | ❌未涉及 |

**地址转换流程**：
1. 检查段号合法性
2. 检查段内地址 < 段长（越界检查）
3. 检查权限
4. 检查段是否在内存
5. 计算：物理地址=基址+段内地址
