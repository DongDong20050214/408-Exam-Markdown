# 动态分区真题汇总

## 2017年第25题

**元数据**
- 年份：[../../01-真题原卷/2017年408真题#25|2017年第25题](../../01-真题原卷/2017年408真题#25|2017年第25题.md)
- 科目：#操作系统

- 难度：⭐⭐⭐
- 关联知识：[知识库/操作系统/动态分区](知识库/操作系统/动态分区.md)

**题干**

某计算机按字节编址，其动态分区内存管理采用最佳适应算法，回收起始地址为 60K、大小为 140KB 的分区后，系统中空闲分区的数量、空闲分区链第一个分区的起始地址和大小分别是（）。A. 3,20K,380KB  B. 3,500K,80KB  C. 4,20K,180KB  D. 4,500K,80KB

**答案**：C

**考点分析**
- 回收后合并分区，最佳适应按大小排序

---


## 解题思路

### 第一步：理解最佳适应算法
最佳适应：空闲分区链按大小升序排列
回收时需合并相邻分区

### 第二步：分析回收过程（需要原始分区表）
```
回收：起始60K，大小140KB（60K-200K）
需要判断：
1. 是否与相邻分区合并
2. 合并后重新排序
3. 统计分区数和第一个分区信息
```

### 第三步：根据答案推理
```
答案C：4个分区，首个20K起始，180KB
说明：
- 回收后有4个空闲分区
- 最小分区：20K起始，180KB大小
- 最佳适应按大小排序，最小的在前
```

### 第四步：确认答案
4个分区，首个20K/180KB → 答案：**C**

## 易错点分析

### 易错点1：忘记合并相邻分区
❌ 错误：回收后直接插入链表
✅ 正确：先检查是否与相邻分区合并

### 易错点2：不理解最佳适应排序
| 算法 | 链表排序 |
|------|---------|
| 首次适应 | 按地址升序 |
| 最佳适应 | 按大小升序 |
| 最坏适应 | 按大小降序 |

**回收步骤**：
1. 检查与前后分区是否相邻
2. 合并相邻分区
3. 按大小重新排序
4. 更新分区链
