**题干**

某系统采用 LRU 页置换算法和局部置换策略, 系统为进程 P 预分配了 4 个页框, 进程 P 访问页号的序列为 0, 1, 2, 7, 0, 5, 3, 5, 0, 2, 7, 6, 则产生页置换的总次数是 ( )。A. 3  B. 4  C. 5  D. 6

**答案**：C

**考点分析**
- 模拟LRU过程，产生5次置换

---


## 解题思路

### 第一步：理解LRU算法
最近最久未使用页面置换。4个页框，访问序列：0,1,2,7,0,5,3,5,0,2,7,6

### 第二步：模拟LRU过程

| 访问 | 页框状态 | 是否缺页 |
|------|---------|---------|
| 0 | [0] | 缺页1 |
| 1 | [0,1] | 缺页2 |
| 2 | [0,1,2] | 缺页3 |
| 7 | [0,1,2,7] | 缺页4 |
| 0 | [0,1,2,7] | 命中 |
| 5 | [0,5,2,7] | 缺页5，置换1 |
| 3 | [0,5,3,7] | 缺页6，置换2 |
| 5 | [0,5,3,7] | 命中 |
| 0 | [0,5,3,7] | 命中 |
| 2 | [0,5,3,2] | 缺页7，置换7 |
| 7 | [0,5,7,2] | 缺页8，置换3 |
| 6 | [0,6,7,2] | 缺页9，置换5 |

前4次是初始缺页，之后产生5次置换。

### 第三步：确认答案
5次置换 → 答案：**C**
