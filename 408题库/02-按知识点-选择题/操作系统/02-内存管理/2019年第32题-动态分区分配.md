# 动态分区分配真题汇总

**元数据**
- 年份：[2019年第32题](../../../01-真题原卷/2019年408真题.md#32)
- 科目：#操作系统

- 难度：⭐⭐
- 关联知识：[动态分区](../../../../AI-知识点原子库/动态分区.md)

**题干**

在下列动态分区分配算法中，最容易产生内存碎片的是（）。

A. 首次适应算法  B. 最坏适应算法  C. 最佳适应算法  D. 循环首次适应算法

**答案**：C

**考点分析**
- 最佳适应算法容易产生大量小碎片

---


## 解题思路

### 第一步：理解各算法特点
动态分区分配：根据进程需求分配内存

### 第二步：逐项分析
**A. 首次适应算法 ❌**
```
从头开始查找第一个合适分区
低地址产生碎片，但不是最多
结论：不是最容易
```

**B. 最坏适应算法 ❌**
```
选择最大的分区
剩余分区较大，碎片相对较少
结论：不是最容易
```

**C. 最佳适应算法 ✅**
```
选择最小的合适分区
每次分配后剩余空间很小
产生大量难以利用的小碎片
结论：最容易产生碎片
```

**D. 循环首次适应 ❌**
```
类似首次适应，从上次位置开始
分布均匀，碎片不是最多
结论：不是最容易
```

### 第三步：确认答案
最佳适应产生最多小碎片 → 答案：**C**

## 易错点分析

### 易错点1：认为最坏适应产生碎片多
❌ 错误：最坏适应选大分区，剩余小碎片多
✅ 正确：最坏适应剩余分区较大，可继续利用

### 易错点2：不理解碎片产生机制
| 算法 | 选择策略 | 剩余分区 | 碎片情况 |
|------|---------|---------|---------|
| 首次适应 | 第一个合适 | 中等 | 中等 |
| 最佳适应 | 最小合适 | 很小 | 很多小碎片✅ |
| 最坏适应 | 最大分区 | 较大 | 较少 |

**最佳适应问题**：
- 选择最接近需求的分区
- 剩余空间极小
- 产生大量无法利用的小碎片
