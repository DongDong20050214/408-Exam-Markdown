# LRU页置换真题汇总

## 2019年第29题

**元数据**
- 年份：[[../../01-真题原卷/2019年408真题#29|2019年第29题]]
- 科目：#操作系统
- 知识点：#操作系统/内存管理/LRU
- 难度：⭐⭐⭐
- 关联知识：[[知识库/操作系统/页面置换]]

**题干**

某系统采用 LRU 页置换算法和局部置换策略, 若系统为进程 P 预分配了 4 个页框, 进程 P 访问页号的序列为 0, 1, 2, 7, 0, 5, 3, 5, 0, 2, 7, 6, 则进程访问上述页的过程中, 产生页置换的总次数是 ( )。

A. 3  B. 4  C. 5  D. 6

**答案**：C

**考点分析**
- 前4次缺页不置换
- 后续访问逐一模拟LRU算法

---


## 解题思路

### 第一步：模拟LRU置换过程
```
页框数=4，访问序列：0,1,2,7,0,5,3,5,0,2,7,6
```

### 第二步：逐步模拟
```
0 * [0]           缺页1
1 * [0,1]         缺页2  
2 * [0,1,2]       缺页3
7 * [0,1,2,7]     缺页4（前4次不置换）
0   [0,1,2,7]     命中
5 * [0,5,2,7]     缺页5，淘汰1（最久未用）
3 * [0,5,3,7]     缺页6，淘汰2
5   [0,5,3,7]     命中
0   [0,5,3,7]     命中
2 * [0,5,3,2]     缺页7，淘汰7
7 * [0,5,7,2]     缺页8，淘汰3
6 * [0,6,7,2]     缺页9，淘汰5

总缺页9次，置换5次（第5-9次缺页需置换）
```

### 第三步：确认答案
置换次数=5 → 答案：**C**

## 易错点分析

### 易错点1：混淆缺页和置换
❌ 错误：缺页次数=置换次数
✅ 正确：前4次缺页不置换，只有后续需置换

### 易错点2：LRU算法理解错误
| 访问 | 操作 | 是否置换 |
|------|------|----------|
| 前4次 | 填充 | ❌ |
| 第5次起 | 缺页 | ✅需置换 |

**关键点**：
- 缺页次数：页面不在内存的次数
- 置换次数：页框满后的缺页次数
- 置换=缺页-页框数（前提是缺页≥页框数）
