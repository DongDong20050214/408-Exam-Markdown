- 关联知识：[2019年第14题-缺页处理](2019年第14题-缺页处理.md)

**题干**

下列关于缺页处理的叙述中，错误的是（ ）。A. 缺页是在地址转换时 CPU 检测到的一种异常  B. 缺页处理由操作系统提供的缺页处理程序来完成  C. 缺页处理程序根据页故障地址从外存读入所缺失的页  D. 缺页处理完成后回到发生缺页的指令的下一条指令执行

**答案**：D

**考点分析**
- D错误：缺页处理后返回发生缺页的指令重新执行

---


## 解题思路

### 第一步：理解缺页处理流程
缺页：访问页面不在内存→异常处理→调入页面→重新执行

### 第二步：逐项分析
**A. CPU检测到的异常 ✅**
```
地址转换时发现存在位=0
CPU产生缺页异常
陷入内核处理
结论：正确
```

**B. OS缺页处理程序完成 ✅**
```
缺页处理在内核态
由操作系统的缺页中断处理程序负责
结论：正确
```

**C. 根据页故障地址读入页面 ✅**
```
获取缺失页的外存地址
从外存读入内存
更新页表
结论：正确
```

**D. 返回下一条指令执行 ❌**
```
错误：缺页指令未完成
正确：返回缺页指令重新执行
因为该指令还未成功完成
结论：错误
```

### 第三步：确认答案
返回缺页指令而非下一条 → 答案：**D**

## 易错点分析

### 易错点1：认为返回下一条指令
❌ 错误：缺页处理后继续执行下一条
✅ 正确：重新执行引发缺页的指令

### 易错点2：不理解指令重执行原因
| 异常类型 | 返回位置 | 原因 |
|---------|---------|------|
| 缺页异常 | 当前指令 | 指令未完成 |
| 系统调用 | 下一条 | 调用已完成 |

**关键点**：
- 缺页发生时指令未完成
- 页面调入后需要重新执行该指令
- PC不前进，返回原指令地址
