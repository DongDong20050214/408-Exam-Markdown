**题干**

若用户进程访问内存时产生缺页，则下列选项中，操作系统可能执行的操作是（）。I. 处理越界错  II. 置换页  III. 分配内存

A. 仅 I、II  B. 仅II、III  C. 仅 I、III  D. I、II 和 III

**答案**：B

**考点分析**
- 缺页时可能置换页和分配内存

---


## 解题思路

### 第一步：理解缺页处理流程
缺页：页面不在内存→调入页面→可能需要置换

### 第二步：逐项分析
**I. 处理越界错 ❌**
```
越界：访问超出合法地址范围
缺页：访问的页面不在内存
两者不同
缺页不会处理越界错
结论：不会执行
```

**II. 置换页 ✅**
```
内存满时需要置换页
选择一个页面换出
为新页面腾出空间
缺页处理的核心操作
结论：可能执行
```

**III. 分配内存 ✅**
```
为新调入的页面分配页框
可能分配空闲页框
或置换后分配
必须的操作
结论：一定执行
```

### 第三步：确认答案
II、III可能执行 → 答案：**B**

## 易错点分析

### 易错点1：混淆缺页和越界
❌ 错误：缺页就是访问越界
✅ 正确：
- 缺页：页面合法但不在内存
- 越界：访问非法地址

### 易错点2：不理解缺页处理流程
| 步骤 | 操作 | 是否必须 |
|------|------|----------|
| 1.检查地址 | 合法性检查 | ✅ |
| 2.查找页面 | 在外存位置 | ✅ |
| 3.分配页框 | 分配或置换 | ✅ |
| 4.调入页面 | 从外存读入 | ✅ |
| 5.更新页表 | 修改页表项 | ✅ |
