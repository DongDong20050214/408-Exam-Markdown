**题干**

下列选项中，不会影响系统缺页率的是（）。

A. 页置换算法  B. 工作集的大小  C. 进程的数量  D. 页缓冲队列的长度

**答案**：D

**考点分析**
- D不影响：页缓冲只是优化性能，不影响缺页率
- ABC都影响缺页率

---


## 解题思路

### 第一步：理解缺页率影响因素
缺页率 = 缺页次数 / 访问次数

### 第二步：逐项分析
**A. 页置换算法 ✅影响**
```
不同算法缺页率不同
LRU < FIFO（通常）
OPT最优
结论：影响
```

**B. 工作集大小 ✅影响**
```
工作集：进程活跃页面集合
工作集大，需要页框多
分配页框少会导致频繁缺页
结论：影响
```

**C. 进程数量 ✅影响**
```
进程多，每个进程分到页框少
页框不足导致缺页率上升
可能引起抖动
结论：影响
```

**D. 页缓冲队列长度 ❌不影响**
```
页缓冲：保存被置换的页面
加快换入换出速度
不改变缺页次数
只是性能优化
结论：不影响
```

### 第三步：确认答案
页缓冲队列长度不影响 → 答案：**D**

## 易错点分析

### 易错点1：认为页缓冲影响缺页率
❌ 错误：缓冲队列减少缺页
✅ 正确：只加快处理速度，不减少缺页次数

### 易错点2：混淆缺页率和缺页处理时间
| 因素 | 影响缺页率 | 影响处理时间 |
|------|-----------|-------------|
| 置换算法 | ✅ | ❌ |
| 工作集大小 | ✅ | ❌ |
| 进程数量 | ✅ | ❌ |
| 页缓冲 | ❌ | ✅ |

**关键点**：
- 缺页率：缺页发生的频率
- 页缓冲：优化缺页处理性能
- 缓冲不能避免缺页，只能加快处理
