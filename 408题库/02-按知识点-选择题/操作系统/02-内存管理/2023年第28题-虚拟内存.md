**题干**

对于采用虚拟内存管理方式的系统，下列关于进程虚拟地址空间的叙述中，错误的是（）。

A. 每个进程都有自己独立的虚拟地址空间
B. C 语言中 malloc() 函数返回的是虚拟地址
C. 进程对数据段和代码段可以有不同的访问权限
D. 虚拟地址空间的大小由内存和硬盘的大小决定

**答案**：D

**考点分析**
- D错误：虚拟地址空间大小由CPU位数决定（如32位系统4GB）
- 与物理内存、硬盘大小无关

---


## 解题思路

### 第一步：理解虚拟地址空间
虚拟地址空间：进程看到的逻辑地址范围

### 第二步：逐项分析
**A. 每个进程独立虚拟地址空间 ✅**
```
每个进程有独立的页表
虚拟地址通过页表映射到物理地址
进程间地址空间隔离
结论：正确
```

**B. malloc()返回虚拟地址 ✅**
```
malloc()分配的是虚拟地址
由MMU转换为物理地址
程序操作的都是虚拟地址
结论：正确
```

**C. 不同段不同访问权限 ✅**
```
代码段：只读可执行
数据段：可读写
页表项中有权限位控制
结论：正确
```

**D. 大小由内存和硬盘决定 ❌**
```
虚拟地址空间大小由地址位数决定
32位系统：最大4GB（2^32）
64位系统：理论上2^64
与物理内存、硬盘大小无关
结论：错误
```

### 第三步：确认答案
虚拟空间大小由CPU位数决定 → 答案：**D**

## 易错点分析

### 易错点1：混淆虚拟空间和物理空间
❌ 错误：虚拟空间受物理资源限制
✅ 正确：虚拟空间只受地址位数限制

### 易错点2：不理解地址空间大小
| 系统 | 地址位数 | 虚拟空间 | 物理限制 |
|------|---------|---------|---------|
| 32位 | 32位 | 4GB | 无关 |
| 64位 | 48位实际 | 256TB | 无关 |

**关键点**：
- 虚拟空间：由CPU地址位数决定
- 物理内存：实际可用内存大小
- 两者独立，虚拟可以远大于物理
