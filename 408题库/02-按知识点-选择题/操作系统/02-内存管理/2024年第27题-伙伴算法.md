**题干**

下列算法中，每次回收分区时仅合并大小相等的空闲分区的是（）。

A. 伙伴算法  B. 最佳适应算法  C. 最坏适应算法  D. 首次适应算法

**答案**：A

**考点分析**
- 伙伴算法：只合并大小相等的伙伴分区

---


## 解题思路

### 第一步：理解伙伴算法特点
伙伴算法：分区大小为2的幂次，只合并相等的伙伴

### 第二步：逐项分析
**A. 伙伴算法 ✅**
```
特点：
- 分区大小为2的幂次（2,4,8,16...）
- 只合并大小相等且相邻的伙伴分区
- 两个64KB的伙伴→128KB
结论：正确
```

**B. 最佳适应算法 ❌**
```
按大小升序排列空闲分区
合并时合并所有相邻分区，不管大小
结论：错误
```

**C. 最坏适应算法 ❌**
```
选择最大的空闲分区
合并时合并相邻分区，不限制大小
结论：错误
```

**D. 首次适应算法 ❌**
```
按地址顺序查找
合并时合并相邻分区，不限制大小
结论：错误
```

### 第三步：确认答案
只有伙伴算法限制合并条件 → 答案：**A**

## 易错点分析

### 易错点1：不理解伙伴算法的特殊性
❌ 错误：所有算法都合并相邻分区
✅ 正确：伙伴算法只合并大小相等的伙伴

### 易错点2：混淆分配算法
| 算法 | 分区大小 | 合并条件 |
|------|---------|---------|
| 伙伴算法 | 2的幂次 | 大小相等的伙伴 |
| 其他算法 | 任意 | 相邻即合并 |

**伙伴算法特点**：
- 分区大小：1,2,4,8,16,32...
- 伙伴：地址相邻且大小相等
- 合并：只合并伙伴分区
- 优点：快速分配和回收
