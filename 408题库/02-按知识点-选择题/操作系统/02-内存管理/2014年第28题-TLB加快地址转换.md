**题干**

下列措施中，能加快虚实地址转换的是（）。I. 增大块表（TLB）容量  II. 让页表常驻内存  III. 增大交换区（swap）

A. 仅I  B. 仅 II  C. 仅I、II  D. 仅 II、III

**答案**：A

**考点分析**
- 仅I能加快地址转换

---


## 解题思路

### 第一步：理解地址转换过程
虚拟地址→TLB查找→页表查找→物理地址

### 第二步：逐项分析
**I. 增大TLB容量 ✅**
```
TLB（快表）：缓存页表项
容量越大，命中率越高
减少访问内存次数
直接加快地址转换
结论：有效
```

**II. 页表常驻内存 ❌**
```
页表本来就在内存中
常驻只是避免页表被换出
不减少访问内存次数
访问TLB miss仍需访问内存
结论：不能加快转换
```

**III. 增大交换区 ❌**
```
交换区用于页面换入换出
与地址转换速度无关
地址转换在内存中完成
结论：无关
```

### 第三步：确认答案
只有增大TLB容量有效 → 答案：**A**

## 易错点分析

### 易错点1：认为页表常驻能加快转换
❌ 错误：页表在内存更快访问
✅ 正确：页表本就在内存，常驻不改变访问速度

### 易错点2：不理解TLB作用
| 操作 | TLB命中 | TLB miss |
|------|---------|----------|
| 访问次数 | 1次（TLB） | 2次（TLB+内存） |
| 加速原理 | 避免访问内存 | 必须查页表 |
| 提升方法 | 增大TLB | - |
