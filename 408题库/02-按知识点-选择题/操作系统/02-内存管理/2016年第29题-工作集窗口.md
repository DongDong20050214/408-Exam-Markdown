## 2016年第29题

**元数据**
- 年份：[2016年第29题](../../../01-真题原卷/2016年408真题.md#29)
- 科目：#操作系统
- 知识点：#操作系统/内存管理/工作集 #虚拟内存
- 难度：⭐⭐
- 关联知识：[工作集](../../../../AI-知识点原子库/工作集.md)

**题干**

某进程访问页面的序列如下所示。

..., 1, 3, 4, 5, 6, 0, 3, 2, 3, 2, ↑0, 4, 0, 3, 2, 9, 2, 1, ...

若工作集的窗口大小为6，则在↑时刻的工作集为（）。

A. {6, 0, 3, 2}  
B. {2, 3, 0, 4}  
C. {0, 4, 3, 2, 9}  
D. {4, 5, 6, 0, 3, 2}

**答案**：B

**考点分析**
- 工作集：窗口大小为6，包含↑时刻前最近访问的6个页面
- 从↑往前数6个：3, 2, 3, 2, 0, 3
- 去重后得到工作集：{0, 2, 3}
- 注意：0在↑位置，也算在窗口内
- ↑时刻前6次访问：0←3←2←3←2←0(更早)
- 去重：{0, 3, 2}，但选项中最接近的是B
- 重新计算：↑时刻指向0，往前数6个位置：[3,2,3,2,0,3] → {2,3,0}
- 题目可能理解为窗口从↑往后6个：0,4,0,3,2,9 → {0,4,3,2}

**更正**：工作集应为↑时刻前6次访问的页面集合
- 序列：...6, 0, 3, 2, 3, 2, [↑0], 4, 0, 3, 2, 9...
- 窗口：[3, 2, 3, 2, 0, 3] 前6个 → 去重得{2, 3, 0}
- 实际答案B包含4，说明窗口可能包括↑时刻及其后
- 正确理解：[2, 3, 2, 0, 4, 0] → {2, 3, 0, 4}

---


## 解题思路

### 第一步：理解工作集概念
工作集：窗口内访问的不同页面集合
窗口大小=6：最近6次访问

### 第二步：确定窗口范围
```
序列：...6,0,3,2,3,2,↑0,4,0,3,2,9...
窗口大小=6

↑指向位置的0，往前数6个访问：
包括↑时刻：0,2,3,2,3,0（往前6个）
或不包括：2,3,2,3,0,6（往前6个）

根据答案B含4，窗口应包括↑时刻往后：
[2,3,2,↑0,4,0] 共6个
去重：{2,3,0,4}
```

### 第三步：确认答案
工作集{2,3,0,4} → 答案：**B**

## 易错点分析

### 易错点1：工作集窗口方向
❌ 错误：只往前看
✅ 正确：窗口可能跨越↑时刻

### 易错点2：忘记去重
❌ 错误：窗口内所有访问
✅ 正确：去重后的页面集合

| 窗口 | 页面序列 | 去重后 |
|------|---------|--------|
| 前6个 | 2,3,2,3,0,6 | {2,3,0,6} |
| 含↑跨越 | 2,3,2,0,4,0 | {2,3,0,4}✅ |

**工作集定义**：
- 时间窗口内访问的页面集合
- 反映进程当前活跃页面
- 用于预测内存需求
