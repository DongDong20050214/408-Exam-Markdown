**题干**

处理外部中断时，应该由操作系统保存的是（）。A. 程序计数器（PC）的内容  B. 通用寄存器的内容  C. 块表（TLB）中的内容  D. Cache 中的内容

**答案**：B

**考点分析**
- 操作系统保存通用寄存器

---


## 解题思路

### 第一步：理解中断保存分工
硬件保存：PC、PSW（自动）
OS保存：通用寄存器（软件）

### 第二步：逐项分析
**A. PC ❌**
```
PC由硬件自动保存
中断发生时，硬件自动压栈断点地址
不需要OS保存
结论：硬件保存
```

**B. 通用寄存器 ✅**
```
通用寄存器由OS保存
进入中断处理程序后，OS手动保存现场
保存到进程的内核栈
结论：OS保存
```

**C. TLB ❌**
```
TLB是快表，硬件管理
中断不影响TLB内容
不需要保存
结论：不保存
```

**D. Cache ❌**
```
Cache由硬件管理
中断不清空Cache
不需要保存
结论：不保存
```

### 第三步：确认答案
只有通用寄存器由OS保存 → 答案：**B**

## 易错点分析

### 易错点1：认为PC由OS保存
❌ 错误：中断保存现场都是OS完成的
✅ 正确：
- 硬件自动保存：PC、PSW
- OS手动保存：通用寄存器

### 易错点2：不理解保存分工原因
| 内容 | 保存者 | 原因 |
|------|--------|------|
| PC | 硬件 | 必须立即保存返回地址 |
| PSW | 硬件 | 包含中断屏蔽位等 |
| 通用寄存器 | OS | 根据需要选择保存 |
| TLB/Cache | 无 | 硬件自动管理 |
