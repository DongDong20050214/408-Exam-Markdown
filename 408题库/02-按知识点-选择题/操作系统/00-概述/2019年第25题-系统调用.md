**题干**

下列关于系统调用的叙述中，正确的是（）。I. 在执行系统调用服务程序的过程中，CPU 处于内核态  II. 操作系统通过提供系统调用避免用户程序直接访问外设  III. 不同的操作系统为应用程序提供了统一的系统调用接口  IV. 系统调用是操作系统内核为应用程序提供服务的接口

A. 仅 I、IV  B. 仅II、III  C. 仅 I、II、IV  D. 仅 I、III、IV

**答案**：C

**考点分析**
- I、II、IV正确，III错误（不同OS接口不统一）

---


## 解题思路

### 第一步：理解系统调用特点
系统调用：应用程序请求OS服务的接口

### 第二步：逐项分析
**I. 执行时CPU处于内核态 ✅**
```
系统调用在内核态执行
需要访问内核数据和特权指令
陷入内核后才执行服务程序
结论：正确
```

**II. 避免用户程序直接访问外设 ✅**
```
用户程序不能直接操作硬件
必须通过系统调用请求OS
OS统一管理硬件资源
结论：正确
```

**III. 不同OS提供统一接口 ❌**
```
不同OS的系统调用接口不同
Windows：Win32 API
Linux：POSIX标准（但也有差异）
Unix：不同版本接口不同
结论：错误
```

**IV. 是OS为应用提供的接口 ✅**
```
系统调用是OS提供的服务接口
应用程序通过它请求OS服务
如文件操作、进程管理等
结论：正确
```

### 第三步：确认答案
I、II、IV正确，III错误 → 答案：**C**

## 易错点分析

### 易错点1：认为系统调用接口统一
❌ 错误：系统调用是标准接口，所有OS都一样
✅ 正确：不同OS的系统调用接口差异很大

### 易错点2：混淆接口层次
| 接口 | 提供者 | 是否统一 |
|------|--------|----------|
| 系统调用 | OS内核 | ❌ 不统一 |
| POSIX | 标准规范 | 部分统一 |
| C标准库 | 编译器 | 较统一 |
