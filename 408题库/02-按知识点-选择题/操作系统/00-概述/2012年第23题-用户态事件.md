## 2012年第23题

**元数据**
- 年份：[../../01-真题原卷/2012年408真题#23|2012年第23题](../../01-真题原卷/2012年408真题#23|2012年第23题.md)
- 科目：#操作系统
- 知识点：#操作系统/概述/用户态核心态
- 难度：⭐⭐⭐
- 关联知识：[2023年第26题-态转换](..\01-进程管理\2023年第26题-态转换.md)

**题干**

下列选项中，不可能在用户态发生的事件是（）。

A. 系统调用  
B. 外部中断  
C. 进程切换  
D. 缺页

**答案**：C

**考点分析**
- A可能：用户态发起系统调用，然后切换到核心态
- B可能：外部中断随时可能发生
- C不可能：进程切换必须在核心态进行
- D可能：缺页异常在用户态触发

---


## 解题思路

### 第一步：理解"发生"vs"执行"
发生：事件在哪个态触发
执行：事件处理在哪个态进行

### 第二步：逐项分析
**A. 系统调用 ✅可能**
```
系统调用在用户态发起
用户程序执行trap指令
陷入内核态后由OS执行
结论：可以在用户态发生
```

**B. 外部中断 ✅可能**
```
外部中断由硬件产生
不管CPU在用户态还是内核态
中断都可能发生
结论：可以在用户态发生
```

**C. 进程切换 ❌不可能**
```
进程切换必须在内核态执行
需要保存/恢复PCB
需要切换页表等特权操作
结论：不可能在用户态发生
```

**D. 缺页 ✅可能**
```
缺页异常在访问页面时触发
用户程序访问不在内存的页面
触发缺页异常，然后陷入内核处理
结论：可以在用户态发生
```

### 第三步：确认答案
只有进程切换不可能在用户态发生 → 答案：**C**

## 易错点分析

### 易错点1：混淆"发生"和"处理"
❌ 错误：系统调用在内核态执行，所以不能在用户态发生
✅ 正确：系统调用在用户态发起（发生），在内核态执行（处理）

### 易错点2：认为中断只能在内核态发生
| 事件 | 发生位置 | 处理位置 | 能否用户态发生 |
|------|----------|----------|----------------|
| 系统调用 | 用户态 | 内核态 | ✅ 是 |
| 外部中断 | 任意态 | 内核态 | ✅ 是 |
| 缺页异常 | 用户态 | 内核态 | ✅ 是 |
| 进程切换 | 内核态 | 内核态 | ❌ 否 |
