**题干**

执行系统调用的过程包括如下主要操作：① 返回用户态  ② 执行陷入（trap）指令  ③ 传递系统调用参数  ④ 执行相应的服务程序

正确的执行顺序是（）。A. ②→③→①→④  B. ②→④→③→①  C. ③→②→④→①  D. ③→④→②→①

**答案**：C

**考点分析**
- 顺序：传参→陷入→服务→返回

---


## 解题思路

### 第一步：理解系统调用流程
传参→陷入→执行→返回

### 第二步：分析各步骤
**③ 传递系统调用参数（第1步）**
```
用户程序准备参数
通过寄存器或栈传递
在用户态完成
```

**② 执行陷入指令（第2步）**
```
执行trap/int指令
触发软中断
切换到内核态
```

**④ 执行服务程序（第3步）**
```
内核态执行系统调用
完成请求的OS服务
获取结果
```

**① 返回用户态（第4步）**
```
服务完成后返回
恢复用户程序现场
切换回用户态继续执行
```

### 第三步：确认答案
顺序：③→②→④→① → 答案：**C**

## 易错点分析

### 易错点1：认为先陷入再传参
❌ 错误：先陷入内核态再传参数
✅ 正确：必须在用户态准备好参数，再陷入

### 易错点2：混淆返回时机
| 步骤 | 操作 | 状态 |
|------|------|------|
| ③传参 | 准备参数 | 用户态 |
| ②陷入 | trap指令 | 用户→内核 |
| ④服务 | 执行调用 | 内核态 |
| ①返回 | 恢复现场 | 内核→用户 |
