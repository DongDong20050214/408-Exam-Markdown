**题干**

下列选项中，操作系统提供给应用程序的接口是（）。A. 系统调用  B. 中断  C. 库函数  D. 原语

**答案**：A

**考点分析**
- A：系统调用是OS提供的接口

---


## 解题思路

### 第一步：理解操作系统接口
操作系统为应用程序提供的接口是应用程序请求OS服务的通道。

### 第二步：逐项分析
**A. 系统调用 ✅**
```
定义：应用程序请求OS服务的唯一接口
例如：open()、read()、write()、fork()
用户程序通过系统调用陷入内核态，由OS执行
结论：这是正确答案
```

**B. 中断 ❌**
```
中断是硬件/软件向CPU发送的信号
不是应用程序主动调用的接口
由OS响应中断，不是应用程序接口
结论：不是接口
```

**C. 库函数 ❌**
```
库函数是编程语言提供的函数
部分库函数内部会调用系统调用
但库函数本身不是OS提供的接口
结论：不是OS接口
```

**D. 原语 ❌**
```
原语是OS内核提供的底层操作
是OS内部使用的，不对外开放
应用程序不能直接调用原语
结论：不是应用程序接口
```

### 第三步：确认答案
系统调用是OS提供给应用程序的唯一接口 → 答案：**A**

## 易错点分析

### 易错点1：混淆库函数和系统调用
❌ 错误理解：printf()、scanf()是OS提供的接口
✅ 正确理解：库函数是编程语言（如C标准库）提供的用户态函数，printf()内部调用write()系统调用陷入内核。系统调用是操作系统提供的内核态接口（如open/read/write/fork），是应用程序请求OS服务的唯一合法通道。库函数可以不调用系统调用（如strlen纯计算），但系统调用必然涉及特权指令和内核态切换

### 易错点2：认为中断是接口
| 概念 | 方向 | 主动性 | 是否接口 |
|------|------|--------|----------|
| 系统调用 | 应用→OS | 主动请求 | ✅ 是 |
| 中断 | 硬件→CPU | 被动响应 | ❌ 否 |
| 库函数 | 应用内部 | 封装调用 | ❌ 否 |
