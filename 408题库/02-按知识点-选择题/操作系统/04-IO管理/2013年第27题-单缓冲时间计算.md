- 科目：#操作系统

- 关联知识：[[知识库/操作系统/缓冲区]]

**题干**

进程从外设读入并分析 2 个数据块的最短时间是（）。A. 200  B. 295  C. 300  D. 390

**答案**：B

**考点分析**
- 第1块：100+5+90=195
- 第2块：max(100,90)+5=100
- 总计：195+100=295

---


## 解题思路

### 第一步：理解题目核心
题目需补充完整条件：设备读入一块数据时间T=100μs，CPU处理数据时间C=5μs，数据传送到用户区时间M=90μs。单缓冲情况下，CPU与I/O可并行工作。

### 第二步：逐项分析

**A. 200 ❌**
```
详细分析：误认为所有时间完全串行相加
判断依据：若200=100+100，说明忽略了并行性和处理时间
结论：未考虑单缓冲的并行特性
```

**B. 295 ✅**
```
详细分析：单缓冲允许CPU处理与I/O操作并行执行
判断依据：
  第1块数据：
    - 读入缓冲区：100μs（I/O设备→缓冲区）
    - 数据送用户区：90μs（缓冲区→用户区，此时可并行读第2块，但第1块还未处理完）
    - CPU处理：5μs
    小计：100+90+5=195μs
    
  第2块数据（关键）：
    - 第1块处理完后，第2块已在等待
    - 读入时间100μs与第1块的max(处理5μs+传送90μs)=95μs并行
    - 因此第2块开始时已读完，只需处理时间
    - 实际耗时：max(100, 90)+5=100μs（读入与上一块传送并行，取较大值）
    小计：100μs
    
  总时间：195+100=295μs
结论：正确答案
```

**C. 300 ❌**
```
详细分析：可能误算为2×(100+90+5)-部分并行
判断依据：390-90=300，但这种减法缺乏理论依据
结论：对并行时机理解有误
```

**D. 390 ❌**
```
详细分析：完全串行处理，未利用缓冲区并行特性
判断依据：2×(100+90+5)=390，忽略了单缓冲的流水线效果
结论：未理解单缓冲的作用
```

### 第三步：确认答案
单缓冲第1块需完整时间195μs，第2块因并行读入只需100μs，总计295μs → 答案：**B**

## 易错点分析

### 易错点1：混淆无缓冲与单缓冲的时间计算
❌ 错误理解：单缓冲只是内存区域，不影响时间计算
✅ 正确理解：无缓冲时读入、传送、处理必须严格串行（读完才能传，传完才能处理下一块），2块需2×195=390μs。单缓冲允许流水线：当CPU处理第1块时（耗时95μs），设备可同时读取第2块（耗时100μs），这100μs与95μs并行，第2块只需等待max(100,95)=100μs即可开始处理。缓冲区的价值不是空间而是时间并行

### 易错点2：单缓冲与双缓冲的并行差异
| 无缓冲 | 单缓冲 | 双缓冲 | 并行程度 |
|--------|--------|--------|---------|
| 2×195=390μs | 195+100=295μs | 195+5=200μs | 无缓冲最慢 |
| 完全串行 | 部分并行（第2块读入可与第1块传送并行） | 完全流水线（读、处理、传送全并行） | 双缓冲最快 |
| CPU等I/O，I/O等CPU | 一个缓冲区限制并行度 | 两个缓冲区交替使用 | 缓冲区越多并行度越高 |
