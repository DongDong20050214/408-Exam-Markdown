**题干**

下列关于SPOOLing技术的叙述中，错误的是（）。

A. 需要外存的支持  
B. 需要多道程序设计技术的支持  
C. 可以让多个作业共享一台独占设备  
D. 由用户作业控制设备与输入/输出井之间的数据传送

**答案**：D

**考点分析**
- A正确：SPOOLing需要磁盘作为缓冲区(输入井/输出井)
- B正确：需要多道程序技术支持并发处理
- C正确：通过假脱机技术实现设备共享
- D错误：数据传送由**系统进程**控制，不是用户作业
- SPOOLing的核心：将独占设备改造为共享设备

---


## 解题思路

### 第一步：理解题目核心
SPOOLing（Simultaneous Peripheral Operations On-Line，假脱机技术）将独占设备改造为共享设备，核心是用磁盘作为缓冲，通过系统进程管理输入/输出井。

### 第二步：逐项分析

**A. 需要外存的支持 ✅**
```
详细分析：SPOOLing必须使用磁盘作为输入井和输出井
判断依据：
  - 输入井：存储输入设备的数据
  - 输出井：缓存要输出到设备的数据
  - 磁盘容量大，可缓存多个作业的数据
  - 内存缓冲区太小，无法支持多作业共享
结论：外存（磁盘）是SPOOLing的物理基础
```

**B. 需要多道程序设计技术的支持 ✅**
```
详细分析：SPOOLing需要多个进程并发协作
判断依据：
  - 输入进程：从输入设备读取数据到输入井
  - 输出进程：从输出井取数据送到输出设备
  - 用户作业：与输入井、输出井交互
  - 多道程序允许这些进程并发执行，实现异步I/O
结论：多道程序是SPOOLing的软件基础
```

**C. 可以让多个作业共享一台独占设备 ✅**
```
详细分析：这是SPOOLing的核心目标
判断依据：
  - 传统打印机是独占设备（一次只能一个作业使用）
  - SPOOLing将打印任务先送入输出井，多个作业可同时提交
  - 打印进程统一从输出井取任务打印
  - 用户感觉打印机是"共享"的，实际是时分复用
结论：将独占设备改造为共享设备是SPOOLing的设计目的
```

**D. 由用户作业控制设备与输入/输出井之间的数据传送 ❌**
```
详细分析：数据传送由系统进程控制，而非用户作业
判断依据：
  - 输入进程（系统守护进程）：设备→输入井
  - 输出进程（系统守护进程）：输出井→设备
  - 用户作业只与输入井/输出井交互，无需知道物理设备细节
  - 这正是SPOOLing实现设备共享的关键：用户与设备解耦
结论：控制权在系统而非用户，此选项错误
```

### 第三步：确认答案
SPOOLing由系统进程管理设备与井的数据传送，用户作业不参与物理设备控制 → 答案：**D**

## 易错点分析

### 易错点1：误认为用户作业控制数据传送
❌ 错误理解：用户程序调用输出函数时，直接控制打印机工作
✅ 正确理解：用户调用print()时数据只是写入磁盘上的输出井（逻辑输出），函数立即返回，用户作业继续执行无需等待。系统的输出守护进程（spooler）后台运行，从输出井读取数据控制打印机物理输出。用户作业完全不知道打印机何时工作、是否空闲，这种解耦正是SPOOLing实现设备共享的关键机制

### 易错点2：SPOOLing与一般缓冲的区别
| 一般缓冲 | SPOOLing技术 | 核心差异 |
|---------|-------------|---------|
| 内存中的缓冲区 | 磁盘上的输入井/输出井 | SPOOLing容量更大 |
| 临时缓存数据块 | 缓存完整作业的输入/输出 | SPOOLing支持整个作业暂存 |
| 加速单个I/O操作 | 实现设备共享 | SPOOLing改变设备使用模式 |
| 数据量小（KB级） | 数据量大（MB级） | SPOOLing用磁盘换空间 |
