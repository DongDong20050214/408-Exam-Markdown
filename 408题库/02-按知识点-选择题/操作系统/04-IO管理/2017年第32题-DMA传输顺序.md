# DMA数据传输顺序真题汇总

## 2017年第32题

**元数据**

**题干**

系统将数据从磁盘读到内存的过程包括以下操作，正确的执行顺序是（）。

① DMA控制器发出中断请求  
② 初始化DMA控制器并启动磁盘  
③ 从磁盘传输一块数据到内存缓冲区  
④ 执行"DMA结束"中断服务程序

A. ③→①→②→④  
B. ②→③→①→④  
C. ②→①→③→④  
D. ①→②→④→③

**答案**：B

**考点分析**
- 顺序：初始化DMA→数据传输→传输完成中断→中断服务
- DMA传输过程不需要CPU干预

---


## 解题思路

### 第一步：理解题目核心
DMA（Direct Memory Access，直接内存访问）允许设备直接与内存交换数据，无需CPU干预。理解DMA工作流程是关键。

### 第二步：逐项分析

**A. ③→①→②→④ ❌**
```
详细分析：未初始化就开始传输，违反硬件工作原理
判断依据：DMA控制器需要先知道传输参数（内存地址、数据量等）才能工作
结论：顺序错误，缺少初始化步骤
```

**B. ②→③→①→④ ✅**
```
详细分析：符合DMA标准工作流程
判断依据：
  步骤②：初始化DMA控制器并启动磁盘
    - CPU设置DMA寄存器：内存起始地址、传输字节数、传输方向
    - 启动磁盘控制器开始读取
    - CPU继续执行其他任务（DMA的关键优势）
  
  步骤③：从磁盘传输一块数据到内存缓冲区
    - DMA控制器接管总线
    - 磁盘→DMA控制器→内存，整个过程CPU不参与
    - 传输期间CPU可执行其他进程
  
  步骤①：DMA控制器发出中断请求
    - 数据块传输完成后，DMA向CPU发中断信号
    - 通知CPU：数据已就绪，可以处理
  
  步骤④：执行"DMA结束"中断服务程序
    - CPU响应中断，保存现场
    - 检查传输状态、处理数据、启动下一次传输等
结论：正确的执行顺序
```

**C. ②→①→③→④ ❌**
```
详细分析：初始化后立即中断，数据传输还未开始
判断依据：DMA中断应在数据传输完成后发出，而非初始化后
结论：中断时机错误
```

**D. ①→②→④→③ ❌**
```
详细分析：未初始化就发中断，逻辑混乱
判断依据：中断是传输完成的通知信号，不可能在传输前发生
结论：完全违背DMA工作原理
```

### 第三步：确认答案
DMA工作流程：初始化→数据传输（CPU不干预）→传输完成中断→中断处理 → 答案：**B**

## 易错点分析

### 易错点1：混淆中断触发时机
❌ 错误理解：初始化DMA后立即产生中断通知CPU
✅ 正确理解：中断是完成信号，在整个数据块传输完成后才发出。初始化DMA后CPU立即返回执行其他任务，DMA控制器接管总线独立完成数据传输（可能传输几千字节耗时几毫秒），传输完成后DMA才发出中断请求。如果初始化后就中断，DMA的核心价值"解放CPU"就荡然无存，还不如用程序直接I/O

### 易错点2：三种I/O控制方式的CPU参与度
| 程序直接I/O | 中断驱动I/O | DMA方式 | CPU占用 |
|-----------|-----------|--------|---------|
| CPU轮询等待设备就绪 | CPU响应中断处理每个字节 | CPU只参与初始化和结束处理 | DMA最省CPU |
| 每个字节CPU都参与 | 每次中断CPU介入 | 整块数据传输CPU不参与 | 程序I/O最浪费 |
| 传输1MB需100万次循环 | 传输1MB需100万次中断 | 传输1MB只需2次CPU干预（初始化+中断） | DMA适合大块数据 |
