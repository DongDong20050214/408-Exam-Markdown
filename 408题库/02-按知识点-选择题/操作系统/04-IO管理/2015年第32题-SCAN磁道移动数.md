## 2015年第32题

**元数据**
- 年份：[[../../01-真题原卷/2015年408真题#32|2015年第32题]]
- 科目：#操作系统
- 知识点：#操作系统/IO管理/磁盘调度
- 难度：⭐⭐⭐
- 关联知识：[[2021年第26题-磁盘调度]]

**题干**

某硬盘有 200 个磁道，磁道访问请求序列为 130, 42, 180, 15, 199，当前磁头位于第 58 号磁道并从外侧向内侧移动。按照 SCAN 调度方法处理完上述请求后，磁头移过的磁道数是（）。A. 208  B. 287  C. 325  D. 382

**答案**：B

**考点分析**
- SCAN先内侧：58→130→180→199，再外侧：199→42→15，总计287

---


## 解题思路

### 第一步：理解题目核心
SCAN算法（电梯调度算法）沿当前方向处理所有请求，到达边界后反向。当前磁头在58号磁道，从外侧向内侧移动（磁道号增大方向），请求序列为130, 42, 180, 15, 199。

### 第二步：逐项分析

**A. 208 ❌**
```
详细分析：此选项可能误认为只处理单向请求
判断依据：58→199的单向距离为141，不等于208
结论：计算方法错误
```

**B. 287 ✅**
```
详细分析：按SCAN算法完整计算移动路径
判断依据：
  第一阶段（向内侧，磁道号增大）：
    58→130（移动72）→180（移动50）→199（移动19）
    小计：72+50+19=141
  
  第二阶段（到达边界后反向，向外侧）：
    199→42（移动157）→15（移动27）
    小计：157+27=184
    
  **注意**：199不是最大磁道199，但因已无更大请求而折返
  
  总移动磁道数：141+184=287
结论：正确答案
```

**C. 325 ❌**
```
详细分析：可能误算为先到0再到199的总距离
判断依据：58+199=257，或其他错误组合
结论：路径规划错误
```

**D. 382 ❌**
```
详细分析：可能误以为必须到达磁道0或199边界
判断依据：若58→199→0→15，则141+199+15=355，仍不等于382
结论：对SCAN算法理解有误
```

### 第三步：确认答案
SCAN算法沿当前方向处理完所有请求后折返，磁头实际移动路径为58→130→180→199→42→15，总移动磁道数287 → 答案：**B**

## 易错点分析

### 易错点1：误认为必须到达磁盘边界
❌ 错误理解：SCAN算法必须移动到磁道0或最大磁道199才能折返
✅ 正确理解：SCAN在当前方向**没有更多请求时**即可折返，不必到达物理边界。本题199之后无更大请求，直接折返处理42和15

### 易错点2：C-SCAN与SCAN的区别
| SCAN（电梯算法） | C-SCAN（循环扫描） | 核心区别 |
|-----------------|-------------------|---------|
| 到达边界后反向服务 | 到达边界后直接返回起点 | SCAN双向服务，C-SCAN单向服务 |
| 58→199→42→15 | 58→199→0→15→42 | SCAN原路返回，C-SCAN跳回起点 |
| 每个磁道最多经过2次 | 磁道只在一个方向被服务 | C-SCAN避免中间磁道等待时间过长 |
