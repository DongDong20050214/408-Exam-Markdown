- 科目：#操作系统

**题干**

下列选项中，不能改善磁盘设备I/O性能的是（）。

A. 重排I/O请求次序  
B. 在一个磁盘上设置多个分区  
C. 预读和滞后写  
D. 优化文件物理块的分布

**答案**：B

**考点分析**
- A正确：磁盘调度算法(SCAN、C-SCAN等)
- B错误：分区不改善性能，只是逻辑划分
- C正确：缓存策略提高性能
- D正确：连续分配减少寻道时间

---


## 解题思路

### 第一步：理解题目核心
题目要求找出**不能**改善磁盘I/O性能的措施。磁盘I/O性能主要受寻道时间、旋转延迟、传输时间三个因素影响。

### 第二步：逐项分析

**A. 重排I/O请求次序 ✅**
```
详细分析：通过磁盘调度算法（SCAN、C-SCAN、SSTF等）优化磁头移动路径
判断依据：
  - 减少寻道时间（磁盘性能瓶颈之一）
  - 例如SCAN算法按磁道号顺序处理请求，避免磁头频繁往复
结论：能改善性能
```

**B. 在一个磁盘上设置多个分区 ❌**
```
详细分析：分区是逻辑划分，不改变磁盘物理特性
判断依据：
  - 分区只是在磁盘上划分不同的逻辑区域（C盘、D盘等）
  - 磁头仍需物理移动，寻道时间不变
  - 数据仍在同一磁盘上，传输速度不变
  - 分区的目的是文件系统隔离、数据管理，与性能无关
结论：不能改善性能，是本题答案
```

**C. 预读和滞后写 ✅**
```
详细分析：缓存策略减少实际磁盘访问次数
判断依据：
  - 预读（Read Ahead）：提前读取后续数据到内存，利用空间局部性
  - 滞后写（Delayed Write）：数据先写入缓存，批量写回磁盘
  - 减少I/O次数是性能优化的核心手段
结论：能改善性能
```

**D. 优化文件物理块的分布 ✅**
```
详细分析：合理安排数据存储位置减少磁头移动
判断依据：
  - 连续分配：文件占用连续磁盘块，减少寻道次数
  - 簇优化：相关文件放在相近位置
  - 磁盘碎片整理：使文件物理连续
结论：能改善性能
```

### 第三步：确认答案
A、C、D都通过减少寻道时间或I/O次数改善性能，只有B分区是逻辑划分，不涉及物理优化 → 答案：**B**

## 易错点分析

### 易错点1：误认为分区能提升性能
❌ 错误理解：多个分区可以"分散"数据，降低磁盘负载
✅ 正确理解：分区只是逻辑概念，所有分区共享同一套磁头和盘片，物理性能不变。若要真正提升性能，需使用**多个物理磁盘**（如RAID）

### 易错点2：分区与RAID的本质区别
| 分区（Partition） | RAID（磁盘阵列） | 性能差异 |
|------------------|-----------------|---------|
| 单磁盘逻辑划分 | 多物理磁盘组合 | 分区无性能提升 |
| 共享磁头、转速 | 并行读写、冗余 | RAID可显著提升I/O性能 |
| 文件系统隔离 | 数据条带化（striping） | RAID 0读写速度翻倍 |
