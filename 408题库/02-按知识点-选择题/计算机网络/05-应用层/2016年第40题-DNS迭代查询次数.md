# DNS查询次数真题汇总

## 2016年第40题

**元数据**
- 年份：[[../../01-真题原卷/2016年408真题#40|2016年第40题]]
- 科目：#计算机网络
- 知识点：#计算机网络/应用层/DNS #迭代查询
- 难度：⭐⭐⭐
- 关联知识：[[知识库/计算机网络/DNS]]

**题干**

设所有域名服务器均采用迭代查询方式进行域名解析。当H4访问规范域名为www.abc.xyz.com的网站时，域名服务器201.1.1.1在完成该域名解析过程中，可能发出DNS查询的最少和最多次数分别是（）。

A. 0, 3  
B. 1, 3  
C. 0, 4  
D. 1, 4

**答案**：C

**考点分析**
- 迭代查询：本地DNS服务器201.1.1.1向根、顶级、权威DNS依次查询
- 域名www.abc.xyz.com需要查询：
  1. 根DNS(.) → 返回.com服务器
  2. .com DNS → 返回.xyz.com服务器
  3. .xyz.com DNS → 返回.abc.xyz.com服务器
  4. .abc.xyz.com DNS → 返回www的A记录
- 最多4次查询
- 最少0次：缓存中已有完整记录

---

## 解题思路

### 第一步：理解题目核心
DNS迭代查询次数计算（含缓存场景）

### 第二步：逐项分析

已知：域名www.abc.xyz.com，迭代查询

**域名层次结构**：
根(.) → com → xyz.com → abc.xyz.com → www

**完整迭代查询流程**（无缓存）：
1. 查根DNS → 返回.com地址
2. 查.com DNS → 返回.xyz.com地址
3. 查.xyz.com DNS → 返回.abc.xyz.com地址
4. 查.abc.xyz.com DNS → 返回www的A记录
**最多4次查询**

**最佳情况**（完全缓存）：
- 缓存中已有www.abc.xyz.com的IP地址
- 无需发起任何查询
**最少0次查询**

**选项验证**：
- A. 0,3 ❌（最多应为4次）
- B. 1,3 ❌（最少可以是0，最多应为4）
- C. 0,4 ✅（最少0次缓存命中，最多4次完整查询）
- D. 1,4 ❌（最少可以是0）

### 第三步：确认答案
迭代查询最少0次（缓存），最多4次（完整查询），答案C

## 易错点分析

### 易错点1：域名层次计算错误
❌ 错误理解：认为只需查询3次（根→.com→最终权威）
✅ 正确理解：xyz.com和abc.xyz.com是两个区域，需分别查询，共4次

### 易错点2：查询次数对比
| 缓存命中 | 完全无缓存 | 查询次数 |
|---------|-----------|---------|
| 0次 | 4次 | 根→com→xyz.com→abc.xyz.com |
