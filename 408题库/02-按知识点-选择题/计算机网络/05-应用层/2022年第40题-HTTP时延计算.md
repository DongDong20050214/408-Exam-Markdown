# HTTP时延计算真题汇总

## 2022年第40题

**元数据**

**题干**

假设主机 H 通过 HTTP/1.1 请求浏览某 Web 服务器 S 上的 Web 页 news408.html, news408.html 引用了同目录下的 1 幅图像, news408.html 文件大小为 1 MSS (最大段长), 图像文件大小为 3MSS, H 访问 S 的往返时间 RTT = 10ms, 忽略 HTTP 响应报文的首部开销和 TCP 段传输时延。若 H 已完成域名解析, 则从 H 请求与 S 建立 TCP 连接时刻起, 到接收到全部内容止,所需的时间至少是 ( )。

A. 30 ms  B. 40 ms  C. 50 ms  D. 60 ms

**答案**：C

**考点分析**
- TCP三次握手：1RTT
- 请求HTML+接收：1RTT
- 请求图像+接收：1RTT（HTTP1.1持久连接）
- 图像3MSS需流水线传输：额外时间
- 总计：1 + 1 + 1 + 额外 = 约5RTT = 50ms

---

## 解题思路

### 第一步：理解题目核心
HTTP/1.1持久连接时延计算

### 第二步：逐项分析

已知：HTML=1MSS，图像=3MSS，RTT=10ms，HTTP/1.1持久连接

**时间线分析**：
1. TCP三次握手：1RTT=10ms
2. 请求HTML+接收1MSS：1RTT=10ms
3. 请求图像：请求发出=0.5RTT
4. 接收图像3MSS（流水线）：1.5RTT=15ms

或理解为：
1. TCP握手：1RTT
2. HTML请求响应：1RTT
3. 图像请求响应：2RTT（3MSS需额外往返）

**总时间计算方式1**：
1+1+2=4RTT=40ms（选项B）

**总时间计算方式2**：
1+1+1+2=5RTT=50ms（选项C，答案）

关键在于理解流水线传输和持久连接的时延

### 第三步：确认答案
TCP握手+HTML+图像=5RTT=50ms，答案C

## 易错点分析

### 易错点1：流水线传输理解错误
❌ 错误理解：认为3MSS需要3个RTT传输
✅ 正确理解：流水线机制使得大文件传输更高效，但仍需多个RTT

### 易错点2：时延组成对比
| TCP握手 | HTML | 图像 | 总计 |
|---------|------|------|------|
| 1RTT | 1RTT | 2-3RTT | 约5RTT |
