**题干**

主机甲通过 128kbps 卫星链路, 采用滑动窗口协议向主机乙发送数据, 链路单向传播延迟为 250ms, 帧长为 1000 字节。不考虑确认帧的开销, 为使链路利用率不小于 80%, 帧序号的比特数至少是 ( )。A. 3  B. 4  C. 7  D. 8

**答案**：B

**考点分析**
- 窗口大小W≥8×0.8/0.8=8，需4位

---

## 解题思路

### 第一步：理解题目核心
链路利用率≥80%需要的窗口大小，再确定序号位数

### 第二步：逐项分析

**计算帧传输时延：**
- T_frame = 1000B×8 / 128kbps = 8000 / 128000 = 62.5ms

**计算往返时延RTT：**
- RTT = 2 × 250ms = 500ms

**计算RTT内可发帧数：**
- N = RTT / T_frame = 500ms / 62.5ms = 8帧

**利用率公式：**
- 利用率 U = W / (1 + 2a)，其中a = 传播时延/传输时延 = 250/62.5 = 4
- 要U ≥ 0.8：W / (1+8) ≥ 0.8 → W ≥ 7.2
- 或直接：U = W×T_frame / (W×T_frame + RTT)，要U≥0.8需W≥8

**序号位数：**
- 如果是GBN：窗口W≤2^n-1，需2^n-1≥8 → 2^n≥9 → n≥4
- 如果是SR：窗口W≤2^(n-1)，需2^(n-1)≥8 → 2^n≥16 → n≥4

**A. 3** ❌  2^3=8，GBN最大窗口7不够
**B. 4** ✅  2^4=16，GBN最大窗口15或SR最大窗口8都满足
**C. 7** ❌  过大
**D. 8** ❌  过大

### 第三步：确认答案
B。需要4位序号

## 易错点分析

### 易错点1：混淆链路利用率和窗口大小的关系
❌ 错误理解：链路利用率80%就需要窗口大小为0.8×某个值
✅ 正确理解：链路利用率U=发送窗口W×帧长/(带宽×往返时延RTT)。要使U≥80%，需W≥0.8×带宽×RTT/帧长。本题带宽128kbps，RTT=2×250ms=500ms，帧长1000字节=8000位，计算得W≥0.8×128000×0.5/8000=8。发送窗口至少8，对应需要4位序号（2^4=16>8），选择重传协议发送窗口最大为2^(n-1)，4位时最大窗口8满足要求

### 易错点2：链路利用率的计算公式
| 参数 | 本题数值 | 单位换算 |
|------|---------|---------|
| 带宽 | 128kbps | 128000 bps |
| 单向传播延迟 | 250ms | 0.25s |
| 往返时延RTT | 500ms | 0.5s |
| 帧长 | 1000字节 | 8000位 |
| 窗口大小W | ≥8帧 | 需4位序号 |
