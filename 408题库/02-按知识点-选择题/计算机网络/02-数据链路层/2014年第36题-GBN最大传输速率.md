**题干**

甲的发送窗口尺寸为1000，数据帧长为1000字节，信道带宽为100Mbps，单向传播延迟是50ms，则甲可以达到的最大平均数据传输速率约为（）。

A. 10Mbps  
B. 20Mbps  
C. 80Mbps  
D. 100Mbps

**答案**：C

**考点分析**
- 一帧传输时延：1000×8/100M=0.08ms
- RTT=2×50=100ms
- 一个RTT可发送帧数：100/0.08=1250帧
- 窗口限制最多1000帧
- 信道利用率：1000/1250=0.8
- 最大速率：100Mbps×0.8=80Mbps

---

## 解题思路

### 第一步：理解题目核心
信道利用率受窗口大小限制，实际速率=带宽×利用率

### 第二步：逐项分析

**计算关键参数：**
- 一帧传输时延：T_frame = 1000B×8 / 100Mbps = 8000/10⁸ = 0.08ms
- 往返时延：RTT = 2×50ms = 100ms
- RTT内理论可发帧数：N = RTT / T_frame = 100ms / 0.08ms = 1250帧

**窗口限制分析：**
- 发送窗口W = 1000帧 < 1250帧
- 窗口成为瓶颈，无法充分利用带宽
- 信道利用率：U = W / N = 1000 / 1250 = 0.8 = 80%

**最大速率：**
实际速率 = 100Mbps × 0.8 = 80Mbps

**A. 10Mbps** ❌  严重低估
**B. 20Mbps** ❌  利用率算成20%
**C. 80Mbps** ✅  正确的窗口受限速率
**D. 100Mbps** ❌  忽略窗口限制

### 第三步：确认答案
C。窗口限制下最大速率80Mbps

## 易错点分析

### 易错点1：误认为发送窗口不是瓶颈
❌ 错误理解：认为RTT内可发1250帧,窗口1000帧足够,利用率100%,最大速率100Mbps
✅ 正确理解：信道利用率=min(窗口大小,RTT内可发帧数)/RTT内可发帧数。窗口1000<1250,窗口成为瓶颈,利用率=1000/1250=0.8,最大速率=100×0.8=80Mbps

### 易错点2：GBN信道利用率计算
| 理想情况 | 窗口受限 | 公式 |
|-------|-------|------|
| W≥RTT/T_frame | W<RTT/T_frame | 利用率=W/(RTT/T_frame) |
| 充分利用带宽 | 窗口太小限制性能 | 实际速率=带宽×利用率 |
