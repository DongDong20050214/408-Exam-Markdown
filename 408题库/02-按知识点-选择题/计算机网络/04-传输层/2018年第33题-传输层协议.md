# 题目

下列 TCP/IP 应用层协议中，可以使用传输层无连接服务的是（）。

A. FTP 

B. DNS 

C. SMTP 

D. HTTP 

# 答案

B

# 考点分析

本题考查应用层协议与传输层协议的对应关系。关键知识点：
1. 传输层无连接服务指UDP协议，特点是无需建立连接、开销小、传输快但不可靠
2. DNS(域名系统)主要使用UDP端口53进行域名解析查询，因为查询数据量小、对实时性要求高，丢失后可重传。当响应超过512字节时才使用TCP
3. FTP使用TCP的20和21端口，需要可靠的文件传输
4. SMTP使用TCP的25端口，邮件传输必须保证可靠性
5. HTTP使用TCP的80端口(HTTPS用443)，网页数据传输需要可靠连接

## 解题思路

### 第一步：理解题目核心
应用层协议与传输层协议对应关系

### 第二步：逐项分析

**A. FTP（文件传输协议）** ❌
- 使用TCP端口20（数据）和21（控制）
- 需要可靠传输，不能用UDP

**B. DNS（域名系统）** ✅
- **主要使用UDP端口53**进行查询
- 查询数据量小，对实时性要求高
- 响应超512字节时才用TCP
- 可以使用传输层无连接服务（UDP）

**C. SMTP（简单邮件传输协议）** ❌
- 使用TCP端口25
- 邮件传输必须可靠，不能用UDP

**D. HTTP（超文本传输协议）** ❌
- 使用TCP端口80（HTTPS用443）
- 网页数据传输需要可靠连接

### 第三步：确认答案
只有DNS可以使用UDP（无连接服务），答案B

## 易错点分析

### 易错点1：DNS传输层协议混淆
❌ 错误理解：认为DNS只能用TCP或必须用TCP
✅ 正确理解：DNS通常用UDP（快速查询），仅超长响应或区域传送时用TCP

### 易错点2：协议对应对比
| 协议 | 端口 | 传输层 |
|------|------|--------|
| FTP | 20/21 | TCP |
| DNS | 53 | UDP/TCP |
| SMTP | 25 | TCP |
| HTTP | 80 | TCP |
