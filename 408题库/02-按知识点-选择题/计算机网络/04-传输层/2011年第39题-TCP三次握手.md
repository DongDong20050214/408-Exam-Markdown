**题干**

主机甲向主机乙发送一个（SYN = 1, seq = 11220）的 TCP 段，期望与主机乙建立 TCP 连接，若主机乙接受该连接请求，则主机乙向主机甲发送的正确的 TCP 段可能是（）。A. (SYN = 0, ACK = 0, seq = 11221, ack = 11221)  B. (SYN = 1, ACK = 1, seq = 11220, ack = 11220)  C. (SYN = 1, ACK = 1, seq = 11221, ack = 11221)  D. (SYN = 0, ACK = 0, seq = 11220, ack = 11220)

**答案**：C

**考点分析**
- 三次握手第二步：SYN=1,ACK=1,ack=seq+1

---

## 解题思路

### 第一步：理解题目核心
三次握手第二步(SYN+ACK)：SYN=1, ACK=1, seq=乙的初始序号, ack=甲的seq+1

### 第二步：逐项分析

**甲发送：SYN=1, seq=11220**

**乙的正确响应(第二次握手)：**
- SYN=1(仍在握手阶段)
- ACK=1(确认字段有效)
- seq=乙自己选择的初始序号(可以是任意值)
- ack=11220+1=11221(确认收到甲的SYN)

**A. SYN=0, ACK=0** ❌  握手期间SYN和ACK应为1
**B. seq=11220, ack=11220** ❌  ack应该是11221
**C. SYN=1, ACK=1, seq=11221, ack=11221** ✅  完全正确
**D. SYN=0, ACK=0** ❌  标志位错误

### 第三步：确认答案
C。第二次握手SYN=1, ACK=1, ack=原seq+1

## 易错点分析

### 易错点1：常见误区
❌ 错误理解：认为第二次握手的seq应该等于甲的seq，或者认为ack应该等于甲的seq而不是seq+1。混淆SYN和ACK标志位的设置规则。
✅ 正确理解：TCP三次握手第二次握手(SYN+ACK段)的正确格式：①SYN=1表示这是连接请求的响应段，仍在握手阶段 ②ACK=1表示确认字段有效 ③seq字段是乙自己选择的初始序列号，与甲无关(可以是任意值，题目选项中是11221) ④ack=甲的seq+1=11220+1=11221(确认收到甲的SYN，期望下一个序号)。因此正确答案C：SYN=1, ACK=1, seq=11221(乙的初始序号), ack=11221(确认甲的SYN)。选项A/D中SYN=0或ACK=0都不符合第二次握手要求。
