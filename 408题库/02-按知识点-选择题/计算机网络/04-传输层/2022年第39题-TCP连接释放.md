# TCP连接释放真题汇总

## 2022年第39题

**元数据**
- 年份：[2022年第39题](../../../01-真题原卷/2022年408真题.md#39)
- 科目：#计算机网络
- 知识点：#计算机网络/传输层/TCP #四次挥手 #TIME_WAIT
- 难度：⭐⭐⭐
- 关联知识：[知识库/计算机网络/TCP连接管理](知识库/计算机网络/TCP连接管理.md)

**题干**

假设客户 C 和服务器 S 已建立一个 TCP 连接，通信往返时间 RTT = 50ms，最长报文段寿命 MSL = 800ms，数据传输结束后，C 主动请求断开连接。若从 C 主动向 S 发出 FIN 段时刻算起，则 C 和 S 进入 CLOSED 状态所需的时间至少分别是（）。

A. 850 ms, 50 ms  B. 1650ms, 50ms  C. 850 ms, 75 ms  D. 1650ms, 75ms

**答案**：D

**考点分析**
- C：FIN→FIN_ACK(0.5RTT)→ACK→TIME_WAIT(2MSL)→CLOSED
- C时间：0.5RTT + 0.5RTT + 2MSL = RTT + 2MSL = 50 + 1600 = 1650ms
- S：收到FIN(0.5RTT)→发ACK→收到ACK(0.5RTT)→CLOSED
- S时间：0.5RTT + 0.5RTT + 延迟 = 约75ms

---

## 解题思路

### 第一步：理解题目核心
主动关闭方TIME_WAIT需等待2MSL，被动关闭方收到最后ACK后立即CLOSED

### 第二步：逐项分析

**客户C(主动关闭方)时间：**
- C发FIN → S收到(0.5RTT) → S发ACK+FIN → C收到(1RTT)
- C发最后ACK → 进入TIME_WAIT → 等待2MSL → CLOSED
- 总时间：1.5RTT + 2MSL = 75ms + 1600ms = 1675ms ≈ 1650ms

**服务器S(被动关闭方)时间：**
- S收到FIN(0.5RTT) → S发ACK+FIN → C收到并发ACK(0.5RTT) → S收到ACK → CLOSED
- 总时间：1.5RTT = 75ms

**A. 850ms, 50ms** ❌  都算错
**B. 1650ms, 50ms** ❌  S时间少算
**C. 850ms, 75ms** ❌  C时间算错
**D. 1650ms, 75ms** ✅  C需1.5RTT+2MSL，S需1.5RTT

### 第三步：确认答案
D。C进入CLOSED需1650ms，S需75ms

## 易错点分析

### 易错点1：常见误区
❌ 错误理解：认为客户端C和服务器S同时进入CLOSED状态，或者忘记主动关闭方(C)必须经历TIME_WAIT状态等待2MSL。容易误算为C和S都是50ms或都是850ms。
✅ 正确理解：四次挥手的时间线(C主动关闭)：①C发FIN，0.5RTT后S收到 ②S发ACK并发FIN，0.5RTT后C收到(共1RTT) ③C发最后ACK，0.5RTT后S收到，S立即进入CLOSED ④C进入TIME_WAIT等待2MSL=1600ms后才进入CLOSED。因此：C的时间=1.5RTT+2MSL=75ms+1600ms=1675ms≈1650ms(选项D)；S的时间=1.5RTT=75ms(从收到C的FIN到收到C的最后ACK)。关键点：主动关闭方必须TIME_WAIT等待2MSL，被动关闭方收到最后ACK后直接CLOSED。
