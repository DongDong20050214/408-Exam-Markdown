# 分组交换传输时间真题汇总

## 2010年第34题

**元数据**
- 年份：[2010年第34题](../../../01-真题原卷/2010年408真题.md#34)
- 科目：#计算机网络
- 知识点：#计算机网络/概述/分组交换时延
- 难度：⭐⭐⭐⭐
- 关联知识：[知识库/计算机网络/交换技术](知识库/计算机网络/交换技术.md)

**题干**

存储转发分组交换网，链路速率100Mbps，分组1000B（头20B），发送980000B文件，需要时间至少是（）。

A. 80ms  
B. 80.08ms  
C. 80.16ms  
D. 80.24ms

**答案**：C

**考点分析**
- 分组数：980000/(1000-20)=1000个分组
- 发送时延：1000×8/100M=0.08ms
- 经过3个节点（2条链路）
- 总时延：(1000+2)×0.08=80.16ms

## 解题思路（逐项分析法）

### 第一步：明确题目条件
- 链路速率：100Mbps
- 分组大小：1000B（包头20B，数据980B）
- 文件大小：980000B
- 网络拓扑：源主机→路由器→路由器→目标主机（3个节点，2条链路）
- 传输方式：存储转发分组交换

### 第二步：回顾分组交换时延计算

**关键概念**：
1. **有效数据量**：每个分组携带 1000-20=980B 数据
2. **分组数量**：总数据/每个分组数据 = 980000/980 = 1000个分组
3. **发送时延**：分组大小/链路速率 = 1000×8bit / 100Mbps = 0.08ms

**存储转发时延模型**：
```
第1个分组：经过3个节点，需要3次发送 = 3×0.08ms
第2-1000个分组：流水线传输，每个分组只需等待1次发送时延
总时延 = 3×0.08 + 999×0.08 = 1002×0.08ms
```

**简化公式**：
对于n个分组，经过m个节点（m-1条链路）：
**总时延 = (n + m - 1) × 单个分组发送时延**

### 第三步：详细计算过程

**步骤1：计算分组数量**
```
文件大小 = 980000B
每个分组有效数据 = 1000 - 20 = 980B
分组数 n = 980000 / 980 = 1000个
```

**步骤2：计算单个分组发送时延**
```
分组总大小 = 1000B = 8000bit
链路速率 = 100Mbps = 100×10^6 bps
发送时延 t = 8000 / (100×10^6) = 0.00008s = 0.08ms
```

**步骤3：确定节点数和链路数**
```
拓扑：主机A → 路由器R1 → 路由器R2 → 主机B
节点数 m = 3（R1、R2、B）
链路数 = m - 1 = 2条
```

**步骤4：应用流水线公式**
```
总时延 = (n + m - 1) × t
       = (1000 + 3 - 1) × 0.08
       = 1002 × 0.08
       = 80.16ms
```

**时序图**：
```
时刻0ms:    源主机开始发送分组1
时刻0.08ms: 分组1到达R1，源主机开始发送分组2
时刻0.16ms: 分组1离开R1到达R2，分组2到达R1，源主机发送分组3
...
时刻80.08ms: 分组1000到达R2
时刻80.16ms: 分组1000到达目标主机 ✓
```

### 第四步：验证选项

**A. 80ms ❌**
```
这是仅计算数据传输的时间：
980000×8 / 100M = 78.4ms ≈ 80ms（忽略了包头）
错误原因：没有考虑分组包头和存储转发延迟
```

**B. 80.08ms ❌**
```
这是1001个分组的时延：
(1000+2)×0.08 = 1002×0.08 = 80.16ms ≠ 80.08ms
80.08 = 1001×0.08（计算错误）
```

**C. 80.16ms ✅**
```
正确计算：
(1000 + 3 - 1) × 0.08 = 1002 × 0.08 = 80.16ms ✓
```

**D. 80.24ms ❌**
```
这是1003个分组的时延：
1003×0.08 = 80.24ms
错误原因：节点数计算错误（可能误认为4个节点）
```

### 第五步：确认答案
答案：**C**

## 易错点分析

### 陷阱1：忘记减去包头计算有效分组数
❌ **错误计算**：分组数 = 980000/1000 = 980个
✅ **正确计算**：分组数 = 980000/(1000-20) = 1000个
- 每个分组有20B包头，只有980B是有效数据
- 必须按有效数据量计算分组数

### 陷阱2：节点数和链路数混淆
❌ **错误理解**：2条链路，所以 (n+2)×t
✅ **正确理解**：3个节点，所以 (n+3-1)×t = (n+2)×t
- 公式中的m是节点数（不含源主机）
- 链路数 = 节点数 - 1

**图示**：
```
主机A --链路1--> R1 --链路2--> R2 --链路3--> 主机B
      (发送点) (节点1) (节点2)    (节点3=目标)
```
注意：源主机不计入m，因为它不需要"接收后转发"

### 陷阱3：包头也要传输
❌ **错误计算**：发送时延 = 980×8 / 100M（只算数据）
✅ **正确计算**：发送时延 = 1000×8 / 100M（包括包头）
- 链路上传输的是完整分组（包头+数据）
- 发送时延基于分组总大小

### 陷阱4：混淆不同时延概念
| 时延类型 | 本题是否考察 | 说明 |
|---------|------------|------|
| 发送时延 | ✅ | 分组从网卡发出的时间 = 分组大小/链路速率 |
| 传播时延 | ❌ | 信号在链路上传播的时间（本题未给出链路长度） |
| 排队时延 | ❌ | 在路由器等待的时间（题目说"至少"，不考虑排队） |
| 处理时延 | ❌ | 路由器处理分组的时间（忽略不计） |

## 知识点对比

| 交换方式 | 时延计算 | 适用场景 |
|---------|---------|---------|
| **电路交换** | 建立连接时延 + 数据传输时延 | 固定时延，适合实时通信 |
| **报文交换** | (1 + m - 1) × 报文发送时延 | 整个文件作为一个单元 |
| **分组交换** | **(n + m - 1) × 分组发送时延** | **文件分成n个分组** |

**分组交换流水线效应**：
```
无流水线（每个分组走完全程再发下一个）：
总时延 = n × m × t = 1000 × 3 × 0.08 = 240ms

有流水线（存储转发）：
总时延 = (n + m - 1) × t = 1002 × 0.08 = 80.16ms

效率提升约3倍！
```
