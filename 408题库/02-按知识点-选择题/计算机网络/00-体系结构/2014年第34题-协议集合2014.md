## 易错点分析

### 陷阱1：混淆ARP的层次归属
❌ **错误理解**：ARP解析MAC地址，属于数据链路层
✅ **正确理解**：
- ARP工作在**网际层**（网络层），为IP服务
- ARP的功能是将IP地址解析为MAC地址
- 虽然涉及MAC地址，但逻辑上属于IP层的辅助协议
- 在TCP/IP模型中归入网际层，在OSI模型中有争议（介于2层和3层之间）

### 陷阱2：混淆协议功能的相似性
容易混淆的协议对：
- **ICMP vs IGMP**：
  - ICMP：差错控制和网络诊断（ping、traceroute）
  - IGMP：组播组管理
- **FTP vs TFTP**：
  - FTP：基于TCP的文件传输协议
  - TFTP：基于UDP的简单文件传输协议
- **HTTP vs HTTPS**：
  - HTTP：明文传输
  - HTTPS：HTTP + SSL/TLS加密

### 陷阱3：忽略协议的端口号
**常见协议端口**：
| 协议 | 端口 | 传输层协议 |
|------|------|-----------|
| FTP | 21(控制)、20(数据) | TCP |
| SSH | 22 | TCP |
| TELNET | 23 | TCP |
| SMTP | 25 | TCP |
| DNS | 53 | UDP/TCP |
| HTTP | 80 | TCP |
| POP3 | 110 | TCP |
| HTTPS | 443 | TCP |

## 知识点对比

### TCP/IP模型各层协议汇总

| 层次 | 主要协议 | 功能 |
|------|---------|------|
| **应用层** | HTTP, FTP, SMTP, DNS, TELNET | 为用户提供网络服务 |
| **传输层** | TCP, UDP | 端到端通信 |
| **网际层** | IP, ICMP, IGMP, ARP | 路由和寻址 |
| **网络接口层** | Ethernet, PPP | 物理传输 |

### 应用层协议分类

| 协议类别 | 协议 | 用途 |
|---------|------|------|
| Web服务 | HTTP, HTTPS | 网页访问 |
| 文件传输 | FTP, TFTP | 文件上传下载 |
| 电子邮件 | SMTP, POP3, IMAP | 邮件收发 |
| 远程登录 | TELNET, SSH | 远程管理 |
| 网络管理 | SNMP | 设备监控 |
| 域名解析 | DNS | 域名到IP映射 |
