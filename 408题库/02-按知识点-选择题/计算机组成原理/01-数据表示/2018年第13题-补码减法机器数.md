**题干**

假定带符号整数采用补码表示，若 int 型变量 x 和 y 的机器数分别是 FFFFFFDFH 和 00000041H，则 x、y 的值以及 x - y 的机器数分别是（）。A. x = -65, y = 41, x - y 的机器数溢出  B. x = -33, y = 65, x - y 的机器数为 FFFFFF9DH  C. x = -33, y = 65, x - y 的机器数为 FFFFFF9EH  D. x = -65, y = 41, x - y 的机器数为 FFFFFF96H

**答案**：C

**考点分析**
- x=-33,y=65,x-y=-98=FFFFFF9EH

---

## 解题思路（补码转换法）

### 第一步：转换真值
**x = FFFFFFDFH (32位补码)**
```
最高位=1，为负数
真值 = -(2^32 - FFFFFFDFH)
    = -(2^32 - 4294967263)
    = -33
```

**y = 00000041H**
```
最高位=0，为正数
真值 = 65
```

### 第二步：计算x - y
```
x - y = -33 - 65 = -98
```

### 第三步：-98的补码
```
-98的补码 = 2^32 - 98
         = 4294967296 - 98
         = 4294967198
         = FFFFFF9EH
```

## 易错点分析

### 易错点1：补码转真值计算错误
❌ 错误理解：FFFFFFDFH首位1是负数，直接读DFH=223
✅ 正确理解：32位补码负数真值=-(2³²-机器数)=-(4294967296-4294967263)=-33。或快速法：FFFFFFDFH距离FFFFFFFFH(-1)差32，所以是-33

### 易错点2：补码减法溢出判断
| 操作数 | x=-33 | y=65 | 结果 |
|--------|-------|------|------|
| x-y | 负-正 | -98 | 未溢出(在范围内) |
| 补码 | FFFFFFDFH | 00000041H | FFFFFF9EH |
| 范围 | -2³¹~2³¹-1 | -98在内 | 无溢出 |
取反 = FFFFFF9DH
加1 = FFFFFF9EH
```

### 第四步：逐项判断
**C. x=-33, y=65, x-y的机器数为FFFFFF9EH ✅**

### 第五步：确认答案
答案：**C**

## 易错点分析

### 易错点1：补码转真值计算错误
❌ **错误**：FFFFFFDFH = -65
✅ **正确**：
```
FFFFFFDFH = 4294967263
真值 = -(2^32 - 4294967263) = -33
```

### 易错点2：-98的补码计算错误  
常见错误：FFFFFF96H或FFFFFF9DH
```
正确方法：
98 = 00000062H
~98 = FFFFFF9DH
-98 = FFFFFF9DH + 1 = FFFFFF9EH
```

## 知识点对比

### 32位补码特征

| 范围 | 最高字节特征 | 示例 |
|------|------------|------|
| 正数(0~2^31-1) | 00-7F | 00000041H=65 |
| 负数(-1~-128) | FFFFFF | FFFFFFDFH=-33 |
| 负数(-128~-2^31) | 80-FFFFFE | 80000000H=-2^31 |

**记忆口诀**：
> **补码减法变加法，减数取反再加一**  
> **负数补码高位F，转真值要用2的n次方减**
