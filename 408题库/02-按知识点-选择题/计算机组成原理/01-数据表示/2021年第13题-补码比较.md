# 补码比较真题汇总

## 2021年第13题

**元数据**
- 年份：[[../../01-真题原卷/2021年408真题#13|2021年第13题]]
- 科目：#计算机组成原理

- 难度：⭐⭐
- 关联知识：[[知识库/计算机组成原理/补码]]

**题干**

已知带符号整数用补码表示，变量 x, y, z 的机器数分别为 FFFDH, FFDFH, 7FFCH，下列结论中，正确的是（）。

A. 若 x, y 和 z 为无符号整数，则 z < x < y
B. 若 x, y 和 z 为无符号整数，则 x < y < z
C. 若 x, y 和 z 为带符号整数，则 x < y < z
D. 若 x, y 和 z 为带符号整数，则 y < x < z

**答案**：D

**考点分析**
- 无符号：FFFD > FFDF > 7FFC，不对
- 有符号：FFFD=-3, FFDF=-33, 7FFC=32764
- 有符号：-33 < -3 < 32764，即 y < x < z

---

## 解题思路（分类比较法）

### 第一步：转换为真值
**x = FFFDH (16位)**
- 无符号：65533
- 有符号补码：-(2^16 - 65533) = -3

**y = FFDFH (16位)**
- 无符号：65503  
- 有符号补码：-(65536 - 65503) = -33

**z = 7FFCH (16位)**
- 无符号：32764
- 有符号补码：+32764（最高位0为正）

### 第二步：逐项判断

**A. 无符号：z < x < y ❌**
```
无符号值：x=65533, y=65503, z=32764
实际：z < y < x（32764 < 65503 < 65533）
错误
```

**B. 无符号：x < y < z ❌**
```
与实际不符，错误
```

**C. 有符号：x < y < z ❌**
```
有符号值：x=-3, y=-33, z=32764
实际：y < x < z（-33 < -3 < 32764）
错误
```

**D. 有符号：y < x < z ✅**
```
y=-33, x=-3, z=32764
-33 < -3 < 32764，正确！
```

### 第三步：确认答案
答案：**D**

## 易错点分析

### 易错点1：混淆有符号和无符号比较规则
❌ **错误理解**：FFxx的数都很大
✅ **正确理解**：
- 无符号：FF开头确实大（接近65535）
- 有符号：最高位1是负数，反而小

### 易错点2：补码转真值错误
**16位补码→真值**：
```
最高位=0：直接转十进制
最高位=1：-(2^16 - 机器数)

示例：
FFDFH = 1111 1111 1101 1111
真值 = -(65536 - 65503) = -33
```

## 知识点对比

### 有符号vs无符号大小比较

| 机器数 | 无符号值 | 有符号值 | 规律 |
|--------|---------|---------|------|
| 7FFCH | 32764 | +32764 | 最高位0→正数 |
| FFDFH | 65503 | -33 | 最高位1→负数 |
| FFFDH | 65533 | -3 | 最高位1→负数 |

**有符号排序**：y(-33) < x(-3) < z(32764)
**无符号排序**：z(32764) < y(65503) < x(65533)

**记忆口诀**：
> **有符号看最高位，为一便是负数值**  
> **无符号全为正，数值越大越靠后**
