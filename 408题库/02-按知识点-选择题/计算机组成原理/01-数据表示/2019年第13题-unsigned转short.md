**题干**

考虑以下 C 语言代码：unsigned short usi = 65535; short si = usi; 执行上述程序段后，si 的值是（）。A. -1  B. -32767  C. -32768  D. -65535

**答案**：A

**考点分析**
- 65535=FFFFH，补码表示-1

---

## 解题思路（位模式不变法）

### 第一步：分析转换过程
```c
unsigned short usi = 65535;  // 16位无符号
short si = usi;              // 转为16位有符号
```

### 第二步：位模式分析
```
usi = 65535 = 0xFFFF = 1111 1111 1111 1111

转换时位模式不变：
si = 0xFFFF（位模式相同）

但解释方式变了：
- 作为unsigned short：65535
- 作为short：最高位1→负数
```

## 解题思路

### 第一步：题目核心
unsigned→signed：位模式不变，解释变

### 第二步：转换分析
**usi = 65535(unsigned short)**
- 二进制：1111111111111111₂ = FFFFH

**转为short(有符号)**
- 位模式：FFFFH不变
- 最高位1 → 负数
- 补码真值：-(2¹⁶-FFFFH) = -(65536-65535) = -1

### 第三步：答案确认
✅ **A. -1**

## 易错点分析

### 易错点1：认为会截断或转换
❌ 错误理解：65535超出short范围会截断或报错
✅ 正确理解：unsigned short和short都是16位，转换只改变解释不改变位模式。FFFFH作为unsigned是65535，作为signed补码是-1

### 易错点2：全1位模式的补码值
| 位数 | 二进制 | unsigned | signed补码 |
|------|--------|----------|-----------|
| 8位 | FFH | 255 | -1 |
| 16位 | FFFFH | 65535 | -1 |
| 32位 | FFFFFFFFH | 2³²-1 | -1 |

### 第四步：确认答案
si = -1 → 答案：**A**

## 易错点分析

### 易错点1：认为会溢出或截断
❌ **错误想法**：65535太大，short只能到32767，会溢出
✅ **正确理解**：
- 类型转换不改变位模式
- FFFF作为short就是-1
- 没有溢出，只是解释方式变化

### 易错点2：补码计算错误
❌ **错误计算**：-65535或-32768
✅ **正确方法**：
```
FFFFH = 全1
补码-1的特征：全1
验证：-1的补码 = ~1 + 1 = FFFEH + 1 = FFFFH
```

## 知识点对比

### unsigned与signed的转换

| 原类型 | 值 | 位模式 | 新类型 | 新值 | 说明 |
|--------|---|--------|--------|-----|------|
| unsigned short | 65535 | FFFF | short | -1 | 重新解释 |
| unsigned short | 32767 | 7FFF | short | 32767 | 不变 |
| unsigned short | 32768 | 8000 | short | -32768 | 最小负数 |

**规律**：
- 值≤32767：转换后值不变
- 值>32767：转换为负数，新值 = 原值 - 65536

**记忆口诀**：
> **位模式保持不改变，解释方式才是关键点**  
> **全一FFFF是负一，补码特征要记牢**
