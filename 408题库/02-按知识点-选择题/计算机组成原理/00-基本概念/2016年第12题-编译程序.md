# 编译程序真题汇总

## 2016年第12题

**元数据**
- 年份：2016年第12题
- 科目：#计算机组成原理
- 知识点：#计算机组成原理/基本概念/编译
- 难度：⭐
- 关联知识：[2022年第20题-程序翻译](..\05-编译系统\2022年第20题-程序翻译.md)

**题干**

将高级语言源程序转换为机器级目标代码文件的程序是（）。

A. 汇编程序  B. 链接程序  C. 编译程序  D. 解释程序

**答案**：C

**考点分析**
- 编译程序：高级语言→机器码

## 解题思路（逐项分析法）

### 第一步：明确题目要求
找出将**高级语言源程序**转换为**机器级目标代码文件**的程序

关键词：
- 输入：高级语言源程序（如C、Java）
- 输出：机器级目标代码文件（二进制）

### 第二步：回顾程序翻译工具

```
源程序 → [翻译工具] → 目标程序
```

主要翻译工具：
1. **编译程序**：高级语言 → 机器码（整体翻译）
2. **汇编程序**：汇编语言 → 机器码
3. **解释程序**：高级语言 → 边翻译边执行（无独立目标文件）
4. **链接程序**：多个目标文件 → 可执行文件

### 第三步：逐项判断

**A. 汇编程序 ❌**
```
功能：汇编语言 → 机器码
输入：.asm文件（汇编语言）
输出：.obj文件（机器码）

示例：
MOV AX, BX  →  汇编程序  →  10001011 11011000

题目要求"高级语言"，汇编语言不是高级语言
结论：错误 ✗
```

**B. 链接程序 ❌**
```
功能：多个目标文件 + 库文件 → 可执行文件
输入：.obj文件（已经是机器码）
输出：.exe文件（可执行文件）

工作内容：
- 地址重定位
- 符号解析
- 库函数链接

不涉及语言翻译，只是"组装"已有的机器码
结论：错误 ✗
```

**C. 编译程序 ✅**
```
功能：高级语言 → 机器码
输入：.c/.cpp文件（高级语言源程序）
输出：.obj/.o文件（机器级目标代码）

示例：
int a = b + c;  →  编译程序  →  MOV AX, [b]
                                ADD AX, [c]
                                MOV [a], AX
                            →  机器码

完全符合题意
结论：正确 ✓
```

**D. 解释程序 ❌**
```
功能：高级语言 → 边翻译边执行
特点：不生成独立的目标代码文件

执行方式：
读取一条语句 → 翻译 → 执行 → 读取下一条...

示例：Python解释器
print("hello")  →  解释器  →  直接输出hello
                           （无.obj文件生成）

题目要求"转换为目标代码文件"，解释程序不生成文件
结论：错误 ✗
```

### 第四步：确认答案
只有编译程序符合 → 答案：**C**

## 易错点分析

### 陷阱1：混淆"汇编程序"和"编译程序"
❌ **错误理解**：汇编程序也生成机器码，应该也对
✅ **正确理解**：
- **汇编程序**：汇编语言 → 机器码（低级到低级）
- **编译程序**：高级语言 → 机器码（高级到低级）
- 题目明确说"高级语言源程序"，排除汇编程序

**记忆技巧**：
- 汇编程序 = Assembly → 处理Assembly Language
- 编译程序 = Compiler → 处理High-level Language

### 陷阱2：混淆"编译"和"链接"
❌ **错误想法**：编译+链接才能生成可执行文件，应该选链接程序
✅ **正确理解**：
- 题目问的是"转换为**机器级目标代码文件**"，不是"可执行文件"
- 编译：源程序 → 目标代码（.obj）✓ 符合题意
- 链接：目标代码 → 可执行文件（.exe）✗ 不符合

**完整流程**：
```
.c文件 → [编译] → .obj文件 → [链接] → .exe文件
        ↑题目问的是这步
```

### 陷阱3：认为"解释程序"也能生成机器码
❌ **错误理解**：解释程序也要把代码转成机器码才能执行，应该也算
✅ **正确理解**：
- 解释程序：翻译后**立即执行**，不保存为文件
- 编译程序：翻译后**保存为.obj文件**
- 题目要求"转换为...文件"，解释程序不符合

**类比**：
- 编译 = 把整本书翻译成另一种语言，出版成书
- 解释 = 同声传译，听一句翻一句，不留记录

## 知识点对比

### 四种程序翻译工具对比

| 工具 | 输入 | 输出 | 翻译方式 | 生成文件 | 执行速度 |
|-----|------|------|---------|---------|---------|
| 编译程序 | 高级语言(.c) | 机器码(.obj) | 整体翻译 | ✅ 是 | 快 |
| 汇编程序 | 汇编语言(.asm) | 机器码(.obj) | 逐条翻译 | ✅ 是 | 快 |
| 解释程序 | 高级语言(.py) | 执行结果 | 边译边执行 | ❌ 否 | 慢 |
| 链接程序 | 机器码(.obj) | 可执行文件(.exe) | 地址重定位 | ✅ 是 | - |

### 程序生成完整流程

```
源程序阶段：
  hello.c (高级语言)
      ↓
  [预处理器] 展开宏、包含头文件
      ↓
  hello.i (预处理后的源文件)
      ↓
  [编译程序] ← 题目问的是这个
      ↓
目标代码阶段：
  hello.obj (机器级目标代码文件)
      ↓
  [链接程序] 链接库文件、重定位
      ↓
可执行文件阶段：
  hello.exe (可执行文件)
```

**记忆口诀**：
> **高级变机器，编译是核心**  
> **汇编处理汇编码，链接组装不翻译**  
> **解释边译又边跑，不留文件在硬盘**
