# 执行时间真题汇总

## 2010年第12题

**元数据**
- 年份：2010年第12题
- 科目：#计算机组成原理
- 知识点：#计算机组成原理/性能/执行时间
- 难度：⭐⭐
- 关联知识：[知识库/计算机组成原理/执行时间](知识库/计算机组成原理/执行时间.md)

**题干**

下列选项中，能缩短程序执行时间的措施是（）。I. 提高 CPU 时钟频率  II. 优化数据通路结构  III. 对程序进行编译优化

A. 仅I和II  B. 仅I和III  C. 仅II和III  D. I、II和III

**答案**：D

**考点分析**
- I、II、III都能缩短执行时间

---

## 解题思路（逐项分析法）

### 第一步：回顾执行时间公式
```
CPU执行时间 = 指令数 × CPI × 时钟周期
           = (指令数 × CPI) / 时钟频率
```

缩短执行时间的方法：
1. 减少指令数
2. 降低CPI
3. 缩短时钟周期(提高时钟频率)

### 第二步：逐项分析

**I. 提高CPU时钟频率 ✅**
```
时钟频率 ↑ → 时钟周期 ↓ → 执行时间 ↓

示例：
原：1GHz，执行时间 = 100万条 × 2 × (1/1G) = 2ms
改：2GHz，执行时间 = 100万条 × 2 × (1/2G) = 1ms

效果：时间减半

结论：正确 ✓
```

**II. 优化数据通路结构 ✅**
```
数据通路优化 → 减少指令执行周期数(CPI) → 执行时间 ↓

优化方式：
- 增加数据旁路(forwarding)：减少数据冒险停顿
- 改进流水线设计：减少结构冒险
- 优化ALU设计：减少运算延迟

示例：
原CPI=3，优化后CPI=2.5
执行时间减少：(3-2.5)/3 = 16.7%

结论：正确 ✓
```

**III. 对程序进行编译优化 ✅**
```
编译优化 → 减少指令数 → 执行时间 ↓

优化技术：
- 循环展开：减少循环控制指令
- 公共子表达式消除：避免重复计算
- 强度削弱：用移位代替乘法
- 死代码删除：删除无用指令

示例：
for(i=0; i<4; i++) a[i]=0;
优化后：a[0]=0; a[1]=0; a[2]=0; a[3]=0;
减少循环控制指令

结论：正确 ✓
```

### 第三步：确认答案
三种措施都能缩短执行时间 → 答案：**D**

## 易错点分析

### 易错点1：认为"提高频率"会增加功耗而无效
❌ **错误理解**：提高频率会增加功耗和发热，不是好方法
✅ **正确理解**：
- 题目问的是"**能否**缩短时间"，不是"性价比最高的方法"
- 提高频率确实能缩短时间，虽然有功耗代价
- 实际应用中需权衡，但技术上完全可行

### 易错点2：混淆"数据通路优化"和"增加功能部件"
❌ **错误想法**：优化数据通路不就是加硬件吗，不是软件优化
✅ **正确理解**：
- 数据通路优化是**硬件级优化**
- 包括：流水线设计、数据旁路、指令调度
- 与软件优化(编译优化)是两个层面

**三种优化层次对比**：

| 优化层次 | 具体措施 | 影响的参数 | 效果 |
|---------|---------|-----------|------|
| 硬件-工艺 | 提高时钟频率 | 时钟周期 ↓ | 直接加速 |
| 硬件-架构 | 优化数据通路 | CPI ↓ | 减少停顿 |
| 软件-编译 | 编译优化 | 指令数 ↓ | 减少指令 |

### 易错点3：认为"编译优化"只是"代码整理"
❌ **错误理解**：编译优化只是让代码更易读，不影响性能
✅ **正确理解**：
- 编译优化直接**减少生成的机器指令数**
- 优化后执行的指令变少，时间自然缩短
- 效果显著：优化级别-O3比-O0快数倍

**编译优化实例**：
```c
// 原代码
int sum = 0;
for(int i=0; i<100; i++) {
    sum += 2 * i;  // 乘法指令
}

// 优化后(强度削弱)
int sum = 0;
for(int i=0; i<100; i++) {
    sum += i << 1;  // 移位指令(更快)
}

// 进一步优化(数学变换)
int sum = 2 * (0+1+2+...+99) = 2 * (99*100/2) = 9900;
// 0条循环指令！
```

## 知识点对比

### 缩短执行时间的三大途径

| 途径 | 优化对象 | 实现方式 | 优点 | 缺点 |
|------|---------|---------|------|------|
| 提高频率 | 时钟周期 | 提升工艺、降压 | 效果直接 | 功耗高、散热难 |
| 优化通路 | CPI | 流水线、旁路 | 硬件并行 | 设计复杂 |
| 编译优化 | 指令数 | 优化算法 | 成本低 | 有上限 |

**记忆口诀**：
> **频率通路加编译，三管齐下时间减**  
> **硬件软件双优化，执行时间全方位**
