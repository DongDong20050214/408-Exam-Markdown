**题干**

程序 P 在机器 M 上的执行时间是 20s，编译优化后，P 执行的指令数减少到原来的 70%，而 CPI 增加到原来的 1.2 倍，则 P 在 M 上的执行时间是（）。A. 8.4s  B. 11.7s  C. 14.0s  D. 16.8s

**答案**：D

**考点分析**
- 新时间=20×0.7×1.2=16.8s

---

## 解题思路（公式计算法）

### 第一步：明确执行时间公式
```
CPU执行时间 = 指令数 × CPI × 时钟周期
```

### 第二步：分析变化
**原始情况**：
- 执行时间 T₀ = 20s
- 指令数 = I₀
- CPI = C₀
- 时钟周期 = τ

**优化后**：
- 指令数 = 0.7I₀ (减少到70%)
- CPI = 1.2C₀ (增加到1.2倍)
- 时钟周期 = τ (不变)

### 第三步：计算新执行时间
```
T₀ = I₀ × C₀ × τ = 20s

T₁ = (0.7I₀) × (1.2C₀) × τ
   = 0.7 × 1.2 × (I₀ × C₀ × τ)
   = 0.84 × T₀
   = 0.84 × 20
   = 16.8s
```

### 第四步：确认答案
新执行时间16.8s → 答案：**D**

## 易错点分析

### 易错点1：误认为CPI减少
❌ **错误理解**：编译优化后效率提高，CPI应该减少
✅ **正确理解**：
- 编译优化删除简单指令(如NOP)，保留复杂指令
- 剩余指令平均周期数增加：CPI↑
- 但总指令数大幅减少：指令数↓

**类比**：班级去掉学习最快的学生后，平均成绩反而下降

### 易错点2：直接用比例相减
❌ **错误计算**：
```
指令数减少30% → 时间减少30% → 20×0.7=14s
或
CPI增加20% → 时间增加20% → 20×1.2=24s
```

✅ **正确方法**：
```
两个因素同时变化，必须相乘：
新时间 = 原时间 × 指令数比例 × CPI比例
      = 20 × 0.7 × 1.2 = 16.8s
```

**公式记忆**：
```
T_new / T_old = (I_new/I_old) × (CPI_new/CPI_old) × (τ_new/τ_old)
```

### 易错点3：混淆"变化后的值"和"变化比例"
❌ **错误理解**：
- "减少到70%"理解成"减少了70%"→ 剩30%
- "增加到1.2倍"理解成"增加了1.2倍"→ 变2.2倍

✅ **正确理解**：
- "减少到70%"= 原来的70% = 0.7倍
- "增加到1.2倍"= 原来的1.2倍

**对比表格**：

| 表述 | 含义 | 比例系数 |
|------|------|----------|
| 减少到70% | 剩余70% | ×0.7 |
| 减少了70% | 剩余30% | ×0.3 |
| 增加到1.2倍 | 变成1.2倍 | ×1.2 |
| 增加了1.2倍 | 变成2.2倍 | ×2.2 |

## 知识点对比

### 编译优化对性能指标的影响

| 指标 | 变化趋势 | 原因 |
|------|---------|------|
| 指令数 | ↓ 减少 | 删除冗余指令、循环展开 |
| CPI | ↑ 增加 | 删掉简单指令，复杂指令占比高 |
| 时钟频率 | → 不变 | 硬件特性，编译不影响 |
| 执行时间 | ↓ 减少 | 指令数减少量 > CPI增加量 |

### 性能优化的两个方向

```
方向1：减少工作量
- 编译优化 → 指令数↓
- 效果：直接减少执行的指令数量

方向2：提高效率  
- 硬件优化 → CPI↓
- 效果：每条指令执行更快

本题：指令数↓30%，CPI↑20%
净效果：0.7×1.2=0.84，时间减少16%
```

**记忆口诀**：
> **指令减少CPI涨，两者相乘算时长**  
> **减到乘以增到乘，莫把减了当减到**
