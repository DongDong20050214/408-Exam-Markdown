# 时间复杂度

## 定义
时间复杂度是衡量算法执行效率的尺度，用 O 符号表示，表示最坏情况下算法执行时间与输入规模的关系。

## 核心概念
- 大 O 记号：表示上界，用于分析最坏情况
- 大 Ω 记号：表示下界，用于分析最好情况
- 大 Θ 记号：表示紧界，最坏和最好相同时用
- 常见阶：O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)
- 系数和低阶项：分析时忽略常数系数和低阶项

## 算法流程/关键公式
1. 基本操作计数：简单语句 1，循环累乘，递归分治
2. 递推关系求解：代换法、递推树、主定理
3. 主定理：T(n) = aT(n/b) + f(n) 的求解
4. 常见递推：
   - T(n) = 2T(n/2) + n → O(n log n)
   - T(n) = T(n-1) + n → O(n²)

## 易混淆点
| 时间复杂度 | 空间复杂度 | 关系 |
|-----------|----------|------|
| 执行步骤数 | 使用存储量 | 可能相关不相关 |
| 越低越好 | 越低越好 | 有时需要平衡 |
