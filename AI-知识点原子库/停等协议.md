# 停等协议

## 定义
停等协议（Stop-and-Wait Protocol）是最简单的可靠传输协议，每发送一个分组都要等待确认后才能发送下一个。

## 核心概念
- 发送方：发送一个分组，等待确认
- 接收方：接收分组，发送确认信号
- 超时重传：未收到确认则重新发送
- 序号：防止分组重复和乱序
- 信道利用率：发送时间 / 总时间
- 往返延迟（RTT）：决定发送效率

## 算法流程/关键公式
1. 发送 → 启动定时器 → 等待确认
2. 确认到达 → 停止定时器 → 发送下一个
3. 超时 → 重新发送
4. 信道利用率 U = 分组长度 / (分组长度 + 2×RTT×带宽)

## 易混淆点
| 停等 | GBN | SR |
|-----|------|------|
| 1 个分组 | N 个分组 | N 个分组 |
| 最低效率 | 中等效率 | 最高效率 |
