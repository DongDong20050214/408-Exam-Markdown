# 408刷题攻略 v2 —— 科班快启动·题驱动计划

> 考试日期：**2026年12月20日** | 制定日期：2026-06-12 | 剩余：**27周**
> 适用人群：科班出身、基础遗忘、**刷课慢→直接放弃系统刷课**
> 本计划取代 [[【刷题攻略】循序渐进三阶段计划]] 和 [[【考研冲刺】6个月高强度刷题计划]]

---

## 🧠 核心思路（与旧计划的三点不同）

1. **不刷课，题驱动**。你是科班生，知识是"忘了"不是"没学过"。做题→错→看本库的解题思路/易错点→读对应原子知识点，唤醒速度远快于看视频。视频只留给"完全看不懂解析"的章节，且只看对应小节、1.5 倍速。
2. **选择题+综合题同章捆绑**。旧计划先 4 周选择再 4 周综合，同一章要捡两遍记忆。改成：每过一章，当天就把该章的选择题和综合题一起消灭。
3. **初期压缩到 5 周**。7 月 19 日前结束知识点轮，**7 月 20 日就开始整套真题**，给 16 套卷留足 5 个月（一刷精做 + 二刷限时 + 三刷错题 + 考前模拟）。

## 阶段0：摸底诊断（6.12 - 6.14，3天）

**目的**：搞清楚到底忘到什么程度，让阶段1有侧重，而不是平均用力。

- **Day 1**：做 `01-真题原卷/2009年408真题.md`（用最老的卷摸底，把 2020-2024 的卷留给后面限时模拟，保证"首做"的诊断价值）。不限时，不会就跳，**不要查资料**。
- **Day 2**：对答案（`408答案/`），按科目×章节统计错误，给每章打标：
  - 🟢 正确率 ≥70%：阶段1 快过（只做高频题）
  - 🟡 40%~70%：正常刷
  - 🔴 <40%：精刷 + 读原子知识点 + 必要时翻王道书对应章节
- **Day 3**：把标记结果写进 [[我的408刷题进度-冲刺版]]，规划阶段1每周的轻重。

## 阶段1：知识点轮（6.15 - 7.19，5周）

**目标**：922 道选择题 + 113 道综合题全部过一遍，**按章捆绑**。

### 每日节奏（全职 ~9 小时）

```
08:00-10:00  主科目：读该章原子知识点15min → 刷选择题35题
10:00-11:00  对答案，错题打 #错题 标签，看易错点分析
11:00-12:00  该章综合题 1~2 题（03-按知识点-综合题/ 对应目录）

14:00-15:30  副科目：刷选择题 25 题
15:30-16:30  对答案 + 错题标记
16:30-18:00  副科目综合题 1 题 + 知识点回看

19:30-21:30  当日错题重做 + 昨日错题快扫（费曼式：能讲清为什么错才算过）
```

> 主/副科目轮换是刻意设计：一天碰两科，防止"刷完网络忘了数构"。

### 周计划（科目两两穿插）

| 周次 | 日期 | 主科目（上午） | 副科目（下午） |
|------|------|----------------|----------------|
| 第1周 | 6.15-6.21 | 数构：线性表、栈队列串、算法基础 | 网络：物理层~网络层 |
| 第2周 | 6.22-6.28 | 数构：树（遍历/BST/堆/哈夫曼/B树⭐） | 网络：传输层、应用层 |
| 第3周 | 6.29-7.5 | 数构：图（拓扑排序⭐/MST/最短路）、查找（散列⭐） | 操统：进程管理、同步与死锁⭐ |
| 第4周 | 7.6-7.12 | 数构：排序（快排/归并/外排⭐） | 操统：内存管理、文件、I/O |
| 第5周 | 7.13-7.19 | 组原：数据表示、运算、存储、Cache⭐ | 组原：指令、数据通路⭐、流水线⭐、总线I/O + 总收尾 |

⭐ = 考频统计 TOP 考点（见 [[AI-统计分析/考频统计|考频统计]]），这些章节的题**一题都不能跳**；🟢章节可只做 frequency 为"高频/中频"的题。

### 卡壳处理规则（防止刷课黑洞）

1. 错了 → 先看题目自带的「解题思路 + 易错点分析」（90% 能解决）
2. 还不懂 → 查 `AI-知识点原子库/` 和 `AI-QA问答库/`，或直接在 Claude Code 里问这道题
3. 整章正确率 <40% → 当晚翻**王道书**对应章节（看书比看课快 3 倍）
4. 书也看不懂的硬骨头（常见：浮点数、流水线、Cache映射）→ 才允许看对应小节视频，1.5倍速，单个知识点不超过 40 分钟

### 红线

- 每天选择题最低 50 题，综合题最低 2 题
- **7 月 19 日必须收尾**。若落后 >3 天：🟢章节全部跳过，只刷🔴🟡

## 阶段2：真题一刷·精做（7.20 - 9.6，7周）

**目标**：16 套卷全部做完第一遍，每题吃透。**不限时**，质量优先。

### 节奏：每周 2~3 套，1 套 = 2 天

```
Day A 上午  完整做一套（01-真题原卷/，可不掐表但一鼓作气）
Day A 下午  对选择题答案，逐题复盘，错题入库
Day B 上午  复盘 7 道综合题（对照解题步骤，重写一遍解答）
Day B 下午+晚上  本套暴露的薄弱知识点回炉（回阶段1对应章节重做）
```

### 顺序：从旧到新（新卷的"新鲜感"留给二刷限时模拟）

| 周次 | 日期 | 试卷 |
|------|------|------|
| 第1周 | 7.20-7.26 | 2010、2011、2012 |
| 第2周 | 7.27-8.2 | 2013、2014 |
| 第3周 | 8.3-8.9 | 2015、2016 |
| 第4周 | 8.10-8.16 | 2017、2018 |
| 第5周 | 8.17-8.23 | 2019、2020 |
| 第6周 | 8.24-8.30 | 2021、2022 |
| 第7周 | 8.31-9.6 | 2023、2024 + 一刷总结 |

**一刷及格线**：2015 年后的卷子裸做 ≥ 90 分（满分150）。低于此分的科目，在 Day B 下午加倍回炉。

## 阶段3：真题二刷·限时模拟（9.7 - 10.31，8周）

**目标**：16 套全部在**严格 3 小时**（8:30-11:30，模拟真实考试时段）内完成。

```
周二上午    限时模拟 1 套 → 下午复盘 + 记分
周五上午    限时模拟 1 套 → 下午复盘 + 记分
其余半天    错题专项：用下方 Dataview 查询调出错题集中营，按章歼灭
周日        休息半天 + 本周错题快扫
```

顺序从新到旧：2024 → 2009（最新卷优先二刷，因为它们最接近现在的命题风格）。

**二刷目标分**：≥110；连续两套 <100 → 触发应急（见文末）。

### 错题 Dataview（贴进任意笔记即可用）

```dataview
TABLE subject as 科目, chapter as 章节, difficulty as 难度, review_count as 复习次数
FROM "408题库"
WHERE contains(tags, "错题")
SORT subject, chapter
```

错题标记规范（每次错都更新）：

```yaml
tags: ["错题", "数据结构", "B树"]
review_count: 2
last_review: 2026-09-15
```

## ⚠️ 应急预案

- **阶段1落后 >3 天**：🟢章节直接跳过；综合题只做高频题型
- **阶段2落后**：2009-2012 年卷降级为"只做综合题"

- **任何时候都不准做的事**：从头刷网课、换资料、重做计划。执行 > 完美。
