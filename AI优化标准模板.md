## 📝 模板1：题目文件优化标准

### Metadata增强（在现有frontmatter后追加）

```yaml
---
# ===== 保留所有现有字段 =====
title: xxx
topic: xxx
chapter: xxx
knowledge_point: xxx
tags: []
aliases: []

# ===== 以下为AI优化新增字段 =====
year: 2024                       # 年份
question_id: "2024-DS-01"        # 唯一ID（年份-科目缩写-题号）
difficulty: 2                    # 难度（1-5星）
exam_frequency: "高频"            # 考频（高频/中频/低频）
exam_count: 8                    # 历年考察次数（同类型考点）
has_image: false                 # 是否含配图
related_questions:               # 相关题目
  - "[[2019年第01题-xxx]]"
related_knowledge:               # 关联知识点
  - "[[408真题知识库/知识库/xxx]]"
---
```

**科目缩写规范**：
- DS：数据结构 (Data Structure)
- CO：计算机组成原理 (Computer Organization)
- OS：操作系统 (Operating System)
- CN：计算机网络 (Computer Network)

**难度星级标准**：
- ⭐ (1星)：基础概念，直接记忆
- ⭐⭐ (2星)：简单应用，一步推理
- ⭐⭐⭐ (3星)：综合应用，多步推理
- ⭐⭐⭐⭐ (4星)：复杂分析，易错陷阱
- ⭐⭐⭐⭐⭐ (5星)：深度理解，计算复杂

**考频判断标准**：
- 高频：16年考察 ≥ 8次，或近5年每年都考
- 中频：16年考察 4-7次
- 低频：16年考察 < 4次

## 🚀 批量处理流程

### 阶段1：Metadata标准化（68小时）

**任务**：为753道题补充标准化元数据

**工作流程**：
1. 按年份批量处理（从2024年倒推）
2. 每处理完一年，验证dataview查询
3. 使用脚本辅助填充year、question_id
4. 人工审核difficulty、exam_frequency

**验证标准**：
```dataview
TABLE difficulty, exam_frequency, year
FROM "408题库/02-按知识点-选择题"
WHERE year = 2024
SORT question_id
```

### 阶段2：知识点补充（50小时）

**任务**：补充100个核心知识点

**优先级**：
1. 高频考点（考频≥8次）
2. 易错点（学生反馈困难）
3. 关联题目多的知识点

**工作流程**：
1. 先补充"易混淆点"章节
2. 再补充"常见提问"章节
3. 关联相关真题

### 阶段3：QA扩充（33小时）

**任务**：每个知识点至少5个QA

**QA来源**：
1. 从真题中提取问题
2. 学生常见疑问
3. 易错点转化为QA

### 阶段4：统计分析（20小时）

**任务**：生成统计报告

**实现方式**：
1. Python脚本统计metadata
2. 生成markdown表格
3. 手动补充分析结论
