# 哈夫曼树

## 常见提问

**Q: 哈夫曼树的结点数有什么规律？**
A: n个叶子结点的哈夫曼树，总结点数=2n-1。其中叶子n个，分支结点n-1个。记住公式：给叶子数求总数，给总数求叶子数n=(总数+1)/2。

**Q: 哈夫曼树为什么没有度为1的结点？**
A: 哈夫曼树构造时，每次选两个最小的合并，新结点必有两个孩子（度为2）。初始叶子度为0，最终只有度0和度2，没有度1。

**Q: 哈夫曼树唯一吗？**
A: 不唯一。当有权值相同的结点时，选择顺序不同会构造出不同形状的树。但所有哈夫曼树的WPL相同，都是最小值。

**Q: 如何手工构造哈夫曼树？**
A: 
1. 把n个权值看作n棵树（森林）
2. 每次选两个权值最小的合并
3. 新树根的权值=两棵树权值之和
4. 重复直到只剩一棵树

**Q: 哈夫曼编码为什么是前缀编码？**
A: 因为每个字符对应一个叶子结点。从根到叶子的路径唯一，任何字符的编码都不可能是另一个字符编码的前缀（叶子不可能是另一个叶子的祖先）。

**Q: 哈夫曼编码的平均长度怎么算？**
A: 加权平均：每个字符的编码长度×该字符出现的频率，求和。公式：Σ(频率i × 编码长度i)

## 解题模板

**题型特征**：给定权值或结点数，求哈夫曼树的结点数/WPL

### 模板1：结点数计算
```
题型：已知叶子数，求总结点数
公式：总结点数 = 2n - 1

例：5个叶子 → 2×5-1 = 9个结点

题型：已知总结点数，求叶子数
公式：n = (总数 + 1) / 2

例：115个结点 → (115+1)/2 = 58个叶子
```

### 模板2：WPL计算
```
第1步：构造哈夫曼树
- 每次选最小两个合并
- 记录每个叶子的深度

第2步：计算WPL
WPL = Σ(权值i × 深度i)

例：权值{5,15,40,30,10}
构造后深度：{4,3,1,2,4}
WPL = 5×4 + 15×3 + 40×1 + 30×2 + 10×4 = 205
```

### 模板3：哈夫曼编码
```
第1步：构造哈夫曼树（权值=频率）

第2步：编码规则
- 左分支：0
- 右分支：1
- 从根到叶子的路径=编码

第3步：计算平均编码长度
平均长度 = Σ(频率i × 编码长度i) / 总频率
```

**常见变式**：
- 给叶子数求结点数 → 2n-1
- 给结点数求叶子数 → (总数+1)/2
- 给权值求WPL → 构造树+加权求和
- 给频率求编码 → 构造树+标记路径

## 定义

## 定义

**哈夫曼树（Huffman Tree）**：也称最优二叉树，是一类带权路径长度WPL最小的二叉树。

**相关概念**：

1. **路径**：从树中一个结点到另一个结点的分支序列
2. **路径长度**：路径上的分支数目
3. **结点的带权路径长度**：从根到该结点的路径长度 × 该结点的权值
4. **树的带权路径长度（WPL）**：树中所有叶子结点的带权路径长度之和

$$WPL = \sum_{i=1}^{n} w_i \times l_i$$

其中：
- $w_i$：第i个叶子结点的权值
- $l_i$：第i个叶子结点的路径长度（深度）
- $n$：叶子结点个数

## 核心性质

### 1. 结点数关系

**n个叶子结点的哈夫曼树**：
- 总结点数 = **2n - 1**
- 分支结点数 = n - 1
- 叶子结点数 = n

**推导**：
- 每次合并操作减少1个结点，增加1个分支结点
- 初始n个叶子，需要n-1次合并
- 总结点 = n个叶子 + (n-1)个分支 = 2n-1

### 2. 度的性质

哈夫曼树是**严格二叉树**：
- 度为0的结点（叶子）：n个
- 度为1的结点：**0个**（没有）
- 度为2的结点：n-1个

**结论**：哈夫曼树中不存在度为1的结点

### 3. WPL性质

- 权值越大的叶子，离根越近
- 权值越小的叶子，离根越远
- WPL最小

## 构造算法（贪心策略）

**哈夫曼算法步骤**：

1. **初始化**：将n个权值作为n棵只有根结点的二叉树，构成森林F
2. **选择**：从F中选取两棵根结点权值**最小**的树作为左右子树
3. **合并**：构造新树，根结点权值 = 左右子树根结点权值之和
4. **删除与加入**：从F中删除被选中的两棵树，将新树加入F
5. **重复**：重复步骤2-4，直到F中只剩一棵树（哈夫曼树）

**示例**：权值集合{5, 15, 40, 30, 10}

```
步骤1：初始森林
[5] [15] [40] [30] [10]

步骤2：选5和10合并
      [15]
     /   \
    5    10
[15] [40] [30] [15]

步骤3：选15和15合并
        [30]
       /   \
     [15]  [15]
     / \    / \
    5  10  ?  ?
[40] [30] [30]

步骤4：选30和30合并
          [60]
         /   \
      [30]   [30]
       / \
     15  15
[40] [60]

步骤5：选40和60合并
            [100]
           /    \
        [40]    [60]
               /    \
            [30]   [30]
             / \
           15  15
          / \
         5  10

最终哈夫曼树WPL = 5×4 + 10×4 + 15×3 + 30×2 + 40×1 = 205
```

## 哈夫曼编码

**应用背景**：数据压缩

**定长编码问题**：
- 每个字符用相同位数编码（如ASCII 8位）
- 浪费空间（高频字符和低频字符编码长度相同）

**哈夫曼编码（变长编码）**：
- 高频字符用短编码
- 低频字符用长编码
- 平均编码长度最短

**编码规则**：
- 左分支标记0，右分支标记1
- 从根到叶子的路径即为编码
- **前缀编码**：任何字符的编码都不是其他字符编码的前缀

**示例**：字符频率{A:5, B:15, C:40, D:30, E:10}

```
构造哈夫曼树后：
           根
          /  \
         /    \
       C:40   \
             / \
           D:30 \
               / \
             B:15 \
                 / \
               E:10 A:5

编码结果：
C: 0        (频率最高，编码最短)
D: 10
B: 110
E: 1110
A: 1111     (频率最低，编码最长)

平均编码长度 = (40×1 + 30×2 + 15×3 + 10×4 + 5×4) / 100 = 2.05位
```

| 操作 | 时间复杂度 | 说明 |

## 典型例题

**例1**：n=5个叶子结点，哈夫曼树有多少个结点？
- 答案：2×5-1 = 9个结点

**例2**：哈夫曼树共有115个结点，有多少个叶子结点？
- 设叶子n个，则2n-1=115
- n = (115+1)/2 = 58

**例3**：为什么哈夫曼树没有度为1的结点？
- 哈夫曼树每次合并两棵树
- 新结点必有两个孩子（度为2）
- 原始叶子结点度为0
- 因此不存在度为1的结点

## 易错点

1. **结点数计算**
   - 总结点 = 2n-1（n为叶子数）
   - 不要忘记"-1"

2. **度的分布**
   - 哈夫曼树只有度0和度2的结点
   - **没有度为1的结点**

3. **编码唯一性**
   - 同一组权值可能构造出不同形状的哈夫曼树
   - 但WPL相同，都是最小值
   - 编码可能不同，但平均长度相同

4. **左右孩子顺序**
   - 权值小的在左还是右不影响WPL
   - 但会影响具体编码

5. **WPL计算**
   - 只计算叶子结点
   - 不包括分支结点

## 记忆要点

- **结点数公式**：2n-1（n为叶子数）
- **度的特点**：只有度0和度2，无度1
- **构造原则**：每次选最小两个合并（贪心）
- **编码特点**：前缀编码，无二义性
- **应用场景**：数据压缩（ZIP、JPEG等）
- **WPL最小**：哈夫曼树的核心优势

## 补充：应用实例

**文件压缩**：
1. 统计字符频率
2. 构造哈夫曼树
3. 生成哈夫曼编码
4. 用编码替换原字符
5. 存储哈夫曼树（用于解码）

**压缩率**：
- 定长编码：8位/字符
- 哈夫曼编码：平均2-4位/字符（取决于频率分布）
- 压缩率可达50%-75%
