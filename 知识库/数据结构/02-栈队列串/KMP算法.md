# KMP算法

## 常见提问

**Q: KMP算法为什么比朴素算法快？**
A: 主串指针i不回退，避免重复比较。朴素算法i会回退到下一个起始位置，重复比较已匹配部分。KMP利用已匹配信息，模式串智能回退。

**Q: next数组怎么手算？**
A: next[j]等于前j个字符的最长公共前后缀长度。
- 步骤1：写出子串（前j个字符）
- 步骤2：列出所有前缀和后缀
- 步骤3：找最长的公共部分，记录长度
- 例如"aba"：前缀{a,ab}，后缀{ba,a}，公共为a，长度1

**Q: next和nextval有什么区别？**
A: nextval是优化版本。当S[j]==S[next[j]]时，会产生无效比较，nextval跳过这种情况：
```
if (S[j] == S[next[j]])
    nextval[j] = nextval[next[j]];  // 继续回退
else
    nextval[j] = next[j];
```

**Q: 失配时为什么j回退到next[j]？**
A: next[j]表示前j个字符中，最长公共前后缀的长度。失配时，这部分前缀与主串中对应的后缀已经匹配，可以直接从next[j]位置继续比较，不用从头开始。

**Q: KMP算法的时间复杂度为什么是O(n+m)？**
A: 
- 计算next数组：O(m)（m为模式串长度）
- 匹配过程：i只增不减，最多比较n次（n为主串长度）
- 总时间：O(m+n)，比朴素算法的O(n×m)快很多

**Q: KMP适用于什么场景？**
A: 适合长模式串匹配。如果模式串很短（2-3个字符），朴素算法可能更快（常数因子小）。模式串越长，KMP优势越明显。

### 模板1：手算next数组
```
第1步：写出模式串和下标
S:    a b a a b c

第2步：逐个计算next[j]
- next[0] = -1（约定）
- next[1] = 0（单字符无前后缀）
- next[2]：看"ab"，无公共前后缀 → 0
- next[3]：看"aba"，前缀{a,ab}后缀{ba,a}，公共a → 1
- next[4]：看"abaa"，前缀{a,ab,aba}后缀{baa,aa,a}，公共a → 1
- next[5]：看"abaab"，前缀{a,ab,aba,abaa}后缀{baab,aab,ab,b}，公共ab → 2

第3步：填表
next: -1  0  0  1  1  2
```

### 模板2：计算匹配比较次数
```
第1步：准备next数组（已知或计算）

第2步：逐字符模拟匹配
- 匹配成功：i++, j++，计数+1
- 失配：j回退到next[j]，计数+1（失配也算一次比较）
- 如果j=-1：i++, j=0，不计数

第3步：统计总比较次数
包括所有成功和失败的比较
```

**常见变式**：
- 求next数组 → 模板1
- 求匹配成功时的比较次数 → 模板2
- 求nextval数组 → 在next基础上优化

## 定义

## 定义

**KMP算法**（Knuth-Morris-Pratt算法）：一种改进的字符串模式匹配算法，通过利用已匹配的信息，避免主串指针回溯，从而提高匹配效率。

**核心思想**：
- 当模式串与主串失配时，利用已匹配部分的信息
- 模式串指针智能回退，主串指针不回退
- 通过next数组记录模式串的部分匹配信息

## 核心概念

### 1. next数组

**定义**：next[j]表示当模式串第j位失配时，j应该回退到的位置。

**计算规则**：
- next[j] = 模式串前j个字符的**最长公共前后缀**的长度
- 特殊值：next[0] = -1（约定）

**示例**：模式串S = "abaabc"

| j | 0 | 1 | 2 | 3 | 4 | 5 |
|---|---|---|---|---|---|---|
| S[j] | a | b | a | a | b | c |
| 子串 | - | a | ab | aba | abaa | abaab |
| 最长公共前后缀 | - | 无 | 无 | a | a | ab |
| next[j] | -1 | 0 | 0 | 1 | 1 | 2 |

**计算说明**：
- next[0] = -1（约定）
- next[1] = 0（单字符无前后缀）
- next[2] = 0（"ab"无公共前后缀）
- next[3] = 1（"aba"的最长公共前后缀是"a"，长度1）
- next[4] = 1（"abaa"的最长公共前后缀是"a"，长度1）
- next[5] = 2（"abaab"的最长公共前后缀是"ab"，长度2）

### 2. nextval数组（优化）

进一步优化next数组，避免不必要的比较：

**规则**：
```
if (S[j] == S[next[j]])
    nextval[j] = nextval[next[j]];
else
    nextval[j] = next[j];
```

## 算法步骤

### 1. 计算next数组

```c
void getNext(char *S, int *next, int m) {
    int i = 0, j = -1;
    next[0] = -1;
    
    while (i < m - 1) {
        if (j == -1 || S[i] == S[j]) {
            i++; j++;
            next[i] = j;
        } else {
            j = next[j];  // 回退
        }
    }
}
```

### 2. KMP匹配

```c
int KMP(char *T, char *S, int *next) {
    int i = 0, j = 0;
    int n = strlen(T);  // 主串长度
    int m = strlen(S);  // 模式串长度
    
    while (i < n && j < m) {
        if (j == -1 || T[i] == S[j]) {
            i++; j++;  // 匹配成功，继续比较下一位
        } else {
            j = next[j];  // 失配，j回退，i不回退
        }
    }
    
    if (j == m)
        return i - m;  // 匹配成功，返回起始位置
    else
        return -1;     // 匹配失败
}
```


## 时间空间复杂度

| 算法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| 朴素算法 | O(n×m) | O(1) | 主串回溯 |
| **KMP算法** | **O(n+m)** | **O(m)** | 主串不回溯 |

**复杂度分析**：
- 计算next数组：O(m)
- 匹配过程：O(n)
- 总时间：O(n+m)
- 空间：O(m)（存储next数组）

## KMP vs 朴素算法

| 特性 | 朴素算法 | KMP算法 |
|------|---------|---------|
| 主串指针 | 会回退 | **不回退** |
| 模式串指针 | 回到起点 | **智能回退** |
| 比较次数 | 多次重复比较 | 利用已匹配信息 |
| 最坏情况 | O(n×m) | O(n+m) |
| 适用场景 | 短模式串 | 长模式串 |

## 关键理解

### 为什么主串指针不回退？

**朴素算法的问题**：
```
T: a b a a b c
S: a b c
   ✓ ✓ ✗ 失配后i回退到1，重复比较
```

**KMP的改进**：
- 已经知道T[0..1]与S[0..1]匹配
- 利用这些信息，避免重复比较
- i只需要向前，不需要回退

### next数组的物理意义

**next[j]的含义**：
- 当S[j]失配时，前面已有j个字符匹配成功
- 这j个字符的最长公共前后缀长度是next[j]
- 因此可以直接从next[j]位置继续比较

**示例**：S = "abaabc"，在j=5失配
- 前5个字符"abaab"已匹配
- "abaab"的最长公共前后缀是"ab"（长度2）
- 主串中已经有"ab"在正确位置，直接从j=2继续

## 易错点

1. **next数组下标从0还是1开始**
   - 本说明从0开始，next[0]=-1
   - 有些教材从1开始，next[1]=0
   - 要根据具体题目调整

2. **next和nextval混淆**
   - next是基础版本
   - nextval是优化版本
   - 两者计算方法不同

3. **失配时的处理**
   - j回退到next[j]
   - i不变，不要回退i

4. **比较次数的统计**
   - 包括成功和失败的比较
   - 不包括指针移动等辅助操作
