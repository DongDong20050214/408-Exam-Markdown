# 信号量

## 常见提问

**Q: P操作和V操作分别做什么？**
A: 
- P操作（申请资源）：S--，若S<0则进程阻塞
- V操作（释放资源）：S++，若S≤0则唤醒一个等待进程
- 两者都是原子操作，不可中断

**Q: 信号量的值表示什么含义？**
A: 
- S>0：可用资源数量
- S=0：资源刚好用完，无等待进程
- S<0：|S|表示等待该资源的进程数

**Q: 互斥信号量和同步信号量有什么区别？**
A: 
- 互斥信号量：初值=1，保护临界区，同一进程的P和V成对
- 同步信号量：初值=0，控制执行顺序，前V后P（不同进程）

**Q: 生产者-消费者问题为什么要先申请资源信号量，再申请互斥信号量？**
A: 防止死锁。如果先P(mutex)再P(empty)，可能所有生产者都持有mutex等待empty，而消费者无法V(empty)（因为拿不到mutex），造成死锁。正确顺序：先资源后互斥。

**Q: 为什么信号量操作必须是原子操作？**
A: 防止竞态条件。如果P操作的"判断S"和"S--"之间被打断，可能两个进程同时通过判断，都执行S--，导致资源分配错误。

**Q: 如何用信号量实现进程的前驱关系？**
A: 设置同步信号量初值为0。在前驱进程末尾V操作，在后继进程开头P操作。例如P1→P2→P3：
```
S12=0, S23=0
P1: {代码; V(S12);}
P2: {P(S12); 代码; V(S23);}
P3: {P(S23); 代码;}
```

## 定义

## 定义

**信号量（Semaphore）**：是一种用于进程同步和互斥的整型变量，只能通过两个原子操作P和V来访问。

**核心思想**：
- 用一个整型变量记录可用资源数量
- 通过原子操作保证多进程对资源的正确访问
- 实现进程间的同步与互斥

## 信号量的类型

### 1. 整型信号量

简单的整数变量，只能取非负值：
```c
int S = 1;  // 初始值表示可用资源数
```

### 2. 记录型信号量（常用）

包含资源数和等待队列：
```c
typedef struct {
    int value;           // 信号量值
    struct process *L;   // 等待队列
} semaphore;
```

## P操作和V操作

### P操作（wait/down/申请资源）

```c
P(S) {
    S.value--;
    if (S.value < 0) {
        // 资源不足，进程阻塞
        将进程加入S.L等待队列;
        block(this);
    }
}
```

**含义**：
- 申请一个资源，S--
- 若S<0，表示无可用资源，进程阻塞
- S的绝对值 = 等待该资源的进程数

### V操作（signal/up/释放资源）

```c
V(S) {
    S.value++;
    if (S.value <= 0) {
        // 有进程在等待
        从S.L等待队列中移出一个进程;
        wakeup(P);
    }
}
```

**含义**：
- 释放一个资源，S++
- 若S≤0，表示有进程在等待，唤醒一个
- 若S>0，表示有可用资源

**关键特性**：
- P和V操作必须成对出现
- P和V操作是原子操作（不可中断）
- 同一进程可以多次执行P和V

## 信号量的应用

### 1. 实现互斥（Mutual Exclusion）

保证临界区同时只有一个进程访问：

```c
semaphore mutex = 1;  // 初始值为1

// 进程P1
P(mutex);
临界区代码;
V(mutex);

// 进程P2
P(mutex);
临界区代码;
V(mutex);
```

**互斥信号量初值 = 1**

### 2. 实现同步（Synchronization）

保证进程执行的先后顺序：

```c
semaphore S = 0;  // 初始值为0

// 进程P1
代码段1;
V(S);  // 通知P2可以执行

// 进程P2  
P(S);  // 等待P1完成
代码段2;
```

**同步信号量初值 = 0**

**规律**：前V后P

### 3. 实现前驱关系

多个进程的复杂执行顺序：

```
P1 → P2
P1 → P3
P2 → P4
P3 → P4
```

```c
semaphore S12 = 0, S13 = 0;
semaphore S24 = 0, S34 = 0;

P1: { code; V(S12); V(S13); }
P2: { P(S12); code; V(S24); }
P3: { P(S13); code; V(S34); }
P4: { P(S24); P(S34); code; }
```

### 4. 生产者-消费者问题

```c
semaphore mutex = 1;    // 互斥访问缓冲区
semaphore empty = n;    // 空缓冲区数量
semaphore full = 0;     // 满缓冲区数量

// 生产者
while(true) {
    生产产品;
    P(empty);    // 申请空缓冲区
    P(mutex);    // 进入临界区
    放入产品;
    V(mutex);    // 退出临界区
    V(full);     // 增加满缓冲区数
}

// 消费者
while(true) {
    P(full);     // 申请满缓冲区
    P(mutex);    // 进入临界区
    取出产品;
    V(mutex);    // 退出临界区
    V(empty);    // 增加空缓冲区数
    消费产品;
}
```

**注意**：P操作顺序很重要，先资源后互斥，防止死锁！

### 5. 读者-写者问题

```c
semaphore rmutex = 1;    // 保护readcount
semaphore wmutex = 1;    // 写互斥
int readcount = 0;       // 读者数量

// 读者
P(rmutex);
readcount++;
if (readcount == 1)      // 第一个读者
    P(wmutex);           // 阻止写者
V(rmutex);

读文件;

P(rmutex);
readcount--;
if (readcount == 0)      // 最后一个读者
    V(wmutex);           // 允许写者
V(rmutex);

// 写者
P(wmutex);
写文件;
V(wmutex);
```

## 信号量值的含义

| S的值 | 含义 |
|-------|------|
| S > 0 | 可用资源数量 |
| S = 0 | 无可用资源，无等待进程 |
| S < 0 | \|S\| = 等待该资源的进程数 |

**例**：
- S = 3：有3个可用资源
- S = 0：资源刚好用完
- S = -2：有2个进程在等待

## 常见错误

### 1. P、V操作不配对

```c
// 错误：只有P没有V
P(mutex);
临界区;
// 忘记V(mutex); → 死锁
```

### 2. P、V操作顺序错误

```c
// 错误：先互斥后资源
P(mutex);  // 可能导致死锁
P(empty);
...
```

应该先申请资源，再申请互斥：
```c
P(empty);  // 正确
P(mutex);
...
```

### 3. 初值设置错误

- 互斥信号量：初值 = 1
- 同步信号量：初值 = 0
- 资源信号量：初值 = 资源数量

## 典型例题

**例1**：某系统有3台打印机，5个进程竞争使用。用信号量S表示可用打印机数量，则：
- S初值 = 3
- 某时刻S = -2，表示有2个进程在等待打印机

**例2**：实现进程P1、P2、P3按顺序执行，需要几个信号量？
- 答案：2个（S12和S23）
- P1: {代码; V(S12);}
- P2: {P(S12); 代码; V(S23);}
- P3: {P(S23); 代码;}

## 记忆要点

- **P操作**：申请资源，S--，可能阻塞
- **V操作**：释放资源，S++，可能唤醒
- **互斥**：初值1，保护临界区
- **同步**：初值0，前V后P
- **生产消费**：先资源后互斥（防死锁）
- **S<0时**：|S|表示等待进程数
