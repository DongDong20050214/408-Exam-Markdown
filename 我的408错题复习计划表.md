# 我的408错题复习计划表

> 配套：[[【刷题攻略v3】每天4小时·取舍版]] | [[我的408刷题进度-冲刺版]]
> 用途：① 错题从"做错"到"移出"的全流程管理 ② 每天滚动复习打卡 ③ **遗忘监测**

---

## 🔁 错题生命周期（每道错题都走这条流水线）

```
做错 → 当场打标（YAML） → 当日40min费曼复盘 → 进入滚动池
     → 按间隔重做（1周后、1个月后） → 连对2次 → 移出滚动池 ✅
                                    → 又做错 → review_count清零，重新排队 🔄
```

**打标规范**（错题当场写进题目 YAML，缺一不可）：

```yaml
tags: ["错题", "数据结构", "B树"]
review_count: 0        # 连续答对次数，做错就清零
last_review: 2026-06-16
```

**移出标准**：`review_count ≥ 2`（连续两次答对）→ 把 `错题` 标签改为 `已掌握`，不再进入滚动池。

## ⚠️ 遗忘监测（每天滚动复习后必填一行）

**硬指标：旧科目错题重做正确率**

| 正确率 | 含义 | 动作 |
|--------|------|------|
| ≥70% | 遗忘可控 | 按计划走 |
| 50~70% | 开始衰减 | 该科目滚动题量 3~5题 → 6~8题 |
| <50% 连续3天 | 🚨 超预期 | 当周周日2h改为该科目薄弱章节**全章重刷** |

## 🔍 Dataview 自动查询

### ① 今日滚动复习取题（最久没碰的10道）
```dataview
TABLE file.name as 题目, subject as 科目, chapter as 章节, last_review as 上次复习, review_count as 连对
FROM "408题库"
WHERE contains(tags, "错题")
SORT last_review ASC
LIMIT 10
```

### ② 错题最密集章节（专项日定靶用）
```dataview
TABLE length(rows) as 错题数
FROM "408题库"
WHERE contains(tags, "错题")
GROUP BY subject + " / " + chapter
SORT length(rows) DESC
LIMIT 10
```

### ③ 快移出的题（连对1次，再对1次就毕业）
```dataview
TABLE file.name as 题目, subject as 科目, last_review as 上次复习
FROM "408题库"
WHERE contains(tags, "错题") AND review_count = 1
SORT last_review ASC
```

### ④ 已掌握题数（成就感专用）
```dataview
TABLE length(rows) as 已掌握
FROM "408题库"
WHERE contains(tags, "已掌握")
GROUP BY subject
```
